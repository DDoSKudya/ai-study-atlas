# Contributing

Спасибо за вклад в AI Study Atlas.
Это проект учебной Markdown-базы, поэтому главный фокус вклада — качество структуры, ясность объяснений и консистентность навигации.
Процесс intentionally простой: минимальная бюрократия, максимум практической пользы.

## Что важно перед изменениями

1. Прочитать `INDEX.md` для общей карты проекта.
2. Прочитать `docs/content_contract.md` и `docs/naming_convention.md`.
3. Следовать workflow из `prompts/HOW_TO_USE_PROMPTS.md`, если вы создаете/улучшаете контент через промпты.

## Типы полезных вкладов

- Улучшение структуры и навигации.
- Улучшение качества объяснений без искажения терминов.
- Добавление/улучшение self-check блоков.
- Исправление несоответствий naming-convention.
- Исправление битых ссылок и неточностей в документации.

## Ограничения проекта

- Не добавлять выдуманный "маркетинговый" или "филлерный" контент.
- Не ломать существующий workflow с промптами.
- Не делать массовые переименования без явной необходимости.

## Правила для PR (облегченный режим)

- По возможности: один PR — одна логическая цель.
- В описании PR указать:
  - что именно изменено;
  - зачем это изменение;
  - какие документы/разделы затронуты.
- Если изменение касается структуры, обновить `INDEX.md` и при необходимости `CHANGELOG.md`.
- Для небольших одиночных правок допускаются прямые коммиты без сложного PR-процесса.

## Changelog

Используйте категории в стиле GitLab changelog:

- `added`
- `fixed`
- `changed`
- `deprecated`
- `removed`
- `security`
- `performance`
- `other`

При коммитах можно добавлять trailer:

`Changelog: <category>`

Например: `Changelog: changed`
