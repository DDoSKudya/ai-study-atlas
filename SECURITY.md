# Security Policy

## Supported Versions

This repository stores educational content and documentation.
Security-sensitive changes usually relate to:

- prompt or workflow instructions that can cause unsafe automation behavior;
- accidental publication of private or sensitive data in markdown files.

Current active branch is considered supported.

## Reporting a Vulnerability

If you find a security issue:

1. Do not open a public issue with sensitive details.
2. Contact maintainers through a private channel if available.
3. If no private channel is available, open a minimal public issue without exploit details and request private follow-up.

Please include:

- affected files and paths;
- impact summary;
- reproduction steps (if safe);
- suggested remediation (optional).
