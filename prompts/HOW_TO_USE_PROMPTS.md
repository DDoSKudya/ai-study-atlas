# Как использовать промпты

Этот документ описывает рекомендуемый порядок работы с промптами из папки `prompts/` для построения учебной базы:

1. сначала создаётся и усиливается глобальный план;
2. затем создаётся контент по каждой части плана;
3. после этого выполняется финальное улучшение связности плана и контента.

Важно: количество вызовов в шагах ниже не является жёстким правилом. Это практический ориентир, который используется "на глаз" в зависимости от качества результата после каждой итерации.

---

## 1) Создание глобального плана

Используй промпты из `prompts/global_plan/` в таком порядке:

1. `create_global_mastery_plan.md` — **1 раз**
2. `enhance_global_plan.md` — **15 раз**
3. `upgrade_global_plan_to_99_percent.md` — **1 раз**

### Почему именно так

- Первый промпт создаёт базовый каркас плана: структуру, шаги, декомпозицию.
- Серия из `enhance_global_plan.md` добавляет недостающие подтемы, крайние случаи, альтернативы и повышает детализацию.
- Финальный `upgrade_global_plan_to_99_percent.md` работает как "контрольный проход" на полноту и методическую зрелость плана.

Идея этапа: получить план, по которому можно создавать учебные части без догадок.

---

## 2) Создание контента по частям глобального плана

Для **каждой части** глобального плана используй промпты из `prompts/pact/` в таком порядке:

1. `create_learning_content.md` — **1 раз**
2. `improve_latest_learning_file.md` — **8 раз**
3. `add_self_check_questions.md` — **1 раз**
4. `validate_and_fix_structure.md` — **1 раз**

### Почему именно так

- `create_learning_content.md` создаёт первичный полный вариант учебной части.
- Повторные вызовы `improve_latest_learning_file.md` нужны, чтобы последовательно убирать "сухость", пробелы и слабые объяснения.
- `add_self_check_questions.md` добавляет контроль понимания, чтобы материал был не только для чтения, но и для закрепления.
- `validate_and_fix_structure.md` выравнивает итог под единый шаблон курса и снижает структурный дрейф между файлами.

Идея этапа: каждая часть должна быть цельной, обучающей и методически согласованной с остальным курсом.

---

## 3) Финальная связка плана и контента

После создания всех файлов контента выполни финальные промпты из `prompts/global_plan/`:

1. `analyze_and_improve_latest_learning_file.md` — **1 раз**
2. `full_audit_and_auto_improvement.md` — **10 раз**
3. `reorganize_structure_and_split_into_parts.md` — **1 раз**

### Почему именно так

- `analyze_and_improve_latest_learning_file.md` помогает сделать точечный финальный проход по последнему/актуальному материалу.
- Многократный `full_audit_and_auto_improvement.md` приводит базу к более ровному качеству: покрытие плана, связность, понятность, равномерная глубина.
- `reorganize_structure_and_split_into_parts.md` завершает работу на уровне архитектуры материалов: навигация, разбиение, структура папок и связей.

Идея этапа: привести весь курс к состоянию единой обучающей системы, а не набора разрозненных файлов.

---

## Практические рекомендации

- После каждого 2-3 прогона быстро проверяй, есть ли реальный прирост качества. Если прироста нет, меняй фокус уточнениями.
- Если часть темы особенно сложная, увеличивай число итераций улучшения локально именно для неё.
- Если структура уже хорошая, не делай лишнюю реорганизацию: цель — удобство обучения, а не "идеальная" перестройка ради перестройки.
- Веди изменения последовательно: план -> контент -> глобальный аудит -> структурная доводка.

---

## Короткая формула процесса

`План -> Усиление плана -> Контент по частям -> Усиление контента -> Самопроверка -> Структурная проверка -> Глобальный аудит -> Реорганизация`
