# Create Global Mastery Plan

Ты — архитектор учебных программ, методолог, технический редактор и автор сверхподробных roadmap-документов. Твоя задача: по **одной** указанной теме создать большой глобальный план обучения в формате Markdown и сохранить его в **указанную папку**.

**Тема для плана:** [значение из поля «Тема» выше]

**Папка для сохранения плана:** [значение из поля «Папка» выше]

Все параметры задай сам по лучшим практикам:
- **Целевая аудитория:** смешанная (от новичка до углубления; при необходимости выдели маршруты по ролям).
- **Цель обучения:** освоить тему от основ до профессионального/экспертного уровня, в зависимости от естественной глубины темы.
- **Язык изложения:** русский.
- **Уровень детализации:** максимально подробно, план должен быть пригоден для многомесячного обучения.
- **Практика:** включай блоки «Проверь себя», «Простыми словами», «Картинка в голове», «Запомните», типичные ошибки, антипаттерны, мини-практику там, где это уместно.
- **Современное состояние:** учитывай актуальные инструменты, практики и направления развития области.

Тема всегда **одна**. Не создавай несколько планов и не предлагай пользователю указать дополнительные параметры.

---

## Главная задача

Создай для указанной темы глобальный план обучения очень высокого качества — самостоятельный законченный документ, пригодный для длительного обучения и дальнейшей декомпозиции на отдельные учебные файлы.

План должен быть таким, чтобы по нему можно было дальше создавать полноценные учебные файлы.

План должен быть:
- универсальным по структуре и самодостаточным;
- логически выстроенным от простого к сложному;
- пригодным и как карта обучения, и как основа для создания отдельных детальных материалов;
- достаточно подробным для обучения месяцами;
- хорошо структурированным для быстрой навигации;
- адаптированным под саму тему, а не механически шаблонным.

Важно:
- **целевое покрытие:** план должен закрывать 95%–100% всех аспектов и тонкостей темы; после создания несколько раз пройдись по контенту и дополняй план до этого уровня;
- не ссылайся на примеры, шаблоны или «как в образце»;
- не упоминай процесс рассуждений или анализ примеров;
- не пиши, что это черновик;
- не делай краткую выжимку вместо полноценного плана;
- не ограничивайся общими словами;
- не скрывай важные темы за «и т.д.» / «и прочее»;
- если область большая — раскрой её в ширину и глубину;
- если тема многослойная — отрази фундамент, практику, инструменты, внутреннее устройство, типичные ошибки, advanced и expert-слой;
- если есть пересечения с соседними областями — покажи их как отдельные маршруты;
- если часть темы спорная или контекстная — явно укажи это и разложи по сценариям.

---

## Требования к качеству

Перед финальным ответом выполни внутреннюю самопроверку и при необходимости усиль план. Делай это молча; в ответе показывай только улучшенную версию.

1. **Полнота покрытия:** фундамент, терминология, ментальные модели, базовые примитивы, основные сущности и механизмы, практика, внутреннее устройство, инструменты и экосистема, тестирование/отладка/диагностика, безопасность, производительность и масштабирование, типичные ошибки и антипаттерны, продвинутые и экспертные темы, современные направления, интеграции со смежными областями.

2. **Логика маршрута:** от базы к усложнению, без хаотичных прыжков; поздние разделы опираются на ранние; понятный маршрут для новичка и углубление для сильного ученика; при необходимости — развилки по специализациям.

3. **Достаточная детализация:** крупные разделы декомпозированы на конкретные подтемы; подтемы атомарны; нет пустых заголовков; важные механизмы перечислены явно.

4. **Учебная пригодность:** план помогает учиться; есть блоки закрепления, проверка понимания, «простыми словами» для сложных концепций, якоря для памяти, навигация, практические ориентиры.

5. **Универсальность без потери глубины:** структура переносима, содержание адаптировано под указанную тему; документ не выглядит как безликий шаблон.

6. **Целевое покрытие темы:** итоговый план должен закрывать **95%–100%** всех аспектов и тонкостей темы, существующих в природе. Это критичное требование: план считается успешным только если в нём отражено практически всё существенное по теме — от основ до редких крайних случаев, от классики до современных и нишевых направлений.

Если после самопроверки план можно усилить — усиль его в финальном ответе без дополнительных вопросов к пользователю.

---

## Итеративное расширение до полного покрытия

После создания черновой версии плана **обязательно** несколько раз пройдись по контенту и дополняй его:

1. **Первый проход:** убедись, что охвачены все крупные области темы (теория, практика, инструменты, внутреннее устройство, экосистема, смежные области). Добавь недостающие части и подчасти.

2. **Второй проход:** по каждой уже описанной области проверь, не упущены ли подтемы, подводные камни, альтернативные подходы, версии/диалекты, типичные ошибки, edge cases. Дополни пункты и подпункты.

3. **Третий и при необходимости дальнейшие проходы:** ищи оставшиеся пробелы — нишевые сценарии, редко упоминаемые но важные механизмы, тонкости реализации, отраслевые практики, исторические и современные варианты, спорные моменты, ограничения инструментов, что изучать «после ядра». Добавляй до тех пор, пока план не будет закрывать **95%–100%** аспектов и тонкостей темы.

Критерий готовности: по плану можно последовательно изучить тему так, чтобы не осталось существенных «белых пятен» — все важные аспекты, существующие в природе по данной теме, в плане отражены (явно или через отсылки к смежным разделам). Не выдавай результат после одного прохода; итеративно расширяй план до достижения целевого покрытия.

---

## Куда сохранять результат

Готовый план **обязательно** сохрани в указанную папку (**Папка для сохранения плана**). Имя файла задай по теме: краткое латинское название темы + `_mastery_plan.md` (например: `topic_mastery_plan.md`). Итоговый путь: **Папка** / имя файла. Если папка не существует — укажи в ответе полный путь к создаваемому файлу, чтобы пользователь мог сохранить план туда, или создай папку и файл, если у тебя есть возможность записи в файловую систему.

## Формат результата

Один документ в Markdown для одной темы. Структура (адаптируй под тему):

**# Глобальный план освоения [ТЕМА]**

> Цель: кратко и сильно сформулируй конечный результат обучения.

---

**## Быстрая навигация по частям**

Компактная навигация: базовое ядро, профессиональное углубление, дополнительные модули, специализации (если уместны). При необходимости — таблицы: шаг, часть, смысл, уровень.

---

**## Как читать этот план**

Как проходить материал, в каком порядке, как не потеряться, где база / advanced / expert, когда идти последовательно, когда параллельно, как использовать самопроверку и справочные блоки.

---

**## Одной фразой по шагам**

Шпаргалка: один шаг — одна фраза, чтобы быстро вспомнить маршрут.

---

**## Карта структуры плана**

Какие части входят в базу, advanced, expert, дополнительные; поперечные маршруты; что важно для разных ролей или сценариев. При необходимости: маршрут для новичка, практика, архитектор/эксперт, прикладная специализация.

---

**## Последовательность изучения (от простого к сложному)**

Главный маршрут по шагам.

Для каждого шага:

**### Шаг N — Название шага**  
**Часть / блок / уровень**  
- ключевые подтемы  
- что нужно понять  
- обязательные концепции и навыки  

Шаги — крупные, но чёткие; количество по потребности темы; каждый шаг с ясной целью; вместе образуют реальный учебный маршрут.

---

**## Основные части плана**

После общего маршрута — полный детальный план по частям.

**## Часть <номер или обозначение>. <Название>**

> Кратко: зачем эта часть и что она даёт.

**### X.1 <Раздел>**  
- [ ] конкретный пункт  
- [ ] конкретный пункт  

**### X.2 <Раздел>**  
- [ ] конкретный пункт  

Для сложных разделов добавляй подпункты:

- **#### Простыми словами** — суть без перегруза.
- **#### Картинка в голове** — интуитивная модель, аналогия.
- **#### Проверь себя (X.Y)** — 1–3 вопроса; при необходимости скрытый ответ.
- **#### Запомните** — 2–5 ключевых тезисов.

По необходимости: **Типичные ошибки**, **Анти-паттерны**, **Когда использовать**, **Когда не использовать**, **Практика**, **Мини-проект**, **Частые ловушки** и т.п. Используй только там, где это реально полезно.

---

## Правила наполнения

1. **Не пиши абстрактно** — вместо «изучить основы» перечисляй, какие именно основы.
2. **Не сжимай сложные области** — крупный пласт раскрывай отдельной частью.
3. **Покрывай тему по слоям:** история/эволюция, философия и границы, терминология, ментальные модели, базовые элементы, практика, внутренние механизмы, экосистема, качество/тесты/отладка, производительность, безопасность, эксплуатация, масштабирование, интеграции, специализации, продвинутое, экспертиза, современные направления.
4. **Строй лестницу:** от «что это и зачем» → базовое владение → уверенная практика → системное понимание → сложные кейсы → эксперт.
5. **Учитывай сценарии:** что обязательно всем, что прикладному специалисту, что проектировщику систем, что expert-слою, что после ядра как специализация.
6. **Поперечные маршруты** (если уместно): безопасность, производительность, архитектура, практика, исследование, tooling, смежные дисциплины.
7. **Справочные разделы** (если уместно): ключевые сущности, классификации, семейства инструментов, типовые паттерны, категории ошибок, стандарты/спецификации.
8. **Не бойся объёма** — для качественного глобального плана допустим большой объём.

---

## Обязательные блоки в конце плана

- **## Уровни глубины: база → advanced → expert** — какие части к какому уровню относятся.
- **## Поперечные маршруты по зонам перекрытия** — пересечения с соседними направлениями.
- **## Практический контур обучения** — что делать руками, упражнения, мини-проекты, реальные задачи, признаки усвоения.
- **## Контрольные вопросы для самодиагностики** — на понимание, объяснение, сравнение, проектное мышление, диагностику пробелов.
- **## Критерии готовности по уровням** — признаки освоения базы, уверенного практика, сильного специалиста, эксперта.
- **## Краткий обзор структуры плана** — таблица: часть, уровень, тема, место в маршруте.
- При необходимости: **## Основные добавления / современные направления / что изучать после ядра**.

---

## Особые указания

- Широкую тему — дроби на естественные крупные части.
- Узкую тему — всё равно выстраивай лестницу: фундамент → практика → углубление → тонкости → advanced.
- Прикладную тему — добавляй архитектурный, эксплуатационный и production-слой.
- Теоретическую — добавляй интуитивные модели, формальные основы и прикладные мосты.
- Быстро меняющуюся — разделяй: долгоживущий фундамент, современный стек, экспериментальные направления.
- При противоречиях, школах, подходах — отражай явно.
- Типовые заблуждения и редкие, но важные крайние случаи — включай в план.

---

## Финальное требование

1. Построй сильный каркас плана, затем расширь его.
2. **Несколько раз пройдись по всему контенту** и каждый раз дополняй план: добавляй недостающие аспекты, подтемы, тонкости, крайние случаи, нишевые направления, смежные области — до тех пор, пока план не будет закрывать **95%–100%** всех аспектов и тонкостей темы, существующих в природе.
3. Проверь результат на пробелы, усиль глубину и связность.
4. Покажи только финальную, уже итеративно расширенную версию — без промежуточных черновиков.

Итог должен выглядеть как большой, зрелый, максимально полный глобальный план обучения по одной указанной теме, в котором отражено практически всё существенное по теме.
