# Reorganize Structure And Split Into Parts

## Входные данные
- GLOBAL_PLAN: <полный путь к файлу глобального плана, например: /path/to/training/topic/mastery_plan.md>

Обозначения (ниже в тексте используй только их, а не явные пути):
- ROOT_DIR  = каталог, в котором лежит GLOBAL_PLAN
- PARTS_DIR = ROOT_DIR/pact
- STRUCT_MD = ROOT_DIR/pact/README_STRUKTURA_IZUCHENIYA.md (если существует)
- (опционально) REFERENCE_ROOT  = <путь к курсу-эталону, если хочешь ориентироваться на его структуру>
- (опционально) REFERENCE_PLAN  = глобальный план в REFERENCE_ROOT
- (опционально) REFERENCE_PARTS = REFERENCE_ROOT/pact или другая папка с частями эталона


## Промпт
Твоя задача: полностью реорганизовать учебные материалы в каталоге ROOT_DIR так, чтобы крупные монолитные файлы в PARTS_DIR были разбиты на подфайлы по частям, каждая часть имела свой `index.md`, навигация была сквозной, а структура — удобной для чтения, навигации и работы в просмотрщике Markdown.

Эталоны, на которые можно ориентироваться:
- Эталон структуры разбиения (если задан): REFERENCE_ROOT/REFERENCE_PARTS
- Эталон глобального плана (если задан): REFERENCE_ROOT/REFERENCE_PLAN
- Структурный эталон частей (если есть): STRUCT_MD
- Глобальный план текущего курса: GLOBAL_PLAN
- Папка с частями текущего курса: PARTS_DIR

Главная цель:
- взять существующие крупные файлы частей в PARTS_DIR и разложить их на иерархию подфайлов по осмысленной структуре;
- сделать для каждой части свой `index.md`, который служит точкой входа и навигацией;
- организовать двусторонние ссылки:
  - GLOBAL_PLAN → index.md части;
  - index.md части → подфайлы;
  - подфайл → index.md своей части (и, при необходимости, → GLOBAL_PLAN);
- привести структуру всей базы в ROOT_DIR к единому, предсказуемому, учебно-удобному виду.

Общий подход:
- не трогай `GLOBAL_PLAN`, кроме ссылок и навигационных блоков;
- основная работа — в PARTS_DIR;
- уважай и используй STRUCT_MD как эталон формы, если он есть;
- если есть REFERENCE_ROOT/REFERENCE_PARTS — используй их как идею того, как должна быть устроена иерархия.

Шаги работы:

1. Понять текущую структуру.

1.1. Проанализируй GLOBAL_PLAN.
- Прочитай GLOBAL_PLAN.
- Определи список «частей»/крупных блоков курса (часто это «Часть N», «Step N», группы параграфов и т.п.).
- Для каждой части пойми:
  - где в PARTS_DIR сейчас лежит её материал (какой файл/файлы);
  - какова примерно внутренняя логика: какие большие разделы внутри.

1.2. Проанализируй PARTS_DIR.
- Собери список всех .md-файлов в PARTS_DIR:
  - какие уже организованы в подпапки;
  - какие являются одинокими «гигантами» без разбивки.
- Отметь:
  - какие части уже имеют структуру с `index.md` и подфайлами;
  - какие части представлены одним-двумя большими файлами и требуют дробления.

1.3. Если есть STRUCT_MD — прочитай и запомни шаблон.
- Пойми, что именно ожидается в начале части, в оглавлении и в разделах;
- это будет твой «формальный» эталон оформления.

1.4. Если есть REFERENCE_ROOT/REFERENCE_PARTS — взгляни на эталонную структуру.
- Посмотри, как там устроены:
  - глобальный план;
  - папка с частями;
  - `index.md` внутри части;
  - подфайлы внутри одной части (имена, порядок, наполнение);
- не нужно копировать дословно, важно уяснить идею.

2. Спроектировать структуру для каждой части.

2.1. Для каждой крупной части плана (из GLOBAL_PLAN):
- Определи, какой(ие) файл(ы) в PARTS_DIR её покрывают сейчас;
- Открой этот файл и набросай «карту»:
  - какие крупные разделы внутри (по заголовкам, логике темы);
  - какие разделы логично выделить в отдельные подфайлы.

2.2. Спроектируй файлы части:
- Придумай структуру:
  - `index.md` — навигация по части;
  - `00_*` — введение + маршрут + оглавление + связь с GLOBAL_PLAN;
  - `01_*`, `02_*`, `03_*`, … — большие тематические блоки;
  - один или несколько финальных файлов (`0N_*`) — практикум, справочник, резюме, вопросы (если уместно).
- Определи имена файлов:
  - используй префикс с номером (`00_`, `01_`, `02_`...), чтобы обеспечить сортировку;
  - slug в имени должен кратко отражать содержание (латиницей или транслитом, но стабильно).

2.3. Реши, какие части нужно дробить.
- Если файл небольшой и логически цельный — можно оставить его как один подфайл + `index.md`;
- если файл огромный (1000+ строк) и содержит много тем — дроби его на подфайлы по смыслу, а не по размеру.

3. Создать папку части и `index.md`.

3.1. Для каждой части:
- создай папку `PARTS_DIR/имя_части/` (если её ещё нет);
  - имя папки выбери по логике плана (например, «part_01_*», «00_vvedenie_*», и т.п., но единообразно для всего курса);
- внутри этой папки создай `index.md`.

3.2. Содержимое index.md:
- заголовок части;
- краткое описание, чему посвящена часть;
- явная ссылка назад на глобальный план:
  - например: `[← Назад к глобальному плану](../RELATIVE_PATH_TO_GLOBAL_PLAN.md)` — путь бери относительно ROOT_DIR;
- список всех подфайлов этой части в правильном порядке:
  - один раздел «Навигация по части»;
  - маркеры/список вида:
    - `[Введение, маршрут и оглавление](00_...md)`
    - `[§1 Тема](01_...md)` и т.п.;
- короткое пояснение, как читать эту часть (если это укладывается в STRUCT_MD).

3.3. Если STRUCT_MD описывает обязательные блоки в начале части:
- убедись, что index.md + `00_...md` удовлетворяют этим требованиям:
  - цель части;
  - «после этой части ты сможешь» (если нужно);
  - что знать заранее;
  - краткая шпаргалка;
  - маршрут;
  - структура материала;
  - пояснение, как устроены разделы и оглавление.

4. Вынести содержимое из монолитных файлов в подфайлы.

4.1. Для каждого крупного исходного файла части:
- разбей его на подфайлы по спроектированной структуре;
- каждый логический блок (крупный раздел) — в свой .md-файл;
- при необходимости слегка перепиши вступления и переходы, чтобы в новом файле текст читался цельно.

4.2. В начале каждого подфайла:
- добавь ссылку назад на index.md своей части:
  - `[← Назад к индексу части](index.md)`;
- при необходимости можно добавить ссылку на соответствующий раздел GLOBAL_PLAN (через относительный путь от ROOT_DIR, если удобно и полезно).

4.3. Убедись, что содержимое не потеряно:
- каждый абзац, пример, таблица из исходного монолитного файла должен оказаться либо в index.md, либо в одном из подфайлов;
- не оставляй неиспользованных «обрезков».

5. Обновить глобальный план (GLOBAL_PLAN).

5.1. Найти все места в GLOBAL_PLAN, где:
- упоминаются части/темы, для которых ты сейчас делал структуру;
- есть старые ссылки на монолитные файлы (если были).

5.2. Обновить ссылки:
- сделай явные блоки вида:
  - `Материалы: [→ Навигация по Части N](RELATIVE_PATH_TO_INDEX_FROM_GLOBAL_PLAN)`;  
  путь должен вести к index.md соответствующей части относительно файла GLOBAL_PLAN;
- если раньше были ссылки на старые файлы, замени их ссылками на index.md.

5.3. Проверь, что:
- структура и нумерация частей/шагов в GLOBAL_PLAN согласованы с тобой созданными папками в PARTS_DIR;
- читатель, глядя в GLOBAL_PLAN, чётко понимает, куда кликнуть, чтобы попасть в нужную часть.

6. Добавить обратные ссылки.

6.1. В index.md каждой части:
- обязательно добавь ссылку назад на GLOBAL_PLAN (относительным путём);
- это позволяет легко вернуться к общей картине.

6.2. В подфайлах части:
- ссылка `[← Назад к индексу части](index.md)` в самом верху обязательна;
- при необходимости:
  - можно добавить «см. также» с ссылками на другие части или на GLOBAL_PLAN, если это помогает навигации.

7. Учесть STRUCT_MD (если есть).

7.1. Если STRUCT_MD существует:
- проверь, какие обязательные блоки он предписывает для:
  - начала части;
  - структуры оглавления;
  - структуры разделов;
  - финального справочника/вопросов.

7.2. Приведи новые index.md и подфайлы в соответствие этим требованиям:
- не обязательно слово-в-слово, важно:
  - цель части в начале;
  - шпаргалка;
  - маршрут;
  - оглавление по этапам (если предусмотрено);
  - разделы по единому шаблону;
  - финальный справочник и вопросы (если оговорено).

8. Проверка совместимости с просмотрщиком Markdown.

8.1. Структура каталогов:
- убедись, что дерево ROOT_DIR выглядит аккуратно:
  - GLOBAL_PLAN наверху;
  - PARTS_DIR с подпапками по частям;
  - в каждой папке части есть index.md и подфайлы.

8.2. Ссылки:
- все ссылки между файлами должны быть относительными;
- не привязывайся к абсолютным путям файловой системы;
- проверь:
  - из GLOBAL_PLAN переход к index.md части работает;
  - из index.md в подфайлы — работает;
  - из подфайлов в index.md и (если есть) в GLOBAL_PLAN — работает.

8.3. Отображение:
- в просмотрщике, который строит дерево по ROOT_DIR, структура должна быть:
  - понятной;
  - предсказуемой;
  - читабельной.

9. Итоговая самопроверка.

9.1. Пройди маршрут «как студент»:
- начни с GLOBAL_PLAN;
- выбери одну часть;
- открой её index.md;
- пройдись по подфайлам по порядку;
- оцени:
  - понятно ли, куда идти;
  - нет ли «провалов» и пропусков;
  - корректно ли возвращение к индексу и к плану.

9.2. Убедись, что:
- никакой старый файл не «оторван» от системы (лежит, но на него никто не ссылается и он не нужен);
- новые файлы логично вставлены в общую структуру.

Формат итогового отчёта (для чата):

1. Кратко опиши новую структуру:
   - какие папки/части появились в PARTS_DIR;
   - какие крупные монолитные файлы были разбиты и на какие подфайлы.

2. Для каждой крупной части:
   - имя части / папки;
   - путь к её index.md;
   - список подфайлов с кратким смыслом каждого.

3. Изменения в GLOBAL_PLAN:
   - какие новые ссылки добавлены;
   - какие старые ссылки заменены;
   - как сейчас выглядит маршрут по плану → частям.

4. Совместимость и навигация:
   - работает ли навигация «план → часть → подфайл → часть → план»;
   - нет ли «висящих» файлов без входящих ссылок (если есть — перечисли).

5. Рекомендации на будущее:
   - если ещё остались части, которые можно/нужно дальше дробить;
   - если есть места, где структура может быть улучшена, но это не критично сейчас.

Важно:
- не оставляй «полумеры» — если начал структурировать часть, доведи её до логичного конца;
- не ломай уже хорошую структуру без необходимости;
- структура должна быть не только красивой, но и реально удобной для учёбы.
