## Часть XIX. Геопространственные данные

### Цель части

Научиться **хранить и обрабатывать геопространственные данные в PostgreSQL с PostGIS**: понимать, что такое геометрии и системы координат, как создавать точки/линии/полигоны, как писать пространственные запросы (расстояния, «находится внутри», «пересекается») и как использовать пространственные индексы (GiST, SP-GiST), чтобы всё это работало быстро на реальных объёмах данных.

После этой части ты сможешь:

- **установить и включить PostGIS** в базе PostgreSQL, понимать, какие типы и функции он добавляет;
- **различать типы `geometry` и `geography`**, понимать, когда использовать плоскую модель, а когда — модель «на сфере»;
- **работать с SRID и системами координат**: знать, что такое EPSG:4326 и EPSG:3857, как перепроецировать данные через `ST_Transform`;
- **создавать геометрии разных типов** (POINT, LINESTRING, POLYGON, MULTI*, GEOMETRYCOLLECTION) из текстового представления (WKT) и с помощью функций `ST_MakePoint`, `ST_MakeLine`, `ST_MakePolygon`;
- **писать пространственные запросы**: искать объекты в радиусе, попадание в полигон, пересечения, расстояния, буферы, объединения геометрий;
- **создавать и использовать пространственные индексы GiST / SP-GiST**, читать планы запросов и понимать, почему запрос с геоданными работает быстро или медленно;
- **избегать типичных ошибок с геоданными**: путаница мест/широт, неправильный SRID, игнорирование проекций и неточный расчёт расстояний.

Материал опирается на:

- **Часть II–III** (основы SQL, работа с типами и функциями),
- **Часть IV–VII** (транзакции, индексы, оптимизация запросов),
- **Часть V** (страницы, хранение, буферный кеш),
- **Часть XIV** (партиционирование и внутренности хранилища),
- **Часть XVI** (аналитика и хранилища данных),
- **Часть XVII** (расширения PostgreSQL, в т.ч. PostGIS).

В этой части мы смотрим на БД **как на геоплатформу**: точки на карте, полигоны районов, маршруты, поиск «объектов рядом» — всё это можно делать прямо в PostgreSQL, без отдельного ГИС‑движка, если понимать базовые концепции (типы, SRID, индексы и операции).

Изучать материал лучше **по шагам**, не перескакивая:

1. **§95. PostGIS: основы** — что такое PostGIS, типы `geometry`/`geography`, SRID и проекции, создание геометрий (WKT, WKB, `ST_MakePoint`, `ST_Transform`). Желательно заранее: основы SQL и расширения PostgreSQL (Часть XVII).

### Краткая шпаргалка по части

| Понятие / конструкция | Суть (что это и когда нужно) |
|-----------------------|------------------------------|
| **PostGIS** | Расширение PostgreSQL для геопространственных данных: добавляет типы `geometry`/`geography`, функции `ST_*`, индексы GiST/SP-GiST. |
| **`geometry`** | Тип для геометрий на плоскости (карта как лист бумаги); быстрый и гибкий, но расстояния — в единицах координат, а не обязательно в метрах. |
| **`geography`** | Тип для геометрий на поверхности эллипсоида (Земли); функции сразу считают расстояния/площади «по шару» в метрах, но вычисления дороже. |
| **SRID (Spatial Reference System Identifier)** | Числовой код системы координат (например, 4326 — WGS84, 3857 — Web Mercator); нужен, чтобы понимать, в какой проекции заданы координаты. |
| **WKT / WKB** | Текстовый (`Well-Known Text`) и бинарный (`Well-Known Binary`) форматы представления геометрий; удобны для обмена и дебага. |
| **`POINT`, `LINESTRING`, `POLYGON`** | Базовые типы: точка, ломаная линия и многоугольник (замкнутая область). |
| **`MULTI*` типы** | Коллекции однотипных геометрий: `MULTIPOINT`, `MULTILINESTRING`, `MULTIPOLYGON` — несколько точек/линий/полигонов в одном объекте. |
| **`GEOMETRYCOLLECTION`** | Смешанная коллекция геометрий разных типов (например, точка + полигон). |
| **`ST_Distance` / `ST_DistanceSphere`** | Функции расстояния: первая — в единицах координат (для `geometry`), вторая — приблизительно по сфере в метрах. |
| **`ST_Within` / `ST_Contains` / `ST_Intersects`** | Отношения между геометриями: внутри, содержит, пересекается. |
| **`ST_Buffer`** | Строит «окружность»/зону вокруг геометрии на заданное расстояние (часто для поиска «в радиусе X метров»). |
| **`ST_Union` / `ST_Intersection`** | Объединение и пересечение геометрий — полезно для агрегации и анализа зон. |
| **`ST_Envelope`** | Ограничивающий прямоугольник (bounding box) геометрии — быстрая «приближённая» оболочка. |
| **`ST_MakeEnvelope`** | Прямоугольник по двум углам: `(x_min, y_min, x_max, y_max, srid)`; удобно в `WHERE geom && ...` для использования индекса. |
| **`ST_Area` / `ST_Length`** | Площадь полигона и длина линии; в метрах — для `geography` или метровой проекции. |
| **`ST_MakeLine`** | Сборка линии из массива точек или из строк таблицы (агрегат с ORDER BY). |
| **GiST индекс** | Обобщённое дерево поиска; основной тип индекса для `geometry`/`geography` в PostGIS. |
| **SP-GiST индекс** | Вариант индекса для некоторых пространственных структур (например, quad-tree); иногда быстрее на определённых данных. |

Используй шпаргалку как карту: нашёл слово в первом столбце — во втором видишь, **зачем оно нужно**.
