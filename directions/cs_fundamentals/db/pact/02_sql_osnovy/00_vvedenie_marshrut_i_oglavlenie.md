## Часть II. SQL — основы

> Цель этой части — **научиться уверенно работать с SQL на уровне, достаточном для реального проекта**: создавать и изменять структуру таблиц (DDL), грамотно формулировать ограничения (constraints), вставлять, обновлять и удалять данные (DML), строить запросы на выборку (SELECT) и агрегировать данные с помощью GROUP BY и агрегатных функций.

После этой части ты сможешь:

- **создавать таблицы** с правильными типами данных и ограничениями — не «как попало», а с пониманием того, что именно ты объявляешь;
- **понимать, зачем нужны ограничения** (PRIMARY KEY, UNIQUE, NOT NULL, CHECK, FOREIGN KEY) и как СУБД их реализует;
- **вставлять, обновлять, удалять** данные в таблицах, понимать разницу между DELETE и TRUNCATE, пользоваться RETURNING и UPSERT;
- **писать SELECT-запросы** от простых до «фильтруй, сортируй, ограничивай, убирай дубли»;
- **агрегировать данные** с GROUP BY, HAVING, использовать стандартные агрегатные функции и понимать GROUPING SETS / ROLLUP / CUBE.

Если при чтении кажется, что «слишком много терминов» или «ничего не запоминается» — так бывает у всех. Материал разбит на шаги и повторяется разными словами; в каждом разделе есть блоки «Простыми словами» и «Запомните». Иди по оглавлению по этапам, после раздела отвечай на вопросы для самопроверки — так проще закрепить тему. Примеры в разделах стоит **пробовать в своей БД**, а не только читать.

### Краткая шпаргалка по части

| Понятие / конструкция | Суть (что это и когда нужно) |
|----------------------|-----------------------------|
| **DDL:** `CREATE TABLE` | Создать таблицу с колонками и типами; объявление новой таблицы в схеме |
| **DDL:** `ALTER TABLE` | Добавить/удалить/изменить колонку или ограничение; изменение схемы без потери данных |
| **DDL:** `DROP TABLE` | Удалить таблицу и все данные; когда структура больше не нужна |
| **DDL:** `TRUNCATE TABLE` | Очистить таблицу быстро (DDL, не DML); когда нужно удалить все строки без построчного перебора |
| **DDL:** временные таблицы | `TEMPORARY` / `TEMP` — таблица живёт в рамках сессии; для промежуточных расчётов |
| **Типы:** `INTEGER`, `BIGINT`, `NUMERIC` | Целые и точные дроби; для ID, счётчиков, денег (NUMERIC) |
| **Типы:** `REAL`, `DOUBLE PRECISION` | Плавающие числа (неточные); не для денег |
| **Типы:** `TEXT`, `VARCHAR(n)`, `CHAR(n)` | Строки; предпочтительно TEXT или VARCHAR(n) |
| **Типы:** `DATE`, `TIME`, `TIMESTAMP`, `TIMESTAMPTZ` | Дата и время; для моментов с поясом — TIMESTAMPTZ |
| **Типы:** `BOOLEAN` | TRUE / FALSE / NULL; флаги, переключатели |
| **Типы:** `UUID` | Уникальный идентификатор; для распределённой генерации id |
| **Типы:** `JSON`, `JSONB` | Документы; JSONB — бинарный, индексируемый |
| **Типы:** `ARRAY` | Массив значений одного типа; теги, списки |
| **Типы:** `ENUM` | Перечисление (фиксированный набор строк); статусы, коды |
| **Типы:** диапазоны (`int4range`, `tsrange`) | Интервал «от и до»; бронирования, периоды |
| **Ограничения:** `PRIMARY KEY` | Уникален + NOT NULL; один на таблицу; главный идентификатор строки |
| **Ограничения:** `UNIQUE` | Уникальность; может быть несколько; NULL не считается дубликатом |
| **Ограничения:** `NOT NULL` | Запрещает NULL в колонке |
| **Ограничения:** `CHECK` | Произвольное логическое условие на значение |
| **Ограничения:** `FOREIGN KEY` | Ссылочная целостность; каскады ON DELETE / ON UPDATE |
| **Ограничения:** `DEFERRABLE` | Проверка откладывается до конца транзакции; взаимные ссылки |
| **Ограничения:** `EXCLUDE` | Запрет перекрывающихся диапазонов (PostgreSQL); бронирования |
| **DML:** `INSERT INTO ... VALUES` | Вставить одну или несколько строк |
| **DML:** `INSERT ... ON CONFLICT` | UPSERT — вставить или обновить при конфликте по уникальному ключу |
| **DML:** `UPDATE ... SET ... WHERE` | Обновить строки по условию; всегда проверяй WHERE |
| **DML:** `DELETE FROM ... WHERE` | Удалить строки по условию; всегда проверяй WHERE |
| **DML:** `MERGE` | Слияние источника с целью (SQL:2003); массовый upsert из подзапроса |
| **DML:** `RETURNING` | Вернуть значения вставленных/обновлённых/удалённых строк |
| **DML:** `COPY` | Массовая загрузка из файла / в файл; быстрее множества INSERT |
| **SELECT:** базовая структура | FROM → WHERE → ORDER BY → LIMIT/OFFSET; выборка и фильтрация |
| **SELECT:** `ORDER BY ... NULLS FIRST/LAST` | Сортировка с управлением позицией NULL |
| **SELECT:** `LIMIT`, `OFFSET` | Ограничение и пропуск строк; для пагинации предпочтительнее keyset |
| **SELECT:** `DISTINCT`, `DISTINCT ON (...)` | Убрать дубликаты; оставить первую строку из каждой группы по выражению |
| **Агрегация:** `GROUP BY` | Сгруппировать строки; в SELECT только столбцы из GROUP BY или агрегаты |
| **Агрегация:** `HAVING` | Фильтр по результату агрегации (после группировки) |
| **Агрегация:** `COUNT`, `SUM`, `AVG`, `MIN`, `MAX` | Стандартные агрегатные функции |
| **Агрегация:** `FILTER (WHERE ...)` | Условный агрегат внутри одной группы |
| **Агрегация:** `GROUPING SETS`, `ROLLUP`, `CUBE` | Несколько уровней группировки за один запрос; итоги и суперитоги |
| **Агрегация:** `string_agg(col, sep)` | Склеить значения группы в одну строку через разделитель |
| **Последовательности:** `SEQUENCE`, `SERIAL`, `IDENTITY` | Автонумерация строк; IDENTITY — современный стандартный способ |

Используй таблицу как карту: нашёл конструкцию в первом столбце → во втором видишь суть и типичное применение. Для повторения после изучения раздела — сверяйся с этой шпаргалкой.

### Маршрут изучения в этой части

1. **DDL и типы данных (раздел 5)**  
   Научиться **создавать таблицы с правильными типами**: целые числа, строки, даты, JSON, UUID, массивы — чтобы данные хранились без потери точности и сюрпризов. Понять, чем отличается `ALTER TABLE` от `DROP` и `TRUNCATE`, и когда что применять. Разобраться с временными таблицами для промежуточных расчётов.

2. **Ограничения (раздел 6)**  
   Разобраться, что такое **ограничения целостности** и как СУБД сама не даёт записать некорректные данные. Ключевые темы: первичный ключ (одна «главная» идентификация строки), уникальность (UNIQUE), запрет пустых значений (NOT NULL), проверки (CHECK), внешние ключи с каскадами (что делать при удалении родителя), отложенная проверка (DEFERRABLE), исключающий оператор EXCLUDE для диапазонов.

3. **DML: изменение данных (раздел 7)**  
   Научиться **вставлять, обновлять и удалять** строки безопасно (всегда WHERE в UPDATE/DELETE, проверка через SELECT). Понять UPSERT (`ON CONFLICT` — вставить или обновить при конфликте), MERGE, зачем нужен `RETURNING`, как работают последовательности и IDENTITY. Разобраться с массовой загрузкой через COPY.

4. **SELECT: выборка данных (раздел 8)**  
   Научиться **строить запросы на чтение**: откуда брать данные (FROM), как фильтровать (WHERE и какие операторы использовать), как сортировать (ORDER BY) и ограничивать вывод (LIMIT/OFFSET), убирать дубли (DISTINCT / DISTINCT ON), давать псевдонимы и считать выражения в SELECT.

5. **Агрегация (раздел 9)**  
   Научиться **суммировать, считать, группировать** строки: GROUP BY, агрегатные функции (COUNT, SUM, AVG и т.д.), фильтр по группам (HAVING). Понять разницу между WHERE и HAVING, освоить условные агрегаты (FILTER), несколько уровней группировки (GROUPING SETS, ROLLUP, CUBE) и строковую агрегацию (string_agg).

### Как связаны темы в этой части (карта идей)

Главные идеи части в одном месте — чтобы привязывать к ним новые примеры по ходу чтения:

- **Типы данных** — точность (деньги → NUMERIC), корректность дат (момент → TIMESTAMPTZ), строки → TEXT, не CHAR.
- **Ограничения** — база сама не допускает некорректные данные: PK, UNIQUE, NOT NULL, CHECK, FK и каскады.
- **DML** — явный список столбцов где возможно; WHERE в UPDATE/DELETE обязателен; RETURNING возвращает изменённые строки.
- **SELECT** — порядок выполнения: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT; поэтому в WHERE нельзя псевдонимы из SELECT и агрегаты.
- **Агрегация** — GROUP BY группирует; в SELECT только столбцы из GROUP BY или агрегаты; фильтр по агрегату — HAVING.

**Логика по шагам:** разделы 5–6 — **структура** (таблицы, типы, ограничения); раздел 7 — **изменение данных** (INSERT, UPDATE, DELETE); разделы 8–9 — **чтение и подсчёт** (SELECT, агрегация). От «как устроена база» к «как с ней работать».
