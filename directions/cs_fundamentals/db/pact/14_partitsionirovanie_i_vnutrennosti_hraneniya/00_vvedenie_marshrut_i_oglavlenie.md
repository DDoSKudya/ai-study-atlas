## Часть XIV. Партиционирование и внутренности хранилища

> Цель этой части — **понять партиционирование таблиц и внутренности хранилища PostgreSQL**: как разбивать большие таблицы по партициям (RANGE, LIST, HASH), как планировщик отсекает партиции (partition pruning) и выполняет partition-wise операции; как устроены WAL, восстановление до точки во времени (PITR) и standby; зачем нужен VACUUM и autovacuum и как их настраивать; как работает репликация (streaming и логическая); как делать бэкапы и восстанавливаться.

После этой части ты сможешь:

- **проектировать и создавать партиционированные таблицы** — RANGE, LIST, HASH; default partition; ATTACH/DETACH; уникальность и FK; локальные индексы;
- **понимать partition pruning и partition-wise операции** — статическое и динамическое отсечение; подготовленные запросы; partition-wise join и aggregate; субпартиционирование;
- **настраивать WAL и PITR** — archive_command, restore_command; recovery_target и standby; восстановление до произвольного момента;
- **управлять VACUUM и autovacuum** — обычный VACUUM, VACUUM FULL, VACUUM FREEZE; пороги autovacuum; мониторинг (pg_stat_user_tables); не допускать bloat и transaction ID wraparound;
- **понимать репликацию PostgreSQL** — streaming (синхронная/асинхронная), слоты; логическая репликация (публикация/подписка), конфликты и replica identity; логическое декодирование;
- **делать бэкапы и восстановление** — pg_basebackup, pg_dump/pg_restore; проверка восстановления; понимать, что реплика не заменяет бэкап.

Материал опирается на **Часть V (хранение, WAL, страницы)** и **Часть IV (транзакции, MVCC)**. Материал сгруппирован по **этапам изучения**: партиционирование (§73) → partition pruning и partition-wise (§74) → WAL и PITR (§75) → VACUUM и autovacuum (§76) → репликация (§77) → бэкап и восстановление (§78). В каждом разделе один и тот же шаблон: **цель**, **термины**, **теория и правила**, **пошагово** (где есть), **простыми словами**, **картинка в голове / как запомнить**, **«Проверь себя»** (2–3 вопроса с ответами под спойлером) и **«Запомните»**. Иди по оглавлению по этапам — так удобнее изучать и возвращаться к нужному месту. В конце части — справочник, вопросы для самопроверки и резюме для закрепления.

### Краткая шпаргалка по части

| Понятие / конструкция | Суть (что это и когда нужно) |
|----------------------|-----------------------------|
| **Партиционирование** | Разбиение одной логической таблицы на физические партиции по ключу (диапазон, список, хеш) |
| **PARTITION BY RANGE** | Партиции по диапазону значений (например, по дате: до 2023, 2023–2024, 2024–2025) |
| **PARTITION BY LIST** | Партиции по списку значений (например, по региону: eu, us, asia) |
| **PARTITION BY HASH** | Партиции по остатку от деления хеша ключа на число партиций; равномерное распределение |
| **Default partition** | Партиция «по умолчанию» для строк, не попавших ни в одну явную партицию (только RANGE/LIST) |
| **ATTACH PARTITION / DETACH PARTITION** | Присоединить существующую таблицу как партицию; отсоединить партицию (например, для архивирования) |
| **Partition pruning** | Отсечение партиций по условию WHERE: планировщик не сканирует партиции, где данных заведомо нет |
| **Partition-wise join** | Соединение партиционированных таблиц по партициям: каждая пара партиций соединяется отдельно |
| **WAL segment** | Сегмент WAL (по умолчанию 16 MB); при заполнении создаётся следующий; LSN — позиция в WAL |
| **Структура записей WAL** | Заголовок (LSN, размер, resource manager) + тело; full page write — первая модификация страницы после checkpoint пишется в WAL целиком (защита от частичной записи); детали — Часть V |
| **archive_command** | Команда, вызываемая при переключении сегмента: копировать сегмент в архив (например, в S3) |
| **restore_command** | Команда при восстановлении: по имени сегмента достать файл из архива |
| **PITR (Point-in-Time Recovery)** | Восстановление до заданного момента времени: полный бэкап + применение WAL до recovery_target_time |
| **standby** | Реплика в режиме только чтение; получает WAL от primary; promote — переключение в primary |
| **VACUUM** | Удаление мёртвых строк, обновление visibility map и FSM; не возвращает место в ОС (без FULL) |
| **VACUUM FULL** | Перезапись таблицы с эксклюзивной блокировкой; возвращает место в ОС; долго и блокирует |
| **VACUUM FREEZE** | Заморозка старых версий строк; защита от transaction ID wraparound |
| **autovacuum** | Фоновый процесс: автоматический VACUUM и ANALYZE по порогам (n_dead_tup, scale_factor) |
| **Streaming replication** | Поток WAL от primary к replica в реальном времени; синхронная или асинхронная |
| **synchronous_standby_names / synchronous_commit** | synchronous_standby_names — какие реплики ждать; synchronous_commit — ждать ли сброс WAL и подтверждение реплик (on — ждать, off — быстрее, возможна потеря) |
| **Replication slot** | Слот сохраняет WAL для подписчика; при отстающей реплике WAL накапливается на primary |
| **Logical replication** | Публикация/подписка: репликация на уровне строк (таблицы/фильтры); CDC, мультимастер |
| **pg_basebackup** | Полная копия кластера (файлы данных + WAL); для создания реплики или полного бэкапа |
| **Инкрементальный бэкап (PostgreSQL)** | Нативно «только изменённые файлы» нет; инкремент = полный бэкап реже + непрерывный архив WAL между полными бэкапами (PITR до любого момента) |
| **pg_dump / pg_restore** | Логический дамп (таблицы, схема, данные); форматы plain, custom, directory; параллельный режим |

Используй таблицу как карту: нашёл понятие в первом столбце → во втором видишь суть и типичное применение.

### Как устроены разделы и оглавление

Материал в разделах приведён к **единому шаблону**, чтобы одну и ту же идею встретить несколько раз: в теории, пошагово (где есть), простыми словами, в образе и в блоке «Запомните». Так контент удобнее изучать.

| Блок | Назначение |
|------|------------|
| **Цель раздела** | Что именно ты освоишь после чтения (одним абзацем). |
| **В этом разделе главное** | Три ключевых тезиса раздела. |
| **Термины** | Ключевые слова с определениями в одном месте. |
| **Теория и правила** | Как устроено явление, какие правила, как пользоваться. |
| **Пошагово** (где есть) | Что происходит по шагам (с числами, где уместно). |
| **Простыми словами** | Та же идея без жаргона; аналогии. |
| **Картинка в голове / Как запомнить** | Образ или краткая формула для запоминания. |
| **Проверь себя** | 2–3 вопроса с ответами под спойлером (`<details><summary>Ответ</summary>...</details>`). |
| **Запомните** | Сжатый вывод раздела; якорь для повторения. |
| **Типичная ошибка** / **Что будет, если…** (где есть) | Граничные случаи и частые промахи. |

**Оглавление** сгруппировано **по этапам изучения** (Этап 1–6): партиционирование → partition pruning → WAL и PITR → VACUUM → репликация → бэкап. В таблицах под каждым этапом указаны ссылка на раздел, **содержание** и **ключевые понятия**. По ссылке переходишь в раздел; по столбцам видно, о чём раздел и какие термины освоить. В конце части — справочник, вопросы для самопроверки и резюме для повторения.
