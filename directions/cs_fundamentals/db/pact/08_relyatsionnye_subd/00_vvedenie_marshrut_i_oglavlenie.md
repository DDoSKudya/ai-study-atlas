## Часть VIII. Реляционные СУБД: продукты

> Цель этой части — **понять основные реляционные СУБД и как с ними работать**: архитектура PostgreSQL (процессы, память, расширения, системные каталоги), типы данных и партиционирование; движки и репликация MySQL/MariaDB; встраиваемая SQLite; обзор Oracle, SQL Server и распределённых NewSQL (CockroachDB, TiDB, YugabyteDB); подключение из приложения (драйверы, пулы, подготовленные выражения); ORM и миграции схемы.

После этой части ты сможешь:

- **понимать архитектуру PostgreSQL** — postmaster, backend-процессы, background workers, shared memory; расширения (CREATE EXTENSION); системные каталоги (pg_class, pg_attribute, pg_index) и полезные представления (pg_stat_*, pg_size_*);
- **использовать возможности PostgreSQL** — типы JSONB, массивы, диапазоны (int4range, tstzrange), UUID, сетевые (inet, cidr); партиционирование (RANGE, LIST, HASH); схемы; логическая репликация и слоты;
- **различать MySQL/MariaDB** — движки InnoDB (транзакционный) и MyISAM (табличные блокировки); binlog и репликация (GTID, group replication); ключевые настройки;
- **знать, когда и как использовать SQLite** — один файл, WAL mode, PRAGMA; ограничения (один писатель, нет сетевого доступа); сценарии (приложения, тесты, прототипы);
- **ориентироваться в других реляционных СУБД** — Oracle (RAC, PL/SQL), SQL Server (T-SQL, Always On), CockroachDB/TiDB/YugabyteDB (распределённый SQL); сравнение по лицензиям и сценариям;
- **подключаться к БД из приложения** — драйверы (psycopg2, asyncpg, JDBC и др.); пулы соединений и PgBouncer (transaction/session); подготовленные выражения и таймауты;
- **выбирать между ORM и raw SQL** — плюсы и минусы; миграции (Alembic, Flyway, Liquibase); connection string и failover в драйвере.

Материал опирается на **Часть VII (оптимизация запросов)** и готовит к **Части IX (NoSQL)** и **Части X (векторные БД)**. Контент сгруппирован по **этапам изучения**: от архитектуры PostgreSQL и типов данных к MySQL/MariaDB, SQLite, обзору других СУБД, подключению из приложения и ORM с миграциями. В конце части — справочник, вопросы для самопроверки (по 2–3 на раздел, ответы под спойлером) и резюме.

### Краткая шпаргалка по части

| Понятие / конструкция | Суть (что это и когда нужно) |
|----------------------|-----------------------------|
| **postmaster** | Главный процесс PostgreSQL: принимает подключения, порождает backend-процессы для каждого клиента |
| **shared memory** | Общая память процесса (shared_buffers, WAL buffers и др.); доступна всем backend-процессам |
| **pg_class** | Системный каталог: одна строка на таблицу, индекс, представление и т.д. (oid, relname, relkind) |
| **pg_attribute** | Системный каталог: атрибуты (столбцы) отношений; привязка к pg_class по attrelid |
| **CREATE EXTENSION** | Установка расширения (pgvector, PostGIS, pg_cron и др.); расширения добавляют типы, функции, операторы |
| **JSONB** | Бинарный JSON в PostgreSQL; индексируемый (GIN); операторы @>, ?, ->, ->> |
| **PARTITION BY RANGE/LIST/HASH** | Партиционирование таблицы по диапазону, списку значений или хешу ключа |
| **InnoDB** | Транзакционный движок хранения MySQL по умолчанию; поддержка FK, MVCC, binlog |
| **MyISAM** | Движок MySQL без транзакций; табличные блокировки; редко для новых проектов |
| **binlog** | Бинарный лог MySQL: последовательность изменений для репликации и PITR |
| **GTID** | Глобальный идентификатор транзакции в MySQL; однозначная идентификация события в репликации |
| **SQLite** | Встраиваемая СУБД: один файл или in-memory; один писатель; без сетевого сервера |
| **WAL (SQLite)** | Режим журнала: запись в отдельный WAL-файл; лучше параллелизм чтение/запись |
| **PRAGMA** | Директивы настройки SQLite (journal_mode, synchronous, cache_size и др.) |
| **PgBouncer** | Прокси пула соединений к PostgreSQL; режимы transaction/session/statement; снижение числа соединений к серверу |
| **Prepared statement** | Запрос, скомпилированный один раз; выполняется с разными параметрами; кэш плана, защита от SQL-инъекций |
| **ORM** | Object-Relational Mapping: объекты приложения ↔ таблицы; плюсы — безопасность, переносимость; минусы — сложные запросы, контроль |
| **Alembic / Flyway** | Инструменты миграций схемы: версионирование DDL, применение и откат миграций |

Используй таблицу как карту: нашёл понятие в первом столбце → во втором видишь суть и типичное применение.
