<summary>Краткие ответы</summary>

1. Когда нужно сохранить все строки «левой» таблицы и подставить данные справа там, где есть совпадение; при отсутствии совпадения справа — NULL.  
2. 50 строк (все пары).  
3. LATERAL позволяет подзапросу ссылаться на столбцы таблицы слева; выполняется по строкам слева.  
4. Потому что x = NULL даёт UNKNOWN; NOT IN с любым NULL в списке даёт UNKNOWN для сравнения с этим NULL, и строка отбрасывается.  
5. Коррелированный ссылается на внешний запрос; выполняется в контексте каждой строки. Заменяют на JOIN или оконную функцию.  
6. Нет; только в SELECT и ORDER BY. Фильтровать по результату — через подзапрос/CTE.  
7. SUM(col) OVER (PARTITION BY ... ORDER BY ...) — рамка по умолчанию «от начала до текущей строки».  
8. Базовый запрос (anchor) и рекурсивный запрос, ссылающийся на имя CTE.  
9. Чтобы рекурсия остановилась (условие выхода).  
10. СУБД должна вычислить и отбросить все строки до OFFSET — при большом OFFSET это медленно.  
11. WHERE key > last_seen_key ORDER BY key LIMIT n; передаётся последний ключ с предыдущей страницы.  
12. В SQL сравнение с NULL даёт UNKNOWN. WHERE оставляет только строки, где условие TRUE; UNKNOWN отбрасывается. Поэтому WHERE col = NULL не находит ни одной строки. Нужно WHERE col IS NULL.  
13. COUNT(*) считает все строки (включая с NULL). COUNT(col) считает только строки, где col не NULL.

## Резюме части III

В этой части рассмотрены **средний уровень SQL**: соединения таблиц, подзапросы, оконные функции, CTE (включая рекурсивные), операции над множествами и осознанная работа с NULL.

### Краткая сводка

- **JOIN:** INNER — только совпадения; LEFT — все слева + совпадения справа (NULL при отсутствии); CROSS — декартово произведение; self-join с разными алиасами; USING для одноимённых столбцов; LATERAL — подзапрос, зависящий от строки слева.
- **Подзапросы:** скалярный (одно значение) в SELECT/WHERE/HAVING; в FROM с алиасом; IN, EXISTS, ANY, ALL; коррелированные подзапросы; при возможности заменять на JOIN.
- **Оконные функции:** OVER(), PARTITION BY, ORDER BY, рамка ROWS/RANGE; ROW_NUMBER, RANK, DENSE_RANK; LAG/LEAD; агрегаты как окно (нарастающий итог); FIRST_VALUE/LAST_VALUE/NTH_VALUE. Строки не свёртываются.
- **CTE:** WITH name AS (SELECT ...); несколько CTE через запятую; WITH RECURSIVE для иерархий; MATERIALIZED/NOT MATERIALIZED в PostgreSQL.
- **Множества:** UNION (без дублей), UNION ALL; INTERSECT, EXCEPT. Одинаковое число и совместимые типы столбцов. Пагинация: предпочтительно keyset (WHERE key > last LIMIT n), а не большой OFFSET.
- **NULL:** трёхзначная логика (TRUE, FALSE, UNKNOWN); проверка только IS NULL/IS NOT NULL; COALESCE, NULLIF; IS DISTINCT FROM для сравнения с учётом NULL; NULL в ORDER BY (NULLS FIRST/LAST), в GROUP BY (одна группа), в агрегатах (COUNT(*) vs COUNT(col), SUM/AVG игнорируют NULL).

**Как повторить часть III за 5 минут (краткий чек-лист).**  
1. **JOIN:** INNER — только пары; LEFT — все слева, справа NULL если нет пары; анти-join — LEFT + WHERE right.key IS NULL.  
2. **Подзапросы:** скалярный — одна строка, один столбец; IN — в списке; EXISTS — есть строка; NOT IN с NULL опасен → NOT EXISTS.  
3. **Оконные:** OVER(), PARTITION BY (не свёртываем строки); ROW_NUMBER/RANK/DENSE_RANK; LAG/LEAD — предыдущая/следующая; нарастающий итог — SUM() OVER (PARTITION BY ... ORDER BY ...).  
4. **CTE:** WITH name AS (SELECT ...); рекурсивный — база + шаг, условие выхода обязательно.  
5. **Множества:** UNION без дублей, UNION ALL с дублями; INTERSECT — общие, EXCEPT — только в первом; keyset-пагинация вместо большого OFFSET.  
6. **NULL:** проверка только IS NULL/IS NOT NULL; COALESCE — первое не-NULL; NULLIF(a,b) — NULL если a=b; COUNT(*) считает все строки, COUNT(col) — только не-NULL.
