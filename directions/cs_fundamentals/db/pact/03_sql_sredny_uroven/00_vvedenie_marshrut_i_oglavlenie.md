## Часть III. SQL — средний уровень

> Цель этой части — **углубить владение SQL**: научиться соединять таблицы (JOIN) во всех вариантах, писать подзапросы и заменять их на JOIN где уместно, использовать оконные функции для расчётов «по окну» без свёртки строк, строить запросы через CTE (в том числе рекурсивные), работать с множествами (UNION, INTERSECT, EXCEPT) и осознанно учитывать NULL и трёхзначную логику.

После этой части ты сможешь:

- **соединять таблицы** осознанно: INNER, LEFT/RIGHT/FULL OUTER, CROSS, self-join, LATERAL, USING, составные ключи;
- **использовать подзапросы** — скалярные, в FROM, с IN/EXISTS/ANY/ALL, коррелированные — и знать, когда заменить их на JOIN;
- **применять оконные функции** — OVER, PARTITION BY, ORDER BY, рамка окна, ROW_NUMBER/RANK/DENSE_RANK, LAG/LEAD, FIRST_VALUE/LAST_VALUE/NTH_VALUE, агрегаты как окно;
- **писать запросы с CTE** — обычные и рекурсивные (иерархии), понимать MATERIALIZED/NOT MATERIALIZED;
- **объединять результаты** через UNION/UNION ALL, INTERSECT, EXCEPT и **пагинировать** корректно (OFFSET vs keyset);
- **работать с NULL** и трёхзначной логикой: IS NULL, COALESCE, NULLIF, IS DISTINCT FROM; NULL в сортировке и группировке.

Материал опирается на **Часть II (SQL — основы)**: предполагается, что ты уже умеешь писать SELECT с WHERE, ORDER BY, LIMIT, GROUP BY и агрегатами. Контент сгруппирован по этапам изучения: в каждом разделе — цель, термины, правила и примеры, граничные случаи и блок «Запомните»; после подразделов — вопросы для самопроверки с ответами под спойлером. Иди по оглавлению по этапам, переходи по ссылке в нужный раздел; в конце части — справочник, вопросы по всей части и резюме для повторения. Если при чтении кажется, что «слишком много терминов» — в начале раздела они расшифрованы; блоки «Простыми словами» и мини-примеры повторяют ту же идею. Примеры стоит пробовать у себя в БД.

### Краткая шпаргалка по части

| Понятие / конструкция | Суть (что это и когда нужно) |
|----------------------|-----------------------------|
| **INNER JOIN** | Только строки с совпадением в обеих таблицах по условию ON |
| **LEFT [OUTER] JOIN** | Все строки слева + совпадения справа; справа NULL при отсутствии совпадения |
| **RIGHT JOIN, FULL OUTER JOIN** | Аналогично LEFT — справа/обе стороны; NULL там, где нет пары |
| **CROSS JOIN** | Декартово произведение: каждая строка слева с каждой справа |
| **Self-join** | Таблица соединяется сама с собой по разным алиасам |
| **USING (col)** | JOIN по одному и тому же имени столбца: ON a.col = b.col короче как USING (col) |
| **LATERAL** | Подзапрос справа может ссылаться на столбцы слева; по строкам |
| **Скалярный подзапрос** | Подзапрос, возвращающий ровно одно значение (одна строка, один столбец) |
| **IN (subquery)** | Условие: значение входит в множество результатов подзапроса |
| **EXISTS (subquery)** | TRUE, если подзапрос вернул хотя бы одну строку; часто с корреляцией |
| **ANY / ALL** | Сравнение со множеством: = ANY (...), > ALL (...) |
| **Коррелированный подзапрос** | Подзапрос ссылается на столбцы внешнего запроса; выполняется для каждой строки |
| **Оконная функция** | Функция OVER (PARTITION BY ... ORDER BY ...): результат по «окну» без свёртки в одну строку |
| **ROW_NUMBER(), RANK(), DENSE_RANK()** | Нумерация/ранг строк в партиции; RANK с пропусками, DENSE_RANK без |
| **LAG(col), LEAD(col)** | Значение из предыдущей/следующей строки в порядке окна |
| **Рамка окна** | ROWS/RANGE BETWEEN ... — какие строки входят в окно для агрегата |
| **CTE (WITH)** | Именованный подзапрос в начале запроса; можно использовать несколько раз в запросе |
| **Рекурсивный CTE** | WITH RECURSIVE: базовый член + рекурсивный член для деревьев и графов |
| **UNION / UNION ALL** | Объединение результатов двух запросов; UNION убирает дубликаты |
| **INTERSECT, EXCEPT** | Пересечение и разность множеств строк |
| **Keyset-пагинация** | WHERE id > last_id ORDER BY id LIMIT n — стабильная производительность вместо OFFSET |
| **Трёхзначная логика** | TRUE, FALSE, UNKNOWN (NULL); WHERE отбрасывает FALSE и UNKNOWN |
| **COALESCE, NULLIF** | COALESCE — первое не-NULL; NULLIF(a,b) — NULL, если a = b |
| **IS DISTINCT FROM** | Сравнение с учётом NULL: NULL IS DISTINCT FROM NULL → FALSE |

### Частые сценарии: задача → решение

Таблица для быстрого поиска: по задаче находишь нужную конструкцию (терминология сохранена).

| Тема | Задача | Решение |
|------|--------|---------|
| **JOIN** | Только строки с парой в обеих таблицах | INNER JOIN … ON … |
| **JOIN** | Все строки одной таблицы + данные из другой (или NULL) | LEFT JOIN … ON … (главная таблица слева) |
| **JOIN** | Все комбинации строк двух таблиц без условия | CROSS JOIN |
| **JOIN** | Связать строки одной таблицы между собой (сотрудник — руководитель) | Self-join: одна таблица дважды с разными алиасами |
| **JOIN** | Найти «кого нет в списке» (пользователи без заказов) | LEFT JOIN … WHERE right.key IS NULL или NOT EXISTS |
| **Подзапросы** | Одно значение из другого запроса в выражении | Скалярный подзапрос (одна строка, один столбец) |
| **Подзапросы** | Проверить «значение входит в список» | IN (подзапрос) |
| **Подзапросы** | Проверить «есть хотя бы одна строка» | EXISTS (подзапрос) |
| **Подзапросы** | Проверить «нет ни одной строки» (безопасно при NULL) | NOT EXISTS, не NOT IN |
| **Оконные функции** | В каждой строке — результат по её группе без свёртки | … OVER (PARTITION BY …) |
| **Оконные функции** | Номер строки / ранг в группе | ROW_NUMBER() или RANK() или DENSE_RANK() OVER (PARTITION BY … ORDER BY …) |
| **Оконные функции** | Значение из предыдущей/следующей строки | LAG(col) / LEAD(col) OVER (PARTITION BY … ORDER BY …) |
| **Оконные функции** | Нарастающий итог по группе | SUM(col) OVER (PARTITION BY … ORDER BY …) |
| **CTE** | Именованный подзапрос для читаемости | WITH name AS (SELECT …) SELECT … |
| **CTE** | Обход дерева/иерархии | WITH RECURSIVE … базовый запрос UNION ALL рекурсивный запрос |
| **Множества** | Объединить два запроса в один (без дублей / с дублями) | UNION / UNION ALL |
| **Множества** | Только общие строки двух запросов | INTERSECT |
| **Множества** | Только из первого запроса минус то, что во втором | EXCEPT |
| **Множества** | Пагинация без тормозов на больших страницах | Keyset: WHERE id > last_id ORDER BY id LIMIT n |
| **NULL** | Проверить «значение отсутствует» | col IS NULL (никогда col = NULL) |
| **NULL** | Подставить значение, если NULL | COALESCE(col, 'значение по умолчанию') |
| **NULL** | Обнулить значение, если оно равно другому | NULLIF(a, b) |

### Как связаны темы в этой части (карта идей)

Главные идеи части в одном месте — чтобы привязывать к ним новые примеры по ходу чтения:

- **JOIN** — собираем данные из нескольких таблиц по условию (чаще всего по ключу); INNER — только пары, LEFT — все слева + справа или NULL; анти-join — «кого нет в списке» через LEFT + WHERE right.key IS NULL или NOT EXISTS.
- **Подзапросы** — запрос внутри запроса: скалярный (одно значение), в FROM (таблица с алиасом), с IN/EXISTS/ANY/ALL; коррелированный выполняется для каждой строки внешнего запроса; при возможности заменять на JOIN или оконную функцию.
- **Оконные функции** — считаем по «окну» (партиции) **без свёртки строк**: в каждой строке появляется новое значение (ранг, нарастающий итог, предыдущая/следующая); ORDER BY в OVER задаёт порядок в партиции; рамка ROWS/RANGE ограничивает, какие строки участвуют в агрегате.
- **CTE** — именованный подзапрос в начале запроса; несколько CTE подряд (цепочка); рекурсивный CTE — база + шаг из уже полученного, условие выхода обязательно; удобно для иерархий (дерево по parent_id).
- **Множества** — UNION (без дублей), UNION ALL (с дублями), INTERSECT (общие), EXCEPT (только в первом); одинаковое число и совместимые типы столбцов; пагинация: keyset (WHERE key > last LIMIT n) стабильнее большого OFFSET.
- **NULL** — трёхзначная логика (TRUE, FALSE, UNKNOWN); проверка только IS NULL / IS NOT NULL; COALESCE — первое не-NULL, NULLIF(a,b) — обнулить при a=b; IS DISTINCT FROM — сравнение с учётом NULL; в агрегатах COUNT(*) считает все строки, COUNT(col) — только не-NULL; SUM/AVG/MIN/MAX пропускают NULL.

**Логика по шагам:** раздел 10 — **соединения** (как склеивать таблицы); раздел 11 — **подзапросы** (запрос внутри запроса и когда заменить на JOIN); раздел 12 — **оконные функции** (считать по группе без потери строк); раздел 13 — **CTE** (имя подзапросу, рекурсия для деревьев); раздел 14 — **множества и пагинация** (объединение результатов, keyset вместо OFFSET); раздел 15 — **NULL** (трёхзначная логика и корректная работа с отсутствующими значениями).
