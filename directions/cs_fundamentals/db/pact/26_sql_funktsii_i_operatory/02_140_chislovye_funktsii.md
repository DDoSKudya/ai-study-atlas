## 140. Числовые функции

### Цель раздела

Научиться **контролировать поведение чисел в SQL**: как они округляются, как считать модули, степени, случайные значения и экстремумы так, чтобы не было сюрпризов в отчётах.

### В этом разделе главное

- Округление (`ROUND`, `FLOOR`, `CEIL`, `TRUNC`) может давать разные результаты — важно понимать, **когда мы хотим математическое округление, а когда усечение**.
- Функции `GREATEST` и `LEAST` позволяют компактно находить максимум/минимум из нескольких выражений.
- Случайные функции (`RANDOM`) полезны для **выборки примеров, тестовых данных и A/B‑разбиений**, но их детерминированность зависит от СУБД.

### Термины

- **Модуль числа** — расстояние от числа до нуля по числовой прямой (всегда неотрицательно).
- **Усечение** — отбрасывание дробной части без округления (3.9 → 3).
- **Округление до n знаков** — приведение числа к ближайшему значению с заданным количеством знаков после запятой.

### Теория и правила

- **ABS**:
  - `ABS(-5)` → `5`, `ABS(5)` → `5`.  
  - Удобно при работе с отклонениями и модулями ошибок.

- **ROUND**:
  - `ROUND(num)` — до целого, `ROUND(num, digits)` — до `digits` знаков после запятой.  
  - Для .5 конкретное поведение (банковское или классическое) зависит от СУБД и типа (`numeric` vs `float`).

- **FLOOR и CEIL**:
  - `FLOOR(3.7)` → `3`, `CEIL(3.2)` → `4`.  
  - Используются для «коридоров», разбиения на страницы, округления до «вниз/вверх».

- **TRUNC**:
  - `TRUNC(3.9)` → `3` (как `FLOOR` для положительных чисел).  
  - Для отрицательных важно: `TRUNC(-3.9)` → `-3`, а `FLOOR(-3.9)` → `-4`.  
  - Полезно, когда нужно просто отрезать дробную часть.

- **MOD**:
  - `MOD(a, b)` возвращает остаток от деления `a` на `b`.  
  - Частый паттерн: разбиение на группы, парность (`MOD(x, 2) = 0`).

- **POWER, SQRT**:
  - `POWER(2, 3)` → `8`, `SQRT(16)` → `4`.  
  - Используются в статистике, нормализации, расстояниях.

- **RANDOM**:
  - В PostgreSQL `RANDOM()` возвращает `double precision` в диапазоне `[0, 1)`.  
  - Для псевдослучайной сортировки: `ORDER BY RANDOM()` (осторожно — тяжело для больших таблиц).  
  - В MySQL — `RAND()`.

- **GREATEST, LEAST**:
  - `GREATEST(a, b, c)` — максимальное значение, `LEAST(a, b, c)` — минимальное.  
  - Работают и со строками (лексикографический порядок).

- **SIGN**:
  - `SIGN(-5)` → `-1`, `SIGN(0)` → `0`, `SIGN(5)` → `1`.  
  - Удобно для сводок «положительно / ноль / отрицательно».

### Разбор числовых функций по одной

Ниже — разбор основных числовых функций с примерами и типичными сценариями.

#### ABS — «взять модуль»

- **Что делает:** превращает отрицательное число в положительное, ноль и положительные остаются без изменений.
- **Синтаксис:**

  ```sql
  ABS(number)
  ```

- **Примеры:**

  ```sql
  SELECT ABS(-10);  -- 10
  SELECT ABS(10);   -- 10
  SELECT ABS(0);    -- 0
  ```

- **Когда использовать:**
  - считать **величину отклонения** без знака («насколько промахнулись»);
  - в отчётах по ошибкам/разнице план‑факт.

#### ROUND — «математическое округление»

- **Что делает:** округляет число до ближайшего значения.
- **Синтаксис:**

  ```sql
  ROUND(num)              -- до целого
  ROUND(num, digits)      -- до digits знаков после запятой
  ```

- **Примеры:**

  ```sql
  SELECT ROUND(3.4);        -- 3
  SELECT ROUND(3.5);        -- 4 (но детали зависят от СУБД и типа)
  SELECT ROUND(3.14159, 2); -- 3.14
  ```

- **Важный момент:** для `0.5` разные СУБД и типы (`numeric` vs `float`) могут использовать разные схемы: «от 0» или «к ближайшему чётному» (банковское округление). Всегда проверяй на своих данных.

#### FLOOR / CEIL — «всегда вниз / всегда вверх»

- **`FLOOR(x)`** — наибольшее целое, не превышающее `x` («вниз»).
- **`CEIL(x)` или `CEILING(x)`** — наименьшее целое, не меньшее `x` («вверх»).

Примеры:

```sql
SELECT FLOOR(3.9);   -- 3
SELECT CEIL(3.1);    -- 4
SELECT FLOOR(-1.2);  -- -2
SELECT CEIL(-1.2);   -- -1
```

**Практика:**
- разбиение на страницы: `(row_number - 1) / page_size + 1` с участием `FLOOR`;
- «коридоры» скидок/категорий с чёткими границами.

#### TRUNC — «просто отрезать хвост»

- **Что делает:** убирает дробную часть **без округления**.

```sql
SELECT TRUNC(3.9);   -- 3
SELECT TRUNC(-3.9);  -- -3
```

Отличие от `FLOOR` на отрицательных числах: `FLOOR(-3.9)` → `-4`, `TRUNC(-3.9)` → `-3`.

Применение: когда нужно **«отбросить копейки»**, а не округлять.

#### MOD — «остаток от деления»

- **Что делает:** возвращает остаток от деления `a` на `b`.

```sql
MOD(a, b)
```

Примеры:

```sql
SELECT MOD(10, 3);   -- 1
SELECT MOD(8, 2);    -- 0
```

Практические паттерны:

- разбить пользователей по группам:

  ```sql
  MOD(user_id, 4)  -- даёт 0..3, можно интерпретировать как id группы
  ```

- определить чётность:

  ```sql
  CASE WHEN MOD(value, 2) = 0 THEN 'even' ELSE 'odd' END
  ```

#### POWER / SQRT — степени и корень

- **`POWER(a, b)`** — возводит `a` в степень `b`.  
- **`SQRT(x)`** — квадратный корень из `x`.

Примеры:

```sql
SELECT POWER(2, 3);  -- 8
SELECT SQRT(16);     -- 4
```

Используются при:

- вычислении расстояний (например, евклидово расстояние);  
- статистических измерениях, стандартном отклонении и т.п.

#### RANDOM / RAND — «случайное число»

- В PostgreSQL: `RANDOM()` → число `double precision` в `[0, 1)`.  
- В MySQL: `RAND()`.

Примеры:

```sql
SELECT RANDOM();          -- 0.123456...
SELECT FLOOR(RANDOM()*6)+1 AS dice_roll;  -- бросок кубика 1..6
```

**Важные моменты:**

- `ORDER BY RANDOM()` на больших таблицах очень тяжёлый (БД должна присвоить случайное значение каждой строке, потом отсортировать). Используй осторожно и с `LIMIT`.
- Для воспроизводимого результата в аналитике лучше использовать псевдослучайность **на основе ключа** (например, `MOD(hash(user_id), 10)`), а не `RANDOM()`.

#### GREATEST / LEAST — «максимум/минимум из нескольких»

- **Что делают:** выбирают наибольшее (`GREATEST`) или наименьшее (`LEAST`) значение из списка.

```sql
SELECT GREATEST(10, 5, 20);  -- 20
SELECT LEAST(10, 5, 20);     -- 5
```

- Работают и со строками (лексикографический порядок):

```sql
SELECT GREATEST('apple', 'pear');  -- 'pear'
```

- **Поведение с `NULL`:**
  - В некоторых диалектах, если хотя бы один аргумент `NULL`, результат может становиться `NULL`.  
  - В других — `NULL` игнорируется.  
  Проверяй конкретную СУБД и по возможности **избегай `NULL` внутри GREATEST/LEAST** (предварительно оборачивай `COALESCE`).

#### SIGN — «знак числа»

- **Что делает:** возвращает `-1`, `0` или `1` в зависимости от знака.

```sql
SELECT SIGN(-10);  -- -1
SELECT SIGN(0);    -- 0
SELECT SIGN(10);   -- 1
```

Очень удобно, когда нужно **быстро понять направление** изменения метрики: «растёт, падает или стоит на месте».

### Примеры

**1. Округлить цену до двух знаков после запятой:**

```sql
SELECT
  price_raw,
  ROUND(price_raw, 2) AS price_rounded
FROM products;
```

**2. Посчитать «коридор» скидок по сумме заказа:**

```sql
SELECT
  order_id,
  total_amount,
  CASE
    WHEN total_amount < 1000 THEN '0%'
    WHEN total_amount < 5000 THEN '5%'
    ELSE '10%'
  END AS discount
FROM orders;
```

Здесь числовые функции могут использоваться, например, для округления `total_amount` перед сравнением.

**3. Разбить пользователей на 10 случайных групп (A/B‑варианты):**

```sql
SELECT
  user_id,
  (FLOOR(RANDOM() * 10) + 1)::int AS bucket
FROM users;
```

**4. Проверить, растёт ли метрика (используем `SIGN`):**

```sql
SELECT
  metric_name,
  value_today,
  value_yesterday,
  SIGN(value_today - value_yesterday) AS trend_sign
FROM metrics;
```

### Простыми словами

Числовые функции — это **линейка и калькулятор** в SQL: они помогают аккуратно измерять, округлять и сравнивать числа так, чтобы отчёты не отличались на «одну сотую» из‑за разных правил округления.

### Картинка в голове

Представь, что у тебя есть **измерительный прибор**, который показывает число с кучей знаков после запятой. Числовые функции — это набор кнопок: «округлить», «обрезать», «взять модуль», «посмотреть тренд». Ты нажимаешь нужные кнопки прямо в запросе.

### Как запомнить

**`FLOOR` — пол (вниз), `CEIL` — потолок (вверх), `ROUND` — по правилам математики, `TRUNC` — просто отрезать хвост.** Это четыре разных поведения с одной и той же дробью.

### Типичные ошибки

- Неосознанное использование `ROUND` вместо `TRUNC` (или наоборот), что даёт отличающиеся суммы в отчётах.
- Применение `ORDER BY RANDOM()` на больших таблицах в продакшене (может убить производительность).
- Непонимание разницы между `TRUNC(-3.9)` и `FLOOR(-3.9)`.

### Проверь себя

1. Чем `TRUNC(-3.9)` отличается от `FLOOR(-3.9)`?  
2. Как равномерно разложить пользователей по 4 группам с помощью `MOD`?  
3. В чём практический смысл функции `SIGN`?

<details>
<summary>Ответы</summary>

1. `TRUNC(-3.9)` даёт `-3` (отрезаем дробную часть), `FLOOR(-3.9)` — `-4` (наименьшее целое, не большее числа).  
2. Например: `MOD(user_id, 4)` даст значения `0..3`; можно интерпретировать как номер группы (`0` → группа 1, и т.д.).  
3. `SIGN` показывает направление изменения: положительное значение — рост, отрицательное — падение, 0 — без изменений; удобно для аналитики трендов.
</details>

### Запомните

Числовые функции делают работу с числами **предсказуемой**. Чётко выбирай: «мне нужно **обрезать** или **округлить**?» — и используй соответствующую функцию, чтобы потом не объяснять, почему отчёт «не сходится на копейки».
