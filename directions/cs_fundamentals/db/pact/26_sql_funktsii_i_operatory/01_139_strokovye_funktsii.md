## 139. Строковые функции

### Цель раздела

Научиться **нормализовать и анализировать текстовые данные в SQL**: очищать пробелы, менять регистр, вырезать подстроки, искать/заменять фрагменты, разбивать строки на части и использовать регулярные выражения.

### В этом разделе главное

- Строковые функции позволяют **подготовить данные к анализу**: привести к единому формату, вытащить нужные фрагменты, убрать лишнее.
- Во многих СУБД **поведение с `NULL` и юникодом (UTF‑8)** отличается: где‑то длина в байтах, где‑то — в символах.
- Регулярные выражения (`REGEXP_REPLACE`, `REGEXP_MATCH`) — мощный, но тяжёлый инструмент: они гибкие, но могут быть медленными.

### Термины

- **Подстрока** — часть исходной строки, заданная начальной позицией и длиной.
- **Разделитель (delimiter)** — символ/строка, по которой мы «режем» исходный текст (`','`, `';'`, `'|'`).
- **Регулярное выражение** — шаблон, описывающий набор строк (например, «любые цифры», «email‑адрес»).

### Теория и правила

- **Конкатенация**:
  - Стандартный способ — функция `CONCAT(str1, str2, ...)`.  
  - В PostgreSQL и ряде других СУБД есть оператор `||`: `'Hello ' || 'world'`.  
  - Важно: многие реализации `CONCAT` интерпретируют `NULL` как пустую строку, а оператор `||` может давать `NULL`, если один из операндов `NULL` — проверяй конкретную СУБД.

- **Извлечение подстрок**:
  - Обобщённая форма: `SUBSTRING(str FROM start FOR length)`.  
  - Индексация обычно **с 1**, не с 0.  
  - В некоторых диалектах есть форма `SUBSTRING(str, start, length)`.

- **Обрезка пробелов и символов**:
  - `TRIM([LEADING|TRAILING|BOTH] [chars] FROM str)` — базовая форма.  
  - Упрощённо: `TRIM(str)` — убирает пробелы слева и справа.  
  - `LTRIM` и `RTRIM` — только слева или справа.

- **Регистр и сравнения**:
  - `LOWER(str)` и `UPPER(str)` — приводят строку к нижнему/верхнему регистру.  
  - Чтобы сделать сравнение без учёта регистра, часто либо приводят обе стороны к одному регистру, либо используют специальные операторы (`ILIKE` в PostgreSQL).

- **Длина строки**:
  - В PostgreSQL `LENGTH` возвращает длину в байтах, `CHAR_LENGTH` — в символах. Для ASCII они совпадают, для UTF‑8 — нет.  
  - В MySQL `CHAR_LENGTH` — в символах, `LENGTH` — в байтах.

- **Поиск в строке**:
  - `POSITION(substr IN str)` или `STRPOS(str, substr)` (PostgreSQL) возвращают позицию подстроки.  
  - При отсутствии совпадения часто возвращается 0 (а не `NULL`) — это важно для условий.

- **Замена и разбиение**:
  - `REPLACE(str, old, new)` меняет **все вхождения** `old` на `new`.  
  - `SPLIT_PART(str, delimiter, field_num)` (PostgreSQL) возвращает n‑й фрагмент строки, разделённой `delimiter`.

- **Регулярные выражения**:
  - `REGEXP_REPLACE(str, pattern, replacement, flags)` — замена по шаблону.  
  - `REGEXP_MATCH` / `REGEXP_MATCHES` — поиск совпадения, обычно возвращают массив или набор строк.  
  - Частые флаги: `'i'` — без учёта регистра, `'g'` — глобально (все совпадения).

### Разбор основных функций по одной

Ниже — **разбор каждой ключевой функции**. Читай последовательно и обязательно пробуй примеры руками.

#### CONCAT и оператор `||` — «склеить строки»

- **Что делает:** соединяет несколько строк в одну.
- **Синтаксис (общее):**

  ```sql
  CONCAT(str1, str2, str3, ...)
  ```

- **Примеры:**

  ```sql
  SELECT CONCAT('Hello', ' ', 'world');         -- Hello world
  SELECT CONCAT(first_name, ' ', last_name)    -- Иван Петров
  FROM users;
  ```

- **Оператор `||` (PostgreSQL, Oracle и др.):**

  ```sql
  SELECT 'Hello ' || 'world';                  -- Hello world
  ```

- **Важный момент — `NULL`:**
  - Во многих СУБД **`CONCAT` ведёт себя «добрее»**: если один из аргументов `NULL`, он считается пустой строкой.  
    `CONCAT('Hello', NULL, 'world')` → `'Helloworld'`.
  - Оператор `||` часто даёт `NULL`, если одна из частей `NULL`:  
    `'Hello ' || NULL` → `NULL`.
- **Когда использовать:**
  - Формировать «человеческие» подписи: `Фамилия Имя`, «город, улица».  
  - Строить ключи для отчётов: `year || '-' || month`.

#### SUBSTRING — «вырезать кусочек»

- **Что делает:** возвращает часть строки.
- **Синтаксис (ANSI‑вариант):**

  ```sql
  SUBSTRING(str FROM start FOR length)
  ```

- **Синтаксис (альтернативный, например, в PostgreSQL/MySQL):**

  ```sql
  SUBSTRING(str, start, length)
  ```

- **Аргументы:**
  - `start` — позиция **с 1** (первый символ — это 1).  
  - `length` — сколько символов взять.

- **Примеры:**

  ```sql
  SELECT SUBSTRING('abcdef' FROM 2 FOR 3);   -- 'bcd'
  SELECT SUBSTRING('abcdef', 2, 3);         -- 'bcd'
  ```

- **Типичные задачи:**
  - Взять код региона из телефона.  
  - Вытащить первые N символов для проверки формата.

#### TRIM / LTRIM / RTRIM — «почистить края»

- **Что делают:** убирают пробелы (или другие символы) слева/справа/с обеих сторон.
- **Самый частый случай:**

  ```sql
  TRIM(str)          -- убирает пробелы слева и справа
  LTRIM(str)         -- только слева
  RTRIM(str)         -- только справа
  ```

- **Пример:**

  ```sql
  SELECT TRIM('   hello   ');   -- 'hello'
  SELECT LTRIM('   hello');     -- 'hello'
  SELECT RTRIM('hello   ');     -- 'hello'
  ```

- **С указанием символов:**

  ```sql
  TRIM(BOTH '.' FROM '...abc...');   -- 'abc'
  ```

- **Практический смысл:** входные данные от пользователей почти всегда с «мусором» по краям — лишние пробелы ломают сравнения и уникальность.

#### LOWER / UPPER — «выравнять регистр»

- **Что делают:** переводят строку полностью в нижний (`LOWER`) или верхний (`UPPER`) регистр.
- **Примеры:**

  ```sql
  SELECT LOWER('HeLLo');  -- 'hello'
  SELECT UPPER('привет'); -- 'ПРИВЕТ'
  ```

- **Типичный паттерн для сравнения без учёта регистра:**

  ```sql
  WHERE LOWER(email) = LOWER(:input_email)
  ```

- **Альтернатива (PostgreSQL):** оператор `ILIKE` (`LIKE` без учёта регистра), см. раздел про операторы.

#### LENGTH / CHAR_LENGTH — «сколько символов / байт»

- **`LENGTH(str)`** — длина в **байтах**.  
- **`CHAR_LENGTH(str)`** (или `CHARACTER_LENGTH`) — длина в **символах**.

**Почему это важно:** в UTF‑8 один «человеческий» символ может занимать 2–4 байта.  
Например, буква `ё` — это один символ, но несколько байт.

```sql
SELECT
  LENGTH('A'),        -- 1
  CHAR_LENGTH('A');   -- 1

SELECT
  LENGTH('Ё'),        -- может быть 2
  CHAR_LENGTH('Ё');   -- 1
```

Если тебе важны **лимиты по символам для пользователя** («не более 50 символов в имени»), используй `CHAR_LENGTH`.

#### POSITION / STRPOS — «найти, где подстрока»

- **`POSITION(substr IN str)`** — стандартный синтаксис.
- **`STRPOS(str, substr)`** — форма PostgreSQL.
- **Возвращают:** номер первого вхождения (с 1). Если нет — 0 или `NULL` (зависит от СУБД).

Пример (PostgreSQL):

```sql
SELECT POSITION('@' IN 'user@example.com');   -- 5
SELECT STRPOS('user@example.com', '@');      -- 5
```

Типичная задача: проверить, есть ли разделитель, перед тем как делать `SPLIT_PART` или `SUBSTRING`.

#### REPLACE — «заменить все вхождения»

- **Что делает:** заменяет все вхождения подстроки `old` на `new`.
- **Синтаксис:**

  ```sql
  REPLACE(str, old, new)
  ```

- **Пример:**

  ```sql
  SELECT REPLACE('2024-01-05', '-', '.');   -- '2024.01.05'
  ```

- **Важно:** заменяются **все** вхождения; если нужно контролировать только первое или использовать шаблон — смотри `REGEXP_REPLACE`.

#### SPLIT_PART — «разбить по разделителю и взять кусок» (PostgreSQL)

- **Что делает:** разбивает строку по разделителю и возвращает N‑й фрагмент.
- **Синтаксис (PostgreSQL):**

  ```sql
  SPLIT_PART(str, delimiter, field_num)
  ```

- **Примеры:**

  ```sql
  SELECT SPLIT_PART('a,b,c', ',', 1);  -- 'a'
  SELECT SPLIT_PART('a,b,c', ',', 3);  -- 'c'
  ```

- **Если фрагмента нет** (например, просим 5‑й элемент, а их только 3) — обычно возвращается пустая строка.

Типичное применение — разбить `full_name`, `path`, `email`, `url`.

#### REGEXP_REPLACE / REGEXP_MATCH — «регулярки»

- **`REGEXP_REPLACE(str, pattern, replacement, flags)`**:
  - `pattern` — шаблон (регулярное выражение).
  - `replacement` — на что заменить.
  - `flags` — дополнительные флаги (`'g'` — все совпадения, `'i'` — без регистра).

- **Пример — оставить только цифры:**

  ```sql
  SELECT REGEXP_REPLACE('+7 (999) 123-45-67', '[^0-9]', '', 'g');
  -- '79991234567'
  ```

- **`REGEXP_MATCH` / `REGEXP_MATCHES`**:
  - Позволяют проверить, **соответствует ли строка шаблону**, и/или забрать группы.

Пример (PostgreSQL, проверить формат email очень упрощённо):

```sql
SELECT REGEXP_MATCH('user@example.com', '^[^@]+@[^@]+$');
-- вернётся массив с совпадением или NULL
```

**Важно:** сложные регулярки тяжёлые для БД — используй их точечно.

#### LPAD / RPAD — «добить строку до длины»

- **Что делают:** добавляют символы слева (`LPAD`) или справа (`RPAD`), чтобы довести строку до нужной длины.
- **Синтаксис (PostgreSQL, MySQL и др.):**

```sql
LPAD(str, length, pad_str)
RPAD(str, length, pad_str)
```

- **Примеры:**

```sql
SELECT LPAD('7', 3, '0');      -- '007'
SELECT RPAD('sql', 5, '_');    -- 'sql__'
```

Нередко применяют для форматирования кодов, номеров, выравнивания в текстовых отчётах.

### Примеры

**1. Нормализация email‑адресов (обрезать пробелы и привести к нижнему регистру):**

```sql
SELECT
  email_raw,
  LOWER(TRIM(email_raw)) AS email_normalized
FROM users;
```

**2. Вырезать домен из email (`user@example.com` → `example.com`):**

```sql
SELECT
  email,
  SPLIT_PART(email, '@', 2) AS domain
FROM users;
```

**3. Заменить все пробелы на подчёркивания:**

```sql
SELECT REPLACE('hello world from sql', ' ', '_');
-- hello_world_from_sql
```

**4. Оставить в строке только цифры (регулярное выражение):**

```sql
SELECT REGEXP_REPLACE('Тел:+7 (999) 123-45-67', '[^0-9]', '', 'g') AS digits;
-- 79991234567
```

### Простыми словами

Строковые функции — это **ножницы, ластик и клей** для текста в БД: ты можешь обрезать лишнее, вырезать нужное, заменить куски и склеить в том формате, который удобно анализировать.

### Картинка в голове

Представь ленту текста, которая проходит через **несколько станций обработки**: первая убирает лишние пробелы, вторая режет по разделителю на блоки, третья берёт нужный блок, четвёртая меняет в нём пару символов. Строковые функции — это как конвейер таких станций, собранный прямо в `SELECT`.

### Как запомнить

**«CONCAT склеивает, SUBSTRING режет, TRIM чистит, REGEXP даёт власть над шаблонами»** — четыре глагола, которые описывают основные действия над строками.

### Типичные ошибки

- Считать, что **индексация начинается с 0** в `SUBSTRING` — в SQL почти всегда с 1.
- Путать **длину в байтах и в символах** при работе с UTF‑8.
- Забывать, что `REPLACE` меняет **все** вхождения, а не только первое (для этого — регулярные выражения с контролем).
- Писать сложные регулярки без ограничений — это может сильно замедлить запрос.

### Проверь себя

1. Чем `LENGTH` отличается от `CHAR_LENGTH` в контексте UTF‑8?  
2. Как вырезать **только фамилию** из строки `Фамилия Имя Отчество`, если части разделены пробелами?  
3. Как с помощью регулярного выражения оставить в строке только буквы латиницы и кириллицы?

<details>
<summary>Ответы</summary>

1. `LENGTH` обычно возвращает количество **байт**, а `CHAR_LENGTH` — количество **символов**. В UTF‑8 один символ может занимать несколько байт, поэтому для не‑ASCII строк `LENGTH` ≥ `CHAR_LENGTH`.  
2. Можно использовать `SPLIT_PART(full_name, ' ', 1)` — это первая часть до первого пробела.  
3. Пример для PostgreSQL: `REGEXP_REPLACE(str, '[^A-Za-zА-Яа-яЁё]', '', 'g')` — заменяем всё, что **не** буква латиницы или кириллицы, на пустую строку.
</details>

### Запомните

Строковые функции — это первый инструмент, который ты берёшь, когда видишь «грязный» текст в колонке. **Не бойся собирать из них конвейер**: несколько функций подряд превращают хаотичный ввод пользователя в аккуратные данные для отчётов и аналитики.
