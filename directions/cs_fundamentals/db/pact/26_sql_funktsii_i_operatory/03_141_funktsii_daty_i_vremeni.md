## 141. Функции даты и времени

### Цель раздела

Научиться **корректно считать интервалы, группировать по времени и работать с таймзонами**: сколько прошло дней/месяцев, в каком месяце/квартале событие, как правильно хранить и отображать время.

### В этом разделе главное

- Время в БД — это не только `DATE`, но и `TIMESTAMP` + `INTERVAL` + таймзоны; каждая часть важна.
- `EXTRACT` и `DATE_TRUNC` — твои главные инструменты для **аналитики по времени** (по дням, неделям, месяцам).
- Арифметика с `INTERVAL` должна быть явной: всегда понимай, что ты прибавляешь (`'1 day'`, `'1 month'`) и в какой таймзоне считаешь.

### Термины

- **Таймзона** — смещение от UTC + правила перехода на летнее/зимнее время.
- **Интервал** — длительность (например, «3 дня», «2 часа 30 минут»), тип `INTERVAL`.
- **Точка во времени** — конкретный момент (`TIMESTAMP` с/без таймзоны).

### Теория и правила

- **Текущее время**:
  - `NOW()` / `CURRENT_TIMESTAMP` — текущий момент времени (обычно `timestamp with time zone`).  
  - `CURRENT_DATE` — только дата (без времени).  
  - `CURRENT_TIME` — только время.

- **Извлечение частей даты/времени (`EXTRACT`)**:
  - `EXTRACT(YEAR FROM ts)` → год, `EXTRACT(MONTH FROM ts)` → месяц, и т.д.  
  - Частый приём: `EXTRACT(DOW FROM ts)` для дня недели (где 0/1 — воскресенье/понедельник — зависит от СУБД).

- **Округление даты/времени (`DATE_TRUNC`)**:
  - `DATE_TRUNC('month', ts)` → первый день месяца, время обнуляется.  
  - Используется для группировки: `GROUP BY DATE_TRUNC('day', created_at)`.

- **Разница во времени (`AGE` и вычитание)**:
  - В PostgreSQL `AGE(ts1, ts2)` возвращает `INTERVAL` (например, «2 years 3 mons 5 days»).  
  - Простое вычитание `ts1 - ts2` тоже даёт интервал.

- **Арифметика с `INTERVAL`**:
  - `ts + INTERVAL '1 day'`, `ts - INTERVAL '2 hours'`.  
  - Важно: `INTERVAL '1 month'` не всегда = 30 дней (зависит от конкретного месяца).

- **Таймзоны (`AT TIME ZONE`)**:
  - Конструкция `ts AT TIME ZONE 'UTC'` приводит значение к указанной таймзоне.  
  - Частый паттерн: хранить в БД в UTC, а при показе пользователю переводить в его зону.

- **Преобразование строк в даты/время**:
  - `TO_TIMESTAMP(str, format)` — строка → `timestamp`.  
  - `TO_DATE(str, format)` — строка → `date`.  
  - Форматные строки вроде `'YYYY-MM-DD HH24:MI:SS'`.

### Разбор функций даты и времени по одной

#### NOW / CURRENT_TIMESTAMP / CURRENT_DATE / CURRENT_TIME

- **`NOW()` / `CURRENT_TIMESTAMP`** — текущий момент времени (обычно `timestamp with time zone`).  
- **`CURRENT_DATE`** — только дата (`date`), время обнуляется.  
- **`CURRENT_TIME`** — только время.

Примеры:

```sql
SELECT NOW();              -- 2026-02-24 12:34:56.789+03
SELECT CURRENT_TIMESTAMP;  -- то же самое
SELECT CURRENT_DATE;       -- 2026-02-24
SELECT CURRENT_TIME;       -- 12:34:56.789+03
```

**Когда использовать:**

- помечать записи временем создания/обновления;
- фильтровать по «сейчас минус N дней/часов»;
- строить отчёты «за сегодня», «за прошлую неделю» и т.п.

#### EXTRACT — «достать кусок времени»

- **Что делает:** извлекает часть даты/времени — год, месяц, день, час, день недели и т.д.
- **Синтаксис:**

```sql
EXTRACT(field FROM source)
```

где `field` — `'year'`, `'month'`, `'day'`, `'dow'` (day of week), `'hour'`, `'minute'`, `'second'`, `'epoch'` и др.

Примеры:

```sql
SELECT
  EXTRACT(YEAR  FROM TIMESTAMP '2024-05-10 15:20:00') AS y,   -- 2024
  EXTRACT(MONTH FROM TIMESTAMP '2024-05-10 15:20:00') AS m,   -- 5
  EXTRACT(DAY   FROM TIMESTAMP '2024-05-10 15:20:00') AS d;   -- 10
```

Используется для:

- построения отчётов «по годам/месяцам/дням»;
- фильтрации по дню недели (`EXTRACT(DOW FROM ts)`).

#### DATE_TRUNC — «обрубить до нужной границы»

- **Что делает:** обрезает дату/время до указанной единицы (дня, месяца, часа и т.д.).
- **Синтаксис (PostgreSQL):**

```sql
DATE_TRUNC('unit', timestamp)
```

- **Примеры:**

```sql
SELECT DATE_TRUNC('day',   TIMESTAMP '2024-05-10 15:20:00');  -- 2024-05-10 00:00:00
SELECT DATE_TRUNC('month', TIMESTAMP '2024-05-10 15:20:00');  -- 2024-05-01 00:00:00
SELECT DATE_TRUNC('hour',  TIMESTAMP '2024-05-10 15:20:00');  -- 2024-05-10 15:00:00
```

Главное применение — **группировка по времени**:

```sql
SELECT
  DATE_TRUNC('day', created_at) AS day,
  COUNT(*) AS cnt
FROM events
GROUP BY DATE_TRUNC('day', created_at)
ORDER BY day;
```

#### AGE / вычитание timestamp — «сколько времени прошло»

- В PostgreSQL `AGE(ts1, ts2)` возвращает интервал между двумя моментами в «человеческом» виде (годы, месяцы, дни).  
- Простое вычитание `ts1 - ts2` тоже даёт `INTERVAL`, но формат и детали могут отличаться.

Примеры:

```sql
SELECT AGE(TIMESTAMP '2020-01-01', TIMESTAMP '2010-01-01');
-- "10 years"

SELECT NOW() - INTERVAL '1 day';    -- время сутки назад
```

Типичная задача — посчитать возраст пользователя по дате рождения:

```sql
SELECT
  user_id,
  AGE(CURRENT_DATE, birth_date) AS age
FROM users;
```

#### INTERVAL — «длительность»

- Тип `INTERVAL` описывает **промежуток времени**, а не конкретный момент.
- Примеры литералов:

```sql
INTERVAL '1 day'
INTERVAL '2 hours'
INTERVAL '3 days 4 hours 30 minutes'
INTERVAL '1 month'
```

- Можно складывать/вычитать с `timestamp`:

```sql
SELECT NOW() + INTERVAL '3 days';    -- через 3 дня
SELECT NOW() - INTERVAL '1 hour';    -- час назад
```

Важно помнить, что **месяц ≠ всегда 30 дней**: `INTERVAL '1 month'` прибавляет ровно один календарный месяц, а не 30 суток.

#### AT TIME ZONE — «перевести время в другой пояс»

- **Что делает:** интерпретирует/переводит `timestamp` в указанную таймзону.
- Частый паттерн: **хранить всё в UTC**, а показывать пользователю в его таймзоне.

Пример (PostgreSQL):

```sql
SELECT
  occurred_at,                               -- timestamp with time zone (UTC)
  occurred_at AT TIME ZONE 'Europe/Moscow'   AS occurred_at_msk
FROM events;
```

Так ты явно видишь, в какой зоне считаешь время и избегаешь «магии» с летним/зимним временем.

#### TO_TIMESTAMP / TO_DATE — «строка → время/дата»

- **`TO_DATE(str, format)`** — превращает строку в `date`.
- **`TO_TIMESTAMP(str, format)`** — превращает строку в `timestamp`.

Форматная строка описывает, как устроена дата в тексте.

Примеры:

```sql
SELECT TO_DATE('31.12.2024', 'DD.MM.YYYY');              -- 2024-12-31
SELECT TO_TIMESTAMP('2024-12-31 23:59:59',
                    'YYYY-MM-DD HH24:MI:SS');
```

Если формат не совпадает (например, `DD` и `MM` местами перепутаны), можно получить либо ошибку, либо некорректный результат — поэтому **формат всегда указывай явно и проверяй на тестовых значениях**.

### Примеры

**1. Посчитать количество заказов по дням:**

```sql
SELECT
  DATE_TRUNC('day', created_at) AS day,
  COUNT(*) AS orders_count
FROM orders
GROUP BY DATE_TRUNC('day', created_at)
ORDER BY day;
```

**2. Найти всех пользователей старше 18 лет по дате рождения:**

```sql
SELECT *
FROM users
WHERE AGE(CURRENT_DATE, birth_date) >= INTERVAL '18 years';
```

**3. Перевести время события из UTC в локальную таймзону:**

```sql
SELECT
  event_id,
  occurred_at AT TIME ZONE 'UTC' AT TIME ZONE 'Europe/Moscow' AS occurred_at_msk
FROM events;
```

**4. Разобрать строку даты нестандартного формата:**

```sql
SELECT
  raw_date,
  TO_DATE(raw_date, 'DD.MM.YYYY') AS parsed_date
FROM uploads;
```

### Простыми словами

Функции даты и времени — это **часы и календарь** внутри SQL: они позволяют спросить у БД «когда это было?», «сколько времени прошло?» и «в какой день/месяц/квартал это попадает» без ручных вычислений в коде.

### Картинка в голове

Представь, что у каждой записи есть **маленький календарь** и **часы**. Функции даты/времени — это как линейка и лупа, которыми ты измеряешь расстояние между датами, разрезаешь время на отрезки и перекладываешь его между часовыми поясами.

### Как запомнить

**`EXTRACT` — достать кусочек времени, `DATE_TRUNC` — обрубить лишнее, `AGE` — посчитать разницу, `AT TIME ZONE` — «переселить» время в другой часовой пояс.**

### Типичные ошибки

- Хранить локальное время без явной таймзоны и потом пытаться корректно считать интервалы.
- Использовать `DATE_TRUNC('month', ...)` и ожидать одинаковое количество дней в каждом месяце.
- Сравнивать `timestamp with time zone` и `timestamp without time zone` без понимания неявных преобразований.

### Проверь себя

1. Как сгруппировать лог‑записи по неделям?  
2. Чем `AGE(ts1, ts2)` отличается от простого `ts1 - ts2`?  
3. Почему часто рекомендуют хранить время в БД в UTC?

<details>
<summary>Ответы</summary>

1. Можно использовать `DATE_TRUNC('week', ts)` и группировать по результату: `GROUP BY DATE_TRUNC('week', ts)`.  
2. В PostgreSQL `AGE` выдаёт интервал в «человеческих» единицах (годы, месяцы, дни), а простое вычитание — в днях/секундах; в простых сценариях обе формы дают интервал, но формат и детали могут отличаться.  
3. UTC не зависит от локальных правил перехода на летнее/зимнее время, что упрощает вычисления и сравнения; локальное время можно всегда получить из UTC через `AT TIME ZONE`.
</details>

### Запомните

Корректная работа с датой и временем — это **одна из самых частых причин багов**. Чем раньше ты привыкнешь осознанно использовать `EXTRACT`, `DATE_TRUNC`, `INTERVAL` и таймзоны, тем меньше сюрпризов будет в отчётах и бизнес‑логике.
