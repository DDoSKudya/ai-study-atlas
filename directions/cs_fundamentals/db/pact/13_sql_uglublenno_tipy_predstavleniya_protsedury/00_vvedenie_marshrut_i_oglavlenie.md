## Часть XIII. SQL углублённо: типы, представления, процедуры

> Цель этой части — **углубить владение SQL и объектами схемы**: точность числовых типов и коллации; дата/время и таймзоны; JSON/JSONB, массивы и диапазоны; полнотекстовый поиск; представления (обычные и материализованные); триггеры; хранимые процедуры и функции; последовательности и IDENTITY.

---

После этой части ты сможешь:

- **выбирать и настраивать типы данных** — DECIMAL для денег, коллации для сортировки строк, таймзоны для дат; понимать ENUM, домены и составные типы;
- **работать с полуструктурированными данными** — JSON/JSONB (операторы, индексы GIN, jsonpath), массивы и диапазоны, XML при необходимости;
- **строить полнотекстовый поиск** — tsvector, tsquery, словари, ранжирование и подсветка; триграммы для нечёткого поиска;
- **использовать представления** — обычные (VIEW) и материализованные (REFRESH CONCURRENTLY), WITH CHECK OPTION; понимать разницу правил и триггеров;
- **писать триггеры** — BEFORE/AFTER, FOR EACH ROW/STATEMENT, INSTEAD OF для представлений; триггерные функции;
- **создавать процедуры и функции** — PL/pgSQL, volatile/stable/immutable, SECURITY DEFINER; курсоры и динамический SQL;
- **управлять последовательностями** — nextval/currval/setval, IDENTITY, понимать разрывы в нумерации.

Материал опирается на **Часть II (SQL основы)** и **Часть VIII (реляционные СУБД)** и готовит к **Части XIV (партиционирование, WAL, репликация)**. Контент сгруппирован по **этапам изучения**: от типов данных к JSON/массивам/диапазонам, полнотексту, представлениям, триггерам, процедурам и последовательностям.

---

### Краткая шпаргалка по части

| Понятие / конструкция | Суть (что это и когда нужно) |
|----------------------|-----------------------------|
| **DECIMAL(p,s)** | Точный числовой тип: p цифр, s после запятой; для денег и расчётов без погрешности float |
| **COLLATE** | Правило сортировки строк (locale); case-sensitive vs insensitive; lc_collate, lc_ctype |
| **TIMESTAMP WITH TIME ZONE** | Дата/время с таймзоной; хранение в UTC; вывод в локальной зоне; AT TIME ZONE, extract() |
| **INTERVAL** | Длительность (дни, часы, минуты); арифметика с датами; extract(), date_trunc() |
| **JSONB** | Бинарный JSON в PostgreSQL; операторы @>, ?, ->, ->>, #>, #>>; индексы GIN; jsonpath |
| **Массивы** | ARRAY[]; операторы &&, @>, <@, []; unnest(); индексы GIN |
| **Диапазоны** | int4range, tstzrange; операторы &&, @>, <@; индексы GiST |
| **tsvector / tsquery** | Полнотекст: лексемы с позициями; запрос с AND/OR/NOT; конфигурации (simple, english) |
| **VIEW** | Виртуальная таблица — запрос с именем; WITH CHECK OPTION для обновляемых представлений |
| **MATERIALIZED VIEW** | Результат запроса хранится на диске; REFRESH / REFRESH CONCURRENTLY |
| **Триггер** | Код при INSERT/UPDATE/DELETE; BEFORE/AFTER; FOR EACH ROW/STATEMENT; INSTEAD OF для VIEW |
| **PL/pgSQL** | Язык хранимых процедур/функций; VOLATILE/STABLE/IMMUTABLE; SECURITY DEFINER |
| **SEQUENCE** | Объект счётчика; nextval(), currval(), setval(); CACHE и разрывы в нумерации |
| **IDENTITY** | Стандартный автоинкремент столбца (GENERATED ALWAYS/BY DEFAULT AS IDENTITY) |

Используй таблицу как карту: нашёл понятие в первом столбце → во втором видишь суть и типичное применение.

### Маршрут изучения в этой части

1. **§66. Типы данных углублённо**  
   Точность числовых (DECIMAL), коллации и кодировки, дата/время и таймзоны (INTERVAL, extract, date_trunc), ENUM, домены и составные типы.

2. **§67. JSON, XML, массивы, диапазоны**  
   JSON/JSONB (операторы, индексы GIN, jsonpath), массивы и диапазоны (операторы, индексы), XML и сетевые типы.

3. **§68. Полнотекстовый поиск**  
   tsvector, tsquery, словари, rank и headline; индексы; триграммы (pg_trgm) для нечёткого поиска.

4. **§69. Представления**  
   Обычные VIEW, WITH CHECK OPTION; материализованные представления (REFRESH CONCURRENTLY); правила vs триггеры.

5. **§70. Триггеры**  
   BEFORE/AFTER, FOR EACH ROW/STATEMENT, INSTEAD OF; триггерные функции (NEW, OLD); условные и constraint-триггеры.

6. **§71. Хранимые процедуры и функции**  
   PL/pgSQL, volatile/stable/immutable, параметры и курсоры, динамический SQL, SECURITY DEFINER, процедуры (PROCEDURE/CALL).

7. **§72. Последовательности и IDENTITY**  
   CREATE SEQUENCE, nextval/currval/setval, CACHE и разрывы; GENERATED AS IDENTITY; SERIAL/BIGSERIAL.

Внутри разделов используется общий шаблон:

| Блок | Для чего он |
|------|-------------|
| **Цель раздела** | Понять, что именно ты освоишь после чтения |
| **«В этом разделе главное»** | Три строки — суть раздела до деталей |
| **Термины (расшифровка)** | Ключевые слова и их определения в одном месте |
| **Теория и правила** | Как устроено явление, какие правила, примеры запросов |
| **Пошагово** (где уместно) | Пошаговое описание процесса (как что-то сделать в SQL) |
| **Что будет, если… / Типичная ошибка / Часто путают** | Граничные случаи и типичные промахи |
| **Простыми словами / Картинка в голове / Как запомнить** | Объяснение без жаргона, образ для запоминания, краткая формула |
| **«Проверь себя»** | 2–3 вопроса с ответами под спойлером (`<details><summary>Ответ</summary>...</details>`) |
| **Запомните** | Краткий вывод: ключевые факты и формулировки |
| **Если прочитал и не понял** | Последняя попытка — одна цепочка самыми короткими фразами |

**Как устроены разделы и оглавление.**  
В каждом разделе материал идёт по одному шаблону: **цель** → **«в этом разделе главное»** → **термины** → **теория и правила** → **пошагово** (где есть) → **граничные случаи и типичные ошибки** → **простыми словами / картинка в голове / как запомнить** → **«Проверь себя»** (2–3 вопроса с ответами под спойлером) → **запомните** → **если прочитал и не понял**. Одна и та же мысль повторяется в разных формулировках — так проще усвоить. Оглавление сгруппировано по этапам изучения (§66–§72); по ссылке из таблицы оглавления переходишь прямо в нужный раздел.

1. **Выбери этап** (например, «Этап 2. JSON, массивы, диапазоны») — это твой следующий шаг в изучении.
2. **Пройди по строкам таблицы** — в «Содержание» указано, о чём раздел; в «Ключевые понятия» — термины, которые нужно знать после изучения.

4. **Справочник и вопросы для самопроверки** в конце части — когда этапы пройдены, используй их для повторения и самопроверки.

Оглавление группирует разделы по этапам и даёт краткое «что внутри» и «зачем это изучать», чтобы удобно планировать изучение и возвращаться к нужному месту.
