## 7. Отладка и интроспекция (§115)

### 7.0. Главная идея §115 (на пальцах)

Отладка — это не «магия». Это просто умение:

- посмотреть, **какие сейчас значения** у переменных;
- понять, **в какой строке** всё пошло не так;
- увидеть **путь выполнения** (traceback) и сделать вывод.

В Python есть быстрые встроенные инструменты, и их реально можно выучить за 15 минут.

### 7.1. `breakpoint()` — встроенная точка входа в отладчик

С Python 3.7:

```python
def foo():
    x = 10
    breakpoint()
    x += 1
    return x
```

Запуск скрипта приведёт к остановке в точке `breakpoint()`  
с открытием отладчика (по умолчанию `pdb`).

Плюсы:

- не нужно импортировать `pdb` вручную;
- поведение настраивается через переменную окружения `PYTHONBREAKPOINT`  
  или через `sys.breakpointhook`.

**Важно:** не оставляйте `breakpoint()` в продакшн‑коде.

---

### 7.1a. Мини‑шпаргалка команд `pdb` (чтобы не паниковать)

Когда вы попали в `breakpoint()`, обычно видно приглашение вроде `(Pdb)`.

Самые нужные команды:

- `l` — показать код вокруг текущей строки (list)
- `n` — выполнить текущую строку и перейти к следующей (next)
- `s` — «зайти внутрь» вызова функции (step)
- `c` — продолжить выполнение до следующего breakpoint/ошибки (continue)
- `p expr` — напечатать значение выражения (print)
- `pp expr` — красиво распечатать (pretty print)
- `where` — показать стек вызовов (где мы находимся)
- `q` — выйти (quit)

Пример (очень грубо):

```text
(Pdb) l
(Pdb) p x
(Pdb) n
(Pdb) where
(Pdb) c
```

### 7.2. `vars`, `dir`, `pprint`, `reprlib`

**`vars(obj)`** — по сути `obj.__dict__` (если есть):

```python
class User:
    def __init__(self, name, active):
        self.name = name
        self.active = active

u = User('Alice', True)
print(vars(u))  # {'name': 'Alice', 'active': True}
```

**`dir(obj)`** — список имён атрибутов:

```python
print(dir(u))  # поможет увидеть доступные методы/поля
```

**`pprint`** — красиво печатает вложенные структуры:

```python
from pprint import pprint

data = {'users': [{'name': 'Alice', 'roles': ['admin', 'user']},
                  {'name': 'Bob', 'roles': ['user']}]}
pprint(data, width=40)
```

**`reprlib.repr`** — «укороченный» `repr`, полезен для огромных структур:

```python
import reprlib

big_list = list(range(1000))
print(reprlib.repr(big_list))
```

### 7.3. f‑строки для отладки: `f'{var=}'`

Начиная с Python 3.8:

```python
x = 42
user = 'Alice'
print(f'{x=}, {user=}')
# x=42, user='Alice'
```

Очень удобно для быстрого логирования без ручного написания `"x=" + str(x)`.

---

### 7.4. `traceback.print_exc()`

Быстрый способ напечатать текущий traceback из блока `except`:

```python
import traceback

try:
    1 / 0
except ZeroDivisionError:
    traceback.print_exc()
```

Полезно в скриптах и черновиках; в продакшене лучше использовать **logging**.

---

### 7.4a. В продакшене чаще нужно `logging.exception(...)`

Когда вы в `except` и хотите не просто «распечатать», а нормально залогировать,
обычно используют:

```python
import logging

log = logging.getLogger(__name__)

try:
    ...
except Exception:
    log.exception("Что-то пошло не так")  # добавит traceback автоматически
```

Это удобнее, чем `print`, потому что:

- лог можно отправлять в файл/систему логов;
- traceback сохраняется вместе с сообщением.

### 7.5. Запомните по §115

- `breakpoint()` — быстрый вход в отладчик, но не забывайте убирать его.
- `vars`, `dir`, `pprint`, `reprlib` помогают **понять форму данных** и доступные атрибуты.
- `f'{var=}'` — маленький, но очень удобный хак для временного логирования.
- `traceback.print_exc()` — базовый инструмент, когда хочется увидеть стек ошибки, но ещё нет нормального логгера.
