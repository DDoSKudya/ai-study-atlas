# Часть XX. Полезные хаки и трюки

> **Цель:** собрать в одном месте практические «хаки» и идиоматичные приёмы Python 3,  
> которые делают код **короче, чище и выразительнее**, но при этом остаются **читабельными**.
>
> Всё объясняется **очень простым языком**, но с аккуратным использованием терминов
> (unpacking, comprehension, decorator, context manager, optional, short‑circuit и т.д.).
>
> Важно: «хак» здесь означает **удобный приём**, а не «грязный трюк». Мы постоянно держим в голове правило:
> **короче — не значит лучше; лучше — значит понятнее**.

После изучения этой части вы:

- будете уверенно использовать **распаковку** (`a, b = b, a`, `first, *rest = ...`, `*_, last = ...`);
- освоите **слияние** коллекций и словарей: `{**d1, **d2}`, `[*a, *b]`, `itertools.chain.from_iterable`;
- начнёте писать аккуратные **условные выражения** и выражения с **значениями по умолчанию**;
- аккуратно работать с **`None` и optional‑значениями**, не путая их с просто «пустыми»/`falsy`;
- сможете оборачивать функции в **декораторы**, использовать `lru_cache`, `wraps`, `partial`;
- будете спокойно управлять ресурсами через **контекстные менеджеры** (`with`, `contextlib`);
- получите набор **инструментов отладки и интроспекции** (`breakpoint()`, `pprint`, `reprlib`, `dir`, `vars`);
- научитесь отличать **полезный однострочник** от «магической каши из символов».

## 0.05. Мини‑словарь терминов (простыми словами)

Это мини‑словарь, чтобы мозг не спотыкался на терминах.

- **iterable (итерируемое)**: объект, по которому можно пройти циклом `for` (например, list, tuple, str, dict, range, генератор).
- **iterator (итератор)**: «вещь, которая помнит, где мы остановились». Её можно кормить в `next(...)`.
- **unpacking (распаковка)**: «разложить пачку значений по переменным» — `a, b = (1, 2)`.
- **star‑unpacking (`*`)**: «собрать/развернуть несколько значений сразу» — `first, *rest = xs`, `[*a, *b]`.
- **truthy/falsy**: «истинное/ложное» в контексте `if`. Например, `[]` falsy, `[1]` truthy.
- **short‑circuit**: логика `and/or`, где вычисление останавливается раньше конца, если ответ уже понятен.
- **optional**: значение «может быть, а может не быть». Обычно `T | None` (или `Optional[T]`).
- **decorator (декоратор)**: функция, которая **берёт функцию** и возвращает **новую функцию‑обёртку**.
- **wrapper (обёртка)**: внутренняя функция, которая делает «до/после» вокруг исходной.
- **context manager (контекстный менеджер)**: объект для `with`, который гарантирует «закрыть/освободить» ресурс.
- **side effect (побочный эффект)**: функция делает что‑то кроме возврата значения (печатает, пишет в файл/БД, меняет состояние).
- **pure function (чистая функция)**: при одинаковых входах всегда одинаковый выход и нет побочных эффектов.
- **memoization (мемоизация)**: кеширование результатов функции, чтобы не считать одно и то же много раз.
- **sentinel (сентинел)**: специальный уникальный объект‑метка «значение не задано».

## 0.1. Быстрая шпаргалка по хакам

| Группа (§) | Приём | Когда использовать | На что обратить внимание |
|-----------|-------|--------------------|---------------------------|
| §109 | `a, b = b, a` | Быстрый обмен значений без временной переменной | Не злоупотреблять в сложных выражениях |
| §109 | `first, *rest = xs` | Взять первый элемент и «хвост» | Работает с любым итерируемым, но материализует в список |
| §109 | `*_, last = xs` | Взять только последний элемент | Для последовательностей, где не страшно создать список |
| §109 | `dict(zip(keys, values))` | Собрать словарь из двух последовательностей | Длины должны совпадать, иначе лишние будут отброшены |
| §110 | `{**d1, **d2}` | Слияние словарей | Конфликты ключей решаются в пользу правого словаря |
| §110 | `[*a, *b]` | Объединение списков | Удобно для небольших коллекций; для больших — `itertools.chain` |
| §110 | `itertools.chain.from_iterable(nested)` | «Сплющивание» (flatten) | Эффективно по памяти, не создаёт промежуточных списков |
| §111 | `x = a or b` | «Значение по умолчанию» | Поймите, что `0`, `''`, `False` считаются ложными |
| §111 | `x = a if cond else b` | Тернарный оператор | Старайтесь, чтобы выражение было коротким |
| §111 | `d.get(k, default)` | Безопасный доступ к словарю | Не создаёт ключ; `setdefault` — создаёт |
| §111 | `next((x for x in xs if cond), default)` | Взять первый подходящий элемент | Удобно вместо цикла с `break` |
| §112 | `x = value if value is not None else default` | Отделить `None` от других пустых значений | Правильная проверка optional‑значений |
| §112 | `getattr(obj, 'attr', default)` | Безопасно читать атрибут | Не перехватывает другие ошибки внутри `__getattr__` |
| §112 | `if (x := f()) is not None:` | Присваивание в условии | Не используйте, если ухудшает читаемость |
| §113 | `@lru_cache` | Кешировать результат чистой функции | Не подходит для функций с побочными эффектами |
| §113 | `@wraps(func)` | Сохранять имя/докстроку у обёрнутой функции | Обязательно для всех «нормальных» декораторов |
| §113 | `partial(func, arg=value)` | Зафиксировать часть аргументов | Удобно при передаче функций как колбеков |
| §114 | `with open(...) as f1, open(...) as f2:` | Несколько ресурсов в одном `with` | Оба ресурса закроются корректно |
| §114 | `contextlib.suppress(Error)` | Игнорировать ожидаемое исключение | Не прячьте «настоящие» баги |
| §114 | `ExitStack()` | Динамическое количество контекстов | Полезно, когда число ресурсов заранее неизвестно |
| §115 | `breakpoint()` | Быстро войти в отладчик | Работает через `PYTHONBREAKPOINT` и `sys.breakpointhook()` |
| §115 | `pprint(obj)` | Красивый вывод сложных структур | Отлично подходит для быстрой диагностики |
| §115 | `f'{var=}'` | Быстрый debug‑вывод | Не оставляйте такие штуки в проде |
| §116 | Comprehensions | Компактный и понятный код | «Читаемость важнее краткости» — не сворачивайте всё в кашу |
