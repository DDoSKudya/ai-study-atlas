## 8. Однострочники и компактный код (§116)

### 8.0. Главная идея §116 (на пальцах)

Однострочники — это как «свернуть одежду». Можно аккуратно сложить,
а можно скомкать в шар — вроде компактно, но пользоваться невозможно.

Наша цель:

- использовать компактность **только там, где она делает понятнее**;
- не превращать код в «головоломку».

### 8.1. List comprehension

Идиоматичный способ построить список:

```python
xs = [1, 2, 3, 4, 5]
evens = [x for x in xs if x % 2 == 0]
print(evens)  # [2, 4]
```

Преимущества:

- короче и **читабельнее**, чем явный цикл с `append`;
- часто быстрее (особенно для простых операций).

Важно: comprehension — это не «обязанность», а инструмент. Сравните:

```python
xs = [1, 2, 3, 4]

# обычный цикл
out = []
for x in xs:
    out.append(x * 2)

# comprehension
out2 = [x * 2 for x in xs]
```

Идея одна, просто comprehension короче.

Плохо:

```python
result = [transform(get_user(id)['data'])
          for id in ids
          if is_valid(get_user(id))]
```

Здесь один и тот же вызов `get_user(id)` повторяется дважды. Лучше сначала вынести в переменную.

---

### 8.1a. Не используйте comprehension «ради побочных эффектов»

Плохая привычка:

```python
[print(x) for x in xs]  # делает список из None, и это выглядит странно
```

Почему плохо:

- comprehension по смыслу создаёт коллекцию;
- здесь коллекция не нужна, мы просто хотим выполнить действие.

Правильно:

```python
for x in xs:
    print(x)
```

### 8.2. Dict и set comprehension

**Dict comprehension:**

```python
users = ['alice', 'bob', 'carol']
user_map = {name: name.upper() for name in users}
```

**Set comprehension:**

```python
words = ['a', 'bb', 'ccc', 'dd', 'e']
lengths = {len(w) for w in words}
```

Используйте, когда вы:

- строите коллекцию из другого итерируемого;
- можете выразить логику в **одном коротком выражении**.

---

### 8.2a. Генераторное выражение: как comprehension, но без списка в памяти

List comprehension создаёт список сразу:

```python
xs = [1, 2, 3, 4]
sq_list = [x * x for x in xs]  # список в памяти
```

Генераторное выражение создаёт «ленивый поток» значений:

```python
xs = [1, 2, 3, 4]
sq_gen = (x * x for x in xs)   # НЕ список, а генератор
```

Где это полезно:

- когда данных много;
- когда вы сразу передаёте значения в `sum`, `any`, `all`, `max`, `min`, `join` и т.д.

Пример:

```python
xs = range(1_000_000)
total = sum(x * x for x in xs)  # без создания огромного списка
```

Запоминалка:

- `[...]` — «сделай список сейчас»
- `(...)` — «давай по одному, по мере надобности»

### 8.3. Swap и другие однострочники

Полезные, но простые:

```python
a, b = b, a                  # обмен значений
min_value = a if a < b else b
names = [u.name for u in users if u.active]
```

Опасные, если перегнуть палку:

```python
# пример "как не надо"
result = [func(x) for xs in nested for x in xs if cond(x)] or default()
```

Лучше разбить:

```python
flattened = [x for xs in nested for x in xs]
filtered = [x for x in flattened if cond(x)]
result = filtered or default()
```

---

### 8.4. Читаемость важнее краткости

**Главное правило Zen of Python:** *Readability counts.*

- Если однострочник **сложно прочитать вслух** — вероятно, это неудачный однострочник.
- Не стесняйтесь разворачивать однострочник в «нормальный» код, если так станет яснее.
- Старайтесь, чтобы каждая строка отражала **одну понятную мысль**.
