## 2. Объединение и слияние (§110)

### 2.0. Главная идея §110 (на пальцах)

Тут мы учимся «склеивать» коллекции так, чтобы понимать две вещи:

1) **Создаём новый объект** или **меняем существующий**?
2) Что происходит при конфликте (например, одинаковые ключи в словарях)?

Два подхода:

- **безопасно (создать новый объект)**: `{**d1, **d2}`, `d1 | d2`, `[*a, *b]`
- **быстро/на месте (изменить существующий)**: `d.update(...)`, `d |= other`, `lst.extend(...)`

Если вы новичок и боитесь «нечаянно сломать» — чаще выбирайте **создать новый объект**.

### 2.1. Слияние словарей: `{**d1, **d2}` и оператор `|`

Современный способ объединить словари:

```python
base_config = {'host': 'localhost', 'port': 5432}
override = {'port': 5433, 'user': 'admin'}

config = {**base_config, **override}
print(config)
# {'host': 'localhost', 'port': 5433, 'user': 'admin'}
```

Порядок важен: **правый словарь перезаписывает левый** по одинаковым ключам.

С Python 3.9 есть ещё более читаемый способ:

```python
config = base_config | override
base_config |= override  # обновить на месте
```

**Когда использовать:**

- сборка final‑конфига из дефолтных настроек и «локальных» перекрытий;
- объединение нескольких словарей с «разумными» правилами приоритета.

---

### 2.1a. Очень важное: это НЕ «глубокое» слияние (deep merge)

Если внутри словаря есть вложенный словарь, то `{**d1, **d2}` **не объединит** вложенности.
Он просто возьмёт значение по ключу целиком из правого словаря.

Пример:

```python
d1 = {'db': {'host': 'localhost', 'port': 5432}}
d2 = {'db': {'port': 5433}}

merged = {**d1, **d2}
print(merged)
# {'db': {'port': 5433}}
```

Почему так: для Python это обычные значения по ключу `'db'`.
Правый словарь сказал: «`db` теперь вот такой», и он полностью заменил левый.

Если вам нужно именно deep‑merge, это отдельная задача (обычно пишут функцию или берут готовую библиотеку).
Главное — **не ожидать deep‑merge от `{**d1, **d2}`**.

### 2.2. Объединение списков: `[*a, *b]`

Простой способ слепить несколько последовательностей в один список:

```python
xs = [1, 2]
ys = [3, 4]
zs = [*xs, *ys]
print(zs)  # [1, 2, 3, 4]
```

Плюсы:

- читабельно;
- работает с любой итерируемой справа: `[*xs, *set(ys), *range(5)]`.

Минусы:

- всегда создаётся **новый список**;
- для очень больших коллекций это может быть дорого по памяти.

---

### 2.2a. `+` vs `extend` vs `[*a, *b]` — что выбрать?

Есть три популярных варианта:

```python
xs = [1, 2]
ys = [3, 4]

a = xs + ys          # новый список
xs.extend(ys)        # изменит xs на месте
b = [*xs, *ys]       # новый список
```

Как запомнить:

- `xs + ys` и `[*xs, *ys]` создают **новый** список.
- `xs.extend(ys)` меняет **старый** `xs`.

Почему `extend` может быть опаснее новичку:

```python
xs = [1, 2]
alias = xs          # alias и xs указывают на один и тот же список
xs.extend([3, 4])
print(alias)        # [1, 2, 3, 4]  <- изменилось «где-то ещё»
```

Если вы не уверены, кто ещё держит ссылку на список — создавайте новый список.

### 2.3. Flatten: comprehension и `itertools.chain.from_iterable`

**Для небольших структур** удобно использовать comprehension:

```python
tuples = [(1, 2), (3, 4), (5, 6)]
flat = [x for pair in tuples for x in pair]
print(flat)  # [1, 2, 3, 4, 5, 6]
```

Порядок `for` важен: сначала внешний (`for pair in tuples`), потом внутренний (`for x in pair`).

**Для более общих и/или больших случаев**:

```python
from itertools import chain

nested = [[1, 2], [3, 4, 5], [], [6]]
flat_iter = chain.from_iterable(nested)

for x in flat_iter:
    print(x)
```

Особенности `chain.from_iterable`:

- **ленивый** — элементы берутся «по мере надобности»;
- не создаёт промежуточных списков.

---

### 2.3a. Анти‑паттерн: `sum(list_of_lists, [])`

Иногда встречается «хак» для flatten:

```python
lists = [[1, 2], [3], [4, 5]]
flat = sum(lists, [])
```

Почему это плохо:

- `sum` будет много раз делать `[] + [1,2] + [3] + ...`,
- а `+` для списков создаёт новый список каждый раз,
- итог: получается квадратичная стоимость по времени и лишние копирования.

Вместо этого:

```python
from itertools import chain
flat = list(chain.from_iterable(lists))
```

или, если данных немного:

```python
flat = [x for xs in lists for x in xs]
```

---

### 2.3b. Мини‑упражнения по §110

1) Есть:

```python
d1 = {'a': 1, 'b': 2}
d2 = {'b': 999, 'c': 3}
```

Слейте так, чтобы `b` стало `999`, а исходные `d1`, `d2` не изменились.

2) Объедините:

```python
xs = [1, 2]
ys = [3, 4]
```

в новый список `[1, 2, 3, 4]` тремя разными способами.

3) «Сплющите» список списков:

```python
nested = [[1, 2], [], [3], [4, 5]]
```

в `[1, 2, 3, 4, 5]` через `chain.from_iterable`.

### 2.4. Запомните по §110

- `{**d1, **d2}` и `d1 | d2` — современный питоничный способ слияния словарей.
- `[*a, *b]` отлично подходит для объединения нескольких небольших последовательностей.
- Для flatten больших структур используйте `itertools.chain.from_iterable`, для маленьких — comprehension.
