## 1. Обмен переменных и распаковка (§109)

### 1.0. Главная идея §109 (на пальцах)

Распаковка в Python — это как открыть коробку и разложить вещи по полкам:

- коробка = кортеж/список/любой iterable;
- полки = переменные слева;
- правило: **количество вещей должно совпадать** с количеством полок  
  (кроме случая со звёздочкой `*rest`, которая означает «всю оставшуюся кучу»).

Если запомнить только одно:

> **`a, b = something` означает: «в something должно быть ровно 2 элемента».**

### 1.1. `a, b = b, a` — обмен без временной переменной

Классический питоновский хак:

```python
a = 1
b = 2
a, b = b, a
print(a, b)  # 2 1
```

**Что здесь происходит на самом деле:**

- справа создаётся **кортеж** `(b, a)`;
- слева этот кортеж **распаковывается** в имена `a` и `b`.

Поэтому такое присваивание:

```python
a, b, c = 1, 2, 3
```

эквивалентно:

```python
tmp = (1, 2, 3)
a = tmp[0]
b = tmp[1]
c = tmp[2]
```

**Когда это уместно:**

- простой обмен двух (иногда трёх) переменных;
- инициализация нескольких переменных сразу (`x, y = get_coords()`).

**Когда лучше не использовать:**

- если справа сложное выражение, в котором «теряется» смысл:

```python
# Слишком много магии
a, b, *rest = sorted(set(xs))[::-1]
```

Такую конструкцию лучше **разбить на несколько строк**.

### 1.1a. Типовые ошибки распаковки (и как их понимать)

Это прям самые частые ошибки, которые пугают новичков. На самом деле они логичные.

#### Ошибка: «слишком много значений»

```python
a, b = (1, 2, 3)
# ValueError: too many values to unpack (expected 2)
```

Перевод на человеческий:

- слева 2 переменные (`a`, `b`);
- справа 3 значения;
- Python не знает, куда деть лишнее.

Как исправить:

```python
a, b, _ = (1, 2, 3)        # если третий элемент не нужен
a, b, *rest = (1, 2, 3)    # если нужен «хвост»
```

#### Ошибка: «слишком мало значений»

```python
a, b, c = (1, 2)
# ValueError: not enough values to unpack (expected 3, got 2)
```

Перевод:

- слева 3 «полки»;
- справа только 2 «вещи».

Как исправить: поменять ожидания (сколько переменных слева) или источник данных.

### 1.1b. Обмен «внутри объектов»: индексы и атрибуты

Обмен работает не только для «обычных переменных», но и для мест,
куда можно присваивать (индексы, атрибуты).

Пример с индексами:

```python
xs = [10, 20, 30]
xs[0], xs[2] = xs[2], xs[0]
print(xs)  # [30, 20, 10]
```

Почему это безопасно:

- сначала вычисляется правая часть (создаётся кортеж значений);
- потом присваиваются значения слева.

Пример с атрибутами:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
p.x, p.y = p.y, p.x
print(p.x, p.y)  # 2 1
```

### 1.2. `first, *rest = iterable`

Приём: взять **первый** элемент и «хвост»:

```python
numbers = [10, 20, 30, 40]
first, *rest = numbers
print(first)  # 10
print(rest)   # [20, 30, 40]
```

Особенности:

- справа может быть **любой итерируемый объект** (list, tuple, range, генератор);
- `rest` всегда будет **списком** (даже если исходник — не список);
- если элементов меньше, чем слева имён — будет `ValueError`.

```python
xs = [1]
first, *rest = xs
print(first)  # 1
print(rest)   # []
```

**Типичный сценарий:**

```python
def process_queue(queue: list[str]) -> None:
    if not queue:
        return

    head, *tail = queue
    handle(head)
    # Можно дальше как-то обработать хвост
```

### 1.2a. Распаковка «голова‑середина‑хвост»

Иногда нужно достать и первый, и последний элемент, а всё остальное собрать в середину:

```python
xs = [1, 2, 3, 4, 5]
first, *middle, last = xs
print(first)   # 1
print(middle)  # [2, 3, 4]
print(last)    # 5
```

Как это читается:

- `first` получает первый элемент;
- `last` получает последний элемент;
- `middle` получает «всё, что осталось между ними» (и это будет список).

**Ограничение:** слева может быть только **одна** переменная со звёздочкой.
Иначе Python не поймёт, как делить «остаток» между двумя списками.

Плохо (SyntaxError):

```python
# a, *b, *c = xs
```

---

### 1.2b. Важное про память: распаковка из итератора превращает данные в список

Распаковка со `*` заставляет Python собрать «остаток» в список.
Это может быть неожиданно, если справа не список, а, например, генератор.

```python
gen = (x for x in range(10_000_000))
first, *rest = gen  # rest станет огромным списком в памяти
```

Если вы работаете с большими потоками данных — лучше использовать итераторные инструменты (`itertools`, `next`, циклы),
а не «тащить всё в память» распаковкой.

### 1.3. `*_, last = iterable`

Аналогично предыдущему приёму, но для **последнего** элемента:

```python
xs = [1, 2, 3, 4]
*_, last = xs
print(last)  # 4
```

Здесь `_` — **осознанное имя**, которым мы помечаем «мне не важно, что здесь».

**Важный момент:** как и раньше, итерируемое **материализуется в список**,
поэтому для огромных коллекций (или бесконечных итераторов) это не подходит.

Для таких случаев лучше:

```python
from collections import deque

def last(iterable):
    d = deque(iterable, maxlen=1)
    return d[0]
```

### 1.3a. Самый простой способ взять последний элемент (если это list/tuple)

Если у вас **точно** список или кортеж — не нужно распаковывать:

```python
xs = [1, 2, 3, 4]
last = xs[-1]
```

Почему так лучше:

- это читается проще (любой сразу узнаёт «последний элемент»);
- не создаётся дополнительный список «остатка».

Распаковка `*_, last = xs` полезна как «самодокументирование» (“мне нужен только last”),
но используйте её там, где это действительно делает код понятнее и где данные небольшие.

### 1.4. `dict(zip(keys, values))`

Сборка словаря из двух последовательностей:

```python
keys = ['id', 'name', 'age']
values = [1, 'Alice', 30]
data = dict(zip(keys, values))
print(data)  # {'id': 1, 'name': 'Alice', 'age': 30}
```

Особенности:

- `zip` идёт **до минимальной длины**:

```python
keys = ['id', 'name', 'age']
values = [1, 'Alice']
print(dict(zip(keys, values)))  # {'id': 1, 'name': 'Alice'}
```

Поэтому, если вы **ожидаете** одинаковую длину, а различие — это ошибка,
используйте `zip(..., strict=True)` (Python 3.10+):

```python
keys = ['id', 'name', 'age']
values = [1, 'Alice']

for pair in zip(keys, values, strict=True):
    print(pair)
# ValueError: zip() argument 2 is shorter than argument 1
```

### 1.4a. Вариант через пары (когда они уже есть)

Если у вас уже есть список пар (ключ, значение), `zip` не нужен:

```python
pairs = [('id', 1), ('name', 'Alice')]
data = dict(pairs)
print(data)  # {'id': 1, 'name': 'Alice'}
```

То есть `dict(zip(keys, values))` — это просто способ сначала сделать пары,
а потом скормить их в `dict(...)`.

### 1.4b. Мини‑упражнения по §109

Попробуйте решить (лучше руками в Python):

1) У вас есть `xs = [1, 2, 3, 4]`.  
   Получите `a=1`, `b=2`, `rest=[3,4]` одной строкой.

2) У вас есть `xs = [1, 2, 3, 4]`.  
   Получите `first=1`, `last=4` и `middle=[2,3]`.

3) Есть `keys = ['a', 'b', 'c']`, `values = [10, 20, 30]`.  
   Сделайте `{'a':10,'b':20,'c':30}`.

Проверка:

```python
xs = [1, 2, 3, 4]
a, b, *rest = xs
first, *middle, last = xs

keys = ['a', 'b', 'c']
values = [10, 20, 30]
data = dict(zip(keys, values))
```

### 1.5. Запомните по §109

- Распаковка — это **не магия**, а всего лишь удобный синтаксис для работы с кортежами/итераторами.
- `first, *rest = xs` удобно, когда вам действительно нужен «голова+хвост».
- `*_, last = xs` — хороший читабельный приём для взятия последнего элемента из небольшой последовательности.
- `dict(zip(keys, values))` отлично подходит для конфигов, CSV‑строк, результатов парсинга.
