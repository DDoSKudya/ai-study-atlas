## 4. Работа с `None` и optional (§112)

### 4.0. Главная идея §112 (на пальцах)

`None` — это не «пусто», а **«значения нет»**.
Это как если бы поле формы вообще не заполнили.

А пустая строка `''` — это «поле заполнили, но пустым текстом».
Ноль `0` — это «значение есть и оно равно нулю».

Поэтому мы часто хотим различать:

- «значение отсутствует» → `is None`
- «значение пустое/нулевое» → отдельные правила (truthiness или явные проверки)

### 4.1. Отличайте `None` от других falsy‑значений

В Python есть несколько «ложных» значений:

- `None`
- `0`, `0.0`
- `''`, `[]`, `{}`
- `False`

```python
if not value:
    ...
```

здесь **не различает** `None` и, например, пустую строку.

Если вам важно именно «значение отсутствует» (optional‑смысл),  
используйте **сравнение с `None` по `is`**:

```python
if value is None:
    print("Значение не задано")
```

---

### 4.1a. Почему сравнивать нужно через `is`, а не `==`

`None` — это **singleton** (единственный объект).
Поэтому правильный стиль:

```python
if x is None:
    ...
```

А не:

```python
if x == None:  # так не делают
    ...
```

Почему:

- `is` проверяет «это тот же объект?» (identity);
- `==` проверяет «равно ли по смыслу?» и может быть переопределено классом.

Практически:

```python
x = None
print(x is None)   # True
print(x == None)   # True, но стиль плохой и может вести к странностям в кастомных типах
```

### 4.2. `x = value if value is not None else default`

Типичный паттерн:

```python
timeout = user_timeout if user_timeout is not None else DEFAULT_TIMEOUT
```

Так вы:

- **не подменяете** `0`, `''`, `False` на дефолт;
- чётко выражаете идею: «если значение не задано (`None`) — взять стандартное».

Если выражение громоздкое, его лучше вынести в отдельную функцию.

---

### 4.2a. Сентинел: «не передали аргумент» ≠ «передали None»

Иногда `None` — это **валидное значение**, а не «не задано».
Тогда паттерн `param=None` ломается.

Решение: использовать **sentinel** — специальную уникальную метку.

```python
MISSING = object()

def connect(timeout=MISSING):
    if timeout is MISSING:
        timeout = 10  # дефолт
    # timeout может быть None, если пользователь так захотел
    return timeout
```

Главная идея: `object()` создаёт уникальный объект, который никто случайно не передаст.

### 4.3. `getattr(obj, 'attr', default)`

Безопасное чтение атрибута:

```python
user = None
name = getattr(user, 'name', 'Anonymous')
print(name)  # 'Anonymous'
```

Особенности:

- если атрибут есть — возвращается его значение;
- если атрибута нет — возвращается `default`;
- **но** если внутри `__getattr__`/`__getattribute__` происходит другая ошибка,
  она **не будет подавлена** — и это хорошо: вы увидите реальный баг.

---

### 4.4. Walrus‑оператор: `if (x := f()) is not None:`

Walrus‑оператор (`:=`) позволяет **одновременно присвоить и проверить** значение:

```python
def get_cached_user():
    # вернёт dict или None
    ...

if (user := get_cached_user()) is not None:
    print(user['name'])
else:
    print("Пользователь не найден")
```

Используйте, когда:

- без него пришлось бы дважды вызывать функцию **или** сначала присвоить, потом писать `if user is not None`;
- выражение короткое, а логика читается естественно.

Не стоит писать длинные многострочные условия с несколькими `:=` —  
это быстро снижает читаемость.

---

### 4.4a. Мини‑упражнения по §112

1) Напишите `timeout`, который:

- равен `user_timeout`, если `user_timeout` **не None**;
- иначе равен `DEFAULT_TIMEOUT`.

2) Сделайте функцию `normalize(name)`:

- если `name is None` → вернуть `'Anonymous'`;
- если `name == ''` → вернуть `''` (то есть НЕ подменять на `'Anonymous'`).

Подсказка: здесь нельзя использовать `name or 'Anonymous'`.

### 4.5. Запомните по §112

- `None` — это **отсутствие значения**, а не «пустая строка» или «ноль».
- Для проверки optional‑значений используйте `is None` / `is not None`.
- `getattr(obj, 'attr', default)` и Walrus‑оператор решают типичную задачу  
  «получить значение и, если оно не `None`, что‑то с ним сделать».
