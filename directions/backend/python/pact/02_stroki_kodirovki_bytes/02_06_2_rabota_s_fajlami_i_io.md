## 6.2 Работа с файлами и I/O

### 6.2.1 open() и encoding

#### Сигнатура open() — полная

```python
open(file, mode='r', buffering=-1, encoding=None, errors=None, newline=None, closefd=True, opener=None)
```

**Ключевые параметры для текста и кодировок:**

| Параметр | Значение по умолчанию | Назначение |
|----------|------------------------|------------|
| **file** | — | Путь к файлу (str, Path) или file-like объект |
| **mode** | `'r'` | Режим: `'r'` (текст, чтение), `'w'` (текст, запись), `'a'` (текст, дополнение), `'rb'`, `'wb'`, `'ab'` (бинарные) и др. |
| **encoding** | UTF-8 (с 3.7+) | Кодировка для текстового режима; **только для текста**, не для `'rb'`/`'wb'` |
| **errors** | `'strict'` | Обработка ошибок декодирования/кодирования: `'strict'`, `'ignore'`, `'replace'`, `'surrogateescape'` и др. |
| **newline** | `None` | Управление переводом строк при чтении/записи в текстовом режиме |

**Важно:** параметр **encoding** имеет смысл **только в текстовом режиме** (`'r'`, `'w'`, `'a'`, `'r+'` и т.д. без `b`). В бинарном режиме (`'rb'`, `'wb'`, `'ab'`) указание encoding вызовет ошибку (TypeError в некоторых версиях; параметр игнорируется).

#### Текстовый режим по умолчанию UTF-8 (PEP 540, 3.7+)

В **Python 3.7+** при **текстовом режиме** кодировка по умолчанию — **UTF-8** на **всех платформах** (включая Windows). Раньше на Windows использовалась системная кодировка (CP1252, CP1251 и т.д.); PEP 540 ввёл UTF-8 по умолчанию, чтобы устранить несовместимость и «кракозябры» при переносе кода между ОС.

**Переменная окружения `PYTHONUTF8=1`** (Python 3.7+): принудительно использовать UTF-8 по умолчанию даже на старых платформах. `PYTHONUTF8=0` — отключить (использовать системную кодировку, устаревшее поведение).

**Практический вывод:** для обычных текстовых файлов в UTF-8 можно не указывать `encoding` (на Python 3.7+):

```python
with open("data.txt") as f:
    text = f.read()   # декодируется из UTF-8
```

**Явное указание `encoding='utf-8'`** полезно для: (1) ясности кода; (2) совместимости со старыми версиями Python (3.6 и ранее на Windows); (3) документирования намерения; (4) линтеров и статического анализа.

#### Если файл в другой кодировке

Указывайте `encoding` явно. **Никогда не полагайтесь на «умолчание» при чтении файлов из внешних источников** — кодировка может отличаться (CP1251, CP866, ISO-8859-1, KOI8-R и т.д.).

```python
# Файл в кодировке Windows-1251 (кириллица Windows)
with open("legacy.txt", encoding="cp1251") as f:
    text = f.read()

# Файл в кодировке консоли DOS (CP866)
with open("dos_file.txt", encoding="cp866") as f:
    text = f.read()

# Запись в UTF-8 (по умолчанию и так UTF-8, но явно — лучше)
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("Привет")
```

#### Граничные случаи open() и encoding

- **Файл не существует при 'r':** `FileNotFoundError`
- **Файл существует при 'x':** `FileExistsError`
- **encoding в бинарном режиме:** при `open(path, 'rb', encoding='utf-8')` параметр encoding **игнорируется** (или может вызвать предупреждение в некоторых версиях); бинарный режим не выполняет decode/encode
- **mode 'x' (exclusive create):** создаёт файл только если он не существует; иначе `FileExistsError`. Удобно для атомарного создания (избежание race condition при многопоточности)
- **mode 'a' (append):** запись всегда в конец файла; позиция seek не влияет на место записи при следующем write
- **Пустой файл:** `open("empty.txt").read()` → `''` (пустая строка)
- **Файл с BOM (UTF-8):** байты `\xef\xbb\xbf` в начале файла; при чтении в UTF-8 они декодируются в символ U+FEFF (ZERO WIDTH NO-BREAK SPACE); некоторые API (например, `codecs.open` с `utf-8-sig`) автоматически удаляют BOM при чтении. Обычный `open(..., encoding='utf-8')` оставляет BOM как первый символ строки.
- **encoding='utf-8-sig':** при *чтении* автоматически пропускает BOM в начале файла (если есть); при *записи* — добавляет BOM. Удобно для совместимости с редакторами/программами, ожидающими BOM.
- **Файл в «неправильной» кодировке:** при `open(path, encoding='utf-8')` и файле в CP1251 декодер может не выдать ошибку сразу (если последовательность байт случайно валидна в UTF-8), но текст будет неверным («кракозябры»). При невалидной последовательности — `UnicodeDecodeError`.

#### Пошагово: что происходит при чтении текстового файла

При выполнении `with open("file.txt") as f: text = f.read()`:

1. **Открытие:** интерпретатор открывает файл в **текстовом режиме** (mode `'r'`).
2. **Обёртка:** создаётся объект `io.TextIOWrapper`, оборачивающий буферизованный бинарный поток (`BufferedReader`).
3. **Чтение:** при вызове `f.read()` читаются **байты** из файла (через буфер).
4. **Декодирование:** байты **декодируются** в строку по кодировке `encoding` (по умолчанию UTF-8).
5. **Возврат:** возвращается объект типа `str` — последовательность Unicode code points.

При **записи** `f.write(s)` строка `s` **кодируется** в байты по той же кодировке и записывается в файл.

#### Теоретическое обоснование

Разделение **текстового** и **бинарного** режима — принцип Python 3: **текст** (str) и **байты** (bytes) не смешиваются неявно. При чтении файла в текстовом режиме происходит **декодирование** байт в строку; при записи — **кодирование** строки в байты. Кодировка определяет, как именно это делается. UTF-8 выбран по умолчанию как универсальный стандарт (веб, JSON, современные ОС, большинство инструментов).

### 6.2.2 Текстовый и бинарный режим

| Режим | Тип данных | encoding | Описание |
|-------|------------|----------|----------|
| `'r'`, `'w'`, `'a'` | str | да (по умолчанию UTF-8) | Текстовый: read/write возвращают/принимают str |
| `'rb'`, `'wb'`, `'ab'` | bytes | нет (игнорируется) | Бинарный: read/write возвращают/принимают bytes |

**Текстовый режим:**

```python
with open("file.txt", "r") as f:
    line = f.readline()   # str
with open("file.txt", "w") as f:
    f.write("Hello\n")    # записывается str, кодируется в UTF-8
```

**Бинарный режим:**

```python
with open("image.png", "rb") as f:
    data = f.read()       # bytes
with open("out.bin", "wb") as f:
    f.write(b"\x00\x01\x02")  # запись сырых байт
```

**Когда использовать бинарный режим:** изображения, исполняемые файлы, протоколы с фиксированной структурой байт, когда нужно читать/писать сырые байты без преобразования. Текстовый режим — для человекочитаемого текста (исходный код, логи, JSON, CSV и т.д.).

**Важно:** смешивание режимов недопустимо. Нельзя открыть файл в `'r'` и вызвать `f.write(b'...')` — write ожидает str в текстовом режиме. Нельзя открыть в `'rb'` и вызвать `f.write("текст")` — нужны bytes.

#### newline — подробно: управление переводами строк

Параметр **newline** в текстовом режиме управляет тем, как интерпретируются и записываются символы перевода строки. Разные ОС используют разные соглашения: Unix/Linux — `\n` (LF); Windows — `\r\n` (CRLF); старый Mac — `\r` (CR).

| Значение newline | При чтении | При записи |
|------------------|------------|------------|
| **`None`** (по умолчанию) | `\n`, `\r`, `\r\n` нормализуются в `\n` | `\n` заменяется на `os.linesep` (на Windows — `\r\n`) |
| **`''`** | Сохраняются как есть (`\r\n`, `\r`, `\n` не преобразуются) | `\n` пишется как есть |
| **`'\n'`** | Как при `None` | `\n` пишется как `\n` |
| **`'\r'`** | Как при `None` | `\n` пишется как `\r` |
| **`'\r\n'`** | Как при `None` | `\n` пишется как `\r\n` |

**Пошагово: что происходит при чтении файла с newline=None.** (1) Читаются байты из файла. (2) При встрече последовательностей `0x0D 0x0A` (CRLF), `0x0A` (LF) или `0x0D` (CR) они преобразуются в один символ `\n` (U+000A). (3) В возвращаемой строке всегда только `\n` как перевод строки — универсальный формат для Python.

**Когда использовать newline='':** когда нужно сохранить оригинальные переводы строк (например, при обработке CSV с разными ОС или при round-trip файла без изменений).

**Пошагово: байты в файле и что получает Python при разных newline**

| Байты в файле (hex) | Описание | newline=None (read) | newline='' (read) |
|---------------------|----------|---------------------|-------------------|
| `41 0A 42 0A` | LF (Unix) | `"A\nB\n"` | `"A\nB\n"` |
| `41 0D 0A 42 0D 0A` | CRLF (Windows) | `"A\nB\n"` (нормализовано) | `"A\r\nB\r\n"` (как есть) |
| `41 0D 42 0D` | CR (старый Mac) | `"A\nB\n"` | `"A\rB\r"` |
| `41 0A 0A 42` | Два LF подряд | `"A\n\nB"` | `"A\n\nB"` |

**При записи (newline):** при `f.write("A\nB\n")` и newline=None на Windows в файл пишутся байты `41 0D 0A 42 0D 0A`; при newline='' — `41 0A 42 0A`. При newline='\r\n' — всегда CRLF независимо от ОС.

```python
# Универсальный режим (по умолчанию)
with open("file.txt") as f:
    lines = f.readlines()  # каждая строка заканчивается на \n

# Сохранить оригинальные переводы
with open("file.txt", newline='') as f:
    content = f.read()  # \r\n остаётся \r\n

# CSV: newline='' предотвращает двойную интерпретацию \r\n в полях
import csv
with open("data.csv", newline='', encoding='utf-8') as f:
    reader = csv.reader(f)
    rows = list(reader)
```

#### Буферизация (buffering) — подробно

Параметр **buffering** управляет буферизацией I/O. От него зависит, когда данные реально попадают на диск или в устройство.

| Значение | Режим | Поведение |
|----------|-------|-----------|
| **0** | Только бинарный | Без буферизации; каждый read/write сразу уходит в ОС (медленно для мелких операций) |
| **1** | Текстовый | Построчная буферизация; буфер сбрасывается при `\n` (при записи) |
| **> 1** | Любой | Буфер указанного размера в байтах |
| **-1** (по умолчанию) | Любой | Буфер полного размера (обычно 8 КБ); оптимально для большинства случаев |

**Практические последствия:**
- При `buffering=0` (бинарный режим) мелкие `write` вызывают много системных вызовов — медленно
- При `buffering=1` (текстовый) `print(..., end='')` без `\n` не сбрасывает буфер до следующего `\n` или `flush()`
- Для гарантированной записи на диск до закрытия файла: `f.flush()` или `os.fsync(f.fileno())`

```python
# Построчная буферизация — вывод сбрасывается на каждом \n
with open("log.txt", "w", buffering=1) as f:
    f.write("line1\n")  # сразу на диск
    f.write("line2")    # может остаться в буфере
    f.flush()           # принудительный сброс

# Без буферизации (бинарный режим)
with open("raw.bin", "wb", buffering=0) as f:
    f.write(b"\x00\x01")  # сразу запись
```

#### Режимы open(): полная таблица

| Режим | Чтение | Запись | Файл существует | Файл не существует |
|-------|--------|--------|-----------------|---------------------|
| `'r'` | да | нет | открывает | FileNotFoundError |
| `'w'` | нет | да | обрезает | создаёт |
| `'a'` | нет | да (дополнение) | открывает, пишет в конец | создаёт |
| `'x'` | нет | да | FileExistsError | создаёт |
| `'r+'` | да | да | открывает | FileNotFoundError |
| `'rb'`, `'wb'`, `'ab'` | бинарный режим | — | аналогично | — |
| `'w+'` | да | да | обрезает | создаёт |
| `'a+'` | да | да (дополнение) | открывает | создаёт |

**Комбинированные режимы:** `'r+b'` или `'rb+'` — чтение и запись в бинарном режиме; `'w+b'` — создание/обрезание с чтением и записью. Режим `'t'` (текстовый) можно указывать явно: `'rt'`, `'wt'` эквивалентны `'r'`, `'w'`.

### 6.2.3 Параметр errors — подробно

Параметр **errors** задаёт поведение при ошибках **декодирования** (байты → str) и **кодирования** (str → байты). Значение по умолчанию — **`'strict'`**: при любой ошибке выбрасывается исключение (`UnicodeDecodeError` или `UnicodeEncodeError`).

#### Полная таблица значений errors

| Значение | decode (байты → str) | encode (str → байты) |
|----------|----------------------|----------------------|
| **`'strict'`** | `UnicodeDecodeError` при невалидных байтах | `UnicodeEncodeError` при не кодируемых символах |
| **`'ignore'`** | Пропустить невалидные байты (тихо) | Пропустить символы, которые нельзя закодировать |
| **`'replace'`** | Заменить невалидные байты на U+FFFD () | Заменить не кодируемые символы на `?` (в ASCII/Latin-1 и др.) |
| **`'surrogateescape'`** | Невалидный байт `b` → U+DC00+b; при encode обратно в `b` | Surrogate code points U+DC80–U+DCFF → соответствующие байты |
| **`'backslashreplace'`** | Заменить на escape `\xNN` | Заменить на `\uNNNN` / `\UNNNNNNNN` |
| **`'namereplace'`** | — | Заменить на `\N{name}` (если есть Unicode name) |
| **`'xmlcharrefreplace'`** | — | Заменить на XML-сущность `&#NNNN;` |
| **`'surrogatepass'`** | Пропустить surrogate code points (для UTF-8 round-trip) | То же для encode |

#### surrogateescape — подробно: round-trip произвольных байт

**surrogateescape** — специальный режим, позволяющий «протащить» **произвольные байты** через текстовый интерфейс без потерь. Типичные случаи:

1. **Имена файлов в POSIX** — могут содержать любые байты, не только валидный UTF-8
2. **Legacy-данные** — бинарные данные, закодированные в «текст» (например, base64, но с «битыми» фрагментами)
3. **Системные вызовы** — `os.listdir()`, `os.environ` и т.д. возвращают str; на POSIX при некорректной кодировке файловой системы используется surrogateescape

**Как работает surrogateescape при decode:**

1. При встрече **невалидных байт** в UTF-8 (например, `0xFF` — невалидный первый байт в UTF-8) вместо выброса `UnicodeDecodeError` каждый невалидный байт `b` (0–255) превращается в символ с code point **U+DC00 + b**
2. Диапазон U+DC00–U+DFFF зарезервирован в Unicode для **low surrogates** (часть UTF-16 surrogate pairs); эти «суррогатные» символы не имеют собственного отображения, но Python использует их для хранения произвольных байт
3. При **encode** с `surrogateescape` символы в диапазоне U+DC80–U+DCFF (т.е. U+DC00 + b для b=0x80–0xFF) превращаются обратно в соответствующие байты; U+DC00–U+DC7F соответствуют байтам 0x00–0x7F, но для них обычно не возникает decode-ошибки (они валидны в UTF-8 как ASCII)

**Формула:** невалидный байт `b` → символ U+DC00+b; символ U+DC00+b при encode с surrogateescape → байт `b`.

**Пошагово: round-trip b'x\xffy' с surrogateescape.** (1) `decode('utf-8', errors='surrogateescape')`: байт 0x78 ('x') — валидный ASCII, декодируется в 'x'. Байт 0xFF — невалидный в UTF-8. Вместо ошибки создаётся символ U+DC00+0xFF = U+DCFF. Байт 0x79 ('y') — валидный, декодируется в 'y'. Результат: строка из трёх «символов»: 'x', '\udcff', 'y'. (2) `encode('utf-8', errors='surrogateescape')`: 'x' кодируется в 0x78. Символ U+DCFF в диапазоне U+DC80–U+DCFF при encode с surrogateescape превращается обратно в байт 0xFF. 'y' кодируется в 0x79. Итог: `b'x\xffy'` — исходные байты восстановлены.

```python
# Round-trip: произвольные байты → str → байты
raw = b'hello\xffworld\xfe'
s = raw.decode('utf-8', errors='surrogateescape')
# s содержит 'hello' + символ U+DCFF (b'\xff') + 'world' + символ U+DCFE (b'\xfe')
back = s.encode('utf-8', errors='surrogateescape')
back == raw   # True

# Пример с os.listdir (POSIX, некорректная кодировка ФС)
# Имя файла с байтами \xff в имени
# os.listdir('.') вернёт str с surrogateescape
# Для работы с байтами имени: name.encode('utf-8', errors='surrogateescape')
```

**Когда использовать surrogateescape:**

- При чтении имён файлов/переменных окружения на POSIX с возможными невалидными UTF-8 байтами
- При необходимости round-trip произвольных байт через str без потерь

**Когда не использовать:**

- При обработке обычного текста — `strict` или `replace` предпочтительнее для отладки
- При записи в JSON/XML — surrogateescape-символы невалидны в JSON (могут вызвать ошибку)

#### Примеры для каждого errors

```python
# strict — ошибка (по умолчанию)
b'\xff\xfe'.decode('utf-8')  # UnicodeDecodeError: invalid start byte

# ignore — пропуск (данные теряются)
b'hello\xffworld'.decode('utf-8', errors='ignore')   # 'helloworld'
len('helloworld')  # 10 — байт \xff пропущен

# replace — символ замены U+FFFD ()
b'hello\xffworld'.decode('utf-8', errors='replace')  # 'hello\ufffdworld'
# Визуально: hello world (с символом замены)

# surrogateescape — round-trip
s = b'x\xffy'.decode('utf-8', errors='surrogateescape')
repr(s)  # "x\udcffy" — surrogate U+DCFF
s.encode('utf-8', errors='surrogateescape')  # b'x\xffy'

# backslashreplace — escape в строке
b'\xff'.decode('utf-8', errors='backslashreplace')  # '\\xff' (отображается как \xff)
```

#### Использование errors в open()

Параметр **errors** можно передать в **open()** для текстового режима:

```python
# Чтение файла с «битыми» байтами — заменять на
with open("possibly_corrupt.txt", encoding="utf-8", errors="replace") as f:
    text = f.read()

# Чтение имён файлов с surrogateescape (через pathlib)
from pathlib import Path
# Path читает имена в str; при некорректной ФС может использовать surrogateescape
```

#### Плохо vs лучше: errors

```python
# Плохо: ignore — тихо теряет данные
b'Важно\xffтекст'.decode('utf-8', errors='ignore')  # 'Важнотекст' — потеряли символ \xff

# Лучше для отладки: replace — видно, где ошибка
b'Важно\xffтекст'.decode('utf-8', errors='replace')  # 'Важно\ufffdтекст'

# Лучше для round-trip: surrogateescape
raw = b'data\xff'
s = raw.decode('utf-8', errors='surrogateescape')
s.encode('utf-8', errors='surrogateescape') == raw  # True
```

### 6.2.4 sys.stdout.encoding и sys.getdefaultencoding()

#### sys.getdefaultencoding()

Возвращает **кодировку по умолчанию** для преобразований, когда кодировка не указана явно. В Python 3 это **'utf-8'** (на современных системах). Используется, например, при `str.encode()` и `bytes.decode()` без аргумента encoding.

```python
import sys
sys.getdefaultencoding()  # 'utf-8'
```

#### sys.stdout.encoding, sys.stdin.encoding, sys.stderr.encoding

Кодировка, в которой **стандартные потоки** (stdout, stdin, stderr) интерпретируют/выводят текст. Зависит от окружения (локаль, переменные, терминал).

```python
import sys
sys.stdout.encoding  # например 'utf-8' в современном терминале
sys.stdin.encoding   # то же
```

**Практический вывод:** при выводе в stdout/в файл Python кодирует строки в кодировку потока. Если терминал ожидает UTF-8, а вы выводите str — всё работает. Если терминал в другой кодировке (например, CP866 в старой консоли Windows), могут появиться «кракозябры». Решение: настроить терминал на UTF-8 или явно кодировать вывод.

**Важно:** `print(s)` выводит `str`; интерпретатор преобразует её в байты с помощью `sys.stdout.encoding`. Для бинарных данных используйте `sys.stdout.buffer.write(bytes)`.

#### Пошагово: что происходит при print("Привет")

1. **print** получает строку `"Привет"` (тип str, шесть Unicode code points).
2. Каждый объект приводится к str через `str()` — здесь уже str, без изменений.
3. Строки объединяются через `sep` (по умолчанию пробел); здесь один аргумент — результат `"Привет"`.
4. Добавляется `end` (по умолчанию `\n`) — итоговая строка `"Привет\n"`.
5. Строка передаётся в `file.write()` (по умолчанию `sys.stdout`).
6. **sys.stdout** — объект `io.TextIOWrapper`; при записи строка **кодируется** в байты по `sys.stdout.encoding` (например, UTF-8).
7. Байты записываются в нижележащий буфер (`sys.stdout.buffer`), который выводит их в терминал/консоль.

Если терминал ожидает UTF-8, вывод корректен. Если терминал в CP866 (старая консоль Windows), символы могут отображаться некорректно — решение: настроить консоль на UTF-8 (`chcp 65001`) или использовать библиотеку, управляющую кодировкой вывода.

#### Практический вывод по sys.stdout.encoding

- Для переносимого вывода в консоль предпочтительно, чтобы терминал использовал UTF-8.
- При перенаправлении вывода в файл (`python script.py > out.txt`) кодировка берётся из окружения; для гарантированного UTF-8 можно открыть файл явно с `encoding='utf-8'`.
- Для бинарного вывода (изображения, сырые байты) используйте `sys.stdout.buffer.write(data)` — обход TextIOWrapper.

### 6.2.5 Потоковая обработка больших файлов

Когда файл не помещается в память целиком (логи гигабайтных размеров, дампы БД, потоковые данные), чтение через `f.read()` приведёт к `MemoryError` или чрезмерному потреблению RAM. Решение — **потоковая (итеративная) обработка**: читать и обрабатывать данные порциями, не загружая весь файл.

#### Текстовые файлы: итерация по строкам

Файловый объект в текстовом режиме **итерируем** — при каждом `for line in f:` читается следующая строка (до `\n` включительно или до EOF). Память: в каждый момент в RAM только одна строка + буфер (обычно 8 КБ).

```python
# Подсчёт строк в файле любого размера
def count_lines(path: str) -> int:
    with open(path, encoding='utf-8') as f:
        return sum(1 for _ in f)

# Поиск строки, содержащей подстроку — останавливаемся при первом совпадении
def find_line_with(path: str, substring: str) -> str | None:
    with open(path, encoding='utf-8') as f:
        for line in f:
            if substring in line:
                return line.rstrip('\n')
    return None

# Фильтрация и запись в новый файл (обработка по одной строке)
def filter_lines(src: str, dst: str, predicate) -> None:
    with open(src, encoding='utf-8') as fin, open(dst, 'w', encoding='utf-8') as fout:
        for line in fin:
            if predicate(line):
                fout.write(line)
```

**Важно:** каждая строка при итерации содержит завершающий `\n` (кроме последней, если файл не заканчивается переводом строки). Для сравнения без `\n` используйте `line.rstrip('\n')` или `line.strip()`.

#### Бинарные файлы: чтение порциями (chunks)

Для бинарных данных нет понятия «строка»; читают фиксированными блоками через `read(size)`:

```python
# Копирование файла порциями (без загрузки в память целиком)
def copy_file(src: str, dst: str, chunk_size: int = 65536) -> None:
    with open(src, 'rb') as fin, open(dst, 'wb') as fout:
        while True:
            chunk = fin.read(chunk_size)
            if not chunk:
                break
            fout.write(chunk)

# Подсчёт вхождений байта в бинарном файле
def count_byte(path: str, byte_val: int, chunk_size: int = 65536) -> int:
    count = 0
    with open(path, 'rb') as f:
        while True:
            chunk = f.read(chunk_size)
            if not chunk:
                break
            count += chunk.count(byte_val)
    return count

# Поиск подпоследовательности в бинарном файле (упрощённо — при перекрытии chunk границ нужна логика склейки)
def find_in_file(path: str, needle: bytes, chunk_size: int = 65536) -> int | None:
    offset = 0
    overlap = len(needle) - 1  # перекрытие для поиска на границе чанков
    with open(path, 'rb') as f:
        prev_tail = b''
        while True:
            chunk = f.read(chunk_size)
            if not chunk and not prev_tail:
                break
            data = prev_tail + chunk
            pos = data.find(needle)
            if pos != -1:
                return offset + pos
            prev_tail = data[-overlap:] if overlap > 0 else b''
            offset += len(chunk)
    return None
```

**Выбор `chunk_size`:** типично 64 КБ (65536) или 1 МБ — баланс между числом системных вызовов и использованием памяти. Для SSD можно увеличить; для сетевых потоков — уменьшить.

#### readline() и readlines() — когда что использовать

| Метод | Поведение | Память |
|-------|-----------|--------|
| `f.read()` | Весь файл в память | O(размер файла) |
| `f.readline()` | Одна строка за вызов | O(длина строки) |
| `f.readlines()` | Все строки в список | O(размер файла) |
| `for line in f:` | Итерация по строкам (лениво) | O(длина строки + буфер) |

**Плохо:** `lines = f.readlines()` для гигабайтного файла — загрузка всего.  
**Лучше:** `for line in f:` — обрабатывать построчно.

#### Паттерн «прочитать N строк»

```python
from itertools import islice

def read_n_lines(path: str, n: int) -> list[str]:
    with open(path, encoding='utf-8') as f:
        return list(islice(f, n))

# Или с ограничением по байтам (пример: первые 1 МБ как строки)
def read_first_mb_as_lines(path: str, mb: float = 1.0) -> list[str]:
    limit = int(mb * 1024 * 1024)
    lines = []
    total = 0
    with open(path, encoding='utf-8') as f:
        for line in f:
            lines.append(line)
            total += len(line.encode('utf-8'))  # байты строки
            if total >= limit:
                break
    return lines
```

#### Запись в поток (streaming write)

При записи `f.write(chunk)` данные обычно попадают в буфер; сброс — при `flush()`, закрытии файла или заполнении буфера. Для логов в реальном времени:

```python
with open("log.txt", "w", encoding="utf-8", buffering=1) as f:
    for event in event_stream:
        f.write(f"{event}\n")  # buffering=1 → сброс на каждый \n
        # или f.flush() после каждой записи при buffering > 1
```

#### Сводка: когда использовать потоковую обработку

| Сценарий | Рекомендация |
|----------|--------------|
| Текстовый файл, обработка по строкам | `for line in f:` |
| Бинарный файл, копирование/сканирование | `while True: chunk = f.read(size); if not chunk: break; ...` |
| Поиск подстроки в тексте | `for line in f: if sub in line:` |
| Поиск байт-последовательности в бинарном | Чтение chunks с перекрытием |
| Подсчёт, агрегация по строкам | `sum(1 for _ in f)` или накопление в цикле |
| JSON/XML большого размера | Специализированные парсеры (ijson, xml.etree iterparse) |
