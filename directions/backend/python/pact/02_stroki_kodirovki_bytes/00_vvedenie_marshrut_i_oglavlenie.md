# Часть II. Строки, кодировки, bytes

> **Цель:** разобрать модель текста и байт в Python 3: Unicode, кодировки, ord/chr, escape, файлы, bytes и bytearray.

**Краткая шпаргалка (суть):**

| Понятие | Суть |
|---------|------|
| Code point | Число 0–1114111, идентифицирует символ |
| str | Последовательность code points (Unicode) |
| bytes | Последовательность байт 0–255 |
| encode / decode | str ↔ bytes; по умолчанию UTF-8 |
| open(encoding=) | Текстовый режим — decode при чтении |
| bytes[i] | int (код байта); срез b[i:j] → bytes |

---

## Маршрут изучения

**Что вы изучите и в каком порядке.** Сначала — **Unicode** (§6.1): как устроен текст в Python 3 (**code points**, **UTF-8**, **ord**/**chr**, escape-последовательности). Это основа работы с кодировками и строками. Дальше — **файлы и I/O** (§6.2): как читать и писать текст и байты (**open**, **encoding**, **newline**, **errors**), потоковая обработка больших файлов. Затем — **bytes и bytearray** (§6.3): индексация, создание, методы, **encode**/**decode**, **memoryview**. В конце части — справочник: сценарии, терминология, вопросы, задания; им удобно пользоваться для повторения и самопроверки.

**Что желательно знать заранее:** Часть I (отличия Py3 от Py2), в частности str/bytes и базовый синтаксис.

## Как устроены разделы и оглавление

В каждом разделе (§6.1–§6.3) материал идёт по одному шаблону: **цель** (что вы освоите) → **термины** (code point, UTF-8, encode/decode и т.д., с кратким определением) → **правила и синтаксис** (как читать и писать текст/байты) → **примеры** (код с пояснением) → **граничные случаи** (ошибки кодировки, newline, errors) → блок **«Запомните»** (краткий вывод). Термины в тексте сохранены (**code point**, **BMP**, **surrogate**, **memoryview** и т.д.); в начале раздела они расшифрованы.

**Оглавление** разбито на этапы изучения. Каждая строка таблицы — раздел: по ссылке переходите к материалу. В столбце «Содержание» — о чём раздел; в «Ключевые понятия» — термины, которые нужно знать после изучения. Идите по этапам сверху вниз; справочник в конце — для повторения и самопроверки.
