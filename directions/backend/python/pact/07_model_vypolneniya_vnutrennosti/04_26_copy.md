**Цель раздела §26:** уметь делать **поверхностную** (**shallow**) и **глубокую** (**deep**) копии объектов; знать, когда какая нужна, и как переопределить поведение через **`__copy__`** и **`__deepcopy__`**.

---

### copy.copy() — поверхностная копия

**`copy.copy(x)`** создаёт **новый** объект того же типа, что и **x**, но для контейнеров **внутренние элементы не копируются** — в копию попадают **те же** ссылки на вложенные объекты. Поэтому изменение вложенного объекта (например, списка внутри списка) отражается и в оригинале, и в копии.

**Пошагово (упрощённо):** для списка **copy.copy** создаёт новый список **new_list = []** и для каждого элемента **item** исходного списка выполняет эквивалент **new_list.append(item)** — то есть копируются **ссылки** на те же объекты. Аналогично для dict: новый словарь с теми же парами ключ→значение (ключи и значения — те же объекты). Вложенные контейнеры (списки, словари внутри) **не** обходятся рекурсивно.

#### list.copy() и copy.copy(list): когда что использовать

Для **списка** **lst.copy()** (метод списка, Python 3.3+) и **copy.copy(lst)** дают **одинаковый результат** — новый список с теми же ссылками на элементы (поверхностная копия). **list(lst)** тоже создаёт поверхностную копию списка. Использовать **lst.copy()** или **list(lst)** удобнее и быстрее для списка. **copy.copy(x)** нужен, когда **x** может быть **другой тип** (например, пользовательский контейнер или общая функция «скопировать что угодно поверхностно»); для списка результат эквивалентен. **copy.deepcopy(x)** — когда нужна полная копия со всеми вложенными объектами.

#### Схема: shallow copy — общие вложенные объекты

```
    Оригинал lst          Shallow copy
    ┌─────────────┐       ┌─────────────┐
    │ [0] → 1     │       │ [0] → 99    │  (присвоили новое значение)
    │ [1] → 2     │       │ [1] → 2     │  (та же ссылка)
    │ [2] ────────┼───┐   │ [2] ────────┼───┐
    └─────────────┘   │   └─────────────┘   │
                      │                     │
                      ▼                     ▼
                 ┌─────────────┐
                 │ [3, 4, 5]   │  ← один и тот же список (добавили 5 через shallow[2].append(5))
                 └─────────────┘
```

#### Пример: shallow copy

```python
import copy
lst = [1, 2, [3, 4]]
shallow = copy.copy(lst)
shallow[0] = 99       # меняем только shallow
print(lst[0])         # 1 — оригинал не изменился
shallow[2].append(5)  # меняем вложенный список
print(lst[2])         # [3, 4, 5] — изменился и оригинал (тот же вложенный объект)
```

### copy.deepcopy() — глубокая копия

**`copy.deepcopy(x)`** рекурсивно копирует объект и **все** достижимые из него объекты. В результате получается полная копия без общих изменяемых подобъектов. Поддерживаются пользовательские классы через **`__deepcopy__(self, memo)`**.

**Параметр memo:** при глубоком копировании **copy** передаёт в **__deepcopy__(self, memo)** словарь **memo** (id объекта → уже скопированный объект). Он нужен, чтобы при циклических ссылках (A → B → A) не уходить в бесконечную рекурсию: при повторном встрече того же объекта возвращается копия, уже созданная ранее. Пользовательский **__deepcopy__** должен вызывать **copy.deepcopy(attr, memo)** для вложенных атрибутов и при необходимости регистрировать копию в **memo**.

#### Пошагово: как memo предотвращает бесконечную рекурсию при deepcopy с циклами

Пусть объект **A** имеет атрибут **next**, ссылающийся на **B**, а **B.next** ссылается обратно на **A** (цикл A → B → A). При **copy.deepcopy(A)** без memo: (1) копируем A, создаём **A'**; (2) копируем **A.next** (B) — создаём **B'**; (3) копируем **B.next** (A) — снова копируем A, создаём **A''**, затем **A''.next** (B) — снова B, и так до бесконечности. С **memo**: (1) копируем A, создаём **A'**, записываем **memo[id(A)] = A'**; (2) копируем **A.next** (B), создаём **B'**, записываем **memo[id(B)] = B'**; (3) копируем **B.next** (A) — **id(A)** уже есть в **memo**, возвращаем **memo[id(A)]** (A') вместо новой копии; (4) присваиваем **B'.next = A'**. Цикл воспроизведён в копии без бесконечной рекурсии.

#### Пример: deep copy

```python
import copy
lst = [1, 2, [3, 4]]
deep = copy.deepcopy(lst)
deep[2].append(5)
print(lst[2])         # [3, 4] — оригинал не изменился
```

### __copy__ и __deepcopy__

Класс может определить:

- **`__copy__(self)`** — вызывается при **`copy.copy(self)`**; должен вернуть **поверхностную** копию (новый объект того же типа, вложенные объекты — те же ссылки).
- **`__deepcopy__(self, memo)`** — вызывается при **`copy.deepcopy(self)`**; должен вернуть **глубокую** копию. **memo** — словарь (id объекта → копия), используемый для избежания бесконечной рекурсии при циклических ссылках. При копировании вложенных атрибутов нужно вызывать **copy.deepcopy(attr, memo)** и при необходимости сохранять результат в **memo**.

#### Пример: __deepcopy__ с memo (избежание циклов)

```python
import copy

class Node:
    def __init__(self, value, next_node=None):
        self.value = value
        self.next = next_node
    def __deepcopy__(self, memo):
        # Без memo при цикле (node.next = node) будет бесконечная рекурсия
        if id(self) in memo:
            return memo[id(self)]
        new_node = Node(copy.deepcopy(self.value, memo))
        memo[id(self)] = new_node
        new_node.next = copy.deepcopy(self.next, memo) if self.next else None
        return new_node
```

**Когда использовать shallow, когда deep:**

| Ситуация | Рекомендация | Причина |
|----------|--------------|---------|
| Плоский список/словарь, вложенные объекты не меняются | **copy.copy()** или **list.copy()** | Достаточно копии верхнего уровня |
| Контейнер с вложенными изменяемыми объектами, не должны быть общими | **copy.deepcopy()** | Изменение вложенного в копии не должно влиять на оригинал |
| Объект с циклическими ссылками | **copy.deepcopy()** с корректным **__deepcopy__(memo)** | memo предотвращает бесконечную рекурсию |

**Запомните:** для контейнеров с вложенными изменяемыми объектами, которые не должны разделяться с оригиналом, используйте **`copy.deepcopy()`**; для «плоских» структур или когда разделение вложенных объектов допустимо — **`copy.copy()`** или встроенные методы типа **`list.copy()`**.

#### Граничные случаи copy (§26)

- **Пустой список/словарь:** **copy.copy([])** и **copy.deepcopy([])** дают новый пустой список; разница проявится только при наличии вложенных изменяемых объектов.
- **Вложенный dict:** **d = {"a": {"b": 1}}**; после **copy.copy(d)** изменение **d["a"]["b"] = 2** меняет и копию (тот же вложенный dict); после **copy.deepcopy(d)** копия полностью независима.
- **Объект с __copy__ без __deepcopy__:** при **copy.deepcopy** для такого объекта по умолчанию вызывается **copy.deepcopy** рекурсивно по атрибутам; если атрибут — тот же объект (цикл), без **memo** возможна бесконечная рекурсия — поэтому в кастомном **__deepcopy__** нужно использовать **memo**.
- **Модули, классы, функции:** **copy.copy(MyClass)** возвращает тот же объект (классы и функции по умолчанию не копируются поверхностно как «новый экземпляр»); **copy.deepcopy** для класса по умолчанию тоже возвращает тот же класс. Копируют обычно экземпляры и контейнеры.
- **__copy__ возвращает self (singleton):** если объект по смыслу единственный (например, конфиг-синглтон), **__copy__** может вернуть **self** — тогда **copy.copy(obj) is obj**. Так «копия» остаётся тем же объектом. Для **__deepcopy__** в таком случае тоже можно вернуть **self**, если глубокое копирование не требуется.
