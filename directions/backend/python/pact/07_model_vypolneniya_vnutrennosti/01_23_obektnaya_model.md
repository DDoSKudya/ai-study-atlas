## §23. Объектная модель

**Цель раздела §23:** твёрдо усвоить, что в Python «всё — объект»: каждое значение — экземпляр какого-то типа, у каждого объекта есть идентификатор и тип. Научиться различать **тождество** (`is`, `id()`) и **равенство** (`==`), понимать, что такое **hashable** объекты и почему мелкие целые числа кешируются. В итоге вы перестанете путать «две переменные указывают на один объект» с «два равных по значению объекта» и будете корректно выбирать типы для ключей словаря и элементов множества.

---

В Python **нет «примитивов» в смысле C или Java**: целое число, строка, список, функция, класс, модуль — всё это **объекты**. Каждое выражение, результат вызова функции, литерал — всё даёт ссылку на объект. У каждого объекта есть три неотъемлемых свойства:

1. **Идентификатор (identity)** — целое число, уникальное в течение жизни объекта; возвращается встроенной функцией **`id(obj)`**. В CPython это обычно адрес объекта в памяти (но полагаться на это не следует: в Jython, IronPython или PyPy семантика может отличаться). Идентификатор **никогда не меняется** с момента создания объекта до его уничтожения; после освобождения памяти тот же целочисленный id может быть выдан другому объекту (reuse).
2. **Тип (type)** — класс объекта; возвращается **`type(obj)`**. Тип определяет, какие операции и атрибуты доступны; тип объекта в Python **не меняется** после создания (объект не «перерождается» в другой тип). Тип сам является объектом (класс — объект типа `type`).
3. **Значение (value)** — данные, которые объект «содержит»; для чисел и строк это очевидно (число 42, строка `"hello"`), для контейнеров — ссылки на другие объекты (список «содержит» ссылки на элементы). У **изменяемых** объектов значение может меняться (добавили элемент в список); у **неизменяемых** — нет (новая строка при конкатенации — новый объект).

**Термины:**

| Термин | Определение |
|--------|-------------|
| **Объект** | Сущность с идентификатором, типом и значением; всё, что можно присвоить переменной или передать в функцию. |
| **Тождество (identity)** | «Один и тот же объект»: два имени или выражения ссылаются на один объект в памяти. Проверяется оператором **`is`** и функцией **`id()`**. |
| **Равенство (equality)** | «Значения считаются равными»: по смыслу типа (например, два списка с одинаковыми элементами). Проверяется оператором **`==`** и методом **`__eq__`**. |
| **Hashable** | Объект, который можно использовать как ключ словаря или элемент множества: у него есть стабильный **`__hash__`** и **`__eq__`**; неизменяемые встроенные типы (int, str, tuple из hashable элементов) — hashable. |

#### Теоретическое обоснование: «Everything is an object»

Идея «всё — объект» закреплена в **The Zen of Python** (PEP 20) и в документации языка. В разделе **Data model** (Python Language Reference) сказано дословно: *«Objects are Python’s abstraction for data. All data in a Python program is represented by objects or by relations between objects.»* То есть не только «данные», но и **отношения** между объектами (ссылки, контейнеры) — тоже часть модели. Далее: *«Every object has an identity, a type and a value. An object’s identity never changes once it has been created.»* — идентификатор неизменен. *«The type of an object determines the operations that the object supports (e.g., does it have a length?) and also defines the possible values for objects of that type.»* — тип задаёт и операции, и множество допустимых значений.

**Практические следствия:**

- **Присваивание не копирует объект:** `a = b` не создаёт копию объекта, на который ссылается `b`. Оно лишь привязывает имя `a` к тому же объекту (в namespace текущего scope). Поэтому после `a = b` выполняется `a is b` и `id(a) == id(b)`.
- **Передача в функцию — передача ссылки (pass-by-object-reference):** при вызове `f(x)` в функцию передаётся не «копия значения», а ссылка на тот же объект. Если внутри `f` изменить изменяемый объект (например, список), изменения видны снаружи; если переназначить имя параметра на другой объект (`x = 0`), внешнее имя `x` не меняется (меняется только локальное имя параметра).
- **Операции вызывают методы объекта:** выражение `a + b` в общем случае приводит к вызову `a.__add__(b)` (или `b.__radd__(a)` при отказе первого). Оператор `in` вызывает `__contains__`. Таким образом, поведение операций полностью определяется типом объекта — нет «магических» примитивов, которые обрабатываются интерпретатором в обход объектной модели.

#### Теоретическое обоснование: Data model (документация Python)

В разделе **The Python Language Reference — Data model** сказано: *«Every object has an identity, a type and a value. An object’s identity never changes once it has been created; you may think of it as the object’s address in memory. The ‘is’ operator compares the identity of two objects; the id() function returns an integer representing its identity.»* Тип объекта определяет операции и атрибуты; значение — данные, которые объект «содержит». Для изменяемых объектов значение может меняться; для неизменяемых — нет. Hashability определяется контрактом: если `a == b`, то `hash(a) == hash(b)`; объект не должен менять значение, влияющее на равенство, пока живёт (по сути — неизменяемость).

**Запомните:** различайте «один объект» (тождество, `is`) и «равные по значению» (равенство, `==`). От этого зависит корректность проверок и использование объектов в словарях/множествах.

### 23.1 id(), type(), is — идентификатор, тип и тождество

**`id(obj)`** возвращает целое число — идентификатор объекта. В течение жизни объекта он не меняется; после того как объект удалён, его id может быть переиспользован для другого объекта. В CPython `id()` обычно возвращает адрес объекта в памяти (для справки), но **не полагайтесь на это** в коде: в Jython или IronPython семантика может отличаться.

**`type(obj)`** возвращает тип объекта (класс). Для проверки типа предпочтительно использовать **`isinstance(obj, SomeType)`** — он учитывает наследование (подклассы считаются подходящими). `type(x) is int` — жёсткая проверка «ровно int».

**`a is b`** — проверка **тождества**: истина только если `a` и `b` ссылаются на **один и тот же** объект. Эквивалентно `id(a) == id(b)`. Используется для сравнения с синглтонами (`x is None`, `x is True`, `x is False`) и когда важно именно «тот же объект», а не «равное значение».

**Почему для None/True/False используют именно `is`:** (1) В языке гарантируется один объект для каждого из них (синглтоны); сравнение по тождеству быстрее и не зависит от переопределения `__eq__` у сравниваемого объекта. (2) **PEP 8** явно рекомендует: *«Comparisons to singletons like None should always be done with is or is not, never the equality operators.»* — то есть `if x is None`, а не `if x == None`.

#### Схема: имя и объект (name binding)

```
    Имя (name)              Объект (object)
    ─────────               ───────────────
    переменная a  ────────►  [1, 2, 3]  (list, id=140234...)
    переменная b  ────────►  тот же объект (a is b → True)
    переменная c  ────────►  [1, 2, 3]  (другой list, id=140235...)
                             (a == c → True, a is c → False)
```

Один объект может иметь много имён (много ссылок); при присваивании создаётся новая ссылка, а не копия объекта.

#### Примеры: id, type, is

```python
# id — уникальный идентификатор объекта
x = [1, 2, 3]
print(id(x))   # например: 140234567890

y = x          # y ссылается на тот же объект
print(id(y))   # то же число, что и id(x)
print(x is y)  # True — один и тот же объект

z = [1, 2, 3] # новый список с теми же значениями
print(id(z))   # другое число
print(x == z)  # True — равны по значению
print(x is z)  # False — разные объекты
```

```python
# type — класс объекта
print(type(42))           # <class 'int'>
print(type("hello"))      # <class 'str'>
print(type([1, 2]))       # <class 'list'>

# isinstance учитывает наследование
class MyList(list):
    pass
ml = MyList([1, 2])
print(type(ml) is list)      # False — тип ровно MyList
print(isinstance(ml, list))  # True — MyList наследует list
```

```python
# is — тождество; для None, True, False принято использовать is
a = None
if a is None:   # правильный способ проверки на None
    print("нет значения")

# Не путать с ==
print((1 == 1) is True)   # True: результат сравнения 1 == 1 — объект True
x = 1
print(x == True)          # True (1 == True в Python)
print(x is True)          # False — разные объекты
```

**Запомните:** для сравнения с `None`, `True`, `False` всегда используйте **`is`**, а не **`==`**. Так вы избежите переопределения `__eq__` и явно показываете намерение «тот же объект».

#### 23.1.1 id(): детали и когда id переиспользуется

Встроенная функция **`id(obj)`** возвращает целое число, уникальное для объекта в течение его **жизни**. После того как объект становится недостижим и сборщик мусора (или reference counting) освобождает память, тот же целочисленный идентификатор может быть выдан **новому** объекту. Поэтому сравнивать **id** двух объектов имеет смысл только пока оба живы: **a is b** эквивалентно **id(a) == id(b)** в момент сравнения.

**Переиспользование id (не полагайтесь в коде):** в CPython при удалении объекта его память может быть возвращена в пул; при создании следующего объекта того же типа интерпретатор может выдать тот же адрес (и тот же **id**). Поэтому **id(obj)** после удаления **obj** может совпасть с **id** другого объекта — не сохраняйте **id** «на потом» для сравнения.

#### 23.1.2 type() и isinstance(): когда что использовать

- **`type(obj)`** возвращает **точный** тип объекта (класс, экземпляром которого является **obj**). **`type(obj) is SomeClass`** — проверка «ровно этот класс»; подклассы **SomeClass** не проходят (например, **type(MyList([1])) is list** — **False**, если **MyList** наследует **list**).
- **`isinstance(obj, SomeClass)`** возвращает **True**, если **obj** является экземпляром **SomeClass** или любого его подкласса. **isinstance(obj, (int, float))** — проверка на несколько типов. Для абстрактных базовых классов (abc) **isinstance** учитывает виртуальное подклассирование (регистрация через **register**).

**Практический вывод:** для проверки типа в обычном коде предпочтительно **isinstance(obj, Type)** — код остаётся корректным при наследовании. Используйте **type(obj) is Type** только когда нужно явно отсечь подклассы (например, разное поведение для «ровно list» и для «подкласс list»).

### 23.2 Сравнение: == и is — равенство и тождество

**`==`** вызывает метод **`__eq__`** объекта. Два объекта могут быть **равны** (`==`), но при этом **разными** объектами (`is` даёт `False`). Например, два списка с одинаковыми элементами: `[1, 2] == [1, 2]` — `True`, но `[1, 2] is [1, 2]` — `False`.

**Когда использовать что:**

- Нужно проверить «то же значение» (для типа) → **`==`**.
- Нужно проверить «тот же объект в памяти» или синглтон (`None`, `True`, `False`) → **`is`**.

Типичная ошибка: писать `if x == None` вместо `if x is None`. Второй вариант предпочтительнее: быстрее и не зависит от возможного переопределения `__eq__` у `x`.

#### 23.2.1 Как работает == : __eq__ и цепочка сравнений

При вычислении **a == b** интерпретатор вызывает **a.__eq__(b)**. Если метод **__eq__** не определён у **a**, или возвращает **NotImplemented**, интерпретатор пробует **b.__radd__**-аналог для равенства — **b.__eq__(a)** (на самом деле для сравнения используется **__eq__** у первого операнда, при **NotImplemented** — у второго). Если оба возвращают **NotImplemented**, то по умолчанию используется сравнение по **is** (то есть **a is b**). Поэтому для пользовательских типов достаточно реализовать **__eq__(self, other)**; возврат **NotImplemented** даёт шанс **other** обработать сравнение (например, при сравнении с числом).

**Важно:** при переопределении **__eq__** рекомендуется также переопределять **__hash__**, если объекты должны быть hashable, и соблюдать контракт «равные — равный хеш». Иначе класс станет unhashable (Python 3 выставляет **__hash__ = None**).

#### Пример: __eq__ с NotImplemented (сравнение с другим типом)

Если ваш тип должен сравниваться с числом (например, **MyInt(5) == 5**), в **__eq__** для «чужого» типа возвращают **NotImplemented** — тогда интерпретатор попробует **other.__eq__(self)** (например, **int.__eq__(5, self)**). Если **other** не умеет сравнивать с вашим типом, в итоге вернётся **False** (или **NotImplemented** приведёт к сравнению по **is**).

```python
class MyInt:
    def __init__(self, val):
        self.val = val
    def __eq__(self, other):
        if isinstance(other, MyInt):
            return self.val == other.val
        if isinstance(other, int):
            return self.val == other
        return NotImplemented  # дать шанс other.__eq__(self)

a = MyInt(5)
print(a == 5)       # True — вызвался a.__eq__(5), вернул True
print(5 == a)       # True — вызвался int.__eq__(5, a); int не знает MyInt, попробует a.__eq__(5)
print(a == "5")     # False — a.__eq__("5") вернул NotImplemented, str.__eq__("5", a) тоже NotImplemented → False
```

#### Таблица: когда использовать is, когда ==

| Ситуация | Использовать | Пример |
|----------|---------------|--------|
| Проверка на **None**, **True**, **False** | **is** | **if x is None** |
| Сравнение значений чисел, строк, списков | **==** | **if a == 100**, **if lst == [1,2]** |
| Проверка «тот же объект в памяти» | **is** | **if obj is cached** |
| Сравнение пользовательских объектов по смыслу | **==** (реализовать **__eq__**) | **if p1 == p2** для точек |
| Сравнение типов с учётом наследования | **isinstance** | **if isinstance(x, list)** |
| Жёсткая проверка «ровно этот тип» | **type(obj) is Type** | **if type(x) is list** |

#### Пример: мутабельный объект и две ссылки

```python
lst = [1, 2, 3]
ref = lst    # ref и lst указывают на один список
ref.append(4)
print(lst)   # [1, 2, 3, 4] — изменился «тот же» объект

lst2 = [1, 2, 3, 4]
print(lst == lst2)  # True — равны по значению
print(lst is lst2)  # False — разные объекты
```

**Запомните:** присваивание не создаёт копию. Если нужно скопировать список — используйте `lst.copy()` или `list(lst)`; для глубокой копии — `copy.deepcopy()` (§26).

### 23.3 Hashability — что можно ключом словаря и элементом множества

Объект **hashable**, если он может быть безопасно использован как ключ словаря или элемент множества. Формально (Data model) объект hashable, если:

1. У него реализован метод **`__hash__`**, возвращающий целое число (int). Вызов встроенной функции **`hash(obj)`** делегируется к **`obj.__hash__()`**.
2. Реализован **`__eq__`** для сравнения с другими объектами (равенство по значению).
3. **Контракт hash/equality:** если **`a == b`** (то есть `a.__eq__(b)` вернул True), то обязательно **`hash(a) == hash(b)`**. Обратное не требуется: разные объекты могут иметь один хеш (коллизия); тогда словарь/множество используют **`__eq__`** для различения. Если бы равные объекты имели разный хеш, при поиске по ключу мы могли бы не найти запись (словарь смотрит в «ведро» по хешу, а объект с тем же значением мог бы попасть в другое ведро).
4. **Неизменяемость по смыслу:** значение **`__hash__`** не должно меняться, пока объект живёт. Иначе объект, уже лежащий в множестве или как ключ словаря, перестал бы находиться (поиск идёт по текущему хешу). Поэтому на практике hashable объекты — те, чьё «значение» не меняется: неизменяемые встроенные типы и пользовательские неизменяемые классы с корректным **`__hash__`** и **`__eq__`**.

**Встроенные типы — сводная таблица:**

| Тип | Hashable? | Примечание |
|-----|-----------|------------|
| **int, float** | Да | Числа; float с NaN не равен сам себе, но хеш определён |
| **str, bytes** | Да | Неизменяемые последовательности |
| **tuple** | Да только если все элементы hashable | `(1, [2])` — не hashable |
| **frozenset** | Да | Неизменяемое множество; элементы должны быть hashable |
| **list** | Нет | Изменяемый |
| **dict** | Нет | Изменяемый |
| **set** | Нет | Изменяемый |
| **bytearray** | Нет | Изменяемый |
| Пользовательский класс | По умолчанию да (по id) | До переопределения **__eq__**; после переопределения **__eq__** без **__hash__** класс становится unhashable |

**Зачем это нужно:** словарь и множество внутри используют хеш-таблицу: по **`hash(key)`** определяется «ведро», в котором искать; при коллизии сравнивают по **`==`**. Если ключ после вставки изменить так, что изменится его хеш, при следующем обращении поиск пойдёт в другое ведро — ключ не будет найден, либо структура может повредиться. Поэтому ключи и элементы множества должны быть hashable и по смыслу неизменяемыми.

#### 23.3.1 Как dict и set используют хеш (упрощённо)

При вставке **d[key] = value** или **s.add(key)** интерпретатор вычисляет **hash(key)** и по значению хеша (по модулю размера таблицы) определяет «ведро» (bucket). Если в этом ведре уже есть элемент с тем же ключом (проверка **key == existing_key**), значение обновляется (dict) или элемент не дублируется (set). Если ключ новый (по **==**), он добавляется в ведро (при переполнении таблица расширяется). При поиске **d[key]** снова вычисляется **hash(key)** — по нему находится ведро; затем по **==** среди ключей в ведре находится нужный. Поэтому: (1) **hash** должен быть стабильным для одного и того же «значения» объекта; (2) равные по **==** объекты обязаны иметь равный **hash**; иначе поиск по ключу даст неверный результат.

#### Примеры: hashable и не hashable

```python
# Hashable: можно ключом dict и элементом set
d = {}
d[1] = "one"           # int — hashable
d["key"] = "value"     # str — hashable
d[(1, 2)] = "tuple"    # tuple из hashable — hashable

s = {1, 2, "hello", (3, 4)}  # все элементы hashable

# Не hashable — нельзя ключом или элементом set
# d[[1, 2]] = "list"   # TypeError: unhashable type: 'list'
# d[{1: 2}] = "dict"   # TypeError: unhashable type: 'dict'
# s.add([1, 2])        # TypeError: unhashable type: 'list'
```

```python
# tuple hashable только если все элементы hashable
t1 = (1, 2, "a")       # hashable
t2 = (1, [2, 3])       # не hashable — список внутри
# hash(t2)             # TypeError: unhashable type: 'list'
```

#### Пользовательский класс как hashable ключ

По умолчанию пользовательские классы имеют **`__hash__`**, наследующийся от **object**: хеш вычисляется по **id** объекта (в CPython — по адресу в памяти). При этом **`__eq__`** по умолчанию сравнивает по **is** (то же тождество). Поэтому два разных экземпляра класса по умолчанию не равны и имеют разный хеш — их можно использовать как ключи словаря. Если вы переопределяете **`__eq__`** (например, сравниваете по полям), вы **должны** либо не переопределять **`__hash__`** (тогда объекты сравниваются по id, а хеш по id — старый контракт сохраняется), либо переопределить **`__hash__`** так, чтобы равные объекты имели равный хеш. При переопределении **`__eq__`** без переопределения **`__hash__`** Python 3 автоматически помечает класс как unhashable: встроенный **object** устанавливает **`__hash__ = None`** для такого класса. Причина: по умолчанию **__hash__** наследуется от **object** и вычисляется по **id**; два разных экземпляра с **id(a) != id(b)** имели бы **hash(a) != hash(b)**. Если при этом **a == b** по вашему **__eq__**, контракт «равные объекты — равный хеш» нарушился бы; словарь и множество могли бы работать некорректно. Поэтому язык запрещает использовать такие объекты как ключи/элементы множества, пока вы явно не определите **__hash__** так, чтобы равные объекты имели равный хеш.

```python
class Point:
    __slots__ = ("x", "y")  # неизменяемость по смыслу — не добавляем поля
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return (self.x, self.y) == (other.x, other.y)
    def __hash__(self):
        return hash((self.x, self.y))  # равные точки — равный хеш

p1 = Point(1, 2)
p2 = Point(1, 2)
print(p1 == p2)   # True
print(hash(p1) == hash(p2))  # True
d = {p1: "first"}
print(d[p2])      # "first" — p2 находит ключ p1 по равенству
```

**Запомните:** если объект должен быть ключом словаря или элементом множества, он должен быть **hashable** и по смыслу **неизменяемым** (чтобы хеш и равенство не менялись).

### 23.4 Кеш мелких целых чисел (implementation detail)

В CPython целые числа в диапазоне от **-5 до 256** включительно **кешируются**: при создании такого числа интерпретатор не аллоцирует новый объект, а возвращает ссылку на уже существующий объект из статического массива (или кеша). Поэтому для этих чисел **`a is b`** может быть **True**, если значения совпадают. Диапазон **-5..256** выбран исторически: числа от -5 до 256 часто встречаются в коде (индексы, счётчики, коды символов ASCII); кеширование экономит память и ускоряет создание объектов. **Почему именно -5:** в ранних версиях CPython в коде интерпретатора использовались малые отрицательные числа (например, -1 для «не найдено»); кеш включал небольшой отрицательный диапазон. **Почему 256:** верхняя граница байта (0–255) и типичные маленькие константы; 256 — естественная граница для «часто встречающихся» целых. **Как проверить кеш в своей сборке:** в интерактивной консоли выполнить **a, b = 100, 100; print(a is b)** — в CPython обычно **True**; **a, b = 257, 257; print(a is b)** — в CPython часто **False** (257 вне кеша). Для одного и того же литерала **257** в одном выражении интерпретатор может вернуть один объект (**257 is 257** в одной строке может быть **True** из-за оптимизации); в разных строках или при вычислении **256 + 1** — уже другой объект. **Полагаться на это в коде нельзя** — это деталь реализации CPython; в других реализациях (Jython, PyPy) кеш может отсутствовать или иметь другой диапазон; в будущих версиях CPython диапазон может измениться. Языковая гарантия — только «равенство по значению» через **`==`**.

#### Пример (иллюстрация, не для опоры в коде)

```python
a = 100
b = 100
# В CPython: a is b может быть True (кеш)
print(a is b)  # True в типичной сборке CPython

c = 300
d = 300
# В CPython: c is d часто False — вне кеша
print(c is d)  # может быть False
print(c == d)  # всегда True
```

**Запомните:** для сравнения чисел всегда используйте **`==`**. Проверка **`is`** для чисел — неправильный стиль и может дать разные результаты в разных реализациях или для чисел вне диапазона кеша.

#### Граничные случаи и типичные ошибки (§23)

- **Проверка на None:** всегда **`x is None`**, не **`x == None`**. У **None** может быть переопределён **__eq__** в теории, но главное — **is** явно выражает «тот же объект» и быстрее; PEP 8 это явно требует.
- **Проверка типа:** предпочтительно **`isinstance(obj, Type)`**, а не **`type(obj) is Type`** — isinstance учитывает наследование (подклассы проходят проверку). Исключение: когда нужно отсечь подклассы (например, «ровно list, не MyList») — тогда **`type(obj) is list`**.
- **Ключи словаря и элементы set:** только hashable объекты. Список или dict нельзя положить в set или использовать ключом dict — будет **TypeError: unhashable type**.
- **tuple как ключ:** tuple hashable только если все элементы hashable. **`(1, [2, 3])`** не hashable — вызов **hash((1, [2, 3]))** даёт **TypeError**.
- **float и NaN:** **float('nan')** не равен сам себе (**`float('nan') == float('nan')`** — False); но **hash(float('nan'))** определён и одинаков для всех NaN. Такие объекты технически hashable, но использование NaN как ключа словаря может привести к неочевидному поведению при поиске по «равенству».
- **Изменять объект после добавления в set/dict:** если объект hashable, но вы после **s.add(obj)** или **d[obj] = v** изменили его так, что изменился **hash(obj)** или результат **obj == other** (для пользовательского класса с **__hash__** и **__eq__** по полям — изменили поле), то поиск **obj in s** или **d[obj]** может не найти запись (поиск идёт по текущему хешу). Поэтому ключи и элементы set должны быть **неизменяемыми по смыслу**: не менять поля, участвующие в **__hash__** и **__eq__**, после добавления в коллекцию.
- **Переопределение __eq__ без __hash__:** в Python 3 класс автоматически становится unhashable (эквивалент **`__hash__ = None`**), чтобы не нарушать контракт hash/equality. Если нужен hashable тип с кастомным равенством — реализуйте **__hash__** так, чтобы равные объекты имели равный хеш.

#### Плохо vs лучше (§23)

| Плохо | Лучше | Причина |
|-------|--------|---------|
| `if x == None` | `if x is None` | Явная проверка тождества, стиль PEP 8 |
| `if type(x) is int` | `if isinstance(x, int)` | Учитывает подклассы |
| `a is 100` (для сравнения значений) | `a == 100` | is для чисел — деталь реализации |
| Использовать list/dict как ключ или в set | Использовать tuple/frozenset или неизменяемые типы | Требуется hashable |

#### Резюме по §23 (объектная модель)

- В Python всё — **объект**; у каждого есть **id**, **type** и **value**.
- **Тождество:** один и тот же объект — проверяйте **`is`** и **`id()`**; для `None`, `True`, `False` используйте **`is`**.
- **Равенство:** равные по значению — проверяйте **`==`** (вызывает `__eq__`).
- **Hashable:** объекты с стабильным `__hash__` и `__eq__`, неизменяемые по смыслу; нужны для ключей dict и элементов set. Встроенные неизменяемые типы (int, str, tuple из hashable) — hashable.
- Мелкие целые (-5..256) в CPython кешируются — не используйте `is` для сравнения чисел.
