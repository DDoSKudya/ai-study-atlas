## 12. functools

Модуль **functools** предоставляет инструменты для работы с функциями: фиксация аргументов, свёртка, кеширование, сохранение метаданных при обёртке, перегрузка по типу и генерация методов сравнения.

### Что такое functools и зачем он нужен

**partial** устраняет повторение одних и тех же аргументов при множественных вызовах. **reduce** реализует свёртку итератора в одно значение (сумма, произведение, конкатенация и т.д.). **lru_cache** и **cache** кешируют результаты вызовов по аргументам — для дорогих или повторяющихся вычислений. **wraps** сохраняет ****name****, ****doc**** обёрнутой функции при написании декораторов. **cmp_to_key** позволяет использовать старый стиль сравнения (функция двух аргументов) там, где API ожидает key function (sorted, min, max). **cached_property** — свойство, вычисляемое один раз при первом обращении и затем кешируемое. **singledispatch** и **singledispatchmethod** дают перегрузку по типу первого аргумента без ручных **if isinstance**. **total_ordering** генерирует недостающие методы сравнения по ****eq**** и одному из ****lt****, ****le****, ****gt****, ****ge****.

### partial

- [ ] **functools.partial(func, \*args, **kwargs)** — возвращает вызываемый объект, который при вызове вызывает **func** с аргументами **args** и **kwargs\*\*, дополненными переданными при вызове. Аргументы, переданные при вызове partial-объекта, добавляются к зафиксированным (позиционные — в конец; именованные могут переопределять зафиксированные).

**Пример**

```python
from functools import partial
def f(a, b, c): return a + b + c
g = partial(f, 1, 2)
g(3)  # 6
g(c=10)  # 13
```

### reduce

- [ ] **functools.reduce(func, iterable, initial=None)** — свёртка: берётся начальное значение (если задан **initial** — оно; иначе первый элемент итератора), затем для каждого следующего элемента вызывается **func(accumulator, element)**; результат становится новым accumulator. Итог — одно значение. Пример: **reduce(lambda a, b: a + b, [1, 2, 3], 0)** → 6.

**Пошагово:** при **reduce(add, [1, 2, 3], 0)**: accumulator = 0; add(0, 1) → 1; add(1, 2) → 3; add(3, 3) → 6. Возвращается 6. Без **initial** первый элемент — начальное значение, свёртка идёт со второго.

### lru_cache и cache

- [ ] **functools.lru_cache(maxsize=128, typed=False)** — декоратор: кеширует результаты вызова по хешируемым аргументам. **maxsize** — максимум записей (при переполнении вытесняется наименее недавно использованная); **typed=True** — аргументы разных типов (1 и 1.0) кешируются отдельно. **@functools.cache** (3.9+) — то же, что **lru_cache(maxsize=None)** (неограниченный кеш).

### wraps

- [ ] **functools.wraps(wrapped)** — декоратор для обёртки: копирует ****name****, ****doc****, ****module****, ****annotations**** и при необходимости ****qualname****, ****wrapped**** с **wrapped** на обёртку. Без **wraps** после декорирования **f.**name**** будет именем обёртки, а не исходной функции.

### cmp_to_key

**Что такое и зачем**

- [ ] **functools.cmp_to_key(cmp_func)** — преобразует **функцию сравнения двух аргументов** (в стиле C/Python 2: возвращает -1, 0 или 1) в **key function** для **sorted**, **min**, **max**. В Py3 **sorted** и др. принимают только **key** (функция одного аргумента); **cmp_to_key** позволяет использовать старый стиль **cmp(a, b)** без переписывания логики на ключ. **Применения:** миграция кода с Py2; сортировка по сложному сравнению, которое проще выразить как «сравнить пару», чем как «вернуть ключ».
- [ ] Внутри: при сравнении двух элементов **x**, **y** type checker вызывает **cmp_func(x, y)**; результат -1/0/1 определяет порядок. Реализация оборачивает каждый элемент в объект-обёртку с ****lt****, вызывающим **cmp_func**. **Пример:** **sorted(items, key=functools.cmp_to_key(my_cmp))** — **my_cmp(a, b)** возвращает -1, 0 или 1.

**Пример cmp_to_key**

```python
import functools
def version_cmp(a, b):
    # сравнить версии "1.2.3" как кортежи (1,2,3)
    ta = tuple(map(int, a.split('.')))
    tb = tuple(map(int, b.split('.')))
    return (ta > tb) - (ta < tb)
versions = ["2.1.0", "1.10.0", "1.9.0"]
sorted(versions, key=functools.cmp_to_key(version_cmp))
# ['1.9.0', '1.10.0', '2.1.0']
```

### cached_property, singledispatch, total_ordering

**cached_property — свойство с кешем**

- [ ] **functools.cached_property** (3.8+) — декоратор для **метода без аргументов** (кроме self): при **первом** обращении к атрибуту с тем же именем вызывается метод, результат **сохраняется в экземпляре** под этим именем; при следующих обращениях возвращается сохранённое значение (метод не вызывается). **Применения:** дорогие вычисления (парсинг, загрузка данных), которые не должны повторяться; ленивая инициализация с кешем. Отличие от **property**: значение кешируется в **instance.**dict****; после первого обращения атрибут можно даже удалить (**del obj.attr**), тогда при следующем обращении метод вызовется снова.
- [ ] **Граничный случай:** не thread-safe — при одновременном первом обращении из нескольких потоков метод может вызваться несколько раз. Для потокобезопасности использовать **lock** внутри метода или внешний кеш.

**Пример cached_property**

```python
from functools import cached_property
class Data:
    def __init__(self, raw):
        self.raw = raw
    @cached_property
    def parsed(self):
        print("Parsing...")  # только при первом обращении
        return json.loads(self.raw)
d = Data('{"a": 1}')
d.parsed   # Parsing...  → {'a': 1}
d.parsed   # {'a': 1}  (без второго вызова parsed())
```

**singledispatch**

- [ ] **functools.singledispatch** — декоратор функции: регистрируются реализации для разных типов **первого аргумента** (**@func.register(str)**); при вызове выбирается реализация по типу первого аргумента. Базовая реализация (по умолчанию) декорируется первой; для других типов — **@func.register(SomeType)**. **singledispatchmethod** — то же для методов класса (тип первого аргумента после self).

**Пример singledispatch**

```python
from functools import singledispatch

@singledispatch
def serialize(obj):
    return str(obj)

@serialize.register(list)
def _(obj):
    return "[" + ", ".join(serialize(x) for x in obj) + "]"

@serialize.register(dict)
def _(obj):
    return "{" + ", ".join(f"{k!r}: {serialize(v)}" for k, v in obj.items()) + "}"

serialize([1, 2, 3])   # вызывает _ для list
serialize({"a": 1})    # вызывает _ для dict
serialize(42)          # базовая реализация
```

**total_ordering**

- [ ] **functools.total_ordering** — декоратор класса: если реализованы ****eq**** и **хотя бы один** из ****lt****, ****le****, ****gt****, ****ge****, декоратор **добавляет остальные** методы сравнения. Удобно для типов, где достаточно задать равенство и один порядок (например ****lt****), а остальные выводятся логически. **Применения:** классы-значения с сортировкой без ручного написания всех шести методов.

**Пример total_ordering**

```python
from functools import total_ordering
@total_ordering
class Version:
    def __init__(self, major, minor):
        self.major, self.minor = major, minor
    def __eq__(self, other):
        return (self.major, self.minor) == (other.major, other.minor)
    def __lt__(self, other):
        return (self.major, self.minor) < (other.major, other.minor)
# __le__, __gt__, __ge__ сгенерированы автоматически
Version(1, 2) <= Version(1, 3)  # True
```

## 13. itertools

Модуль **itertools** предоставляет итераторы и функции для комбинаторики, срезов, группировки и объединения итераторов. Все функции работают **лениво** — элементы порождаются по мере итерации, без создания полных списков в памяти.

### Что такое itertools и зачем он нужен

**Проблема без itertools:** объединение нескольких списков в один цикл, «разветвление» одного итератора на несколько потребителей, группировка подряд идущих элементов, комбинации и перестановки — всё это можно написать вручную, но код получается длинным и легко ошибиться (например, забыть сортировку перед **groupby**). **itertools** даёт готовые, эффективные и проверенные примитивы.

**Теоретическое обоснование:** Итераторы и генераторы в Python позволяют обрабатывать потоки данных без загрузки всего в память. **itertools** построен на этой модели: **chain**, **tee**, **cycle**, **islice** и т.д. возвращают итераторы; потребление элементов откладывается до итерации. Комбинаторика (**permutations**, **combinations**, **product**) тоже ленива — число комбинаций может быть огромным, но память не растёт.

### chain и chain.from_iterable

- [ ] **chain(\*iterables)** — итератор по элементам первого итератора, затем второго, и т.д. **chain.from_iterable(iterable_of_iterables)** — то же, но аргумент один итератор, элементы которого — итераторы (удобно для «плоского» обхода вложенных структур).

**Пример**

```python
from itertools import chain
list(chain([1, 2], [3, 4]))           # [1, 2, 3, 4]
list(chain.from_iterable([[1, 2], [3]]))  # [1, 2, 3]
```

### tee

- [ ] **tee(iterable, n=2)** — разветвляет один итератор на **n** независимых итераторов. Внутри используется кеш: элементы исходного итератора запрашиваются по мере потребления любым из tee-итераторов и сохраняются для остальных. **Исходный итератор не должен использоваться после вызова tee** — иначе он «обгонит» кеш и данные будут пропущены.

**Пример**

```python
from itertools import tee
it1, it2 = tee([1, 2, 3])
list(it1), list(it2)  # [1, 2, 3], [1, 2, 3]
```

### cycle, repeat

**Что такое и зачем**

- [ ] **cycle(iterable)** — **бесконечный** итератор: выдаёт элементы iterable по порядку, по исчерпании снова с начала. **Применения:** циклические паттерны (очередь воркеров по кругу), бесконечный поток из конечного набора значений. **repeat(object[, times])** — повтор **object** **times** раз (или **бесконечно**, если **times** не задан). **Применения:** заполнение (например **zip(keys, repeat(value))**), бесконечная константа в **map**/ **zip**.

**Пример cycle и repeat**

```python
from itertools import cycle, repeat, islice
list(islice(cycle([1, 2, 3]), 7))   # [1, 2, 3, 1, 2, 3, 1]
list(repeat(10, 3))                 # [10, 10, 10]
list(zip([1, 2, 3], repeat(0)))    # [(1, 0), (2, 0), (3, 0)]
```

### islice, takewhile, dropwhile

**Что такое и зачем**

- [ ] **islice(iterable, start, stop[, step])** — **срез** итератора без материализации всего в память: как **iterable[start:stop:step]**, но для любого итератора (в т.ч. генератора). **start** по умолчанию 0, **step** по умолчанию 1. **Применения:** «первые N элементов», «пропустить первые K», пагинация по потоку. **takewhile(pred, iterable)** — элементы **пока pred(elem)** истинен; как только pred вернёт False, итерация **останавливается** (остаток iterable не потребляется). **dropwhile(pred, iterable)** — **пропускает** элементы, пока pred истинен; затем отдаёт **все остальные** без дальнейшей проверки pred. **Применения:** пропуск заголовков/комментариев («пока строка начинается с #»), взятие «пока выполняется условие».

**Пример islice, takewhile, dropwhile**

```python
from itertools import islice, takewhile, dropwhile
list(islice(range(10), 2, 7, 2))   # [2, 4, 6]
list(takewhile(lambda x: x < 5, [1, 3, 5, 2, 7]))  # [1, 3]  (остановка на 5)
list(dropwhile(lambda x: x < 5, [1, 3, 5, 2, 7]))  # [5, 2, 7]
```

### groupby

- [ ] **groupby(iterable, key=None)** — группирует **подряд идущие** элементы с одинаковым значением **key(elem)**. Возвращает итератор пар **(key_value, group_iterator)**. **Важно:** **groupby** не сортирует; он объединяет только соседние элементы с одинаковым key. Поэтому **перед groupby обычно нужна сортировка** по тому же key: **groupby(sorted(iterable, key=key), key=key)**.

**Пример**

```python
from itertools import groupby
data = [("a", 1), ("a", 2), ("b", 3)]
for k, g in groupby(data, key=lambda x: x[0]):
    print(k, list(g))  # a [(a,1), (a,2)], b [(b,3)]
```

**Граничный случай: groupby без сортировки**

Если не отсортировать, элементы с одним ключом, но разнесённые по итератору, попадут в **разные** группы:

```python
data = [("a", 1), ("b", 2), ("a", 3)]  # два "a", но не подряд
for k, g in groupby(data, key=lambda x: x[0]):
    print(k, list(g))
# a [(a,1)]
# b [(b,2)]
# a [(a,3)]   — вторая группа с ключом "a"
```

После сортировки по ключу: **sorted(data, key=lambda x: x[0])** → **[("a", 1), ("a", 3), ("b", 2)]**; тогда **groupby** даст одну группу **"a"** с двумя элементами и одну группу **"b"**.

### zip_longest

- [ ] **zip_longest(\*iterables, fillvalue=None)** — как **zip**, но итерация идёт до **самой длинной** последовательности; для коротких недостающие позиции заполняются **fillvalue**. **Применения:** выравнивание списков разной длины, обработка «по самой длинной» с явным заполнением. **Пример:** **list(zip_longest([1, 2], [10, 20, 30], fillvalue=0))** → **[(1, 10), (2, 20), (0, 30)]**.

### permutations, combinations, product

- [ ] **permutations(iterable, r=None)** — все **перестановки** длины **r** (порядок важен); при **r=None** — полная длина. **combinations(iterable, r)** — все **сочетания** длины **r** (порядок не важен). **combinations_with_replacement(iterable, r)** — сочетания с повторениями. **product(\*iterables, repeat=1)** — **декартово произведение**: все комбинации по одному элементу из каждого итератора; **repeat=N** — один и тот же итератор N раз. **Применения:** перебор вариантов, тесты (все пары настроек), комбинаторика. **Пример:** **list(combinations([1,2,3], 2))** → **[(1,2), (1,3), (2,3)]**; **list(product([0,1], repeat=2))** → **[(0,0), (0,1), (1,0), (1,1)]**.

### accumulate, pairwise

- [ ] **accumulate(iterable, func=operator.add)** — итератор **накопленных** значений: первый — первый элемент, второй — **func(первый, второй)**, третий — **func(предыдущий результат, третий)** и т.д. По умолчанию **func=operator.add** — накопленная сумма. **Применения:** бегущая сумма, префиксные минимумы/максимумы (передать **min**/ **max**). **pairwise(iterable)** (3.10+) — пары **(a, b)** подряд идущих элементов: **(s[0], s[1])**, **(s[1], s[2])**, … **Применения:** разности соседних, «окно из двух», границы интервалов.

**Пример accumulate и pairwise**

```python
from itertools import accumulate, pairwise
list(accumulate([1, 2, 3, 4]))       # [1, 3, 6, 10]
list(pairwise([1, 2, 3, 4]))        # [(1, 2), (2, 3), (3, 4)]
```

### starmap, compress, filterfalse

- [ ] **starmap(func, iterable)** — для каждого элемента iterable (каждый — **последовательность**) вызывается **func(\*element)**. Удобно, когда данные уже в виде кортежей аргументов. **compress(data, selectors)** — оставляет элементы **data[i]**, для которых **selectors[i]** истинен (по длине обрезается по короткому). **filterfalse(pred, iterable)** — элементы, для которых **pred(elem)** **ложен** (обратное к **filter**). **Применения:** **starmap** — вызовы с распаковкой из списка пар; **compress** — маска/фильтр по булеву списку; **filterfalse** — «все, что не подходит под pred».

## 14. contextlib

Модуль **contextlib** предоставляет утилиты для создания и комбинирования контекстных менеджеров: **@contextmanager** (генераторный контекст), **suppress**, **redirect_stdout**/ **redirect_stderr**, **ExitStack**, **nullcontext**.

### Что такое contextlib и зачем он нужен

**Контекстный менеджер** — объект с методами ****enter**** и ****exit****, используемый в **with**; при входе в блок вызывается ****enter****, при выходе (нормальном или по исключению) — ****exit****. Типичное применение: гарантированное освобождение ресурсов (файл, блокировка, соединение). Писать класс с ****enter****/ ****exit**** для простых случаев многословно. **@contextmanager** позволяет описать контекст **генератором**: код до **yield** — вход, код после **yield** — выход; исключения из блока **with** пробрасываются в генератор и могут быть обработаны. **suppress** подавляет указанные исключения в блоке. **redirect_stdout**/ **redirect_stderr** перенаправляют потоки вывода в другой объект (например, **io.StringIO**) в контексте — удобно для тестов и перехвата вывода. **ExitStack** управляет несколькими контекстами динамически (количество и набор контекстов могут зависеть от условий). **nullcontext** — пустой контекст; полезно, когда контекст нужен только в одной ветке (например, **cm = open(f) if f else nullcontext()**).

### @contextmanager

- [ ] **@contextmanager** — декоратор генераторной функции. При входе в **with** выполняется код до **yield**; значение **yield** (если есть) попадает в переменную после **as**. При выходе из блока выполняется код после **yield**; если было исключение, оно пробрасывается в генератор и может быть поймано в **except** внутри генератора; **raise** без аргументов пробрасывает то же исключение дальше.

**Пример**

```python
from contextlib import contextmanager

@contextmanager
def managed_resource():
    resource = acquire()
    try:
        yield resource
    finally:
        release(resource)

with managed_resource() as r:
    use(r)
```

### suppress

**Что такое и зачем**

- [ ] **contextlib.suppress(\*exceptions)** — контекстный менеджер: внутри блока **with** указанные исключения **подавляются** (не пробрасываются наружу). Остальные исключения пробрасываются как обычно. **Применения:** игнорировать ожидаемые исключения (например **FileNotFoundError** при удалении файла, который может отсутствовать) без пустого **except** или **try/except** на несколько строк. Явнее и безопаснее, чем голый **except**.

**Пример suppress**

```python
from contextlib import suppress
with suppress(FileNotFoundError):
    os.remove("maybe_missing.txt")  # если файла нет — ничего не делать
# вместо:
# try: os.remove("maybe_missing.txt")
# except FileNotFoundError: pass
```

### redirect_stdout, redirect_stderr

- [ ] **contextlib.redirect_stdout(new_target)**, **redirect_stderr(new_target)** — в контексте **with** подменяют **sys.stdout** или **sys.stderr** на **new_target** (объект с методом **write**). При выходе восстанавливают прежние значения. **Применения:** перехват вывода в тестах (**io.StringIO**, затем **.getvalue()**); перенаправление в файл или логгер; подавление вывода сторонней функции. **Пример:** **with redirect_stdout(io.StringIO()) as buf: print("hello"); buf.getvalue()** → **'hello\n'**.

### ExitStack

**Что такое и зачем**

- [ ] **contextlib.ExitStack** — контекстный менеджер для **динамического** набора контекстов и callbacks: контексты добавляются в цикле или по условию, при выходе из **with** все откатываются в **обратном** порядке. **Применения:** открыть N файлов из списка (число неизвестно заранее); комбинировать контексты из конфига; гарантированно закрыть всё при ошибке после частичного открытия.
- [ ] **.enter_context(cm)** — войти в контекст **cm** и зарегистрировать его ****exit**** при выходе из ExitStack; возвращает результат **cm.**enter**()**. **.push(callback)** — зарегистрировать callback (вызовется при выходе в LIFO). **.close()** — явно откатить все зарегистрированные (вызывается автоматически при выходе из **with**).

**Пример ExitStack**

```python
from contextlib import ExitStack
paths = ["a.txt", "b.txt", "c.txt"]
with ExitStack() as stack:
    files = []
    for p in paths:
        f = stack.enter_context(open(p))  # при ошибке уже открытые закроются
        files.append(f)
    for f in files:
        process(f)
```

### nullcontext

- [ ] **contextlib.nullcontext(enter_result=None)** — контекстный менеджер: при входе ничего не делает и возвращает **enter_result** (по умолчанию **None**); при выходе тоже ничего не делает. **Применения:** условный контекст, когда в одной ветке нужен реальный контекст (например **open(path)**), в другой — «ничего не делать»: **ctx = open(path) if path else nullcontext(); with ctx: ...**. Или **with nullcontext(42) as x:** — **x** будет 42.
