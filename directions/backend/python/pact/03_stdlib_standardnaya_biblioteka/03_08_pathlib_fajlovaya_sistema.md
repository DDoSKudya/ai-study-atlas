## 8. pathlib и файловая система

Модуль **pathlib** (PEP 428, с Python 3.4) предоставляет **объектно-ориентированный API** для работы с путями и файловой системой. Путь представляется объектом, а не строкой: операции выражаются методами и оператором **/**; кроссплатформенность (Unix/Windows) учитывается через подклассы **PosixPath** и **WindowsPath**.

### Что такое pathlib и зачем он нужен

**Проблема без pathlib:** пути задавались строками (`'/etc/hosts'`, `'data\\file.txt'`); конкатенация через **os.path.join()**; проверка существования — **os.path.exists()**; чтение файла — **open(path).read()**. Разделители путей различаются на Windows (**\\**) и Unix (**/**); логика с **os.path** работала, но код был многословным и смешивал строки с вызовами функций.

**Решение pathlib:** путь — это **объект** (экземпляр **Path** или **PurePath**). Конкатенация — оператор **/** (`p / 'file.txt'`). Чтение/запись — методы **.read_text()**, **.write_text()**, **.read_bytes()**, **.write_bytes()**. Итерация по каталогу — **.iterdir()**, **.glob()**, **.rglob()**. Класс **Path** на каждой ОС — свой (**PosixPath** на Unix, **WindowsPath** на Windows), поэтому разделители и нормализация учитываются автоматически. Код становится короче и читается как «действия с путём», а не как манипуляции со строками.

**Теоретическое обоснование:** Абстракция «путь как объект» соответствует принципу «объектно-ориентированного» дизайна: данные (путь) и операции (чтение, запись, обход) сгруппированы. Один тип **Path** инкапсулирует различия ОС; пользователь не пишет **if os.name == 'nt'**. PEP 428 явно мотивирует pathlib как предпочтительный способ работы с путями в новом коде.

### Path, PurePath, варианты по ОС

- [ ] **PurePath** — базовый класс для **манипуляций строками путей без обращения к диску**. Не проверяет существование файлов, не читает/не пишет. Подклассы: **PurePosixPath** (Unix-стиль с **/**), **PureWindowsPath** (Windows-стиль с **\\** и дисками). Используется когда путь нужен только для конкатенации, разбора частей (**.name**, **.suffix**, **.parent**) или передачи в API, ожидающее путь-строку (**.as_posix()**, **str(p)**).
- [ ] **Path** — наследует **PurePath** и добавляет **операции I/O**: **.read_text()**, **.write_text()**, **.exists()**, **.is_file()**, **.is_dir()**, **.mkdir()**, **.unlink()**, **.iterdir()**, **.glob()**, **.rglob()** и т.д. Подклассы: **PosixPath** (Unix), **WindowsPath** (Windows). При импорте **from pathlib import Path** на текущей ОС **Path** — это **PosixPath** или **WindowsPath**.
- [ ] **PurePath vs Path:** PurePath — только «арифметика» путей (части, конкатенация, нормализация); Path — ещё и доступ к файловой системе. PurePath подходит для кроссплатформенной логики без открытия файлов (например, формирование имён выходных файлов по шаблону). Path — когда нужно читать/писать или проверять существование.

**Терминология**

| Термин            | Определение                                                                      |
| ----------------- | -------------------------------------------------------------------------------- |
| **Pure path**     | Путь без доступа к ФС; только строковые операции над путём                       |
| **Concrete path** | Path (PosixPath/WindowsPath); путь с методами I/O                                |
| **Anchor**        | На Windows — префикс диска (например `C:`); на Posix — может быть пустым или `/` |
| **Drive**         | Буква диска на Windows (`C:`)                                                    |
| **Root**          | Корень пути: `/` на Posix, `\` или `C:\` на Windows                              |
| **Parent**        | Родительский каталог (**.parent**); у корня — сам корень                         |
| **Name**          | Последний компонент пути (**.name**); для каталога — имя каталога                |
| **Suffix**        | Расширение файла (**.suffix**); **.suffixes** — список всех суффиксов            |

### Основные методы Path

- [ ] **.read_text(encoding=None, errors=None)** — открывает файл в текстовом режиме, читает всё содержимое, закрывает файл. Возвращает **str**. Кодировка по умолчанию — из **locale** (обычно UTF-8). При ошибке (файл не найден, нет прав) выбрасывается **OSError** (или **FileNotFoundError**).
- [ ] **.write_text(data, encoding=None, errors=None)** — открывает файл для записи (перезаписывает), записывает строку **data**, закрывает файл. Родительские каталоги не создаёт автоматически; при необходимости вызвать **.parent.mkdir(parents=True, exist_ok=True)**.
- [ ] **.read_bytes()**, **.write_bytes(data)** — то же для бинарного режима; возвращают/принимают **bytes**.
- [ ] **.glob(pattern)** — итератор по путям в **этом** каталоге, совпадающим с **pattern** (например `*.py`, `data/*.json`). Синтаксис как в **pathlib**: **\*** — любой символ в компоненте, **\*\*** — любое число каталогов (в **rglob**). **.rglob(pattern)** — рекурсивный обход всех подкаталогов; эквивалент **.glob('**/' + pattern)\*\*.
- [ ] **Path.walk(top_down=True)** (3.12+) — рекурсивный обход, аналог **os.walk()**: генератор кортежей **(root, dirs, files)**; **root** — Path текущего каталога, **dirs** — список имён подкаталогов (можно изменять in-place, чтобы пропустить обход), **files** — имена файлов. **top_down=True** — сначала родитель, потом дети; **False** — сначала дети, потом родитель.
- [ ] **Оператор /** — конкатенация: **Path('/etc') / 'hosts'** → **PosixPath('/etc/hosts')**; **Path('data') / 'sub' / 'file.txt'**. Правая часть может быть строкой или другим Path. Разделитель вставляется автоматически в соответствии с ОС.
- [ ] **.resolve(strict=False)** — абсолютный **канонический** путь: разрешаются символические ссылки (.**.resolve()**), нормализуются **.** и **..**. Если **strict=True** и путь не существует — **FileNotFoundError**. **.absolute()** — абсолютный путь без разрешения symlink (просто присоединяет текущий **Path** к **Path.cwd()** при необходимости).
- [ ] **Path.cwd()** — текущий рабочий каталог (как **os.getcwd()**, но возвращает **Path**). **Path.home()** — домашний каталог пользователя (как **os.path.expanduser('~')**).

**Пошагово: что происходит при вызове (Path('data') / 'file.txt').read_text()**

1. **Path('data')** создаёт объект Path для каталога **data** (относительно текущего каталога).
2. **/ 'file.txt'** даёт новый Path **data/file.txt** (или **data\\file.txt** на Windows).
3. **.read_text()** открывает этот путь в текстовом режиме (**open(path, 'r', encoding=...)**, кодировка по умолчанию), читает всё содержимое через **read()**, закрывает файл и возвращает строку. Если файл или каталог не существует — **FileNotFoundError**.

**Примеры**

```python
from pathlib import Path

# Текущий каталог и конкатенация
p = Path('.') / 'data' / 'file.txt'
p.write_text('hello')
print(p.read_text())   # hello

# Рекурсивный поиск всех .py
for f in Path('.').rglob('*.py'):
    print(f)

# Части пути
p = Path('/etc/hosts')
p.name      # 'hosts'
p.parent    # PosixPath('/etc')
p.suffix    # ''
p = Path('report.pdf')
p.stem      # 'report'
p.suffix    # '.pdf'

# Существование и тип
Path('file.txt').exists()
Path('file.txt').is_file()
Path('dir').is_dir()

# Создание каталога (родители при необходимости)
Path('a/b/c').mkdir(parents=True, exist_ok=True)

# walk (3.12+)
for root, dirs, files in Path('.').walk():
    for f in files:
        print(root / f)
```

**Граничные случаи**

- Пустой путь: **Path()** на Posix — **'.'**; на Windows — текущий каталог текущего диска.
- **Path('.')** и **Path('')** — текущий каталог; **.resolve()** даёт абсолютный путь к нему.
- Символические ссылки: **.resolve()** разворачивает их в реальный путь; **.absolute()** — нет. **.is_symlink()** — проверка, является ли путь symlink.
- Несуществующий путь: **.exists()** вернёт **False**; **.read_text()** вызовет **FileNotFoundError**. **.resolve(strict=False)** не выбрасывает ошибку для несуществующего пути; **strict=True** — выбрасывает.

### Сравнение pathlib и os.path

| Задача            | os.path / os                               | pathlib                                                |
| ----------------- | ------------------------------------------ | ------------------------------------------------------ |
| Склеить путь      | **os.path.join('dir', 'sub', 'file.txt')** | **Path('dir') / 'sub' / 'file.txt'**                   |
| Имя файла         | **os.path.basename(p)**                    | **Path(p).name**                                       |
| Родитель          | **os.path.dirname(p)**                     | **Path(p).parent**                                     |
| Расширение        | **os.path.splitext(p)[1]**                 | **Path(p).suffix**                                     |
| Существует?       | **os.path.exists(p)**                      | **Path(p).exists()**                                   |
| Это файл?         | **os.path.isfile(p)**                      | **Path(p).is_file()**                                  |
| Прочитать текст   | **open(p).read()** (нужно закрыть)         | **Path(p).read_text()** (закрывает сам)                |
| Записать текст    | **open(p, 'w').write(s)**                  | **Path(p).write_text(s)**                              |
| Список файлов     | **os.listdir(p)** (имена)                  | **list(Path(p).iterdir())** (Path объекты)             |
| Рекурсивный обход | **os.walk(p)**                             | **Path(p).rglob('\*')** или **Path(p).walk()** (3.12+) |

**Почему pathlib предпочтителен в новом коде:** объектный API, один стиль для всех операций, автоматическая нормализация разделителей под ОС, методы **read_text**/ **write_text** без явного open/close, цепочки вида **Path('.') / 'data' / 'file.txt'** читаются естественно. **os.path** и **os** по-прежнему нужны для низкоуровневых вызовов (**os.scandir**, **os.sendfile**, переменные окружения и т.д.) и совместимости со старым кодом.

### Частые сценарии с pathlib

- **Прочитать конфиг из каталога рядом со скриптом:** **Path(**file**).resolve().parent / 'config.json'**; затем **.read_text()** или **json.loads(...)**.
- **Создать каталог и записать в него файл:** **p = Path('out') / 'sub' / 'file.txt'; p.parent.mkdir(parents=True, exist_ok=True); p.write_text(data)**.
- **Найти все .py в проекте:** **list(Path('src').rglob('\*.py'))**.
- **Временно сменить текущий каталог:** **with contextlib.chdir(new_path):** в stdlib нет; использовать **os.chdir** или **with pushd** (сторонние). pathlib даёт путь, смена каталога — операция **os**.

### Отладка и диагностика pathlib

- **Проверить, что путь существует и это файл:** **Path(p).exists() and Path(p).is_file()**. **Path(p).is_dir()** — каталог. **Path(p).is_symlink()** — символическая ссылка.
- **Узнать абсолютный путь к текущему скрипту:** **Path(**file**).resolve()**. Родительский каталог скрипта: **Path(**file**).resolve().parent**.
- **Проверить кодировку при чтении:** **.read_text(encoding='utf-8')** по умолчанию зависит от локали; при кракозябрах явно указать **encoding='utf-8'** или **encoding='cp1251'** и т.д. При ошибке декодирования — **UnicodeDecodeError**; можно указать **errors='replace'** или **errors='ignore'**.
- **Ошибка «No such file» при read_text:** путь не существует или это каталог. Проверить **.exists()**, **.is_file()**. При записи в несуществующий каталог — **FileNotFoundError**; перед записью вызвать **.parent.mkdir(parents=True, exist_ok=True)**.
- **Символические ссылки:** **.resolve()** разворачивает их в реальный путь; **.absolute()** не разворачивает. **.read_text()** по symlink читает целевой файл; **.exists()** по умолчанию проверяет существование цели (на большинстве платформ).

## 8a. Файловая система: низкоуровневые детали

### os.scandir() vs os.listdir()

- [ ] **os.listdir(path)** возвращает **список имён** (строк) файлов и подкаталогов в каталоге **path**. Чтобы узнать, каталог это или файл, размер или время изменения, для каждого имени нужно вызывать **os.path.isdir()**, **os.stat()** и т.д. — при большом числе файлов это даёт множество системных вызовов (stat на каждый элемент).
- [ ] **os.scandir(path)** возвращает итератор объектов **DirEntry**. У каждого **DirEntry** при первом обращении к атрибутам (**.is_dir()**, **.is_file()**, **.stat()**) выполняются нужные системные вызовы, и результат **кешируется** внутри объекта. Поэтому при обходе каталога с фильтрацией (например, «только каталоги» или «файлы больше N байт») **scandir** эффективнее: один проход, кешированные атрибуты, меньше обращений к ядру. В Python 3.5+ **os.walk()** внутри использует **scandir** там, где это доступно.

**Пример**

```python
import os
# Медленнее: для каждого имени — отдельный stat
for name in os.listdir('.'):
    if os.path.isdir(name):
        print(name, "is dir")

# Быстрее: DirEntry кеширует is_dir/stat
for entry in os.scandir('.'):
    if entry.is_dir():
        print(entry.name, "is dir")
    # entry.stat() при первом вызове кешируется
```

### mmap — memory-mapped files

**Что такое и зачем**

- [ ] **mmap** — **отображение файла** (или анонимной области) в адресное пространство процесса: участок виртуальной памяти напрямую связан с файлом. Чтение и запись идут через обращение к памяти (**mv[i:j]**, **mv[i:j] = b'...'**); ядро подкачивает страницы из файла и сбрасывает изменения на диск. **Преимущества:** **zero-copy** доступ (при read не копируем в пользовательский буфер); **разделяемая память** между процессами (один файл, отображённый в несколько процессов); удобно для **больших файлов** при **случайном доступе** по смещению. **Недостатки:** размер отображения обычно кратен размеру страницы; синхронизация с диском не мгновенная (**.flush()** при необходимости); на Windows/разных ОС API может отличаться.
- [ ] **mmap.mmap(fileno, length, access=mmap.ACCESS_WRITE, ...)** — отобразить **length** байт файла с дескриптором **fileno**. **access**: **ACCESS_READ** (только чтение), **ACCESS_WRITE** (чтение/запись в файл), **ACCESS_COPY** (запись в копию, не в файл). Срез **mv[i:j]** — bytes; **mv[i:j] = b'...'** — запись (при ACCESS_WRITE). **.seek(pos)**, **.read(n)**, **.flush()** — сбросить на диск. **.close()** — отменить отображение. **Применения:** случайный доступ к большим файлам (индексы, бинарные форматы), общая память между процессами (shared memory через файл в /dev/shm или обычный файл).

**Пример mmap: чтение по смещению**

```python
import mmap
with open("data.bin", "r+b") as f:
    with mmap.mmap(f.fileno(), 0, access=mmap.ACCESS_READ) as mv:
        # прочитать 100 байт с смещения 1000
        chunk = mv[1000:1100]
        # поиск подстроки
        pos = mv.find(b"signature")
```

### os.sendfile()

**Что такое и зачем**

- [ ] **os.sendfile(out_fd, in_fd, offset, count)** (где доступно) — **копирование данных из файла в сокет/файл внутри ядра**: данные не копируются в пользовательское пространство, что быстрее и экономит память при отдаче больших файлов по сети. **Применения:** статические файлы веб-сервера (отдача файла в сокет), копирование файлов с минимальными копиями. Доступность и сигнатура зависят от ОС: на **Linux** — **os.sendfile(out_fd, in_fd, offset, count)**; на **Windows** — другой API (**os.sendfile** может отсутствовать или иметь другую сигнатуру). В коде нужно проверять наличие (**hasattr(os, 'sendfile')**) или использовать **shutil.copyfileobj** как переносимую альтернативу (с буфером).

## 8b. Время и даты: тонкости

### time.monotonic() vs time.time()

- [ ] **time.time()** возвращает **системные часы** (секунды с эпохи, обычно 1970-01-01 UTC). Они могут **идти назад** при корректировке времени (NTP, ручная смена) или в виртуальных средах при паузе. Использовать **time.time()** для измерения **интервалов** (таймауты, замер производительности) — плохая практика: интервал может получиться отрицательным или искажённым.
- [ ] **time.monotonic()** возвращает монотонно **неубывающие** часы (от произвольной точки отсчёта). Они не идут назад при смене системного времени. **Рекомендуется** для измерения интервалов: **start = time.monotonic(); ...; elapsed = time.monotonic() - start**. **time.perf_counter()** — ещё один монотонный счётчик, часто с более высоким разрешением; для профилирования и замеров предпочтителен он.

**Пример**

```python
import time
start = time.monotonic()
do_work()
print("Elapsed:", time.monotonic() - start)
```

### datetime naive vs aware

- [ ] **Naive** объект **datetime** — без привязки к временной зоне (**tzinfo is None**). Он интерпретируется как «локальное время» в неопределённой зоне; сравнение двух naive дат корректно (это моменты «по локальным часам» без указания каких). **Aware** объект — с заданным **tzinfo** (например, **datetime.timezone.utc** или **zoneinfo.ZoneInfo('Europe/Moscow')**); он однозначно соответствует моменту в UTC.
- [ ] **Сравнение и арифметика** naive с aware в Python 3 **запрещены** (TypeError). Сравнивать и вычитать можно два naive или два aware. Смешивание зон (например, Moscow и UTC) через арифметику даёт результат в типе того объекта, у которого есть tzinfo. **Рекомендация:** хранить и передавать между слоями в **UTC**; переводить в локальную зону только при выводе пользователю.

### Летнее время (DST) и атрибут fold

- [ ] При переходе на **летнее время** один и тот же момент UTC может соответствовать **двум** локальным временам (например, 2:30 утра бывает дважды). При обратном переходе один час «пропадает». В **datetime** атрибут **fold** (PEP 495, 0 или 1) различает, какой из двух раз имеется в виду при неоднозначности. Это важно при конвертации из UTC в локальное время и обратно в зонах с DST.

### zoneinfo (3.9+)

- [ ] Модуль **zoneinfo** даёт доступ к **IANA timezone database** (названия зон вида **Europe/Moscow**, **America/New_York**). **ZoneInfo('Europe/Moscow')** подставляется в **tzinfo=** при создании **datetime** или в **datetime.astimezone()**. База зон поставляется ОС или пакетом **tzdata**; правила перехода DST и исторические изменения учитываются автоматически. Предпочтительная замена **pytz** для нового кода (pytz имел особенности с нормализацией; **zoneinfo** ведёт себя единообразно).
