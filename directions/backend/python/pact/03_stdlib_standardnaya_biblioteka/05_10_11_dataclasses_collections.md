## 10. dataclasses

Модуль **dataclasses** (PEP 557, с Python 3.7) автоматически генерирует ****init****, ****repr****, при необходимости ****eq**** и методы сравнения (****lt****, ****le**** и т.д.) для классов, которые в основном хранят данные («data classes»). Цель — убрать шаблонный код и уменьшить ошибки (например, забытое поле в ****init**** или в ****repr****).

### Что такое dataclass и зачем он нужен

**Проблема без dataclass:** типичный класс-контейнер данных требует вручную писать ****init**(self, a, b, c)**, ****repr****, ****eq****; при добавлении поля нужно обновить все эти методы. Код получается длинным и легко ошибиться (не добавить поле в сравнение или в repr).

**Решение:** декоратор **@dataclass** и аннотации полей. По аннотациям генерируются ****init**** (из полей класса), ****repr**** (перечисление полей и значений), ****eq**** (сравнение по полям). Опционально — **order** (сравнения **<**, **<=**, **>**, **>=** по полям), **frozen** (неизменяемый объект после создания). Для полей с значениями по умолчанию используется **field()**; для мутабельных по умолчанию (list, dict) обязательно **default_factory**, иначе один и тот же объект будет разделяться между экземплярами (известная ловушка mutable default).

**Теоретическое обоснование:** Data classes — компромисс между «полноценным» классом с ручными методами и «namedtuple» (неизменяемый, без default_factory для мутабельных). Dataclass мутабелен по умолчанию, поддерживает наследование, ****post_init**** для валидации и производных полей, **asdict**/ **astuple**/ **replace** для сериализации и копирования с изменениями. PEP 557 мотивирует сокращением шаблона и единообразием с другими языками (Kotlin data class, Scala case class).

### Основные параметры и методы

- [ ] **@dataclass** — декоратор класса; поля задаются аннотациями с типами. По умолчанию генерируются ****init****, ****repr****, ****eq****.
- [ ] **@dataclass(slots=True)** (3.10+) — автоматический ****slots**** у класса; экономия памяти (нет ****dict****); несовместимо с динамическим добавлением атрибутов и с некоторыми приёмами (например, **weakref** требует ****weakref**** в слотах).
- [ ] **field(default=..., default_factory=...)** — значение по умолчанию. Для мутабельных (list, dict, set) **обязательно** **default_factory**: **field(default_factory=list)**. Иначе **field(default=[])** приведёт к одному общему списку для всех экземпляров (ловушка mutable default).
- [ ] Параметры декоратора: **repr=True**, **eq=True**, **order=False**, **frozen=False** — включение/выключение ****repr****, ****eq****, методов сравнения, неизменяемости после создания.
- [ ] ****post_init**(self)** — вызывается после автогенерированного ****init****; здесь можно валидировать поля или вычислить производные атрибуты (например, поле **full_name** из **first_name** и **last_name**).
- [ ] Наследование: поля родителя и потомка объединяются; порядок полей в ****init**** определяется порядком объявления (сначала родитель, потом потомок). Поля с default в родителе и без default в потомке — недопустимая комбинация (как и в обычном ****init****).
- [ ] **dataclasses.asdict(obj)** — рекурсивное преобразование в dict (вложенные dataclass тоже превращаются в dict). **astuple(obj)** — в tuple. **replace(obj, **changes)** — создаёт копию объекта с указанными изменениями полей; для **frozen\*\* dataclass это основной способ «изменить» объект.
- [ ] **field(init=False)** — поле не входит в ****init**** (вычисляется в ****post_init**** или задаётся по умолчанию). **repr=False**, **compare=False**, **hash=None** — исключить из repr, из ****eq****/сравнений, из ****hash****. **field(metadata={})** — произвольные метаданные (используются инструментами сериализации, валидации). **fields(obj)** или **fields(MyClass)** — список полей (объекты **Field**).
- [ ] **make_dataclass(name, fields, ...)** — программное создание dataclass по имени и списку полей. **is_dataclass(obj)** — проверка, является ли объект/класс dataclass.
- [ ] **KW_ONLY** (3.10+) — маркер-разделитель: все поля, объявленные **после** него, становятся **keyword-only** в ****init**** (их нельзя передать позиционно).

**Пример**

```python
from dataclasses import dataclass, field, replace

@dataclass
class Point:
    x: float
    y: float

@dataclass
class User:
    name: str
    tags: list[str] = field(default_factory=list)  # не default=[]!

    def __post_init__(self):
        if not self.name.strip():
            raise ValueError("name must be non-empty")

p = Point(1.0, 2.0)
u = User("Alice")
u.tags.append("admin")
u2 = replace(u, name="Bob")  # копия с другим name
```

### Типичные ошибки dataclass

- **Мутабельное значение по умолчанию:** **tags: list[str] = []** — **неправильно**. Один и тот же список будет разделяться между всеми экземплярами. Нужно **tags: list[str] = field(default_factory=list)**.
- **Порядок полей при наследовании:** поля с default в родителе и без default в потомке недопустимы (как в обычном ****init****). Сначала объявляют поля без default, затем с default; в потомке нельзя добавить поле без default после поля с default в родителе.
- **Забыть default_factory для list/dict/set:** **items: list[int] = field(default=[])** вызовет предупреждение (или ошибку в части версий); использовать **field(default_factory=list)**.
- **frozen и изменение после создания:** у **@dataclass(frozen=True)** нельзя присваивать атрибуты после создания. Для «изменения» использовать **dataclasses.replace(obj, field=new_value)**.

## 11. collections

Модуль **collections** предоставляет специализированные контейнеры и обёртки, дополняющие встроенные **dict**, **list**, **tuple**, **set**. Ниже — разбор с «что такое и зачем», примерами и граничными случаями.

### defaultdict

- [ ] **defaultdict(factory)** — подкласс **dict**, у которого при обращении к **отсутствующему** ключу вызывается **factory()** без аргументов, результат сохраняется под этим ключом и возвращается. Типичное использование: **defaultdict(list)** — группировка элементов в списки по ключу (при **d[key].append(item)** не нужно проверять **if key not in d**). **defaultdict(int)** — счётчики (при **d[key] += 1** новый ключ получает 0).

**Пример**

```python
from collections import defaultdict
by_letter = defaultdict(list)
for w in ["apple", "banana", "apricot"]:
    by_letter[w[0]].append(w)
counts = defaultdict(int)
for c in "abracadabra":
    counts[c] += 1
```

**Граничные случаи:** **factory** вызывается только при обращении к отсутствующему ключу. **d.default_factory = None** — тогда поведение как у обычного dict (KeyError при отсутствующем ключе).

### Counter

**Что такое и зачем**

- [ ] **Counter** — подкласс **dict** для **подсчёта** хешируемых объектов: ключ — элемент, значение — число вхождений. При обращении к отсутствующему ключу возвращается **0** (не KeyError). **Применения:** частотный анализ (слова в тексте, символы, события), топ-N наиболее частых элементов, гистограммы, проверка «достаточно ли элементов» для составления слова из букв.
- [ ] **Counter(iterable)** — построить счётчик из итератора (подсчёт каждого элемента). **Counter(d)** или **Counter(a=1, b=2)** — из словаря или именованных аргументов. **c[key]** — количество (0 если ключа нет). **c[key] += 1** — увеличить счётчик; **c[key] = 0** не удаляет ключ (для удаления **del c[key]**). **.most_common(n)** — список пар **(elem, count)** для n самых частых (или все при **n=None**). **.elements()** — итератор по элементам с повторениями (каждый элемент повторён count раз). **.subtract(iterable)** — вычесть счётчики (могут стать отрицательными). Арифметика **c1 + c2** — сложение счётчиков (только положительные); **c1 - c2** — вычитание (минимальный 0); **c1 & c2** — минимум по ключам; **c1 | c2** — максимум по ключам.

**Пример Counter**

```python
from collections import Counter
words = "one two one two three one".split()
c = Counter(words)           # Counter({'one': 3, 'two': 2, 'three': 1})
c.most_common(2)             # [('one', 3), ('two', 2)]
list(c.elements())           # ['one', 'one', 'one', 'two', 'two', 'three']
c["four"]                    # 0 (нет KeyError)
c["one"] += 1                # Counter({'one': 4, ...})
```

### OrderedDict

**Что такое и зачем**

- [ ] **OrderedDict** — подкласс **dict**, **гарантирующий порядок** вставки ключей. С Python 3.7 обычный **dict** тоже сохраняет порядок вставки; **OrderedDict** добавляет поведение **move_to_end(key, last=True)** — переместить ключ в конец (или в начало при **last=False**) и поддержку **reversed(od)** — итерация в обратном порядке вставки. **Применения:** LRU-кеш (в связке с **move_to_end** при доступе), конфиги с важным порядком ключей, когда код должен работать на старых версиях (до 3.7), явная семантика «упорядоченный словарь».
- [ ] **move_to_end(key, last=True)** — переместить ключ в конец (или в начало при **last=False**); при **last=False** ключ оказывается «самым старым». **popitem(last=True)** — удалить и вернуть последнюю пару (LIFO); **last=False** — первую (FIFO). **reversed(od)** — итерация по ключам в обратном порядке вставки.

### namedtuple

**Что такое и зачем**

- [ ] **namedtuple(typename, field_names)** — фабрика **неизменяемого** типа с **именованными полями**. **field_names** — строка **'x y z'** или список **['x', 'y', 'z']**. Возвращается класс; экземпляры создаются позиционно или по имени: **Point(1, 2)** или **Point(x=1, y=2)**. Доступ по имени **p.x**, по индексу **p[0]**; итерация, **len(p)**. **Применения:** лёгковесные записи (координаты, записи из БД), возврат нескольких значений с именованными полями без объявления класса. Для мутабельных записей с методами и наследованием предпочтительны **dataclass** (3.7+).
- [ ] **.\_asdict()** — преобразовать в **dict**. **.\_replace(**kwargs)** — создать копию с заменой указанных полей. **.\_fields\*\* — кортеж имён полей.

**Пример namedtuple**

```python
from collections import namedtuple
Point = namedtuple('Point', 'x y')
p = Point(1, 2)
p.x, p.y   # 1, 2
p[0]       # 1
p._asdict()  # {'x': 1, 'y': 2}
p._replace(x=10)  # Point(x=10, y=2)
```

### deque

**Что такое и зачем**

- [ ] **deque** — **двусторонняя очередь**: добавление и извлечение с **обоих концов** за O(1). Реализована как связный список блоков; в отличие от **list**, **append**/ **pop** с левого конца не сдвигают все элементы. **deque(maxlen=N)** — очередь с ограничением длины: при добавлении нового элемента, когда очередь полна, элемент с **противоположного** конца отбрасывается (скользящее окно). **Применения:** очередь задач (producer-consumer), скользящее окно по потоку данных, BFS (очередь вершин), стек (если использовать только один конец).
- [ ] **append(x)**, **appendleft(x)** — добавить справа/слева. **pop()**, **popleft()** — извлечь справа/слева. **extend(iterable)**, **extendleft(iterable)** — добавить несколько элементов (extendleft добавляет по одному слева, порядок элементов в iterable «переворачивается»). **rotate(n)** — циклический сдвиг на **n** позиций (n > 0 — вправо, n < 0 — влево). **clear()**, **count(x)**, **remove(x)**. Индексация **d[i]** и срезы поддерживаются, но для частого произвольного доступа **list** эффективнее (deque оптимизирован для операций с концами).

**Пример deque: скользящее окно**

```python
from collections import deque
window = deque(maxlen=3)
for x in [1, 2, 3, 4, 5]:
    window.append(x)
    print(list(window))  # [1], [1,2], [1,2,3], [2,3,4], [3,4,5]
```

### ChainMap

**Что такое и зачем**

- [ ] **ChainMap(\*maps)** — **цепочка** словарей: при поиске ключа перебираются словари по порядку; при записи — только в **первый** словарь. Удаление ключа удаляет только первое вхождение. **Применения:** конфигурация с переопределениями (defaults + user overrides + env), вложенные области видимости (локальные поверх глобальных), объединение нескольких dict для чтения без копирования.
- [ ] **.new_child(m=None)** — новый ChainMap с дополнительным словарём в начале (типично пустой dict для «локальных» переопределений). **.parents** — ChainMap без первого словаря. **.maps** — список словарей (можно изменять для добавления/удаления слоёв).

**Пример ChainMap**

```python
from collections import ChainMap
defaults = {"theme": "light", "lang": "en"}
user = {"theme": "dark"}
cm = ChainMap(user, defaults)
cm["theme"]   # "dark" (из user)
cm["lang"]   # "en" (из defaults)
cm["lang"] = "ru"  # запись в user
```

### UserDict, UserList, UserString

**Что такое и зачем**

- [ ] **UserDict**, **UserList**, **UserString** — обёртки над **dict**, **list**, **str** через **композицию**: внутри хранится **self.data** (настоящий dict/list/str), все операции делегируются ему. При **наследовании** от UserDict/UserList/UserString переопределённые методы (****setitem****, **update**, **append** и т.д.) **гарантированно вызываются** при любых операциях, в отличие от подкласса встроенного **dict**/ **list**, где методы вроде **update** или **sort** могут обходить ваш ****setitem**** (см. §11a). **Применения:** логирование изменений, валидация при записи, синхронизация с другим хранилищем, кастомное поведение при добавлении/удалении элементов.
- [ ] Конструктор **UserDict(initial)** — **self.data** инициализируется копией **initial**. **UserList(initlist=None)** — **self.data = list(initlist or [])**. **UserString(seq)** — **self.data = str(seq)**. Переопределяйте нужные методы (****setitem****, ****getitem****, **update**, **append** и т.д.) и вызывайте **super()** или **self.data**.

## 11a. Подклассирование встроенных типов: тёмные углы

### Почему list.sort() и dict.update() не вызывают **setitem**

- [ ] При подклассировании **list** или **dict** вы можете переопределить ****setitem**(self, key, value)** в расчёте на то, что любое изменение элемента будет идти через него. Но многие методы встроенных типов реализованы **на C** и оптимизированы: они напрямую меняют внутреннее хранилище, **не вызывая** Python-методы ****setitem****, ****delitem**** и т.д. В результате ваш переопределённый ****setitem**** при вызове **lst.sort()** или **d.update(other)** **не вызывается** — сортировка и обновление идут «в обход» вашей логики. Это ломает инварианты (например, логирование изменений или синхронизацию с другим хранилищем).

**Пример (иллюстрация проблемы)**

```python
class LoggingDict(dict):
    def __setitem__(self, key, value):
        print("set", key, value)
        super().__setitem__(key, value)

d = LoggingDict()
d["a"] = 1        # напечатает: set a 1
d.update({"b": 2})  # во многих реализациях НЕ напечатает set b 2 — update обходит __setitem__
```

- [ ] **Обход:** использовать **collections.UserDict**, **UserList**, **UserString**. Они реализованы **через композицию**: внутри хранится настоящий **dict**/ **list**/ **str**, а все операции делегируются ему через явные вызовы **self.data[key] = value** и т.д. Поэтому переопределение ****setitem**** у подкласса **UserDict** гарантированно вызывается при любых операциях, в том числе при **.update()**. Аналогично **UserList** и **.sort()** — если **sort** реализован через присваивания в **self.data**, ваши методы будут вызваны (в текущей реализации **UserList.sort** вызывает **self.data.sort()**, а **self.data** — обычный list, так что ****setitem**** вашего класса при sort не вызывается; но **append**, **extend** при добавлении элементов в UserList идут через ваш код, если вы их переопределили). Для **dict**-подобного поведения с полным контролем — **UserDict**.

### obj.**class** = NewClass

- [ ] Присвоение **obj.**class** = NewClass** меняет класс объекта «на лету». Для **обычных** экземпляров классов (с ****dict****) это может работать, если **NewClass** совместим по атрибутам. Для встроенных типов (**int**, **str**, **tuple**, **bytes** и т.д.) внутреннее представление жёстко привязано к типу; смена ****class**** может привести к краху интерпретатора или неопределённому поведению. **Не полагайтесь** на смену класса для встроенных типов.

### **slots** при наследовании

- [ ] В подклассе ****slots**** родителя **не наследуются** автоматически в смысле «подкласс тоже не будет иметь **dict**». Чтобы подкласс тоже использовал слоты, в нём нужно объявить ****slots**** (пустой кортеж или свои имена). При множественном наследовании несколько родителей со ****slots**** объединяются в один кортеж слотов у потомка; конфликт имён слотов недопустим. Если нужен **weakref** на объект с ****slots****, в ****slots**** нужно включить **'**weakref**'**.
