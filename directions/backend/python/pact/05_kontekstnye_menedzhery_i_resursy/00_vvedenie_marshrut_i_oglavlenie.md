# Часть V. Контекстные менеджеры и ресурсы

> **Цель:** уверенно использовать контекстные менеджеры (синхронные и асинхронные): протокол `__enter__`/`__exit__`, `@contextmanager`, вложенные `with`, `ExitStack`, утилиты `contextlib`; понимать гарантии освобождения ресурсов и обработку исключений.

**Краткая шпаргалка (суть):**

| Концепция | Синтаксис / API | Назначение |
|-----------|-----------------|------------|
| Контекстный менеджер | `with obj as x:` | Вход → блок → выход; гарантированный вызов `__exit__` |
| Протокол | `__enter__`, `__exit__` | Методы объекта для `with` |
| Генераторный контекст | `@contextmanager` + `yield` | Описать контекст одной функцией |
| Динамический набор | `ExitStack` | Несколько контекстов, число неизвестно заранее |
| Асинхронный контекст | `async with obj:` | `__aenter__`, `__aexit__` |

## Маршрут изучения

**Что вы изучите и в каком порядке.** Сначала — **синхронные контекстные менеджеры** (§20): протокол **__enter__**/**__exit__** (§20.1), обработка исключений в **__exit__** (§20.2), **@contextmanager** и **yield** (§20.3), утилиты **contextlib** (§20.3a), вложенные **with** и **ExitStack** (§20.4–20.5), **closing** и **chdir** (§20.6). Затем — **асинхронные контекстные менеджеры** (§20a): **__aenter__**/**__aexit__**, **async with**, **@asynccontextmanager**, **asyncio.timeout**. В конце части — справочник (сценарии, терминология, вопросы, задания) для повторения и самопроверки.

**Что желательно знать заранее:** исключения (try/except, raise), генераторы (yield). Полезно Часть III (contextlib в §14) и базовый async/await для §20a.

## Как устроены разделы и оглавление

В каждом разделе (§20.1–§20a.3) материал идёт по одному шаблону: **цель** (что вы освоите) → **термины** (контекстный менеджер, протокол __enter__/__exit__, ресурс и т.д., с кратким определением) → **правила и синтаксис** (как писать **with**, **async with**) → **примеры** (код с пояснением) → **граничные случаи** (исключения в __exit__, подавление) → блок **«Запомните»** (краткий вывод). Термины в тексте сохранены (**context manager**, **ExitStack**, **asynccontextmanager** и т.д.); в начале раздела они расшифрованы.

**Оглавление** разбито на этапы изучения. Каждая строка таблицы — раздел: по ссылке переходите к материалу. В столбце «Содержание» — о чём раздел; в «Ключевые понятия» — термины, которые нужно знать после изучения. Идите по этапам сверху вниз; справочник в конце — для повторения и самопроверки.
