## 6. CLI и упаковка (§122)

### 6.1. `click`: декларативное CLI

```python
import click

@click.command()
@click.option("--count", default=1, help="Сколько раз вывести")
@click.argument("name")
def hello(count: int, name: str):
    for _ in range(count):
        click.echo(f"Hello, {name}!")

if __name__ == "__main__":
    hello()
```

`click` берёт на себя:

- парсинг аргументов;
- помощь (`--help`);
- обработку ошибок ввода.

Разбор по шагам:

1. `@click.command()`:
   - помечает функцию `hello` как **CLI‑команду**;
   - `click` на основе этой функции строит интерфейс командной строки.
2. `@click.option("--count", default=1, help="...")`:
   - добавляет опцию `--count` с типом `int` (определяется из аннотации параметра);
   - значение по умолчанию — `1`;
   - текст в `help` попадёт в автосгенерированную справку.
3. `@click.argument("name")`:
   - позиционный аргумент (обязательный);
   - `name` становится параметром функции.
4. Внутри функции:
   - обычный Python‑код, который делает работу (печатает приветствия).
5. `if __name__ == "__main__": hello()`:
   - точка входа при запуске файла как скрипта;
   - `click` внутри `hello()` разбирает `sys.argv` и передаёт параметры.

Пример вызова:

- `python script.py Alice` → один раз `Hello, Alice!`;
- `python script.py --count 3 Alice` → три раза `Hello, Alice!`.

Мини‑упражнение:

- добавьте ещё одну опцию `--uppercase/--no-uppercase`, которая делает имя заглавными буквами (`ALICE`), если включена.

### 6.2. `typer`: CLI из аннотаций типов

```python
import typer

app = typer.Typer()

@app.command()
def hello(name: str, count: int = 1):
    for _ in range(count):
        print(f"Hello, {name}")

if __name__ == "__main__":
    app()
```

`typer` поверх Click, но позволяет описывать интерфейс через **type hints** и дефолты параметров.

Чем `typer` приятен:

- он «читается как обычная функция»:
  - аргументы = параметры командной строки;
  - их типы = аннотации типов;
  - значения по умолчанию = дефолты параметров.
- автоматически создаёт:
  - `--help`;
  - проверку типов аргументов;
  - удобные сообщения об ошибках.

Мини‑упражнение:

- добавьте в пример ещё одну команду `goodbye`, которая печатает прощание, и посмотрите, как `typer` создаёт подкоманды.

### 6.3. `rich`: красивый терминал

```python
from rich.console import Console
from rich.table import Table

console = Console()
table = Table(title="Users")

table.add_column("ID", justify="right")
table.add_column("Name", style="cyan")

table.add_row("1", "Alice")
table.add_row("2", "Bob")

console.print(table)
```

Также `rich` умеет:

- прогресс‑бары,
- подсветку кода,
- форматирование трейсбеков.

Практические сценарии использования:

- отображение **таблиц** (например, список пользователей, статус задач);
- **индикаторы прогресса** при длительных операциях (скачивание, миграции);
- акцентирование ошибок/предупреждений цветом.

Мини‑упражнение:

- сделайте маленький скрипт, который выводит таблицу из трёх колонок («ID», «Название», «Статус») и пары строк с разными статусами (например, зелёный «OK», жёлтый «WARN»).

### 6.4. Инструменты упаковки: `setuptools`, `hatch`, `flit`

Современный стандарт — описывать проект в `pyproject.toml`:

- `setuptools` — «классика», умеет почти всё;
- `hatch` / `flit` — более современные и opinionated‑инструменты, часто с удобным управлением окружениями/релизами.

Главное — научиться читать `pyproject.toml` и понимать:

- имя пакета;
- зависимости;
- точки входа (console_scripts).

Упрощённая картина:

- `setuptools`:
  - исторический стандарт;
  - очень гибкий, но иногда многословный;
  - много старых проектов на нём.
- `hatch`, `flit`:
  - более «современные» инструменты;
  - часто проще стартовать новый проект (меньше шаблонного кода);
  - лучше интегрированы с `pyproject.toml`.

`console_scripts`:

- в `pyproject.toml` можно описать:

  ```toml
  [project.scripts]
  mytool = "package.module:main"
  ```

- это создаст исполняемый скрипт `mytool` в окружении, который вызывает функцию `main` из указанного модуля.

Мини‑упражнение:

- откройте любой `pyproject.toml` и найдите секцию с консолными скриптами;  
  попробуйте сопоставить имена скриптов с функциями в коде.

### 6.5. Запомните по §122

- Хороший CLI экономит часы рутины; `click` и `typer` позволяют сделать его почти «на автомате».
- `rich` делает вывод понятным и приятным, особенно в админских/диагностических утилитах.
- Учитесь хотя бы на базовом уровне читать/писать `pyproject.toml` — это «паспорт» вашего проекта.
