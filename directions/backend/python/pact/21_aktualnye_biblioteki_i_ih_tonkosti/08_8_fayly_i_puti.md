## 8. Файлы и пути (§124)

### 8.1. `pathlib`: Path вместо строк

```python
from pathlib import Path

base = Path(__file__).parent
config_path = base / "config" / "settings.yml"

text = config_path.read_text(encoding="utf-8")
config_path.write_text(text, encoding="utf-8")
```

Преимущества:

- кроссплатформенное сложение путей (`/` вместо `os.path.join`);
- удобные методы чтения/записи (`read_text`, `write_bytes`, `glob`).

Почему это удобно:

- вместо:

  ```python
  import os
  config_path = os.path.join(base_dir, "config", "settings.yml")
  ```

- вы пишете:

  ```python
  from pathlib import Path
  base = Path(__file__).parent
  config_path = base / "config" / "settings.yml"
  ```

- и получаете:
  - код, который проще читать;
  - меньше шансов ошибиться со слэшами (`/` vs `\`).

Мини‑упражнение:

- перепишите несколько старых путей в своём коде с `os.path.join` на `Path / "..."`.

### 8.2. `watchdog`: мониторинг изменений

`watchdog` отслеживает изменения в файловой системе (создание/удаление/изменение файлов).

Полезен для:

- dev‑серверов (автоперезапуск при изменении кода);
- синхронизации директорий;
- простых «наблюдателей» за логами.

Идея:

- вы создаёте «наблюдателя», который:
  - следит за каталогом;
  - при создании/удалении/изменении файла вызывает ваш обработчик.

Применение:

- автоперезагрузка веб‑сервера при изменении кода;
- пересборка документации/фронта при изменении исходников;
- мониторинг логов в реальном времени.

### 8.3. `aiofiles`: async‑файлы

```python
import aiofiles
import asyncio

async def main():
    async with aiofiles.open("data.txt", "w", encoding="utf-8") as f:
        await f.write("hello")

asyncio.run(main())
```

Используйте, когда чтение/запись файлов должно быть неблокирующим по отношению к event loop.

Почему это важно для async‑кода:

- обычный `open(...).read()` блокирует поток, пока файл читается;
- если вы делаете это внутри `async def`, вы **замораживаете event loop**;
- `aiofiles` оборачивает файловые операции так, чтобы они выполнялись в пуле потоков и не блокировали главный цикл.

Мини‑упражнение:

- напишите две версии функции:
  - sync: читает большой файл в цикле и считает количество строк;
  - async: делает то же самое через `aiofiles`;
  - измерьте время работы при запуске нескольких таких задач параллельно в `asyncio.gather`.

### 8.4. Архивы: `py7zr`, `rarfile` и др.

- `py7zr` — работа с 7z‑архивами;
- `rarfile` — пайтон‑обвязка вокруг `unrar`/`rar`.

В большинстве случаев достаточно `zipfile` и `tarfile` из stdlib; дополнительные пакеты нужны для специфичных форматов.

---

### 8.5. Запомните по §124

- Привыкайте использовать `pathlib.Path` вместо «голых строк» с путями.
- Для долгоживущих приложений полезно мониторить изменения файлов (особенно конфигов).
- В async‑приложениях любой блокирующий I/O (включая файлы) должен быть либо async, либо вынесен в `run_in_executor`.
