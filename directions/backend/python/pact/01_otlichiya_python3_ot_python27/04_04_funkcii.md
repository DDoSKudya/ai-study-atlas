## 4. Функции

В разделе разобраны **объявление и вызов** (позиционные, именованные, \*args, **kwargs, keyword-only, positional-only, порядок параметров), **области видимости LEGB**, **global** и **nonlocal**, **замыкания** и **late binding**, **mutable default** (почему опасен и как правильно), **hasattr/getattr** (ловушка с property), а также ключевые **builtins\*\* при переходе с Py2 (input, map/filter/zip, enumerate(start), super(), zip(strict), eval/exec, breakpoint). Для обучения: сначала закрепите порядок параметров в сигнатуре и момент вычисления значений по умолчанию; затем разберите LEGB пошагово и пример с nonlocal на трёх уровнях; обязательно разберите mutable default (id(), **defaults**, почему if a is None) и hasattr/getattr с property; в конце — таблицу builtins и отличия Py2→Py3.

### 4.1 Объявление и вызов

- **Позиционные и именованные аргументы** — вызов вида **`f(a, b, key=value)`**; порядок: сначала позиционные, затем именованные.
- **`*args`** — произвольное число **позиционных** аргументов; в теле функции **args** — кортеж.
- **`**kwargs`** — произвольное число **именованных** аргументов; в теле функции **kwargs\*\* — словарь.
- **Keyword-only аргументы** (после **`*`** в списке параметров): **`def f(a, *, b):`** — **`b`** можно передать **только по имени**; вызов **f(1, 2)** вызовет ошибку, нужен **f(1, b=2)**. Полезно для явного указания опций и избежания путаницы с позиционными аргументами: вызывающий код не может случайно передать лишний позиционный аргумент «не в тот» параметр; все опции после **`*`** должны быть названы явно.
- **Positional-only аргументы** (Python 3.8+): **`def f(a, /, b):`** — **`a`** можно передать **только позиционно**; **f(a=1, b=2)** вызовет ошибку для **a**. Используется в C-API и встроенных функциях для совместимости и ясности API: имя параметра можно менять в будущем без поломки вызовов, так как вызывающий код не может ссылаться на него по имени.
- **Аннотации типов**: **`def f(x: int) -> str:`** — аннотации **не проверяются** интерпретатором во время выполнения; они сохраняются в атрибуте функции **`__annotations__`** (словарь «имя параметра → аннотация», плюс ключ **'return'** для возвращаемого значения). Используются статическими анализаторами (mypy, pyright), инструментами документации (Sphinx) и в runtime для интроспекции. Чтобы отложить оценку аннотаций (избежать ошибок при forward reference или тяжёлом импорте), в начале файла можно указать **`from __future__ import annotations`** (Python 3.7+) — тогда аннотации хранятся как строки и не вычисляются при определении функции.

#### Когда вычисляются значения по умолчанию

Выражения в значениях по умолчанию параметров (**def f(a=expr):**) вычисляются **один раз** — в момент **выполнения инструкции def** (при загрузке модуля или при входе в блок, где определена функция). Не при каждом вызове функции. Поэтому **def f(t=time.time()):** сохранит одно и то же время при всех вызовах без аргумента; **def f(a=[]):** сохранит один и тот же список (см. mutable default). Если нужно «свежее» значение при каждом вызове (например, текущее время или новый список), создавайте объект **внутри** тела функции: **def f(a=None): a = [] if a is None else a**.

#### Специальный объект-маркер (sentinel) вместо None

Иногда **None** — допустимое значение аргумента, и нужно различать «аргумент не передан» и «передан явно None». В таком случае используют объект-маркер (sentinel), созданный один раз: **sentinel = object()**; в сигнатуре **def f(a=sentinel):**; внутри **if a is sentinel: a = default_value**. Так вызывающий может передать **f(None)** и получить иное поведение, чем **f()**.

```python
def f(a, b, *args, key_only, **kwargs):
    # a, b — позиционные; args — кортеж остальных позиционных; key_only — только по имени; kwargs — словарь именованных
    pass
f(1, 2, 3, 4, key_only=5, x=6)   # a=1, b=2, args=(3,4), key_only=5, kwargs={'x': 6}

def g(a, /, b, *, c):  # a — только позиционно, c — только по имени
    pass
g(1, 2, c=3)   # OK
g(1, b=2, c=3) # OK
g(a=1, b=2, c=3)  # TypeError: a — positional-only
```

**Сочетание \*args, keyword-only и **kwargs:** после **\*args** могут идти только keyword-only параметры (без значения по умолчанию они обязательны по имени), затем **\*\*kwargs**. Именованные аргументы в вызове сначала сопоставляются с keyword-only параметрами; остальные попадают в **kwargs\*\*.

```python
def h(a, *args, opt=10, **kwargs):
    # a — позиционный; args — остальные позиционные; opt — можно передать только по имени (есть default);
    # kwargs — остальные именованные
    print(a, args, opt, kwargs)
h(1, 2, 3, opt=20, x=4)   # a=1, args=(2, 3), opt=20, kwargs={'x': 4}
h(1, 2, 3, x=4)           # a=1, args=(2, 3), opt=10, kwargs={'x': 4}
# h(1, 2, 3, 20)          # TypeError: 20 не может попасть в opt (keyword-only по факту после *args)
```

#### Порядок параметров в сигнатуре функции

Порядок параметров в объявлении функции строго задан:

1. **Positional-only** (до **`/`**, если есть): можно передать только позиционно.
2. **Обычные позиционные и именованные** (между **`/`** и **`*`**): можно передать и позиционно, и по имени.
3. **Keyword-only** (после **`*`**, если есть): можно передать только по имени.
4. **`*args`** (если есть): собирает остальные позиционные аргументы в кортеж.
5. **`**kwargs`\*\* (если есть): собирает остальные именованные аргументы в словарь.

Пример полной сигнатуры: **`def f(a, b, /, c, d, \*, e, f, **kwargs):`** — **a, b** только позиционно; **c, d** позиционно или по имени; **e, f** только по имени; **kwargs** — любые дополнительные именованные аргументы. Смешивать ***args** и keyword-only после **\*** можно: **`def f(\*args, key_only):`** — **key_only\*\* только по имени.

**Порядок параметров со значением по умолчанию:** параметры **без** значения по умолчанию должны идти **перед** параметрами **с** значением по умолчанию (в группе «обычных» параметров между **/** и **\***). Иначе будет **SyntaxError**: **def f(a=1, b):** — ошибка, так как **b** без default идёт после **a** с default. Правильно: **def f(b, a=1):** или **def f(a, b=1):**. Исключение — **\*args** и **\*\*kwargs** и keyword-only параметры: keyword-only могут иметь default и идти после **\*** без нарушения этого правила внутри своей группы.

**Типичная ошибка: порядок параметров в сигнатуре.** Если написать **def f(\*args, key_only, **kwargs)** без того, чтобы **key_only** шёл после **\*args**, то **key_only** будет считаться частью **args** (позиционные аргументы). Keyword-only параметры должны идти **после** одиночной **\*** (не после **\*\*kwargs**). Ошибка: **def f(a, **kwargs, b):** — **SyntaxError**, так как **\*\*kwargs** должен быть последним. Правильно: **def f(a, \*, b, **kwargs):** — **b** только по имени, **kwargs\*\* собирает остальные именованные.

#### Пошагово: связывание аргументов с параметрами при вызове

**Что происходит при вызове f(1, 2, key=3):** (1) Интерпретатор собирает все позиционные аргументы в порядке следования и все именованные аргументы (пары имя–значение). (2) Сначала сопоставляются **позиционные** аргументы с параметрами слева направо: первый позиционный — первому параметру (не *args, не keyword-only), второй — второму и т.д. (3) Затем сопоставляются **именованные** аргументы: имя аргумента ищется в списке параметров; если параметр уже получил значение позиционно, дублирование по имени вызовет **TypeError**. (4) Оставшиеся позиционные аргументы (если есть **\*args**) попадают в кортеж **args**; оставшиеся именованные (если есть **\*\*kwargs**) — в словарь **kwargs**. (5) Параметры без переданного аргумента получают значение из **значения по умолчанию** (из \***\*defaults\*\*** или \***\*kwdefaults\*\***). Если какой-то обязательный параметр остался без значения или передано лишнее (например, keyword-only без имени) — **TypeError**. Итог: порядок в вызове — сначала все позиционные, потом все именованные; в сигнатуре — positional-only, обычные, *args, keyword-only, \*\*kwargs.

**Граничные случаи аргументов:** (1) **Обязательный keyword-only без передачи** — **def f(a, \*, b):** при вызове **f(1)** — **TypeError: f() missing 1 required keyword-only argument: 'b'**. Параметр **b** не имеет default и не может быть передан позиционно (после **\***). (2) **Positional-only по имени** — **def f(a, /):** при вызове **f(a=1)** — **TypeError: f() got some positional-only arguments passed as keyword arguments: 'a'**. (3) **Лишний позиционный для keyword-only** — **def f(a, \*, b):** при вызове **f(1, 2)** — **TypeError: f() takes 1 positional argument but 2 were given**; второй аргумент **2** не может попасть в **b** позиционно. (4) **Дублирование параметра** — **f(1, a=2)** для **def f(a):** — **TypeError: f() got multiple values for argument 'a'**. (5) **Неизвестный именованный** — **f(x=1)** для **def f(a):** без **\*\*kwargs** — **TypeError: f() got an unexpected keyword argument 'x'**.

### 4.2 Области видимости (LEGB)

Порядок поиска имени: **L**ocal → **E**nclosing (замыкания) → **G**lobal → **B**uilt-in.

- **Local** — имена, определённые внутри текущей функции (включая параметры).
- **Enclosing** — имена из внешних функций (замыкания); актуально при вложенных функциях.
- **Global** — имена на уровне модуля.
- **Built-in** — встроенные имена (например, **len**, **print**, **open**).

**`global name`** — объявляет, что присваивание и чтение **name** относятся к **глобальной** области видимости модуля, а не к локальной. Без **global** присваивание **name = ...** внутри функции создало бы локальную переменную.
**`nonlocal name`** — объявляет, что присваивание **name** относится к **ближайшей внешней** области видимости (enclosing), где **name** уже определён. Без **nonlocal** присваивание создало бы локальную переменную во вложенной функции и не изменило бы переменную во внешней функции.

#### Пошаговый пример поиска имени (LEGB)

Интерпретатор ищет имя **сначала** в локальной области (тело функции, включая параметры), **затем** в enclosing (внешние функции), **затем** в глобальной (модуль), **затем** во встроенной (**builtins**). Как только имя найдено, поиск останавливается.

**Пошагово: как интерпретатор ищет имя при выполнении print(x) внутри inner.** (1) Выполняется **print(x)** внутри функции **inner**. Интерпретатор встречает имя **x** и начинает поиск по LEGB. (2) **L (Local):** проверяется локальный словарь функции **inner** — там есть **x** (присвоено **"local"**). Имя найдено, поиск останавливается. Значение **"local"** передаётся в **print**. (3) Если бы в **inner** не было **x = "local"**, то после проверки Local поиск перешёл бы в **E (Enclosing):** локальный словарь функции **outer** — там **x = "enclosing"**; тогда **print(x)** вывело бы **"enclosing"**. (4) Если бы **x** не было и в **outer**, проверялась бы **G (Global):** глобальный словарь модуля — **x = "global"**. (5) Если бы **x** не было и в модуле, проверялась бы **B (Built-in):** встроенные имена (**len**, **print**, **open** и т.д.). Итог: порядок жёсткий — L → E → G → B; первое совпадение определяет значение.

```python
x = "global"
def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)   # "local" — найдено в Local
    inner()
outer()
```

Если в **inner** убрать присваивание **x = "local"**, то **print(x)** выведет **"enclosing"** (найдено в Enclosing). Если в **outer** тоже убрать **x = "enclosing"**, то **print(x)** выведет **"global"**. Если и в модуле **x** не было бы, то искалось бы в Built-in (например, встроенная функция или исключение с таким именем).

#### Когда использовать global, когда nonlocal

- **global** — когда нужно читать и **менять** переменную уровня **модуля** из функции (счётчик вызовов, флаг, кэш в модуле). Без **global** присваивание создало бы локальную переменную с тем же именем, а глобальная не изменилась бы.
- **nonlocal** — когда нужно менять переменную **внешней функции** (но не модуля) из вложенной функции (замыкание с состоянием: счётчик, аккумулятор). Без **nonlocal** присваивание во вложенной функции создало бы локальную переменную, а переменная во внешней функции осталась бы прежней.

**Пример ошибки без nonlocal:**

```python
def counter():
    n = 0
    def inc():
        n = n + 1   # UnboundLocalError: local variable 'n' referenced before assignment
        return n
    return inc
```

Присваивание **n = n + 1** делает **n** локальной переменной в **inc**; правая часть **n + 1** обращается к этой же локальной **n** до присваивания — отсюда ошибка. Правильно: **nonlocal n** и затем **n = n + 1**.

#### Вложенные функции (несколько уровней) и nonlocal

**nonlocal** ищет имя в **ближайшей** внешней области (enclosing). Если вложенность три уровня (модуль → f → g → h), то в **h** выражение **nonlocal x** найдёт **x** в **g** (если оно там есть), а не в **f** и не в модуле. Чтобы изменить **x** в **f** из **h**, в **g** тоже должно быть **nonlocal x** (тогда **x** в **g** будет ссылаться на **x** из **f**), и в **h** — **nonlocal x**. Или передавать значение через аргументы и возвращаемые значения, чтобы не полагаться на длинные цепочки nonlocal.

**Пошаговый пример LEGB с тремя уровнями вложенности:**

```python
x = "global"
def f():
    x = "f"
    def g():
        x = "g"
        def h():
            # Без nonlocal: x = "local" создало бы локальную x в h
            nonlocal x   # ссылаемся на x из g
            x = "h"      # теперь x в g стала "h"
        h()
        print(x)   # "h"
    g()
f()
# Если в h написать global x, то изменилась бы глобальная x, а не x в g
```

**Замыкание (closure)** — функция, использующая переменные из **охватывающей** (enclosing) области видимости. Важно помнить про **late binding**: переменная из замыкания берётся **в момент вызова**, а не в момент определения функции. Классическая ловушка — цикл и замыкание:

```python
# Неправильно: все функции при вызове вернут 4 (значение i в момент вызова)
funcs = [lambda: i for i in range(5)]
[f() for f in funcs]   # [4, 4, 4, 4, 4]

# Правильно: захват текущего значения i через значение по умолчанию аргумента
funcs = [lambda i=i: i for i in range(5)]
[f() for f in funcs]   # [0, 1, 2, 3, 4]
```

**nonlocal** нужен, когда вложенная функция **присваивает** имя из внешней области (без **nonlocal** присваивание создало бы локальную переменную):

```python
def counter():
    n = 0
    def inc():
        nonlocal n
        n += 1
        return n
    return inc
c = counter()
c(), c(), c()   # 1, 2, 3
```

#### Область видимости в list/dict/set comprehensions (Python 3)

В Python 3 **list comprehension**, **dict comprehension**, **set comprehension** и **generator expression** имеют **собственную локальную область видимости**: переменная цикла (и другие имена, присвоенные внутри выражения) **не** попадают в окружающую область. После **x = [i for i in range(5)]** имя **i** в Python 3 **не** будет равно 4 в окружающей области — в Python 2 оно бы «протекало» и оставалось последним значением. Это устраняет типичный баг, когда переменная цикла перезаписывала внешнее имя.

### 4.3 Ловушки

#### Mutable default argument

**Опасный антипаттерн:** изменяемый объект (список, словарь, множество) в значении по умолчанию создаётся **один раз** при **определении** функции (при загрузке модуля или при выполнении **def**) и **переиспользуется** при всех последующих вызовах. То есть один и тот же объект (один и тот же **id**) используется как значение по умолчанию для параметра при каждом вызове без явной передачи аргумента. Если внутри функции этот объект **изменяется** (append, extend, pop и т.д.), изменения сохраняются между вызовами — и следующий вызов без аргумента «увидит» уже изменённый объект.

```python
# Плохо
def f(a=[]):
    a.append(1)
    return a
f()       # [1]
f()       # [1, 1]  — тот же список!
f()       # [1, 1, 1]
f([0])    # [0, 1]  — передан новый список, он не связан с «дефолтным»

# Хорошо
def f(a=None):
    if a is None:
        a = []
    a.append(1)
    return a
f()       # [1]
f()       # [1]  — каждый раз создаётся новый список
```

**Почему так происходит:** значения по умолчанию для параметров хранятся в атрибуте функции **`__defaults__`** (или **`__kwdefaults__`** для keyword-only). При **определении** функции (при выполнении **def**) вычисляется выражение по умолчанию (например, **[]**) и полученный объект сохраняется в \***\*defaults\*\***. При каждом вызове без соответствующего аргумента используется **тот же** объект из \***\*defaults\*\***, а не создаётся новый. Для неизменяемых типов (числа, строки, кортежи) это безопасно — объект не меняется. Для изменяемых (списки, словари, множества) один и тот же объект переиспользуется при всех вызовах; если функция его изменяет (append, pop и т.д.), изменения накапливаются.

**Пошагово: что происходит при def f(a=[]) и при вызовах f() и f().** (1) **Момент определения:** при выполнении **def f(a=[]):** интерпретатор вычисляет выражение **[]** **один раз** — создаётся один пустой список в памяти. Ссылка на этот объект сохраняется в **f.**defaults\***\* (кортеж значений по умолчанию для позиционных параметров). (2) **Первый вызов f():** аргумент **a** не передан, поэтому **a** связывается с объектом из **f.**defaults**[0]** — тем самым пустым списком. В теле **a.append(1)** изменяет этот список; он становится **[1]**. Функция возвращает **a** (тот же объект). (3) **Второй вызов f():** снова аргумент не передан; **a** снова связывается с **f.**defaults**[0]** — это **тот же объект**, который уже **[1]**. После **a.append(1)** он становится **[1, 1]**. Поэтому **id(a)** в первом и втором вызове одинаковый — один и тот же объект. (4) **Вызов f([0]):** передан явный аргумент; **a** связывается с **новым** списком **[0]**, а не с \*\***defaults\***\*; \*\***defaults\***\* не используется. Итог: объект по умолчанию создаётся при **def**, хранится в \*\***defaults\*\*\*\* и переиспользуется при каждом вызове без аргумента; мутация этого объекта видна при следующих вызовах.

**Проверка через id():**

```python
def f(a=[]):
    print(id(a))
    a.append(1)
    return a
f()   # например id 140234567890
f()   # тот же id 140234567890 — тот же объект!
f([]) # другой id — передан новый список
```

**Правило:** не используйте изменяемые объекты (list, dict, set) в значениях по умолчанию. Используйте **None** и внутри функции создавайте новый объект: **`if a is None: a = []`**. Альтернатива (менее распространённая): **`def f(a=None): a = [] if a is None else a`** — но проверка **`is None`** предпочтительна, чтобы явно различать «не передано» и «передан None».

**Почему проверка именно «a is None», а не «if not a»:** если допустимо передать пустой список **[]** и вы хотите отличать «не передано» от «передан пустой список», то **if not a** не подходит: и для «не передано» (тогда **a is None**), и для «передан []» выражение **not a** истинно. Поэтому идиоматично писать **if a is None: a = []** — новый список создаётся только когда аргумент не передан; если передали **[]** или **None** явно, поведение будет разным (при **a is None** создаём новый список, при **a=[]** используем переданный).

**Плохо vs лучше: mutable default в реальном сценарии.** Допустим, функция накапливает переданные теги: **def add_tags(item, tags=[]): tags.append(item); return tags**. При первом вызове **add_tags("a")** вернётся **["a"]**. При втором **add_tags("b")** без аргумента вернётся **["a", "b"]** — тот же список. Если вызывающий не знает об этом, он ожидает **["b"]**. Правильно: **def add_tags(item, tags=None): tags = [] if tags is None else tags; tags.append(item); return tags** — при каждом вызове без **tags** создаётся новый список. Или, если нужно именно накапливать между вызовами, это должно быть явно (например, глобальный список или объект-контейнер), а не побочный эффект значения по умолчанию.

**isinstance(obj, (A, B))** — второй аргумент может быть **кортеж типов**; тогда возвращается **True**, если объект является экземпляром **любого** из перечисленных типов. Удобно для проверки «число или строка»: **isinstance(x, (int, float))**. **issubclass(cls, (A, B))** — аналогично для классов.

#### eval, exec, compile: риски безопасности

**eval(source)** выполняет **одно выражение** (expression) и возвращает его значение; **exec(source)** выполняет **код** (одну или несколько инструкций), возвращает **None**. Оба принимают опциональные **globals** и **locals** — словари, в контексте которых выполняется код. **compile(source, filename, mode, ...)** компилирует строку в объект кода (mode: **'exec'**, **'eval'**, **'single'**); результат можно передать в **eval** или **exec**.

**Почему это опасно:** если **source** приходит от пользователя (ввод с клавиатуры, из файла, из сети), злоумышленник может передать произвольный код — например, **`__import__('os').system('rm -rf /')`** или чтение секретов из окружения. **Никогда** не передавайте в **eval**/ **exec** непроверенные данные от пользователя. Для разбора выражений (числа, простые формулы) используйте **ast.literal_eval** (безопасно только для литералов) или парсеры/валидаторы (например, для конфигов — **json**, **toml**). **eval**/ **exec** допустимы только для кода, полностью контролируемого вами (например, сгенерированного вашей программой).

#### Важные отличия builtins при переходе с Python 2 на Python 3

- **input():** в Py2 **input(prompt)** выполнял введённую строку как код Python (опасно!); для ввода строки использовали **raw_input(prompt)**. В Py3 **raw_input** удалён, **input(prompt)** всегда возвращает **строку** (str) — ввод интерпретируется как текст, а не как код. Миграция: заменить **raw_input** на **input**.
- **map, filter, zip:** в Py2 возвращали **списки**; в Py3 возвращают **итераторы** (ленивые). Код вида **map(f, seq)[0]** в Py3 сломается — нужно **next(map(f, seq))** или **list(map(f, seq))[0]**.
- **enumerate(iterable, start=0):** второй аргумент **start** задаёт начальное значение индекса (по умолчанию 0). Пример: **enumerate(["a", "b"], start=1)** даёт **(1, "a")**, **(2, "b")**. Удобно для нумерации с единицы или с заданного смещения.
- **super():** в Py3 внутри метода класса можно вызывать **super()** без аргументов — интерпретатор подставит текущий класс и экземпляр (или класс для classmethod). В Py2 писали **super(CurrentClass, self)**. Без аргументов **super()** работает только внутри определения метода класса.
- **zip(\*iterables, strict=False):** в Python 3.10+ добавлен аргумент **strict**: если **True**, то **zip** требует одинаковую длину итерабельных объектов; при разной длине выбрасывается **ValueError**. По умолчанию **strict=False** — поведение как раньше (остановка по кратчайшей последовательности).

#### hasattr и getattr: почему это ловушка

**getattr(obj, name[, default])** — возвращает значение атрибута **name** объекта **obj**. Если атрибут не найден и задан **default**, возвращается **default**; иначе выбрасывается **AttributeError**. Удобно для доступа к атрибутам по строковому имени (динамический выбор поля) или когда отсутствие атрибута не ошибка: **getattr(obj, 'optional_field', None)**.

**Когда использовать getattr с default:** когда имя атрибута задаётся строкой (например, из конфига или от пользователя) и отсутствие атрибута допустимо — **getattr(obj, key, None)** или **getattr(obj, key, default_value)** избавляет от **try/except AttributeError**. Если отсутствие атрибута — ошибка, можно вызывать **getattr(obj, name)** без default и обрабатывать **AttributeError** выше, либо проверять через **hasattr** (с оговоркой про property, см. ниже). Для безопасного доступа к вложенным атрибутам по цепочке имён иногда пишут цикл с **getattr** или используют **functools.reduce(getattr, names.split('.'), obj)**.

**hasattr(obj, name)** реализован как вызов **getattr(obj, name)** с перехватом **исключений**: если **getattr** выбрасывает исключение, **hasattr** возвращает **False**, иначе **True**. Таким образом, **hasattr** «проглатывает» не только **AttributeError** (атрибут отсутствует), но и **любое** исключение, которое может выбросить **getattr** — в том числе исключения из **свойства (property)** или **дескриптора** при обращении к атрибуту (например, **ValueError**, **TypeError**, или кастомные исключения внутри геттера). В результате **hasattr(obj, 'x')** может вернуть **False** не потому, что атрибута нет, а потому, что при обращении к **obj.x** произошла ошибка. Поэтому в критичном коде предпочтительнее явная проверка: **try: getattr(obj, name); except AttributeError: ...** — так вы не скрываете другие исключения. Для простых объектов без свойств и дескрипторов **hasattr** обычно безопасен.

**Пошагово: что происходит при hasattr(obj, 'x'), когда x — property с геттером, выбрасывающим исключение.** (1) **hasattr(obj, 'x')** внутри вызывает **getattr(obj, 'x')**. (2) Интерпретатор ищет **x** в классе объекта; находит **property** (дескриптор). (3) Вызывается **property.**get**(obj, type(obj))** — то есть выполняется **ваш геттер** (функция, переданная в **@property**). (4) Геттер выполняется и выбрасывает, например, **ValueError**. (5) Исключение «всплывает» из **getattr**; **hasattr** перехватывает **любое** исключение (не только **AttributeError**) и возвращает **False**. Итог: **hasattr** не различает «атрибута нет» и «при доступе к атрибуту произошла ошибка» — в обоих случаях возвращается **False**. Поэтому при атрибутах, реализованных через **property** или дескрипторы с побочными эффектами, предпочтительнее **try: getattr(obj, name); except AttributeError: ...**.

**breakpoint()** (Python 3.7+) — встроенная функция, вызывающая отладчик (по умолчанию **pdb**). Эквивалентна **import pdb; pdb.set_trace()**, но учитывает переменную окружения **PYTHONBREAKPOINT** (можно отключить или подменить отладчик). Удобно для условной остановки в коде без ручного импорта **pdb**.

**Пример, когда hasattr «обманывает»:**

```python
class C:
    @property
    def x(self):
        raise ValueError("атрибут x временно недоступен")

obj = C()
hasattr(obj, "x")   # False! Атрибут есть (property), но getattr бросил ValueError
# Ожидали True (атрибут существует), получили False из-за исключения в геттере

# Явная проверка — видим реальное исключение
try:
    getattr(obj, "x")
except AttributeError:
    pass   # атрибута нет
# ValueError пробросится — мы узнаем о проблеме в геттере
```

#### dir(), round(), reversed()

- **dir([obj])** — без аргументов возвращает список имён в текущей области видимости (локальные имена). С аргументом **dir(obj)** возвращает список имён атрибутов объекта (в т.ч. методов). Удобно для интроспекции: посмотреть, какие атрибуты есть у объекта. Не гарантирует полный список (может быть переопределён \***\*dir\*\***); порядок имён не гарантирован.
- **round(x[, ndigits])** — округление **x** до **ndigits** знаков после запятой (по умолчанию 0, т.е. до целого). В Python 3 используется **banker's rounding** (округление к ближайшему чётному при 0.5): **round(2.5)** → **2**, **round(3.5)** → **4**. Для предсказуемого «школьного» окружения 0.5 вверх используйте **math.floor(x + 0.5)** или **int(x + 0.5)** для неотрицательных.
- **reversed(seq)** — возвращает **итератор**, выдающий элементы последовательности в обратном порядке. Требует от объекта поддержки \***\*reversed\*\*** или \***\*len\*\*** и \***\*getitem\*\***. Результат **одноразовый** (итератор). Для списка в виде списка используйте **list(reversed(lst))** или срез **lst[::-1]** (создаёт копию).

```python
dir()   # имена в текущей области
dir([]) # атрибуты списка: append, extend, ...
round(2.5), round(3.5)   # (2, 4) — banker's rounding
list(reversed([1, 2, 3]))   # [3, 2, 1]
```

**Пошагово: что возвращает dir(obj) vs dir().** (1) **dir()** без аргументов — возвращает список строковых имён в текущей локальной области видимости (переменные, функции, классы и т.п., определённые в текущем блоке). Порядок имён соответствует порядку определения; содержимое зависит от контекста (модуль, функция, класс). (2) **dir(obj)** — возвращает список строковых имён атрибутов объекта **obj**. Реализовано как **dir(obj) ≈ sorted(obj.**dict**.keys() + [attr for attr in dir(obj.__class__) if not attr.startswith('_')])** (упрощённо); в реальности Python собирает атрибуты из **obj.**dict**** и из классов в MRO, исключает дубликаты. Метод **obj.**dir**()** может переопределить результат; если не определён, используется стандартная логика. **dir(obj)** не гарантирует, что все имена доступны (часть может быть свойством с побочными эффектами); порядок не гарантирован (в старых версиях Py3 порядок мог отличаться).

**Граничные случаи round:** (1) **round(2.5)** → **2**, **round(3.5)** → **4** — banker's rounding: при 0.5 округляется к **ближайшему чётному**. (2) **round(x, ndigits)** — округление до **ndigits** знаков после запятой: **round(2.675, 2)** может дать **2.67** (из‑за представления float в двоичной системе: 2.675 не представляется точно). Для денежных расчётов используйте **Decimal**. (3) **round(x, 0)** возвращает **float** (не int): **round(2.5, 0)** → **2.0**. (4) Отрицательный **ndigits**: **round(1234, -2)** → **1200** — округление до сотен.

#### Краткие примеры ключевых builtins

```python
# enumerate со start
for i, x in enumerate(["a", "b", "c"], start=1):
    print(i, x)   # 1 a, 2 b, 3 c

# next с default
it = iter([1, 2])
next(it, 99)   # 1
next(it, 99)   # 2
next(it, 99)   # 99 — итератор исчерпан, возвращён default

# sorted с key
sorted(["aa", "b", "ccc"], key=len)   # ["b", "aa", "ccc"]
sorted([(1, "b"), (2, "a")], key=lambda p: p[1])   # [(2, "a"), (1, "b")]
```

#### Практический вывод по функциям

- Для опций и флагов, которые не должны передаваться «по ошибке» позиционно, используйте **keyword-only** аргументы (после **`*`** в сигнатуре).
- Для параметров, имена которых могут меняться в будущем (совместимость API), используйте **positional-only** (до **`/`**, Python 3.8+).
- В замыканиях помните о **late binding**: переменная из внешней области берётся в момент **вызова** функции. В циклах используйте значение по умолчанию аргумента для захвата текущего значения: **lambda i=i: ...**.
- Никогда не используйте изменяемые объекты (list, dict, set) в значениях по умолчанию; используйте **None** и внутри функции создавайте новый объект.

- **map(function, iterable, \*iterables)** — итератор результатов function(elem)
- **max(iterable)** / **max(..., key=..., default=...)**
- **memoryview(obj)** — buffer protocol; представление памяти без копирования
- **min(iterable)** / **min(..., key=..., default=...)**
- **next(iterator[, default])** — следующий элемент; default при StopIteration
- **object()** — базовый класс; экземпляр без атрибутов
- **oct(x)** — строка восьмеричного представления с "0o"
- **open(file, mode='r', buffering=-1, encoding=None, errors=None, ...)** — файл; encoding по умолчанию UTF-8 в Py3
- **ord(c)** — Unicode код символа; обратное к **chr()**
- **pow(base, exp[, mod])** — base\*\*exp; с mod — эффективнее для больших чисел
- **print(\*objects, sep=' ', end='\\n', file=sys.stdout, flush=False)** — вывод; функция в Py3
- **property(fget=None, fset=None, fdel=None, doc=None)** — дескриптор свойства
- **range(stop)** / **range(start, stop[, step])** — ленивый диапазон; не список
- **repr(obj)** — строковое представление для отладки; \***\*repr\*\***
- **reversed(seq)** — обратный итератор; \***\*reversed\*\*** или \***\*len\*\***+\***\*getitem\*\***

- **slice(stop)** / **slice(start, stop[, step])** — объект среза; **seq[slice(...)]**
- **sorted(iterable, \*, key=None, reverse=False)** — новый отсортированный список
- **staticmethod(func)** — декоратор; метод без self/cls
- **str(object)** — строка Unicode; \***\*str\*\***, \***\*repr\*\***
- **sum(iterable, /, start=0)** — сумма
- **super([type[, object_or_type]])** — без аргументов в Py3: автоматический MRO
- **tuple([iterable])** — кортеж
- **type(obj)** — тип объекта; **type(name, bases, dict)** — создание класса (метакласс)
- **vars([obj])** — **obj.**dict\***\*; без аргумента — **locals()\*\*
- **zip(\*iterables, strict=False)** — итератор кортежей (3.10+ strict: равная длина)
- \***\*import**(name, globals=None, locals=None, fromlist=(), level=0)** — низкоуровневый импорт; предпочитать **importlib.import_module()\*\*

##### **import**(name, ...) — низкоуровневый импорт

****import**(name, globals=None, locals=None, fromlist=(), level=0)** — встроенная функция, которую **интерпретатор вызывает** при выполнении инструкции **import** (и **from ... import ...**). Прямой вызов в коде возможен, но **рекомендуется использовать importlib.import_module(name)** — он проще и предсказуемее.

**Параметры:** **name** — строка имени модуля (например, **"os"**, **"package.submodule"**). **globals** и **locals** — словари для контекста (при обычном вызове из кода можно **globals()**, **locals()** или **None**). **fromlist** — при **from package import submodule** сюда передаётся список имён (например, **["submodule"]**); если **fromlist** не пуст, возвращается подмодуль (иначе возвращается верхний пакет). **level** — уровень относительного импорта: **0** (абсолютный), **-1** (относительно текущего пакета родителя) и т.д.

**Пример:** ****import**("os")** возвращает модуль **os** (то же, что **import os** и затем **os**). ****import**("os.path", fromlist=["join"])** возвращает модуль **os.path** (при **from os.path import join** интерпретатор получает **os.path** через ****import**** с **fromlist=["join"]**). Динамический импорт по строке: **mod = importlib.import_module("mymod")** — предпочтительнее **mod = **import**("mymod")**, так как **import_module** всегда возвращает «конечный» модуль и не путается с пакетами.

**Когда использовать:** для **динамического импорта по строке** (имя модуля из конфига, плагины) используйте **importlib.import_module(name)**. ****import**** можно подменить (например, в тестах или для кастомной загрузки модулей); в обычном коде явно вызывать ****import**** не требуется.

**Пошагово: что происходит при **import**("os.path", fromlist=["join"]).** (1) Интерпретатор ищет модуль по имени: сначала **"os"** (верхний пакет), загружает его при необходимости. (2) Затем при **fromlist=["join"]** (не пустой) возвращается подмодуль **os.path** — то есть объект модуля **os.path**, а не **os**. (3) Если бы **fromlist** был пуст, ****import**("os.path")** вернул бы модуль **os** (верхний пакет), а не **os.path**. Поэтому при эмуляции **from os.path import join** вызывают ****import**("os.path", fromlist=["join"])**, чтобы получить именно **os.path**. **importlib.import_module("os.path")** всегда возвращает «конечный» модуль **os.path** без необходимости помнить про **fromlist**.

**Граничные случаи:** **level** для относительного импорта: ****import**("pkg.mod", globals(), locals(), [], 1)** — уровень 1 означает «импорт относительно текущего пакета» (один уровень вверх от текущего ****package****). Обычно относительный импорт делают через **importlib.import_module** с учётом ****package****. Пустая строка ****import**("")** — **ValueError**. Имя с точками **"a.b.c"** — загружается **a**, затем **a.b**, затем **a.b.c**; возвращаемое значение зависит от **fromlist** (если пуст — вернётся **a**).

**Типичные ошибки:** ожидать, что ****import**("os.path")** без **fromlist** вернёт **os.path** — вернётся **os**. Для динамического импорта подмодуля использовать **importlib.import_module("os.path")**. Подменять ****import**** в **builtins** без учёта побочных эффектов — многие части интерпретатора и сторонние библиотеки полагаются на стандартный импорт; подмена для тестов или плагинов должна быть аккуратной.

**Полный пример: динамический импорт через importlib (рекомендуется).**

```python
import importlib

# Имя модуля из конфига или пользовательского ввода (валидированного)
module_name = "json"
mod = importlib.import_module(module_name)
mod.loads('{"a": 1}')   # вызов функции из динамически загруженного модуля

# Подмодуль пакета — import_module всегда возвращает «конечный» модуль
path_module = importlib.import_module("os.path")
path_module.join("a", "b")   # 'a/b' (на Unix)
```

**Пример: **import** с fromlist и без (для понимания разницы).**

```python
# Без fromlist возвращается верхний пакет
os_top = __import__("os.path")
os_top.path   # модуль os.path (доступ через атрибут)
os_top.name   # 'os'

# С fromlist — возвращается запрошенный подмодуль
os_path = __import__("os.path", fromlist=["path"])
os_path.join("a", "b")   # сразу os.path
```

**Практический вывод:** для динамического импорта по строке всегда используйте **importlib.import_module(name)** — не нужно помнить про **fromlist** и **level**; для относительного импорта — **importlib.import_module(name, package=**package**)**.

<details>
<summary>Вопрос: что вернёт <code>__import__("os.path")</code> без второго аргумента? И как правильно получить модуль <code>os.path</code> через <code>__import__</code>?</summary>

****import**("os.path")** без **fromlist** (или с **fromlist=()**) возвращает модуль **os** (верхний пакет), а не **os.path**. Интерпретатор при **import os.path** загружает оба модуля, но ****import**** по контракту возвращает «самый левый» модуль в имени, если **fromlist** пуст.

Чтобы получить именно **os.path**, нужно вызвать ****import**("os.path", fromlist=["path"])** или ****import**("os.path", fromlist=["join"])** (любое непустое **fromlist** с именем из подмодуля). Проще и надёжнее: **importlib.import_module("os.path")** — всегда возвращает конечный модуль. См. [§4.4a](#44a-все-встроенные-функции-builtins--справочник).

</details>

**Когда что использовать (кратко):** **iter(callable, sentinel)** — когда нужно итерировать по вызовам функции до «стоп-значения»: например, читать из файла блоками **iter(lambda: f.read(1024), b'')** или читать строки до пустой **iter(f.readline, '')**. **divmod(a, b)** — когда нужны одновременно частное и остаток (страницы, разбиение на часы/минуты): **q, r = divmod(seconds, 60)**. **enumerate(iterable, start=0)** — когда нужен индекс вместе с элементом; **start=1** для нумерации с единицы: **for i, x in enumerate(items, 1): print(f'{i}. {x}')**. **zip(\*iterables, strict=True)** (3.10+) — когда длины должны совпадать; **strict=False** — по кратчайшей.

#### 4.4b Встроенные константы (справочник)

Встроенные константы доступны без импорта; их имена зарезервированы. Для обучения: **True**/ **False** — единственные экземпляры **bool**; **None** — единственный экземпляр **NoneType** (отсутствие значения); **Ellipsis** (**...**) используется в срезах и type hints; **NotImplemented** — сигнал «операция не реализована» в бинарных операциях (не путать с **NotImplementedError**).

- **True**, **False** — булевы; подкласс int; единственные экземпляры bool
- **None** — единственный экземпляр NoneType; отсутствие значения
- **quit**, **exit** — для интерактивного выхода; не использовать в скриптах
- **copyright**, **credits**, **license** — информационные объекты

##### Ellipsis ( ... )

**Ellipsis** — единственный экземпляр типа **types.EllipsisType**; литерал **...** (три точки). Используется в нескольких контекстах:

- **Срезы:** в **numpy** и других библиотеках **arr[..., 0]** означает «все оси до последней, затем индекс 0 по последней оси»; **...** заменяет «сколько угодно осей». В стандартном списке **lst[..., 0]** не поддерживается (только **slice** с **start**, **stop**, **step**); для многомерных массивов **...** — Ellipsis как объект в слоте среза.
- **Type hints:** **Tuple[int, ...]** — кортеж произвольной длины с элементами типа **int**; **Callable[..., int]** — функция с произвольной сигнатурой, возвращающая **int**. **...** здесь обозначает «произвольное/пропуск».
- **Заглушки в коде:** иногда пишут **...** (Ellipsis) в теле функции или класса как заглушку вместо **pass** (синтаксически допустимо: **def f(): ...**).

**Проверка:** **type(...)** → **<class 'ellipsis'>**; **... is Ellipsis** → **True**. Не путать с **pass** (инструкция) и с многоточием в строке (**"..."** — строка из трёх точек).

**Граничные случаи:** в **type hints** **...** в **Tuple[T, ...]** означает «кортеж произвольной длины с элементами типа T»; в **Callable[..., R]** — «произвольная сигнатура, возвращает R». В **срезах** стандартный **list** не поддерживает **Ellipsis**; **lst[..., 0]** даёт **TypeError** (поддержка есть в **numpy** и других библиотеках). В сигнатурах функций **...** может обозначать «обязательные аргументы до сюда» (PEP 570, positional-only): **def f(a, b, /):** эквивалентно «все аргументы до / только позиционные».

**Полный пример: Ellipsis в заглушках и type hints.**

```python
# Заглушка в теле функции или класса (синтаксически допустимо)
def stub():
    ...

class Stub:
    ...

# Type hints
from typing import Tuple, Callable
def process(items: Tuple[int, ...]) -> None:   # кортеж int произвольной длины
    ...
def on_event(cb: Callable[..., None]) -> None:  # функция с любой сигнатурой, возвращает None
    ...
```

**Практический вывод:** **...** в типах — стандартный способ обозначить «произвольную длину» или «любую сигнатуру»; в коде как заглушка — альтернатива **pass** (стилистический выбор).

<details>
<summary>Вопрос: что вернёт <code>lst[..., 0]</code> для обычного списка <code>lst = [[1,2],[3,4]]</code>?</summary>

Для стандартного **list** срез с **Ellipsis** не поддерживается: **lst[..., 0]** вызовет **TypeError: list indices must be integers or slices, not ellipsis**. В **numpy** и других библиотеках с многомерными массивами **arr[..., 0]** означает «все оси до последней, затем индекс 0 по последней оси» (например, для матрицы — первый столбец). См. [§4.4b](#44b-встроенные-константы-справочник).

</details>

##### NotImplemented

**NotImplemented** — единственный экземпляр типа **types.NotImplementedType**; встроенная константа. Используется как **возвращаемое значение** из методов бинарных операций (\***\*add\*\***, \***\*eq\*\***, \***\*lt\*\*** и т.д.), чтобы сообщить интерпретатору: «операция для данного типа операнда не реализована — попробуй отражённый метод у другого операнда». Интерпретатор при получении **NotImplemented** из **a.**add**(b)** пробует **b.**radd**(a)**; если и там **NotImplemented** — **TypeError**.

**Пример:** в ****eq**** можно вернуть **NotImplemented**, если сравнение с другим типом не определено — тогда Python попробует **other.**eq**(self)**. Не путать с **NotImplementedError** (исключение): **NotImplemented** — константа для возврата из метода; **raise NotImplementedError** — явное исключение «метод не реализован».

**Пошагово: что происходит при возврате NotImplemented из **eq**.** (1) Вызов **a == b**: интерпретатор вызывает **a.**eq**(b)**. (2) Если **a.**eq**(b)** возвращает **NotImplemented**, интерпретатор пробует **b.**eq**(a)** (отражённый метод). (3) Если **b.**eq**(a)** тоже возвращает **NotImplemented**, результат сравнения **False** (для **==**); для **!=** — **True**. (4) Если **a.**eq**(b)** возвращает **True** или **False**, это и есть результат; отражённый метод не вызывается. Поэтому в ****eq**** для «не знаю, как сравнивать с этим типом» возвращают **NotImplemented**, а не **False** — иначе **MyClass() == None** было бы **False** без попытки **None.**eq**(MyClass())**.

**Проверка:** **type(NotImplemented)** → **<class 'NotImplementedType'>**. В **if** контексте **NotImplemented** считается **True** (это не **False**).

**Типичные ошибки:** путать **NotImplemented** и **NotImplementedError** — первое возвращают из метода бинарной операции, второе выбрасывают (**raise**). Возвращать **False** вместо **NotImplemented** в ****eq****, когда сравнение с типом **other** не определено — тогда **a == b** и **b == a** могут дать разный результат (если у **b** реализован ****eq**** для типа **a**).

**Полный пример: **eq** с return NotImplemented.**

```python
class MyValue:
    def __init__(self, x):
        self.x = x
    def __eq__(self, other):
        if not isinstance(other, MyValue):
            return NotImplemented   # пусть other попробует сравнить с собой
        return self.x == other.x

v = MyValue(1)
v == MyValue(1)   # True
v == 1            # False — MyValue.__eq__ вернул NotImplemented, int.__eq__(1, v) вернул False
1 == v            # False — int.__eq__ вернул NotImplemented, MyValue.__eq__(v, 1) вернул False
v == None         # False — MyValue.__eq__ вернул NotImplemented, None.__eq__(v) вернул NotImplemented → False
```

**Практический вывод:** в ****eq****, ****add**** и других бинарных методах для «не знаю, как с этим типом» возвращайте **NotImplemented**, а не **False** и не исключение — тогда интерпретатор попробует отражённый метод у **other**.

<details>
<summary>Вопрос: чем отличается <code>return NotImplemented</code> от <code>raise NotImplementedError</code>?</summary>

**return NotImplemented** — возвращаемое **значение** из метода бинарной операции (\***\*eq\*\***, \***\*add\*\*** и т.д.). Интерпретатор воспринимает его как сигнал «попробуй отражённый метод у другого операнда»; исключение не выбрасывается.

**raise NotImplementedError** — **выброс исключения** «метод не реализован». Используется в абстрактных методах или заглушках, когда вызов метода в текущем классе не поддерживается и должен быть переопределён в подклассе. При вызове такого метода выполнение прерывается и исключение пробрасывается вверх. См. [§4.4b](#44b-встроенные-константы-справочник).

</details>

##### **debug**

****debug**** — встроенная константа **bool**. Значение **True**, если интерпретатор запущен **без** флага **-O** (оптимизация); **False** при запуске с **-O** или **-OO**. Используется для условной компиляции: инструкция **assert** выполняется **только** когда ****debug**** истинно; при **-O** проверки **assert** не выполняются (и не генерируется байткод для них).

**Пример:** **if **debug**:** ... — блок кода (например, дополнительные проверки или логирование) будет выполняться только в «отладочном» режиме; при **python -O script.py** этот блок можно исключить из исполнения. Не изменяйте ****debug**** вручную — значение задаётся интерпретатором при старте.

**Пошагово: как -O влияет на assert и **debug**.** При запуске **python script.py** (без **-O**) ****debug**** равен **True**; инструкции **assert cond, msg** компилируются и выполняются — при **cond == False** выбрасывается **AssertionError**. При запуске **python -O script.py** ****debug**** равен **False**; инструкции **assert** **не генерируют** байткод (они удаляются на этапе компиляции), поэтому проверки не выполняются и не тратят время. Условие **if **debug**:** тоже может быть оптимизировано — блок может не выполняться при **-O**. Флаг **-OO** дополнительно удаляет docstrings из байткода.

**Типичные ошибки:** полагаться на выполнение кода внутри **assert** в продакшене — при **-O** этот код не выполнится (например, **assert setup(), "fail"** — **setup()** не вызовется при **-O**). Для обязательной логики используйте обычный **if**, а не **assert**. Менять ****debug** = False** в коде — не гарантирует отключение **assert** (байткод уже сгенерирован); режим задаётся при запуске интерпретатора.

**Полный пример: assert и -O.**

```python
# script.py
if __debug__:
    print("Отладочный режим")

def check(x):
    assert x > 0, "x должен быть положительным"
    # При -O эта строка не выполнится — assert удаляется при компиляции
    return x

# python script.py        → напечатает "Отладочный режим"; check(-1) вызовет AssertionError
# python -O script.py     → не напечатает; check(-1) вернёт -1 без ошибки (assert отключён)
```

**Практический вывод:** используйте **assert** только для проверок, которые можно отключить в продакшене (инварианты, отладочные проверки). Для обязательной валидации ввода или состояния используйте **if ... raise**.

<details>
<summary>Вопрос: выполнится ли <code>setup()</code> при запуске с <code>python -O</code>, если написать <code>assert setup(), "fail"</code>?</summary>

**Нет.** При **-O** инструкция **assert** **не компилируется** — для неё не генерируется байткод. Поэтому выражение **setup()** вообще не вычисляется: ни вызов **setup()**, ни проверка результата не выполняются. Для кода, который должен выполняться всегда (в том числе в продакшене), используйте обычный **if**: **if not setup(): raise RuntimeError("fail")**. См. [§4.4b](#44b-встроенные-константы-справочник).

</details>

#### 4.4c Встроенные типы и исключения (справочник)

Типы и исключения — встроенные имена; их можно использовать для **isinstance**, **type**, **raise**. #### 4.4d Методы встроенных объектов (справочник)

Ниже перечислены **методы и атрибуты** встроенных типов. Используйте справочник для быстрого поиска: «есть ли у str метод X», «какие методы у dict», «что возвращает dict.popitem». При обучении сначала закрепите методы из основных разделов (§1–§3: str.encode/decode, split/partition, list.append/extend, dict.get/setdefault, set.union/update и т.д.); полный список — для повторения и справки.

##### memoryview — представление памяти без копирования

**memoryview(obj)** — встроенная функция, возвращающая **представление памяти** (buffer) объекта, поддерживающего **buffer protocol** (bytes, bytearray, array.array и др.). Элементы не копируются — **memoryview** даёт доступ к тем же байтам, что и исходный объект; изменение через **memoryview** (если объект изменяемый) видно в оригинале.

**Методы и атрибуты:**

- **mv.tobytes()** — копия данных в виде **bytes** (создаёт новый объект).
- **mv.tolist()** — копия данных в виде списка целых (0–255 для формата 'B').
- **mv.hex([sep[, bytes_per_sep]])** — hex-строка (аналог **bytes.hex()**).
- **mv.release()** — освободить буфер (после вызова дальнейшее использование **mv** не допускается; для **with** контекста удобно **memoryview** в **with** не обязателен, но **release()** явно освобождает ресурс).
- **mv.cast(format[, shape])** — привести представление к другому формату (например, 'B' → 'i' для int) или форме; возвращает новый **memoryview** на те же данные.
- Атрибуты: **itemsize** (размер одного элемента в байтах), **nbytes** (общий размер в байтах), **ndim** (число измерений), **readonly** (только чтение?), **shape** (размеры), **strides**, **suboffsets**, **obj** (исходный объект).

**Пример: чтение и изменение через memoryview.**

```python
ba = bytearray(b"hello")
mv = memoryview(ba)
mv[0]   # 104 (int)
mv[0] = 72   # меняет ba[0] — ba станет b'Hello'
mv.tobytes()   # b'Hello'
mv[1:4].tobytes()   # b'ell' — срез memoryview
# cast: представить байты как int (little-endian, 4 байта на int)
import array
arr = array.array('i', [1, 2, 3])
mv = memoryview(arr)
mv.tolist()   # [1, 2, 3]; изменение mv меняет arr
```

**Когда использовать:** когда нужен доступ к памяти **без копирования** (большие буферы, срезы, передача в C-расширения через buffer protocol); для простого преобразования bytes ↔ int удобнее **int.from_bytes** / **to_bytes**. **memoryview** поддерживает срезы **mv[i:j]** без копирования данных.

**Пошагово: что происходит при mv[i] = x и при mv.cast().** (1) **mv[i] = x**: интерпретатор записывает значение **x** в **i**-й элемент буфера. Для **memoryview** на **bytearray** или **array.array** это меняет **исходный объект** — **obj[i]** станет равен приведённому **x** (для байт — целое 0–255). Для **memoryview** на **bytes** (неизменяемый) присваивание **mv[i] = x** вызовет **TypeError: cannot modify read-only memory**. (2) **mv.cast(format)**: создаётся **новый** memoryview на **те же** данные в памяти, но с другим форматом интерпретации. Например, **mv = memoryview(bytearray(4)); mv.cast('i')** — те же 4 байта интерпретируются как один int (формат **'i'** — 4 байта, платформозависимый порядок). Изменение **mv.cast('i')[0]** изменит те же байты в исходном bytearray. Форматы: **'B'** (unsigned byte), **'b'** (signed byte), **'i'** (int), **'I'** (unsigned int), **'f'** (float) и др.; размер должен быть кратен **itemsize** нового формата.

**Граничные случаи:** **memoryview(bytes_obj)** — только чтение; **mv[0] = 1** даёт **TypeError**. Срез **mv[1:3]** — новый memoryview на ту же память (без копирования); изменение среза меняет исходный объект. После **mv.release()** любой доступ к **mv** даёт **ValueError: operation on released memoryview object**. **mv.tobytes()** и **mv.hex()** создают копию данных; для больших буферов это расход памяти.

**Типичные ошибки:** присваивать в **memoryview** на **bytes** — только чтение. Путать **mv.tolist()** (копия) с изменением через **mv[i]** — изменение **mv[i]** не создаёт копии, оно меняет исходный объект. Не вызывать **release()** при работе с буфером, полученным из C-расширения с ограниченным временем жизни — возможен use-after-free.

**Пример: readonly vs writable и cast в разные форматы.**

```python
# Readonly: memoryview на bytes — только чтение
b = b"hello"
mv_b = memoryview(b)
mv_b[0]   # 104
mv_b.readonly   # True
# mv_b[0] = 72   # TypeError: cannot modify read-only memory

# Writable: memoryview на bytearray — можно менять
ba = bytearray(b"hello")
mv_ba = memoryview(ba)
mv_ba[0] = 72   # ba станет b'Hello'
mv_ba.readonly   # False

# cast: одни и те же байты как int (4 байта на int, little-endian)
import array
arr = array.array('i', [0x01020304, 0x05060708])
mv = memoryview(arr)
mv.tobytes()   # b'\x04\x03\x02\x01\x08\x07\x06\x05' (little-endian)
mv_int = mv.cast('i')
mv_int[0], mv_int[1]   # (67305985, 134678021)  — те же 8 байт как два int
mv_int[0] = -1   # меняет первые 4 байта в arr
arr[0]   # -1
```

**Пример: release() и использование после release.**

```python
ba = bytearray(4)
mv = memoryview(ba)
mv[0] = 1
mv.release()
# mv[0]   # ValueError: operation on released memoryview object
# ba[0]   # 1 — исходный объект по-прежнему доступен
```

**Практический вывод:** используйте **memoryview** для доступа к большим буферам без копирования (срезы, передача в C); для **bytes** представление только для чтения. Перед изменением проверяйте **mv.readonly** или используйте **bytearray**/ **array.array**.

<details>
<summary>Вопрос: чем отличается <code>mv.tobytes()</code> от <code>bytes(mv)</code>? И можно ли изменить исходный bytearray через срез memoryview?</summary>

**mv.tobytes()** и **bytes(mv)** для memoryview дают один и тот же результат — копию данных в виде **bytes**. Оба создают **новый** объект; изменение этого bytes не влияет на исходный буфер.

Срез **mv[1:4]** — это **новый memoryview** на **ту же** память (без копирования данных). Присваивание **mv[1:4][0] = 72** изменит **второй** байт исходного объекта (bytearray или array), потому что срез memoryview разделяет память с оригиналом. См. [§4.4d](#44d-методы-встроенных-объектов-справочник).

</details>

##### file object — методы файлового объекта

Объект, возвращаемый **open()**, поддерживает методы ввода-вывода. В **текстовом** режиме (**'r'**, **'w'**) работа идёт со **строками** (str); в **бинарном** (**'rb'**, **'wb'**) — с **bytes**.

**Чтение:**

- **f.read([size])** — прочитать до **size** символов (текст) или байт (бинарный режим); без аргумента — до конца файла. Возвращает **str** или **bytes**; при достижении конца файла — пустая строка **''** или **b''**.
- **f.readline([size])** — одна строка (до перевода строки или до **size** символов/байт); в конце файла — **''** или **b''**.
- **f.readlines([hint])** — список всех строк (или до **hint** байт/символов, если задан). Для больших файлов предпочтительнее итерация **for line in f:** (файл — итератор по строкам).

**Запись:**

- **f.write(s)** — записать строку **s** (текст) или **bytes** (бинарный); возвращает число записанных символов/байт. Буфер может откладывать запись до **flush()** или закрытия.
- **f.writelines(lines)** — записать последовательность строк (без добавления переводов строк; если нужны переводы, добавляйте **'\\n'** в каждую строку или используйте **write** в цикле).

**Позиционирование:**

- **f.tell()** — текущая позиция в файле (в байтах для бинарного режима; для текстового — непрозрачное число).
- **f.seek(offset[, whence])** — перейти к позиции **offset**; **whence**: **0** (начало файла, по умолчанию), **1** (текущая позиция), **2** (конец файла). В текстовом режиме **seek** обычно только к позициям, полученным от **tell()** (из-за кодировки и переводов строк).
- **f.seekable()** — поддерживает ли файл **seek** (True для обычных файлов; False для потоков ввода и т.п.).
- **f.readable()**, **f.writable()** — можно ли читать/писать.

**Буфер и закрытие:**

- **f.flush()** — сбросить буфер записи в ОС (данные могут ещё не попасть на диск; для гарантии на диск — **os.fsync(f.fileno())**).
- **f.close()** — закрыть файл; последующие операции вызовут **ValueError**. Рекомендуется **with open(...) as f:** для автоматического закрытия.
- **f.closed** — атрибут **True**, если файл закрыт.
- **f.fileno()** — файловый дескриптор (int); для **os.read**/ **os.write** и т.п.
- **f.isatty()** — является ли файл TTY (интерактивный терминал).
- **f.truncate(size=None)** — обрезать файл до **size** байт (текущая позиция, если **size** не задан); режим записи обязателен.

**Пример: чтение по строкам и запись.**

```python
with open("in.txt", encoding="utf-8") as f:
    for line in f:          # итератор по строкам
        print(line.rstrip())
# tell/seek (бинарный режим)
with open("data.bin", "rb") as f:
    f.seek(10)
    chunk = f.read(5)
    pos = f.tell()   # 15
```

**Граничные случаи:** после **f.close()** вызов **f.read()** или **f.write()** даёт **ValueError: I/O operation on closed file**. Для текстового файла **seek(0, 2)** (в конец) и **tell()** не обязательно дают «число символов» из-за кодировки. **read(0)** возвращает **''** или **b''** без чтения.

**Параметр newline в open():** в текстовом режиме **newline** управляет универсальными переводами строк: **newline=None** (по умолчанию) — входные **\\n**, **\\r**, **\\r\\n** нормализуются в **\\n**; при записи **\\n** заменяется на **os.linesep**. **newline=''** — переводы строк не преобразуются (как есть); **newline='\\n'** — при записи всегда **\\n**. Для CSV и фиксированного формата часто используют **newline=''**, чтобы не дублировать **\\r** в Windows.

**Пошагово: что происходит при for line in f.** Файл в текстовом режиме реализует протокол итератора: при каждой итерации вызывается **f.readline()**; когда **readline()** возвращает пустую строку **''**, итерация прекращается (конец файла). Буфер чтения заполняется блоками из ОС; **readline()** возвращает данные из буфера до следующего **\\n** (или до конца буфера). Поэтому **for line in f:** не загружает весь файл в память — строки читаются по одной (с буферизацией на уровне ОС).

**Плохо vs лучше (файлы):**

```python
# Плохо: открыть файл без with — при исключении файл может остаться открытым
f = open("data.txt")
data = f.read()
f.close()   # при ошибке выше close не вызовется

# Лучше: with гарантирует закрытие при любом выходе
with open("data.txt", encoding="utf-8") as f:
    data = f.read()

# Для больших файлов: не читать целиком, итерировать по строкам
with open("huge.txt", encoding="utf-8") as f:
    for line in f:
        process(line)
```

**Типичные ошибки:** не указывать **encoding** при текстовом режиме — на разных ОС разное умолчание (см. §1.2). Вызывать **read()** после **readlines()** без **seek(0)** — указатель в конце файла, **read()** вернёт **''**. Писать в файл, открытый только для чтения (**'r'**) — **io.UnsupportedOperation: not writable**.

**Пример: newline и CSV.**

```python
# Без newline='' в Windows при записи \n может превратиться в \r\n; при чтении CSV

import csv
with open("out.csv", "w", encoding="utf-8", newline="") as f:

    writer.writerow(["a", "b"])
with open("out.csv", "r", encoding="utf-8", newline="") as f:
    reader = csv.reader(f)
    next(reader)   # ['a', 'b']
```

with open("config.txt", encoding="utf-8") as f:

**Практический вывод:** для текстовых файлов всегда указывайте **encoding**; для больших файлов итерируйте **for line in f:** вместо **readlines()**; для гарантированной записи на диск после **flush()** используйте **os.fsync(f.fileno())**; для CSV и бинарных форматов с переводами строк используйте **newline=''**.

<details>
<summary>Вопрос: почему после <code>f.readlines()</code> вызов <code>f.read()</code> возвращает пустую строку?</summary>

У файлового объекта одна **текущая позиция** (курсор). **readlines()** читает строки до конца файла и сдвигает позицию в **конец**. После этого **read()** читает «от текущей позиции до конца» — то есть 0 байт/символов, поэтому возвращается **''** или **b''**. Чтобы снова прочитать файл с начала, нужно **f.seek(0)** (в текстовом режиме надёжнее открыть файл заново). См. [§4.4d](#44d-методы-встроенных-объектов-справочник).

</details>
