## Краткое повторение терминологии

| Термин                        | Краткое определение                                                                                                                                                                                         |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Iterable**                  | Объект, по которому можно итерировать; поддерживает **iter()** (или \***\*iter\*\***).                                                                                                                      |
| **Iterator**                  | Объект с \***\*iter\*\*** (возвращает self) и \***\*next\*\***; выдаёт элементы по одному, в конце выбрасывает **StopIteration**.                                                                           |
| **Generator**                 | Функция с **yield** или генераторное выражение **`(x for x in ...)`**; возвращает объект-генератор (подвид итератора).                                                                                      |
| **View**                      | Объект, отражающий текущее состояние словаря (**dict_keys**, **dict_values**, **dict_items**).                                                                                                              |
| **LEGB**                      | Порядок поиска имени: Local → Enclosing → Global → Built-in.                                                                                                                                                |
| **Closure**                   | Функция, использующая переменные из внешней области видимости.                                                                                                                                              |
| **Keyword-only argument**     | Аргумент, который можно передать только по имени (после **`*`** в сигнатуре).                                                                                                                               |
| **Positional-only argument**  | Аргумент, который можно передать только позиционно (до **`/`** в сигнатуре, Python 3.8+).                                                                                                                   |
| **Exception chaining**        | \***\*cause\*\*** (явная: **raise ... from ...**), \***\*context\*\*** (неявная при raise внутри except).                                                                                                   |
| **View (dict)**               | Объект **dict_keys**, **dict_values** или **dict_items**, отражающий текущее состояние словаря без копирования.                                                                                             |
| **Floor division**            | Целочисленное деление с округлением вниз (**//**); результат типа **int** при целых операндах.                                                                                                              |
| **Late binding**              | Переменная из замыкания берётся в момент **вызова** функции, а не в момент её определения.                                                                                                                  |
| **Shallow copy**              | Копия объекта, при которой вложенные объекты не копируются — копируются только ссылки на них.                                                                                                               |
| **Hashable**                  | Объект с неизменяемым хешем и определённым \***\*eq\*\***; может быть ключом dict и элементом set (например, int, str, tuple из hashable элементов).                                                        |
| **Mutable / Immutable**       | Изменяемый (list, dict, set) можно менять на месте; неизменяемый (tuple, str, frozenset) после создания не меняется.                                                                                        |
| **Banker's rounding**         | Округление 0.5 к ближайшему **чётному** (round(2.5)==2, round(3.5)==4); используется в **round()** в Python 3.                                                                                              |
| **MRO**                       | Method Resolution Order — порядок поиска метода при наследовании; в Python — C3 linearization.                                                                                                              |
| **Дескриптор**                | Объект с \***\*get\*\***, \***\*set\*\***, \***\*delete\*\*** в атрибуте класса; перехватывает доступ к атрибуту экземпляра. **property**, **classmethod**, **staticmethod** реализованы через дескрипторы. |
| **Метакласс**                 | Класс класса; тип объекта **type** (или подкласс **type**). Задаётся в Py3 как **class C(metaclass=Meta):**.                                                                                                |
| \***\*slots\*\***             | Объявление фиксированного набора атрибутов экземпляра; экономит память, запрещает добавление новых атрибутов.                                                                                               |
| **Наследование**              | Подкласс получает атрибуты и методы базового класса; **class B(A):** — одиночное; **class D(B, C):** — множественное; порядок поиска — MRO (C3).                                                            |
| **Инкапсуляция**              | Сокрытие внутренней реализации; в Python: **\_name** — соглашение «внутренний»; **\_\_name** — name mangling (**\_Класс\_\_name**), защита от переопределения в подклассе.                                  |
| **Полиморфизм**               | Один интерфейс, разное поведение; переопределение методов в подклассах; duck typing — код полагается на наличие методов, а не на тип.                                                                       |
| **Композиция**                | Включение объекта в другой как атрибут («A имеет B», has-a); альтернатива наследованию, когда отношение — «содержит», а не «является».                                                                      |
| **Name mangling**             | Переименование атрибутов **\_\_name** в **\_ИмяКласса\_\_name**; защищает от случайного переопределения в подклассе; магические методы \***\*init\*\*** и т.д. не манглятся.                                |
| **Атрибут класса**            | Имя в **C.**dict\***\*; общее для всех экземпляров; при **obj.x** виден, если у экземпляра нет своего **x\*\*.                                                                                              |
| **Атрибут экземпляра**        | Имя в **obj.**dict\***\*; присваивание **obj.x = v\*\* создаёт/обновляет его; «затеняет» атрибут класса для этого объекта.                                                                                  |
| **ABC (abstract base class)** | Класс с **@abstractmethod**; нельзя инстанцировать; подклассы обязаны реализовать абстрактные методы (**abc.ABC**, **abc.abstractmethod**).                                                                 |
| **Миксин**                    | Класс для множественного наследования, добавляющий поведение (методы); не используется сам по себе; имя часто с суффиксом **Mixin**.                                                                        |
| \***\*getattr\*\***           | Вызывается при **чтении** атрибута **только если** атрибут не найден обычным поиском; fallback для «виртуальных» атрибутов.                                                                                 |
| \***\*getattribute\*\***      | Вызывается при **каждом** чтении атрибута; внутри нужен **object.**getattribute**(self, name)** во избежание рекурсии.                                                                                      |
| \***\*init_subclass\*\***     | Метод базового класса (Python 3.6+); вызывается при создании **подкласса**; альтернатива метаклассу для регистрации и проверок.                                                                             |
| **dataclass**                 | Декоратор **@dataclass** (Python 3.7+); генерирует \***\*init\*\***, \***\*repr\*\***, при необходимости \***\*eq\*\*** для классов-данных с аннотированными полями.                                        |

## Вопросы для самопроверки

Ответы скрыты под «Показать ответ»; в каждом вопросе есть ссылка на раздел, где разобрана тема.

---

1. Чем в Python 3 сигнатура и поведение **print** отличаются от Python 2.7?  
   См. [§1.1 print](#11-print).

   <details>
   <summary>Показать ответ</summary>

   В Py3 **print** — встроенная функция с сигнатурой **print(\*values, sep=' ', end='\\n', file=sys.stdout, flush=False)**. В Py2 **print** был оператором: **print x** или **print x, y**. В Py3 нельзя писать **print x**; вывод без перевода строки в конце — **print(..., end='')**. Аргументы **sep**, **end**, **file**, **flush** позволяют менять разделитель, окончание строки, поток вывода и сброс буфера.
   </details>

2. Что такое **str** и **bytes** в Python 3? Как перейти от текста к байтам и обратно?  
   См. [§1.2 Строки (str / bytes / unicode)](#12-строки-str--bytes--unicode).

   <details>
   <summary>Показать ответ</summary>

   **str** — неизменяемая последовательность Unicode-символов (текст). **bytes** — неизменяемая последовательность байтов (0–255). Переход: **текст → байты**: **s.encode(encoding='utf-8', errors='strict')**; **байты → текст**: **b.decode(encoding='utf-8', errors='strict')**. Кодировка по умолчанию — **utf-8**. Тип **unicode** из Py2 удалён; в Py3 **str** по умолчанию Unicode.
   </details>

3. Что возвращает **`b'AB'[0]`** в Python 3 и почему?  
   См. [§1.2 Строки (str / bytes / unicode)](#12-строки-str--bytes--unicode).

   <details>
   <summary>Показать ответ</summary>

   **65** (целое число типа **int**). В Py3 при индексации **bytes** возвращается **int** — код байта (0–255). В Py2 **b'AB'[0]`** возвращало строку **'A'**. Это подчёркивает, что **bytes** — последовательность чисел, а не однобайтовых символов.
   </details>

4. Как в Python 3 работают операторы **`/`** и **`//`** для целых чисел?  
   См. [§1.3 Деление и числовые операции](#13-деление-и-числовые-операции).

   <details>
   <summary>Показать ответ</summary>

   **/** — вещественное деление: **7 / 2** → **3.5** (в Py2 при двух целых было **3**). **//** — целочисленное (floor division): **7 // 2** → **3**; результат округляется вниз. Для целых операндов **//** даёт **int**; при отрицательных — в сторону минус бесконечности: **-7 // 2** → **-4**.
   </details>

5. Зачем нужна конструкция **`raise New from orig`** и чем \***\*cause\*\*** отличается от \***\*context\*\***?  
   См. [§1.4 raise и исключения](#14-raise-и-исключения).

   <details>
   <summary>Показать ответ</summary>

   **raise New from orig** задаёт **явную причину** исключения: у **New** будет атрибут \***\*cause** = orig**; при выводе трассировки показывается цепочка «New вызвано из-за orig». \*\***context\***\* — неявный контекст: если **raise** выполняется внутри **except**, интерпретатор автоматически записывает перехваченное исключение в \*\***context\***\*. **from None\*\* подавляет вывод контекста.
   </details>

6. Что возвращают **dict.keys()**, **.values()**, **.items()** в Python 3 и чем это отличается от Python 2.7?  
   См. [§2.2 dict](#22-dict).

   <details>
   <summary>Показать ответ</summary>

   В Py3 они возвращают **view-объекты** (**dict_keys**, **dict_values**, **dict_items**) — динамические представления текущего состояния словаря: изменения в словаре сразу видны в view. В Py2 возвращались **списки** (копии). View поддерживают операции множества (для keys/items: **&**, **|**, **-**, **^**); итерация по view безопасна при изменении размера словаря только если не удалять текущий ключ. В Py2 методы **iterkeys**, **itervalues**, **iteritems** удалены — используйте **iter(d.keys())** и т.д.
   </details>

7. В чём разница между **range** в Python 3 и **range** / **xrange** в Python 2.7?  
   См. [§2.1 range](#21-range).

   <details>
   <summary>Показать ответ</summary>

   В Py3 **range** — единственный тип «диапазона»: ленивый, не создаёт список в памяти, поддерживает **start**, **stop**, **step**, атрибуты **.start**, **.stop**, **.step**, не поддерживает хеширование (нельзя ключом dict). В Py2 **range** создавал список; **xrange** был ленивым аналогом. **xrange** в Py3 удалён; **range** в Py3 ведёт себя как старый **xrange**.
   </details>

8. Как устроен протокол итератора (\***\*iter\*\***, \***\*next\*\***)? Чем iterable отличается от iterator?  
   См. [§3.1 Итераторы](#31-итераторы).

   <details>
   <summary>Показать ответ</summary>

   **Iterable** — объект, по которому можно итерировать: у него есть \***\*iter\*\***, возвращающий итератор (или в fallback — \***\*getitem\*\*** с индексами с 0). **Iterator** — объект с \***\*iter\*\*** (возвращает **self**) и \***\*next\*\***: \***\*next\*\*** возвращает следующий элемент либо выбрасывает **StopIteration**. **iter(obj)** вызывает **obj.**iter**()**; **next(it)** — **it.**next**()**. Итератор одноразовый: после исчерпания повторная итерация не даёт элементов.
   </details>

9. Что такое **yield from** и для чего используются **send**, **throw**, **close** у генератора?  
   См. [§3.2 Генераторы](#32-генераторы).

   <details>
   <summary>Показать ответ</summary>

   **yield from subgen** делегирует выдачу значений и пересылку **send**/ **throw**/ **close** подгенератору **subgen**; упрощает цепочки генераторов. **gen.send(value)** — передаёт **value** в генератор как результат текущего **yield**; следующий **yield** выдаёт значение. **gen.throw(exc)** — бросает исключение в точке **yield**. **gen.close()** — бросает **GeneratorExit** в генератор (не перехватывать его, иначе нарушается закрытие).
   </details>

10. Как работает поиск имён LEGB? Что делают **global** и **nonlocal**?  
    См. [§4.2 Области видимости (LEGB)](#42-области-видимости-legb).

   <details>
   <summary>Показать ответ</summary>

**LEGB**: поиск имени идёт в порядке **L**ocal → **E**nclosing (внешние функции) → **G**lobal (модуль) → **B**uilt-in. **global name** объявляет, что присваивание **name** в функции изменяет глобальную переменную модуля. **nonlocal name** — что присваивание изменяет переменную во внешней (но не глобальной) функции. Без **global**/ **nonlocal** присваивание создаёт локальную переменную.

   </details>

11. В чём опасность **def f(a=[])** и как правильно задать изменяемое значение по умолчанию?  
    См. [§4.3 Ловушки](#43-ловушки).

   <details>
   <summary>Показать ответ</summary>

Значения по умолчанию вычисляются **один раз** при определении функции. Один и тот же объект **list** используется при всех вызовах; если функция изменяет **a** (например, **a.append(x)**), изменения накапливаются между вызовами. Правильно: **def f(a=None):** и внутри **if a is None: a = []** (создаём новый список при каждом вызове без аргумента). Не использовать **a = a or []** — тогда пустой переданный список **[]** заменится на новый.

   </details>

12. Чем **partition(sep)** отличается от **split(sep, maxsplit=1)**? Когда что использовать?  
    См. [§1.2 Строки (str / bytes / unicode)](#12-строки-str--bytes--unicode).

   <details>
   <summary>Показать ответ</summary>

**partition(sep)** всегда возвращает кортеж из **трёх** частей: **(до_sep, sep, после_sep)**; если **sep** не найден — **(s, '', '')**. **split(sep, maxsplit=1)** возвращает **список** из максимум двух частей (без самого **sep**); если разделитель не найден — список из одного элемента **[s]**. **partition** удобен, когда нужен сам разделитель и фиксированное число частей; **split** — когда нужны подстроки без разделителя и переменное количество частей.

   </details>

13. Что такое пустой **range** и при каких **start**, **stop**, **step** он получается?  
    См. [§2.1 range](#21-range).

   <details>
   <summary>Показать ответ</summary>

Пустой **range** — такой, по которому **for** не выполняет ни одной итерации. Формула: элемент **i** входит, если **(i - start) % step == 0** и **start ≤ i < stop** при **step > 0**, или **stop < i ≤ start** при **step < 0**. Примеры пустых: **range(0)**, **range(5, 3)** (step=1), **range(0, 10, -1)** (шаг в другую сторону). **list(range(2, 2))** → **[]**.

   </details>

14. Почему элементы **set** и **frozenset** должны быть **hashable**? Какие встроенные типы hashable?  
    См. [§2.4 set и frozenset](#24-set-и-frozenset).

   <details>
   <summary>Показать ответ</summary>

Множество реализовано как хеш-таблица: для проверки **in**, добавления и удаления нужен стабильный **hash** и \***\*eq\*\***. Изменяемый объект не должен менять хеш, иначе он «потеряется» в множестве. Hashable встроенные типы: **int**, **float**, **str**, **bytes**, **tuple** (если все элементы hashable), **frozenset**. Не hashable: **list**, **dict**, **set**, большинство пользовательских классов по умолчанию (если не переопределён \***\*hash\*\*** и объект считается неизменяемым).

   </details>

15. Пошагово: что происходит при вызове **gen.send(value)** в генераторе с **x = yield**?  
    См. [§3.2 Генераторы](#32-генераторы).

   <details>
   <summary>Показать ответ</summary>

Генератор приостановлен на **yield**. **gen.send(value)** передаёт **value** как результат выражения **yield**; переменная **x** получает значение **value**. Затем выполнение продолжается до следующего **yield** (который выдаёт значение наружу) или до конца функции. При первом вызове нужно использовать **next(gen)** или **gen.send(None)** — до первого **yield** нельзя отправить значение.

   </details>

16. Чем **divmod(a, b)** отличается от пары **a // b** и **a % b**? Как ведёт себя **a % b** при отрицательных **a** или **b** в Python?  
    См. [§1.3 Деление и числовые операции](#13-деление-и-числовые-операции).

   <details>
   <summary>Показать ответ</summary>

**divmod(a, b)** возвращает пару **(a // b, a % b)** за один вызов; для целых это одна операция. В Python остаток **a % b** всегда имеет знак делителя **b** и связан с целочисленным делением: **a == (a // b) \* b + (a % b)**. Пример: **-7 // 2** → **-4**, **-7 % 2** → **1**; **7 // -2** → **-4**, **7 % -2** → **-1**. Это отличается от C/Java, где остаток может иметь знак делимого.

   </details>

17. Почему **range** в Python 3 нельзя использовать как ключ словаря? Чем **s.union(other)** отличается от **s.update(other)** для множества?  
    См. [§2.1 range](#21-range), [§2.4 set и frozenset](#24-set-и-frozenset).

   <details>
   <summary>Показать ответ</summary>

**range** не реализует \***\*hash\*\*** (он не определён), потому что **range** считается потенциально большим и нецелесообразно делать его ключом. Только hashable объекты могут быть ключами **dict** и элементами **set**. **s.union(other)** возвращает **новое** множество (объединение), не изменяя **s**. **s.update(other)** изменяет **s** на месте, добавляя элементы из **other**; возвращает **None**.

   </details>

18. Что такое **slice assignment** (**lst[i:j] = iterable**) и зачем он нужен? Почему **lst = [[]] \* 3** опасно для списка списков?  
    См. [§2.3 list](#23-list).

   <details>
   <summary>Показать ответ</summary>

**Slice assignment** — замена среза списка на элементы итерабельного объекта; длина списка может измениться. Пример: **lst[1:2] = [10, 20]** вставляет два элемента вместо одного. **lst = [[]] \* 3** создаёт список из **трёх ссылок на один и тот же** пустой список; изменение **lst[0].append(1)** затронет все три элемента. Правильно: **[[] for \_ in range(3)]** — три независимых списка.

   </details>

19. Когда вычисляются значения по умолчанию параметров функции? Зачем использовать объект-маркер (sentinel) вместо **None**?  
    См. [§4.3 Ловушки](#43-ловушки).

   <details>
   <summary>Показать ответ</summary>

Значения по умолчанию вычисляются **в момент определения функции** (один раз), а не при каждом вызове. Поэтому изменяемые объекты (**[]**, **{}**) становятся общими между вызовами. Sentinel (**\_sentinel = object()**) используют, когда **None** — допустимое значение аргумента и нужно различать «аргумент не передан» и «передан явно None». Проверка **if a is \_sentinel: a = default**.

   </details>

20. Что делает **breakpoint()** и чем **getattr(obj, name, default)** полезнее **hasattr** в критичном коде?  
    См. [§4.4 Встроенные функции (builtins)](#44-встроенные-функции-builtins--обзор).

   <details>
   <summary>Показать ответ</summary>

**breakpoint()** (Py3.7+) вызывает отладчик (по умолчанию **pdb**); учитывает **PYTHONBREAKPOINT**. **hasattr(obj, name)** реализован как **getattr** с перехватом **любого** исключения; если при обращении к атрибуту (например, property) выбрасывается не **AttributeError**, **hasattr** вернёт **False** и скроет ошибку. В критичном коде безопаснее **try: getattr(obj, name); except AttributeError: ...** — так другие исключения не проглатываются.

   </details>

21. Чем **str.split()** без аргумента отличается от **str.split(" ")**: когда что использовать?  
    См. [§1.2 Строки (str / bytes / unicode)](#12-строки-str--bytes--unicode).

   <details>
   <summary>Показать ответ</summary>

**split()** без аргумента (или **split(None)**) разбивает по **любому** последовательному пробелу (пробел, табуляция, newline и т.д.), убирая пустые строки по краям. **split(" ")** разбивает строго по одному символу пробела; при нескольких подряд появятся пустые строки. Для «слов» из строки с произвольными пробелами используйте **split()**; для фиксированного разделителя — **split(" ")** или **split(sep)**.

   </details>

22. Чем **list.index(x)** отличается от **str.find(sub)** при отсутствии элемента? Что возвращает **reversed(lst)** — список или итератор?  
    См. [§2.3 list](#23-list), [§4.4 builtins](#44-встроенные-функции-builtins--обзор).

   <details>
   <summary>Показать ответ</summary>

**list.index(x)** при отсутствии **x** выбрасывает **ValueError**. **str.find(sub)** при отсутствии возвращает **-1**. Для поиска в списке с проверкой: **if x in lst: i = lst.index(x)** или обрабатывать **ValueError**. **reversed(lst)** возвращает **итератор**, а не список; для списка используйте **list(reversed(lst))** или срез **lst[::-1]**.

   </details>

23. В каком порядке должны идти блоки **except (A, B)** и **except Exception**, чтобы перехватывать оба случая корректно?  
    См. [§1.4 raise и исключения](#14-raise-и-исключения).

   <details>
   <summary>Показать ответ</summary>

Сначала более **конкретные** исключения, затем более **общие**. Блок **except Exception** перехватывает все исключения (кроме BaseException и подклассов, не наследующих Exception). Поэтому порядок: **except (ValueError, TypeError):** ... затем **except OSError:** ... затем **except Exception:** ... Если **except Exception** поставить первым, до блоков с **ValueError** и **OSError** выполнение не дойдёт — они подклассы **Exception**.

   </details>

24. Что такое **dict comprehension** и когда его использовать вместо **dict(iterable_of_pairs)**?  
    См. [§2.2 dict](#22-dict).

   <details>
   <summary>Показать ответ</summary>

**Dict comprehension** — **{key_expr: value_expr for ... in ... if ...}**. Создаёт словарь в одном выражении. Удобно, когда ключ и значение выражаются через элемент итерации (например **{x: x**2 for x in range(5)}**). **dict(iterable_of_pairs)** удобен, когда уже есть пары **(key, value)** или ключи/значения заданы отдельно (**dict(zip(keys, values))\*\*). Comprehension читаемее при простых формулах ключа и значения.

   </details>

24a. В каком порядке выполняются циклы во вложенном list comprehension **\[x+y for x in [1,2] for y in [10,20]\]**? Какой получится результат?  
 См. [§2.3 list](#23-list).

   <details>

Внешний цикл **x** (медленный), внутренний **y** (быстрый): пары **(1,10) → 11**, **(1,20) → 21**, **(2,10) → 12**, **(2,20) → 22**. Результат: **[11, 21, 12, 22]**. Внутренний цикл (**y**) проходит полностью для каждого значения **x**.

   </details>

#### Вопросы по §5 (Классы и ООП)

46. Чем в Python 3 вызов **super()** без аргументов отличается от **super(CurrentClass, self)** в Python 2? Зачем это нужно при множественном наследовании?  
    См. [§5.1 Базовые механизмы](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>

В Py3 **super()** без аргументов внутри метода экземпляра эквивалентен **super(**class**, self)** — интерпретатор подставляет текущий класс и **self** из контекста (в т.ч. из замыкания \***\*class\*\***). В Py2 нужно было явно писать **super(CurrentClass, self)**. Без аргументов при переименовании класса не нужно менять вызовы **super**; при множественном наследовании MRO соблюдается автоматически, и цепочка вызовов родительских методов работает корректно.

   </details>

47. Что такое MRO и как его посмотреть? Почему порядок баз в **class D(B, C):** важен?  
    См. [§5.1 Базовые механизмы](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>

**MRO (Method Resolution Order)** — порядок, в котором интерпретатор ищет метод при наследовании. В Python используется C3 linearization. Посмотреть: **C.mro()** или **C.**mro\***\* — список классов от самого класса до **object**. Порядок баз в **class D(B, C):** определяет MRO: при поиске метода сначала проверяется **D**, затем **B**, затем **C\*\*, затем их предки; от порядка зависит, какой метод будет найден первым при множественном наследовании.

   </details>

    См. [§5.2 Декораторы методов](#52-декораторы-методов).

   <details>
   <summary>Показать ответ</summary>

Обычный метод получает **self** (экземпляр); **@staticmethod** не получает ни **self**, ни **cls**; **@classmethod** получает **cls** (класс) первым аргументом. **@classmethod** для создания экземпляра (фабричный метод) удобен, когда конструктор нужно выразить через другие аргументы (например, **from_polar(r, phi)**); внутри вызывают **cls(...)** — тогда при наследовании создаётся экземпляр подкласса, а не базового класса.

   </details>

49. Чем \***\*init\*\*** отличается от \***\*new\*\***? В каких случаях переопределяют \***\*new\*\***?  
    См. [§5.3 Магические методы](#53-магические-методы-dunder).

   <details>
   <summary>Показать ответ</summary>

\***\*new**(cls, ...)** вызывается первым и **создаёт** объект (возвращает экземпляр); \*\***init**(self, ...)** вызывается после и **инициализирует** уже созданный экземпляр (не возвращает значение). \***\*new\*\*** переопределяют для неизменяемых типов (подкласс **str**, **int**), синглтонов, пулов объектов или когда нужно вернуть объект другого типа. В большинстве классов достаточно \***\*init\*\***.

   </details>

50. Что такое дескриптор и как связаны **property**, **classmethod**, **staticmethod** с дескрипторами?  
    См. [§5.4 Дескрипторы](#54-дескрипторы).

   <details>
   <summary>Показать ответ</summary>

**Дескриптор** — объект класса с методами \***\*get\*\***, \***\*set\*\***, \***\*delete\*\*** (хотя бы \***\*get\*\***). При обращении к атрибуту экземпляра **obj.x** интерпретатор, если **x** в классе — дескриптор, вызывает **x.**get**(obj, type(obj))**. **property**, **classmethod**, **staticmethod** реализованы как дескрипторы: при **obj.x** или **C.x** вызывается их \***\*get\*\***, что и даёт доступ к свойству или подстановку **cls**/отсутствие аргумента.

   </details>

51. Зачем используют \***\*slots\*\***? Какие ограничения он накладывает на экземпляр и на наследование?  
    См. [§5.5 **slots**](#55-__slots__).

   <details>
   <summary>Показать ответ</summary>

\***\*slots\*\*** задаёт фиксированный набор атрибутов экземпляра; у экземпляра не создаётся \***\*dict\*\*** (экономия памяти) и нельзя добавить атрибут, не указанный в \***\*slots\*\***. Используют при большом числе мелких объектов с фиксированными полями или для жёсткого контракта. При наследовании подкласс наследует ограничение; если у подкласса свой \***\*slots\*\***, в нём перечисляют только дополнительные имена.

   </details>

52. Как в Python 3 задать метакласс класса? Когда метакласс реально нужен, а когда достаточно \***\*init_subclass\*\***?  
    См. [§5.6 Метаклассы](#56-метаклассы).

   <details>
   <summary>Показать ответ</summary>

В Py3 метакласс задаётся в объявлении класса: **class C(metaclass=Meta):**. Класс создаётся вызовом **Meta('C', bases, dict)** вместо **type(...)**. Метакласс нужен, когда нужно изменить или проверить создание класса (атрибуты, базы, имя), зарегистрировать класс, сгенерировать методы. \***\*init_subclass**(cls)\*\* вызывается при создании подкласса и позволяет реагировать на появление подкласса без метакласса — часто этого достаточно для регистрации и проверок.

   </details>

53. Чем в Python отличаются **\_name** и **\_\_name** (один и два подчёркивания)? Что такое name mangling?  
    См. [§5.1 Инкапсуляция](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>
   **_name** — соглашение: атрибут считается «внутренним», не частью публичного API; доступ **obj._name** возможен. **__name** — **name mangling**: интерпретатор переименовывает атрибут в **\_ИмяКласса__name** (например, **\_C__name**), что защищает от случайного переопределения в подклассе; снаружи **obj.__name** не найдёт атрибут (он хранится под переименованным именем). Магические методы **__init__**, **__str__** и т.д. **не** манглятся (подчёркивания с обеих сторон).
   </details>

54. Что такое полиморфизм в контексте классов Python? Как он связан с переопределением методов и duck typing?  
    См. [§5.1 Полиморфизм](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>
   **Полиморфизм** — один интерфейс (вызов метода), разное поведение в зависимости от типа объекта. В Python: (1) **переопределение методов** — подкласс определяет метод с тем же именем, что и базовый; при **obj.method()** вызывается реализация класса **obj**. (2) **Duck typing** — код полагается на наличие методов у объекта, а не на тип; если у объекта есть нужный метод, код работает. Контракты можно задавать через **abc** или **typing.Protocol**.
   </details>

55. Когда предпочтительнее **композиция** (объект содержит другой), а когда **наследование** (подкласс — разновидность базового)?  
    См. [§5.1 Композиция и наследование](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>
   **Наследование** — когда отношение «**B** является разновидностью **A**» (is-a): подкласс расширяет или уточняет поведение базового (например, **Dog(Animal)**). **Композиция** — когда «**A** имеет **B**» (has-a): объект содержит другой как атрибут (например, **Car** с **self.engine = Engine()**). При сомнении предпочтительнее композиция: проще менять «части» и тестировать; наследование — при явном is-a и полиморфизме через переопределение методов.
   </details>

56. Зачем вызывать **super().**init**()** в \***\*init\*\*** подкласса? Что даёт вызов **super().method()** при переопределении метода?  
    См. [§5.1 Базовые механизмы](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>
   **super().__init__(...)** в подклассе нужен, чтобы инициализировать состояние базового класса; без него атрибуты родителя не создаются, и возможны ошибки при обращении к ним. **super().method()** вызывает реализацию метода из **следующего** класса в MRO (родителя или «соседа» при множественном наследовании), что позволяет расширять поведение, а не полностью заменять его; при множественном наследовании цепочка **super()** обеспечивает ровно один вызов каждого класса по MRO.
   </details>

57. Чем **атрибут класса** отличается от **атрибута экземпляра**? Где они хранятся? Почему **obj.items.append(1)** при **C.items = []** в классе меняет список у всех экземпляров?  
    См. [§5.1 Атрибуты класса и экземпляра](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>
   **Атрибут класса** хранится в **C.__dict__**; общий для всех экземпляров. **Атрибут экземпляра** хранится в **obj.__dict__**; создаётся присваиванием **obj.x = value**. При чтении **obj.x** сначала проверяется **obj.__dict__**; если там нет **x**, берётся из класса по MRO. При **C.items = []** в классе один и тот же список у всех экземпляров; **obj.items.append(1)** изменяет этот общий список. Нужно создавать атрибут экземпляра в **__init__**: **self.items = []**.
   </details>

58. Чем \***\*getattr\*\*** отличается от \***\*getattribute\*\***? Почему внутри \***\*getattribute\*\*** и \***\*setattr\*\*** нужно вызывать **object.**getattribute\***\* / **object.**setattr\*\***?  
    См. [§5.3 **getattr**, **getattribute**](#53-магические-методы-dunder).

   <details>
   <summary>Показать ответ</summary>
   **__getattr__** вызывается **только когда** атрибут **не найден** обычным поиском — fallback для «виртуальных» атрибутов. **__getattribute__** вызывается **при каждом** чтении атрибута. Внутри **__getattribute__** обращение к **self.attr** снова вызовет **__getattribute__** — бесконечная рекурсия; поэтому нужно вызывать **object.__getattribute__(self, name)**. Аналогично в **__setattr__**: присваивание **self.x = value** вызовет **__setattr__** снова — нужно **object.__setattr__(self, name, value)** или **self.__dict__[name] = value**.
   </details>

59. Что такое **ABC** (abstract base class) и зачем **@abstractmethod**? Что такое **миксин** и когда его используют?  
    См. [§5.1 ABC и миксины](#51-базовые-механизмы).

   <details>
   <summary>Показать ответ</summary>
   **ABC** — класс с абстрактными методами (**@abstractmethod**); его нельзя инстанцировать; подклассы обязаны реализовать эти методы (модуль **abc**, **abc.ABC**). Задаёт контракт без реализации. **Миксин** — класс для множественного наследования, добавляющий поведение (методы); не используется сам по себе; имя часто с суффиксом **Mixin**; ставят не первым в списке баз.
   </details>

60. Когда вызывается \***\*init_subclass\*\*** и зачем он нужен? Что даёт **@dataclass** (Python 3.7+)?  
    См. [§5.6 **init_subclass**, dataclasses](#56-метаклассы).

   <details>
   <summary>Показать ответ</summary>
   **__init_subclass__(cls, **kwargs)** вызывается **при создании подкласса** этого класса (один раз при определении класса). Позволяет реагировать на появление подкласса без метакласса: регистрация, проверка атрибутов, передача параметров через **kwargs**. **@dataclass** генерирует **__init__**, **__repr__**, при необходимости **__eq__** для классов с аннотированными полями — удобно для классов-данных (DTO, конфиги).
   </details>

13. **divmod и остаток:** Для целых **a** и **b** (b != 0) напишите проверку: **a == (a // b) \* b + (a % b)**. Проверьте на примерах **(-7, 2)** и **(7, -2)**.  
    См. [§1.3 Деление и числовые операции](#13-деление-и-числовые-операции).

   <details>
   <summary>Показать решение</summary>

**a == (a // b) \* b + (a % b)** — всегда **True** в Python для целых. **(-7, 2)**: **-7 // 2 == -4**, **-7 % 2 == 1**; **-4\*2+1 == -7**. **(7, -2)**: **7 // -2 == -4**, **7 % -2 == -1**; **-4\*(-2)+(-1) == 7**.

   </details>

14. **set и get/setdefault:** Дан список пар **(key, value)**. Постройте словарь, в котором каждому ключу соответствует **список** всех значений для этого ключа (используйте **setdefault** или **get** с инициализацией).  
    См. [§2.2 dict](#22-dict).

   <details>
   <summary>Показать решение</summary>

**d = {}**; **for k, v in pairs: d.setdefault(k, []).append(v)**. Или **d.get(k, [])** но тогда нужно присваивать: **lst = d.get(k, []); lst.append(v); d[k] = lst** — но для нового ключа получится тот же список, поэтому правильнее **d.setdefault(k, []).append(v)** или **if k not in d: d[k] = []; d[k].append(v)**.

   </details>

14a. **set.remove vs discard:** Дано множество **s = {1, 2, 3}**. Напишите код, который «безопасно» удаляет элемент **4** (если он есть) — без исключения. Затем напишите код, который удаляет элемент **2** и при отсутствии выбрасывает исключение. Объясните разницу между **remove** и **discard**.  
 См. [§2.4 set и frozenset](#24-set-и-frozenset).

   <details>
   <summary>Показать решение</summary>

**s.discard(4)** — при отсутствии **4** ничего не делает, исключения нет. **s.remove(2)** — при наличии удаляет; при отсутствии — **KeyError**. Для «безопасного» удаления используйте **discard**; для «строгого» (элемент обязан быть) — **remove** и обрабатывайте **KeyError**.

   </details>

15. **itertools:** С помощью **itertools.tee** создайте два итератора из **range(5)** и выведите **list** первого и **list** второго — убедитесь, что оба дают **[0,1,2,3,4]**.

16. **str.split:** Для строки **" a b c "** вызовите **split()** без аргумента и **split(" ")** — сравните результаты и объясните разницу.

17. **list и dict:** Постройте словарь **{x: x**2 for x in range(5)}** (dict comprehension). Для списка **[1, 2, 3, 2]** найдите индекс первого вхождения **2** через **.index()** и удалите все элементы через **.clear()\*\*.

17a. **append vs extend:** Для списка **lst = [1, 2]** выполните **(a)** **lst.append([3, 4])** и **(b)** **lst.extend("ab")**. Что получится в каждом случае? Почему **extend** от строки добавляет символы, а не строку целиком?  
 См. [§2.3 list](#23-list).

   <details>
   <summary>Показать решение</summary>

**(a)** **lst.append([3, 4])** → **lst** станет **[1, 2, [3, 4]]** — один элемент (список) добавлен. **(b)** **lst.extend("ab")** → **lst** станет **[1, 2, 'a', 'b']** — строка итерируема, **extend** добавляет каждый элемент (символ) по отдельности. **append** добавляет **один** объект; **extend** перебирает итерабельное и добавляет каждый элемент.

   </details>

18. **Исключения:** Напишите **try/except**, перехватывающий **ValueError** и **TypeError** одним блоком **except (ValueError, TypeError) as e**, и отдельно **OSError**. Убедитесь, что блок с **Exception** не идёт перед блоком с **ValueError**.

Сначала конкретные, затем общие: **except (ValueError, TypeError) as e:** ... **except OSError:** ... **except Exception:** ... Если **except Exception** поставить первым, он перехватит всё и блоки с **ValueError** и **OSError** не выполнятся.

25. **super() и MRO:** Напишите класс **B** с методом **greet()**, возвращающим **"B"**, и класс **C(B)** с методом **greet()**, который вызывает **super().greet()** и добавляет **" -> C"** к результату. Создайте экземпляр **C()** и вызовите **greet()** — убедитесь, что результат **"B -> C"**.

   <details>
   <summary>Показать решение</summary>


26. **@classmethod и @property:** Напишите класс **Point** с конструктором \***\*init**(self, x, y)** и методом класса **from_polar(cls, r, phi)**, создающим точку по полярным координатам (**x = r\*cos(phi)**, **y = r\*sin(phi)**). Добавьте свойство **radius** (только чтение), возвращающее **sqrt(x² + y²)\*\*.  
    См. [§5.2 Декораторы методов](#52-декораторы-методов).

   <details>
   <summary>Показать решение</summary>

```python
import math
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y
    @classmethod
    def from_polar(cls, r, phi):
        return cls(r * math.cos(phi), r * math.sin(phi))
    @property
    def radius(self):
        return math.hypot(self.x, self.y)
p = Point.from_polar(1, 0)
print(p.x, p.y, p.radius)   # 1.0 0.0 1.0
```

   </details>

## Резюме по Части I (отличия Python 3 от Python 2.7)

Часть I охватывает **синтаксис**, **типы**, **итераторы и генераторы**, **функции** (включая справочники builtins §4.4a–d) и **классы и ООП** (§5) — то, что чаще всего ломается или меняется при переходе с Python 2.7 на Python 3.

**Синтаксис (§1):** **print** — функция с аргументами **sep**, **end**, **file**, **flush**. **str** — всегда Unicode; **bytes** — байты; тип **unicode** удалён. **/** — вещественное деление; **//** — целочисленное; тип **long** удалён, **int** неограниченной точности. **raise** только в форме **raise Exception(...)**; цепочки исключений — **raise New from e**; **except E as e** — имя **e** удаляется после блока.

**Типы (§2):** **range** — ленивый тип (аналог **xrange**); **xrange** удалён. **dict.keys()**, **.values()**, **.items()** возвращают **view**, не списки; **iterkeys** и т.д. удалены. Порядок ключей в **dict** гарантирован с Python 3.7. **list**, **set**, **tuple** — семантика та же; распаковка **\*rest** и слияние словарей **`{**d1, **d2}`**, **`d1 | d2`** (3.9+) доступны в Py3.

**Итераторы и генераторы (§3):** метод итератора — \***\*next\*\*** (в Py2 был **next**). **next(iterator, default)** — встроенная функция. В генераторах **raise StopIteration** превращается в **RuntimeError** (PEP 479, 3.7+). **yield from**, **send**, **throw**, **close** — как в Py2; асинхронные генераторы (**async def** с **yield**) — с Python 3.6.

**Функции (§4):** **keyword-only** аргументы (после **\***), **positional-only** (до **/**, 3.8+), аннотации типов. **global** и **nonlocal**; замыкания и **late binding**. Mutable default argument — по-прежнему ловушка; использовать **None** и создавать объект внутри функции. **§4.4a–d** — справочники: полный список встроенных функций, константы, типы и исключения, методы встроенных объектов (str, bytes, list, dict, set, int, float, file, iterator, generator, Exception, object, type, function, property, module).

**Классы и ООП (§5):** все классы наследуют от **object**; **super()** без аргументов в Py3; MRO (C3). Декораторы **@staticmethod**, **@classmethod**, **@property**. Магические методы (\***\*init\*\***, \***\*new\*\***, \***\*str\*\***, \***\*repr\*\***, \***\*eq\*\***, \***\*hash\*\***, \***\*len\*\***, \***\*getitem\*\***, \***\*iter\*\***, \***\*enter\*\***/ \***\*exit\*\***, \***\*call\*\***). Дескрипторы (\***\*get\*\***, \***\*set\*\***, \***\*delete\*\***); \***\*slots\*\***; метаклассы (**class C(metaclass=Meta):**).

При миграции кода с Py2 на Py3 ориентируйтесь на таблицы сравнения и разделы «Миграция» в каждом подразделе; используйте **2to3**, **six** или **future** для автоматизации части замен (подробнее — в плане §35–37).
