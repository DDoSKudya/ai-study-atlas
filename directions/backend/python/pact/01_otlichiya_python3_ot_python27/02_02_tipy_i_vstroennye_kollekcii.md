## 2. Типы и встроенные коллекции

В этом разделе разобраны **range**, **dict**, **list**, **set** и **tuple** — как они устроены в Python 3, чем отличаются от Python 2.7, и когда что использовать. Важно понимать: **range** в Py3 ленивый (не список); **dict.keys()/values()/items()** возвращают view, не списки; порядок ключей в **dict** с Py3.7 совпадает с порядком вставки; **list** и **tuple** различаются изменяемостью и hashable; **set** требует hashable элементы и не гарантирует порядок. Для обучения: сначала разберите сигнатуры и примеры, затем «Плохо vs лучше» и типичные ошибки, в конце — миграцию с Py2.

### 2.1 range

В **Python 2.7** были два варианта: **`range()`** возвращал **список** (все элементы в памяти), **`xrange()`** — ленивый объект (элементы вычислялись по мере итерации). В **Python 3** тип **`xrange`** удалён; **`range`** — отдельный встроенный тип с ленивым вычислением, по смыслу аналог прежнего **`xrange`**.

#### Сигнатура

```python
range(stop)                    # 0, 1, ..., stop-1
range(start, stop[, step])      # start, start+step, ... до stop (не включая)
```

- Элементы **не создаются сразу** — вычисляются по мере итерации. Это экономит память при больших диапазонах.
- Поддерживает **индексацию** `r[i]`, **len(r)**, проверку **x in r** (за O(1) для типичного range), но **не** срезы вида `r[1:5]`. Для «среза» по итератору/последовательности используют **`itertools.islice`**.

**Отрицательный шаг (step < 0):** при **step < 0** диапазон идёт от **start** вниз к **stop** (не включая **stop**); **start** должен быть больше **stop**, иначе range пустой. Пример: **range(5, 0, -1)** → **5, 4, 3, 2, 1**; **range(0, 5, -1)** → пустой.

**Пустой range:** если при заданных **start**, **stop**, **step** не существует ни одного элемента (например, **start >= stop** при **step > 0** или **start <= stop** при **step < 0**), **range** пустой — **len(r) == 0**, **list(r) == []**. Пример: **range(10, 5)** → пустой; **range(0)** → пустой.

**Формула элемента:** элемент с индексом **i** равен **start + i \* step**; индексация **r[i]** и проверка **x in r** используют эту формулу, поэтому не требуют перебора всех элементов.

**Как это работает внутри (упрощённо):** объект **range** хранит только три числа — **start**, **stop** и **step** (при вызове **range(stop)** фактически **start=0**, **step=1**). Элементы не хранятся в памяти списком: при итерации или при обращении **r[i]** значение вычисляется по формуле **start + i \* step**. Поэтому **range(10**9)** занимает мало памяти (несколько десятков байт), в отличие от **list(range(10**9))**, который создал бы миллиард целых в памяти. Длина **len(r)** вычисляется по формуле: при **step > 0** это **max(0, (stop - start + step - 1) // step)** (упрощённо: сколько целых шагов укладывается от start до stop); при **step < 0** — аналогично с учётом направления.

**Пошагово: как вычисляются r[i], len(r) и x in r на примере range(2, 10, 2).** Объект **r = range(2, 10, 2)** хранит **start=2**, **stop=10**, **step=2**. (1) **r[i]** — элемент с индексом **i** равен **start + i \* step**: **r[0]** → **2**, **r[1]** → **4**, **r[2]** → **6**, **r[3]** → **8**; для **i >= 4** индекс выходит за границы — **IndexError**. Отрицательный индекс трактуется как **i + len(r)**. (2) **len(r)** — число элементов: при **step > 0** это **max(0, (stop - start + step - 1) // step)**; подставляем: **(10 - 2 + 2 - 1) // 2** = **9 // 2** = **4**. Итерация выдаёт ровно 4 значения. (3) **x in r** — проверка принадлежности: число **x** входит в range, если **x** совпадает с одним из значений **start + k \* step** для целого **k** в диапазоне **[0, len(r))**, при этом **(x - start) % step == 0** и **start <= x < stop** (при **step > 0**) или **stop < x <= start** (при **step < 0**). Например, **3 in r** — **(3 - 2) % 2 == 1** ≠ 0, значит **False**; **6 in r** — **(6 - 2) % 2 == 0** и **2 <= 6 < 10** — **True**. Итог: ни **r[i]**, ни **len(r)**, ни **x in r** не перебирают все элементы в памяти — всё считается по формулам.

#### range не hashable: нельзя как ключ dict или элемент set

В отличие от кортежа, тип **range** в Python 3 **не** является hashable: объект **range** нельзя использовать как ключ словаря или элемент множества. Причина — возможность изменения «смысла» range в будущих версиях или детали реализации. Если нужен ключ по диапазону, используйте кортеж **(start, stop, step)** или **(start, stop)**.

```python
r = range(5)
hash(r)   # TypeError: unhashable type: 'range'
# d = {r: 1}   # TypeError
```

#### Представление (repr) и равенство

**repr(range(5))** даёт строку **'range(0, 5)'** (или **'range(5)'** для **range(stop)**); два объекта **range** равны (**==**), если совпадают **start**, **stop** и **step**. Сравнение **range(0, 5)** и **range(0, 5)** даёт **True**; **range(0, 5)** и **range(5)** тоже равны (оба — 0, 1, 2, 3, 4). Атрибуты **r.start**, **r.stop**, **r.step** доступны для чтения: **range(2, 10, 2).start** → **2**, **.stop** → **10**, **.step** → **2**.

#### Примеры

```python
r = range(5)
list(r)           # [0, 1, 2, 3, 4]

r = range(2, 10, 2)
list(r)           # [2, 4, 6, 8]
r[1]              # 4
len(r)            # 4
3 in r            # False
6 in r            # True

# Эквивалент «среза» — создать новый range по формулам (вручную) или islice для произвольного итератора
from itertools import islice
list(islice(range(100), 10, 20))   # [10, 11, ..., 19]
```

#### Теоретическое обоснование

Объединение **range** и **xrange** в один тип упрощает язык: один очевидный способ получить диапазон целых. Ленивость по умолчанию соответствует принципу «не создавать данные, пока они не нужны».

#### Миграция с Python 2 на Python 3

- В Py2 **`range(n)`** возвращал **список** **[0, 1, ..., n-1]**; в Py3 **`range(n)`** возвращает объект **range** (ленивый). Если код в Py2 полагался на то, что **range** — список (например, **`range(5)[2:4]`** или **`range(5).append(10)`**), в Py3 это сломается: у **range** нет метода **append**, а срезы **range** не поддерживаются. Решение: для списка использовать **`list(range(n))`**; для среза — **`list(range(n))[start:stop]`** или **`itertools.islice(range(n), start, stop)`**.
- В Py2 **`xrange(n)`** был ленивым аналогом **range**; в Py3 **xrange** удалён — везде используйте **`range(n)`**.
- В Py2 **`range`** возвращал список, поэтому **`for i in range(1000000)`** создавал список из миллиона элементов в памяти. В Py3 **`for i in range(1000000)`** создаёт только объект **range** и итерирует по нему — память не растёт. Миграция с **xrange** на **range** в Py3 не требует изменений кода.

#### Типичные ошибки при работе с range в Python 3

- **Попытка среза:** **`range(10)[2:5]`** в Py3 даёт **TypeError** (тип **range** не поддерживает срезы). Используйте **`list(range(10))[2:5]`** или **`range(2, 5)`** (если шаг 1).
- **Попытка изменить:** **range** неизменяем; методов **append**, **extend**, **sort** нет. Для изменяемой последовательности целых используйте **list**.
- **Ожидать список от range:** в Py2 **range(5)** возвращал **[0,1,2,3,4]**; в Py3 **range(5)** — объект range. Код вида **range(5).append(10)** или **range(5)[0]** в Py3: первый сломается (нет метода append), второй сработает (индексация у range есть). Но **range(5)[2:4]** в Py3 вызовет **TypeError**. Если нужен именно список — пишите **list(range(5))**.
- **Пустой range при неправильном порядке start/stop:** при **step > 0** нужен **start < stop**; при **step < 0** — **start > stop**. Иначе **list(range(...))** будет **[]** — проверяйте границы при отрицательном шаге.

### 2.2 dict

#### View-объекты

В Python 2.7 методы **`.keys()`**, **`.values()`**, **`.items()`** возвращали **списки** (копии). В Python 3 они возвращают **view-объекты** (**dict_keys**, **dict_values**, **dict_items**), которые отражают текущее состояние словаря и не копируют данные.

```python
d = {'a': 1, 'b': 2}
k = d.keys()
d['c'] = 3
list(k)   # ['a', 'b', 'c']  — view обновился
```

- Итерация по view идёт по текущему словарю; изменение размера словаря во время итерации по view может привести к ошибке.
- Методы **`.iterkeys()`**, **`.itervalues()`**, **`.iteritems()`** в Python 3 удалены: сами view итерируемы.

#### Порядок вставки (Python 3.7+)

Начиная с **Python 3.7** порядок перебора ключей в **dict** совпадает с **порядком вставки**: первый добавленный ключ — первый при итерации, последний добавленный — последний. В Python 3.7 это гарантировано спецификацией языка для стандартного **dict** (в Py3.6 это было деталью реализации CPython). Практические следствия: **for k in d** и **list(d)** дают предсказуемый порядок; **json.dumps(d)** выводит ключи в порядке вставки; логирование **dict** воспроизводимо. При обновлении значения существующего ключа (**d[key] = new_value**) позиция ключа **не** меняется; при удалении ключа и повторной вставке он оказывается в конце.

**Пример:**

```python
d = {}
d["a"] = 1
d["b"] = 2
d["c"] = 3
list(d)   # ['a', 'b', 'c']
d["a"] = 10   # позиция "a" не меняется
list(d)   # ['a', 'b', 'c']
del d["b"]
d["b"] = 20   # "b" теперь в конце
list(d)   # ['a', 'c', 'b']
```

#### Слияние словарей

- **Распаковка:** **`{**d1, **d2}`** — создаёт **новый** словарь; при совпадении ключей побеждает значение из **следующего** по порядку словаря (справа). Порядок ключей в результате: сначала ключи из **d1** (в порядке их следования), затем ключи из **d2**, при совпадении ключа позиция остаётся от первого вхождения, значение берётся из **d2**.
- **Оператор `|`** (Python 3.9+): **`d1 | d2`** — новый словарь (аналог **{**d1, **d2}**); **`d1 |= d2`** — обновление **на месте** (эквивалент **d1.update(d2)**). При совпадении ключей значение из **d2** перезаписывает значение в **d1**.

**Три и более словарей:** распаковка **`{**d1, **d2, **d3}`** — объединяет слева направо; при повторяющихся ключах побеждает значение из самого правого словаря. Оператор **|** можно вызывать цепочкой: **`d1 | d2 | d3`\*\* (порядок применения слева направо, результат — новый словарь).

**Пошагово: как формируется результат слияния при совпадающих ключах.** При **{**d1, **d2}** интерпретатор создаёт новый пустой словарь, затем последовательно выполняет «обновление» ключами и значениями: сначала перебирает **d1** и для каждого ключа **k** делает **result[k] = d1[k]**; затем перебирает **d2** и для каждого **k** делает **result[k] = d2[k]** — то есть перезаписывает значение, если ключ уже был. Итоговый порядок ключей: сначала все ключи из **d1** в порядке их следования, затем ключи из **d2**, которых ещё не было; для ключей, присутствующих в обоих, позиция в результате остаётся от первого вхождения (из **d1**), значение берётся из **d2**. Поэтому **{**d1, **d2}** и **d2 | d1** дают разный порядок ключей при одинаковом наборе пар.

**Когда что использовать:** **{**d1, **d2}** и **d1 | d2** дают **новый** объект; исходные словари не меняются. **d1 |= d2** и **d1.update(d2)** изменяют **d1** на месте. Если нужно сохранить **d1** неизменным — используйте **d1 | d2** или **{**d1, **d2}**; если **d1** можно менять и важен порядок ключей из **d2** — **d1.update(d2)** или **d1 |= d2**.

```python
d1 = {'a': 1, 'b': 2}
d2 = {'b': 20, 'c': 3}
{**d1, **d2}       # {'a': 1, 'b': 20, 'c': 3}
d1 | d2            # {'a': 1, 'b': 20, 'c': 3}
d1 |= d2           # d1 теперь {'a': 1, 'b': 20, 'c': 3}

# Три словаря: при совпадении ключа побеждает самый правый
d3 = {'b': 200, 'd': 4}
{**d1, **d2, **d3}   # {'a': 1, 'b': 200, 'c': 3, 'd': 4}
d1 | d2 | d3         # то же
```

#### Теоретическое обоснование

View-объекты дают **единый интерфейс** к ключам/значениям/парам без копирования; итерация и проверка **`in`** работают за константное время и отражают текущее состояние словаря. Удаление **iterkeys/itervalues/iteritems** устраняет дублирование: в Python 3 сами view итерируемы и эффективны по памяти.

#### Миграция dict с Python 2 на Python 3

- В Py2 **`d.keys()`**, **`d.values()`**, **`d.items()`** возвращали **списки** (копии). Код, который полагался на это (например, **`d.keys().sort()`** или **`d.keys()[0]`**), в Py3 сломается: у view нет метода **sort**, а индексация по целому индексу недоступна. Решение: при необходимости списка использовать **`list(d.keys())`**, **`list(d.values())`**, **`list(d.items())`**; для сортировки ключей — **`sorted(d.keys())`** или **`sorted(d)`**.
- Методы **`.iterkeys()`**, **`.itervalues()`**, **`.iteritems()`** в Py3 удалены — используйте **`d.keys()`**, **`d.values()`**, **`d.items()`**; они уже итерируемы и не копируют данные.
- Порядок ключей в Py2 не гарантировался (зависил от реализации и истории вставок). В Py3.7+ порядок вставки гарантирован; код, который полагался на «произвольный» порядок, может неожиданно зависеть от порядка — это обычно желаемое поведение, но при миграции стоит проверить логику.

#### Типичные ошибки при работе с dict в Python 3

- **Изменение словаря во время итерации по view:** итерация по **d.keys()** или **d.items()** и одновременное изменение **d** (добавление/удаление ключей) может привести к **RuntimeError** или неожиданному поведению. Итерируйте по копии ключей: **`for k in list(d): ...`** или создайте список пар **`list(d.items())`** и итерируйте по нему.
- **Ожидание списка от .keys()/.values()/.items():** в Py3 это view, не списки. Для индексации или многократного прохода по «снимку» используйте **`list(d.keys())`** и т.д.

#### Как устроены view-объекты (упрощённо)

View не хранит копию ключей, значений или пар — он хранит **ссылку на словарь** и при итерации или проверке **in** обращается к текущему состоянию словаря. Поэтому **len(d.keys())** и **key in d.keys()** работают за O(1) и отражают текущее состояние; изменение словаря (добавление/удаление ключей) сразу видно при следующем обращении к view. Операции **d.keys() | other** (объединение с другим set-like) и **d.keys() & other** (пересечение) поддерживаются для **dict_keys**, так как ключи словаря образуют множество (уникальные hashable значения). Аналогично для **dict_items** в ряде операций; **dict_values** не поддерживает операции множеств, так как значения могут повторяться и быть не hashable.

**Пошагово: что происходит при итерации по view и одновременном изменении словаря**

1. Итератор по **d.keys()**, **d.values()** или **d.items()** привязан к внутренней структуре словаря (к количеству и составу записей на момент начала итерации или к текущему состоянию при каждом **next**).
2. Если во время итерации **добавить** или **удалить** ключ в **d**, интерпретатор может выбросить **RuntimeError: dictionary changed size during iteration** — итератор обнаруживает, что размер словаря изменился, и прерывает цикл для предсказуемости.
3. **Безопасный способ** — итерировать по **копии** ключей или пар: **for k in list(d):** или **for k, v in list(d.items()):** — тогда **list(d)** создаётся один раз в начале цикла, и изменение **d** внутри цикла не влияет на уже созданный список. Удаление ключа **k** из **d** во время **for k in list(d):** безопасно; удаление других ключей тоже. Добавление ключей во время такого цикла не приведёт к RuntimeError (но логика «обработать все текущие ключи» может быть нарушена, если добавленные ключи тоже должны быть обработаны — тогда нужна другая схема, например очередь).

**Пример опасного и безопасного кода:**

```python
d = {'a': 1, 'b': 2, 'c': 3}
# Опасно: for k in d: ... del d[k] или d[new] = ... может дать RuntimeError
# Безопасно:
for k in list(d):
    if k == 'skip':
        del d[k]
    print(k, d.get(k))
```

**Пример операций с view ключей:**

```python
d = {'a': 1, 'b': 2, 'c': 3}
other_keys = {'b', 'c', 'd'}
d.keys() | other_keys   # {'a', 'b', 'c', 'd'} — объединение множеств ключей
d.keys() & other_keys   # {'b', 'c'} — пересечение (общие ключи)
d.keys() - other_keys   # {'a'} — ключи из d, которых нет в other
```

Результат — set-like объект (для **|** это **set**); исходный словарь **d** не изменяется. Удобно проверять «есть ли в словаре хотя бы один из ключей» или «какие ключи общие у двух словарей» без создания промежуточных списков.

#### get(key[, default]), setdefault(key[, default]), fromkeys(iterable, value=None)

- **d.get(key, default)** — возвращает **d[key]** при наличии ключа; иначе возвращает **default** (по умолчанию **None**). Не выбрасывает **KeyError**; удобно, когда отсутствие ключа не ошибка.
- **d.setdefault(key[, default])** — если **key** есть в **d**, возвращает **d[key]**. Если ключа нет — записывает **d[key] = default** (если **default** не передан — **None**) и возвращает это значение. Удобно для «инициализации при первом обращении» (например, список по ключу: **d.setdefault(k, []).append(v)**).

**Пошагово: что происходит при d.get(k, default) и d.setdefault(k, default).** (1) **d.get(k, default)**: интерпретатор ищет ключ **k** в словаре **d** (через хеш и сравнение). Если ключ найден — возвращается **d[k]**; если не найден — возвращается **default** (второй аргумент, по умолчанию **None**). Словарь **d** **не изменяется**. (2) **d.setdefault(k, default)**: сначала ищется ключ **k** в **d**. Если найден — возвращается **d[k]**. Если не найден — выполняется **d[k] = default** (если **default** не передан — **d[k] = None**), затем возвращается **d[k]**. Итог: **get** — только чтение, словарь не меняется; **setdefault** — чтение и **запись при отсутствии**. При группировке по ключу **d.setdefault(k, []).append(v)** делает: при первом **k** создаётся **d[k] = []** и возвращается этот список; затем **.append(v)** добавляет **v** в него; при следующих **k** **setdefault** возвращает уже существующий список без перезаписи.

- **dict.fromkeys(iterable, value=None)** — классовый метод; создаёт новый словарь с ключами из **iterable** и одним и тем же **value** для всех (по умолчанию **None**). Удобно для инициализации множества ключей с одним значением: **dict.fromkeys(['a', 'b', 'c'], 0)** → **{'a': 0, 'b': 0, 'c': 0}**.

**Пошагово: что происходит при dict.fromkeys(iterable, value).** (1) Интерпретатор создаёт новый пустой словарь. (2) Перебирает **iterable** и для каждого элемента **k** выполняет **result[k] = value** — то есть один и тот же объект **value** присваивается всем ключам. (3) Важно: если **value** — изменяемый объект (например, список **[]**), то **все** ключи будут указывать на **один и тот же** объект; изменение **d[k1]** затронет **d[k2]**. Пример: **d = dict.fromkeys(['a', 'b'], [])**; **d['a'].append(1)**; **d['b']** тоже станет **[1]**. Для счётчиков по умолчанию используйте **dict.fromkeys(keys, 0)** (int неизменяем — безопасно); для списков по ключу — **{k: [] for k in keys}**, чтобы каждый ключ получил **свой** список.

```python
d = {'a': 1}
d.get('a')        # 1
d.get('b')        # None
d.get('b', 99)    # 99
d.setdefault('b', 2)   # 2; d теперь {'a': 1, 'b': 2}
d.setdefault('b', 99)  # 2 — ключ уже есть
dict.fromkeys([1, 2, 3], "x")   # {1: 'x', 2: 'x', 3: 'x'}
```

**Плохо vs лучше:** в цикле не делайте **if k not in d: d[k] = []** и затем **d[k].append(v)** — короче и яснее **d.setdefault(k, []).append(v)** (одна строка, без лишней проверки).

#### Dict comprehension (генератор словаря)

**Dict comprehension** — синтаксис **`{ключ: значение for элемент in итерабельное [if условие]}`**: фигурные скобки с двоеточием внутри. Результат — **новый словарь**. Удобно строить словарь из итерабельного с преобразованием ключа и значения. При совпадении ключей побеждает последнее значение (как при присваивании **d[k] = v**).

```python
# Квадраты чисел как ключ → число
{x**2: x for x in range(5)}   # {0: 0, 1: 1, 4: 2, 9: 3, 16: 4}

# Нормализация ключей (нижний регистр)
{s.lower(): s for s in ["Hello", "World"]}   # {'hello': 'Hello', 'world': 'World'}

# С условием
{k: v for k, v in [("a", 1), ("b", 2), ("a", 3)] if v > 1}   # {'b': 2, 'a': 3}
```

**Пошагово: порядок выполнения dict comprehension.** При **{k: v for elem in iterable if cond}** интерпретатор: (1) создаёт новый пустой словарь; (2) перебирает **iterable** по элементам **elem**; (3) для каждого **elem** проверяет **cond** (если есть); если **cond** ложно — пропускает элемент; (4) если **cond** истинно (или нет условия), вычисляет **k** и **v** по **elem** и выполняет **result[k] = v**. При совпадении ключей последнее значение перезаписывает предыдущее. Порядок ключей в результате — порядок **первого** присваивания каждого ключа (в Py3.7+ порядок вставки сохраняется). Условие **if** относится к ближайшему слева **for**; вложенные циклы работают как в list comprehension — самый правый **for** «внутренний».

**Когда использовать:** когда нужно получить словарь из последовательности пар или из одной последовательности с вычислением ключа/значения. Для простого **dict(iterable_of_pairs)** достаточно конструктора **dict**; comprehension нужен при фильтрации или преобразовании.

#### Практический вывод по dict

- В Py3 для итерации по ключам/значениям/парам используйте **d.keys()**, **d.values()**, **d.items()** — они итерируемы и не копируют данные.
- Если нужен именно **список** (индексация, **.sort()**), вызывайте **list(d.keys())** и т.д.
- При итерации с изменением словаря итерируйте по **list(d)** или **list(d.items())**, чтобы избежать **RuntimeError** или неожиданного пропуска элементов.

### 2.3 list

#### sort и sorted

- **`list.sort(*, key=None, reverse=False)`** — сортировка **на месте**; возвращает `None`.
- **`sorted(iterable, *, key=None, reverse=False)`** — возвращает **новый** список; исходная последовательность не меняется.

Сортировка в Python **стабильная**: равные элементы сохраняют относительный порядок.

#### Умножение списка на число (lst \* n): копируются ссылки

Операция **lst \* n** (или **n \* lst**) создаёт список из **n** копий **элементов** списка. Копируются **ссылки** на объекты, а не сами объекты. Поэтому если элемент — изменяемый объект (вложенный список, словарь), то все «копии» в результате указывают на **один и тот же** объект; изменение одного вложенного списка затронет все «копии».

**Как это работает внутри: почему lst \* n копирует ссылки.** Интерпретатор при вычислении **lst \* n** создаёт **новый** список длины **len(lst) \* n** и заполняет его: для индекса **i** в новом списке берётся элемент **lst[i % len(lst)]** — то есть **та же ссылка**, что и в исходном списке. Сами объекты-элементы **не клонируются**: новый список содержит **n** раз одни и те же ссылки на одни и те же объекты в памяти. Поэтому **id(rows[0]) == id(rows[1])** для **rows = [[]] \* 3** — все три элемента указывают на один и тот же список. Для неизменяемых элементов (числа, строки) это безопасно; для изменяемых (списки, dict) — одна мутация отражается во всех «копиях».

```python
# Плохо: один и тот же вложенный список во всех элементах
rows = [[]] * 3   # [ [], [], [] ] — но все три элемента это один и тот же []
rows[0].append(1)
# rows теперь [[1], [1], [1]]

# Лучше: создать отдельный список для каждой строки
rows = [[] for _ in range(3)]
rows[0].append(1)   # [[1], [], []]
```

**Практический вывод:** для списка списков (матрицы, таблицы) не используйте **lst \* n**; используйте **[[...] for \_ in range(n)]** или **[[] for \_ in range(n)]**, чтобы каждый элемент был отдельным объектом.

#### Отрицательные индексы и срезы

Индекс **-1** — последний элемент, **-2** — предпоследний и т.д. Срез **lst[start:stop]** при отрицательных **start**/ **stop** трактуется как **start + len(lst)** и т.п. Срез **lst[-2:]** — последние два элемента; **lst[:-1]** — все кроме последнего.

**Пошагово: как вычисляются срезы lst[i:j:k] (start, stop, step).** (1) **lst[i:j]** (без **step**): **start** = **i** (при отрицательном — **i + len(lst)**); **stop** = **j** (при отрицательном — **j + len(lst)**). Элементы с индексами от **start** до **stop-1** (включительно) попадают в результат; если **start >= stop** или **start** за пределами длины — результат **[]**. (2) **lst[i:j:k]** (с **step**): элементы берутся с индексами **start**, **start+k**, **start+2k**, ... пока индекс не достигнет или не переступит **stop**. При **step < 0** индексы идут вниз; **stop** обычно меньше **start**. Пример: **lst[1:4]** при **lst = [0,1,2,3,4]** → индексы 1, 2, 3 → **[1,2,3]**; **lst[::-1]** → **start = len(lst)-1**, **stop = -1**, **step = -1** → весь список в обратном порядке. (3) **lst[10:20]** при **len(lst)=5** — результат **[]** (start > len, диапазон пуст). **lst[-5:]** — последние пять; **lst[:5]** — первые пять.

**Присваивание срезу (slice assignment):** **lst[i:j] = iterable** заменяет элементы с **i** по **j** (не включая **j**) на элементы из **iterable**; длина списка может измениться. **lst[1:3] = [10, 20, 30]** вставит три элемента вместо двух (список удлинится). **lst[1:3] = []** удалит элементы с индексами 1 и 2. Это единственный способ «на месте» изменить длину списка в произвольном месте (вставка/удаление блока).

**Пошагово: что происходит при присваивании срезу lst[i:j] = iterable.** (1) Интерпретатор вычисляет срез **lst[i:j]** — это диапазон индексов от **i** до **j** (не включая **j**); отрицательные индексы приводятся к положительным (i + len(lst) и т.п.). (2) Итерабельный объект **iterable** полностью consumed (преобразуется в последовательность элементов). (3) Элементы с индексами **i .. j-1** в списке **удаляются** (или заменяются); на их место **вставляются** элементы из **iterable** в том же порядке. (4) Если элементов в **iterable** больше, чем было в срезе — список **удлиняется**; если меньше — **укорачивается**. Исходные объекты в срезе заменяются ссылками на объекты из **iterable**; сам список **lst** остаётся тем же объектом (меняется только его содержимое и длина). Поэтому **a[1:3] = [10, 20, 30]** при **a = [1,2,3,4,5]** даёт **a = [1, 10, 20, 30, 4, 5]** — два элемента (индексы 1 и 2) заменены на три.

```python
a = [1, 2, 3, 4, 5]
a[-1]        # 5
a[-2:]       # [4, 5]
a[1:3] = [10, 20, 30]   # a теперь [1, 10, 20, 30, 4, 5]
a[2:5] = []  # a теперь [1, 10, 5]
```

#### extend и append

- **`.append(x)`** — добавляет **один** элемент (в том числе другой список как один элемент).
- **`.extend(iterable)`** — добавляет **все** элементы итерабельного объекта.

**Пошагово: что происходит при lst.append(x) и lst.extend(iterable).** (1) **lst.append(x)**: интерпретатор добавляет ссылку на объект **x** в конец списка **lst**; **len(lst)** увеличивается на 1. Объект **x** может быть **любым** — числом, строкой, списком, словарём и т.д. Если **x** — список **[3, 4]**, то в **lst** попадёт **одна** ссылка на этот список; в результате **lst[-1]** будет тем же объектом, что и **x** (не копия). Метод **append** возвращает **None**. (2) **lst.extend(iterable)**: интерпретатор перебирает **iterable** и для каждого элемента вызывает логику добавления в конец **lst**; по сути эквивалентно **for item in iterable: lst.append(item)**. В итоге **len(lst)** увеличивается на **len(list(iterable))**. Если **iterable** — строка **"ab"**, добавятся два элемента **'a'** и **'b'** (символы), а не строка целиком. Метод **extend** возвращает **None**. Итог: **append** — один элемент; **extend** — «развернуть» итерабельное и добавить каждый элемент по отдельности.

```python
a = [1, 2]
a.append([3, 4])   # [1, 2, [3, 4]]
b = [1, 2]
b.extend([3, 4])   # [1, 2, 3, 4]
# extend от строки — добавит символы
c = [1]
c.extend("ab")     # [1, 'a', 'b']
```

#### Распаковка

Присваивание с **`*`** позволяет собрать «остаток» в список (работает в циклах, функциях, кортежах):

```python
a, *rest = [1, 2, 3, 4]   # a=1, rest=[2, 3, 4]
first, *mid, last = [1, 2, 3, 4, 5]   # first=1, mid=[2,3,4], last=5
```

#### List comprehension (списковые включения)

**List comprehension** — компактный способ построить список из итерабельного объекта (или из другого списка) с опциональным фильтром и преобразованием. Синтаксис: **`[выражение for элемент in итерабельное]`** или с условием: **`[выражение for элемент in итерабельное if условие]`**.

**Базовый синтаксис:** **`[x for x in iterable]`** — эквивалент **`list(iterable)`** (если нужна копия) или построение списка по формуле: **`[x**2 for x in range(5)]`** → **`[0, 1, 4, 9, 16]`\*\*.

**С условием:** **`[x for x in iterable if условие]`** — в список попадают только элементы, для которых условие истинно. Пример: **`[x for x in range(10) if x % 2 == 0]`** → **`[0, 2, 4, 6, 8]`**. Условие проверяется для каждого элемента; можно использовать **несколько** условий (через **and** / **or**).

**Вложенные list comprehension:** внутри выражения или в цикле **for** можно использовать ещё один цикл: **`[x + y for x in [1,2] for y in [10,20]]`** → **`[11, 21, 12, 22]`** (внешний цикл по **x**, внутренний по **y**). Вложенность быстро ухудшает читаемость; при двух и более уровнях часто лучше заменить на обычный цикл **for** с **append** или вынести логику в отдельную функцию — «плоский лучше вложенного» (Zen of Python).

**Когда не злоупотреблять:** одна строка с несколькими **for** и **if** трудно читается и отлаживается. Если выражение или условие длинные — вынесите в переменную или функцию. List comprehension хорош для простых преобразований и фильтрации; для сложной логики предпочтительнее явный цикл.

**Граничные случаи list comprehension:** (1) **Пустой итерабельный объект:** **\[x\*\*2 for x in \[\]\]** → **\[\]**; **\[x for x in range(0)\]** → **\[\]**. (2) **Условие всегда ложно:** **\[x for x in range(5) if False\]** → **\[\]**. (3) **Вложенное условие:** условие **if** относится к **ближайшему** слева **for**; **\[x for x in a for y in b if y\]** — **if y** проверяется для каждой пары **(x, y)**; **\[x for x in a if x for y in b\]** — **if x** фильтрует **x**, затем для каждого допустимого **x** перебирается **y**. (4) **Walrus operator в comprehension (Py3.8+):** **\[y for x in data if (y := f(x)) > 0\]** — присваивание **y = f(x)** в условии и использование **y** в выражении; экономит двойной вызов **f(x)**. (5) **Область видимости:** в Py3 переменная цикла **не** утекает в окружающую область; после **\[i for i in range(3)\]** имя **i** в Py3 **не** определено в модуле (в Py2 было бы **i == 2**).

```python
# Простое преобразование
[x * 2 for x in [1, 2, 3]]   # [2, 4, 6]

# С условием
[x for x in range(5) if x > 2]   # [3, 4]

# Вложенный (плоскость лучше не увлекаться)
[(i, j) for i in range(2) for j in range(2)]   # [(0,0), (0,1), (1,0), (1,1)]
```

**Порядок вычисления:** интерпретатор обрабатывает list comprehension слева направо: сначала **for** (перебор элементов), затем **if** (если есть) — фильтр применяется к каждому элементу, затем **выражение** — результат записывается в список. Порядок циклов при вложенности: **сначала** выполняется самый правый **for**, затем левее (как во вложенных циклах **for**). Условие **if** относится к ближайшему слева **for**.

**Пошагово: порядок выполнения вложенного list comprehension [x + y for x in [1,2] for y in [10,20]].** (1) Внешний цикл — **for x in [1,2]**: **x** принимает значения **1**, затем **2**. (2) Для каждого **x** выполняется внутренний цикл **for y in [10,20]**: **y** принимает **10**, затем **20**. (3) На каждой итерации (для каждой пары **x, y**) вычисляется **x + y** и результат добавляется в список. Порядок пар: **(1,10) → 11**, **(1,20) → 21**, **(2,10) → 12**, **(2,20) → 22**. Результат: **[11, 21, 12, 22]**. Это эквивалентно вложенным циклам **for x in [1,2]: for y in [10,20]: result.append(x+y)**. Важно: внутренний цикл (**y**) «крутится» быстрее — он завершает полный проход для каждого значения **x**.

**Условное выражение в результате (if-else в выражении):** если нужно по условию подставлять **разные** значения в список (не фильтровать, а выбирать одно из двух), используйте тернарный оператор в выражении: **`[x if x > 0 else 0 for x in data]`** — для каждого **x** в список попадёт **x**, если **x > 0**, иначе **0**. Здесь **if-else** — часть **выражения**, а не фильтр после **for**; фильтр — это **if** в конце без **else**: **`[x for x in data if x > 0]`** (только положительные).

**Плохо vs хорошо:**

```python
# Плохо: длинное выражение и вложенность в одну строку
result = [str(x).upper() if isinstance(x, str) else x * 2 for x in mixed for y in [1, 2] if x]

# Лучше: вынести преобразование в функцию или разбить на шаги
def normalize(x):
    return str(x).upper() if isinstance(x, str) else x * 2
result = [normalize(x) for x in mixed for y in [1, 2] if x]
# или явный цикл, если логика сложная
```

#### Копирование списка

- **`lst[:]`** — срез по всей длине, создаёт **shallow copy** (поверхностную копию).
- **`list(lst)`** — конструктор от итерабельного, тоже shallow copy.
- **`lst.copy()`** — метод списка (Python 3.3+), shallow copy.
- **`copy.copy(lst)`** — модуль **copy**, shallow copy; **`copy.deepcopy(lst)`** — полная рекурсивная копия.

#### Почему extend, а не append для «добавить несколько элементов»

Метод **`.append(x)`** добавляет **один** объект в конец списка. Если **x** — список, то в списке окажется **вложенный список** как один элемент: **`[1, 2].append([3, 4])`** → **`[1, 2, [3, 4]]`**. Метод **`.extend(iterable)`** добавляет **каждый элемент** итерабельного объекта по отдельности: **`[1, 2].extend([3, 4])`** → **`[1, 2, 3, 4]`**. Путаница между **append** и **extend** — частая причина багов: ожидают «плоский» список, а получают вложенную структуру.

#### Стабильная сортировка

Сортировка в Python **стабильная**: при равных ключах относительный порядок элементов сохраняется. Это важно при **многоуровневой** сортировке: сначала сортируете по одному полю, затем по другому — порядок по первому полю не нарушается для элементов, равных по второму полю.

**Пример многоуровневой сортировки:** нужно отсортировать список записей сначала по имени, затем по возрасту. Сначала сортируем по возрасту (второстепенный ключ), затем по имени (главный ключ) — тогда при равных именах порядок по возрасту сохранится:

```python
people = [("Alice", 30), ("Bob", 25), ("Alice", 25)]
# Сначала по возрасту, потом по имени
sorted(people, key=lambda p: (p[1], p[0]))   # [("Alice", 25), ("Bob", 25), ("Alice", 30)]
# Или: сначала по имени, потом по возрасту
sorted(people, key=lambda p: (p[0], p[1]))   # [("Alice", 25), ("Alice", 30), ("Bob", 25)]
```

Без стабильности при второй сортировке порядок по первому полю мог бы «перемешаться» для равных элементов; в Python этого не происходит.

#### sort() и sorted(): в чём различие и когда что использовать

- **`list.sort(*, key=None, reverse=False)`** — метод **списка**; сортирует **на месте** (изменяет сам список); возвращает **None**. Вызывается только у списка: **lst.sort()**.
- **`sorted(iterable, *, key=None, reverse=False)`** — **встроенная функция**; принимает любой итерабельный объект (список, кортеж, ключи словаря, range и т.д.); возвращает **новый список** с отсортированными элементами; исходная последовательность **не изменяется**.

**Когда использовать sort():** когда у вас уже есть список и вы хотите отсортировать его на месте, не создавая копию. Экономит память и явно показывает намерение «изменить этот список».

**Когда использовать sorted():** когда исходную последовательность менять нельзя (кортеж, ключи словаря, результат функции) или когда нужен новый список, а оригинал сохранить. Пример: **sorted(d.keys())**, **sorted(my_tuple)**.

```python
lst = [3, 1, 2]
lst.sort()        # lst теперь [1, 2, 3]; sort() возвращает None
result = lst.sort()   # result есть None!

lst = [3, 1, 2]
new_lst = sorted(lst)   # new_lst == [1, 2, 3], lst по-прежнему [3, 1, 2]
```

**Пошагово: что происходит при lst.sort() и при sorted(lst).** (1) **lst.sort():** интерпретатор вызывает метод **sort** объекта **lst**. Алгоритм сортировки (Timsort в CPython) переупорядочивает **элементы внутри того же списка** — меняются местами ссылки в **lst**, сам объект **lst** (его **id**) не меняется. Метод **sort** возвращает **None**; поэтому **result = lst.sort()** присваивает **result** значение **None**, а не отсортированный список. После **lst.sort()** переменная **lst** ссылается на тот же объект, но его содержимое уже отсортировано. (2) **sorted(lst):** встроенная функция **sorted** принимает итерабельный объект (**lst** или любой другой), **создаёт новый список** (копия элементов в новом объекте списка), сортирует этот новый список и возвращает его. Исходный **lst** **не изменяется** — **id(lst)** до и после вызова **sorted(lst)** один и тот же, содержимое **lst** то же. Итог: **sort()** — мутация на месте, один объект; **sorted()** — новый объект, исходный не трогается.

#### Распаковка списка: дополнительные примеры

Распаковка с **`*`** работает в присваивании, в вызовах функций (передача элементов как позиционных аргументов), в циклах:

```python
# Присваивание
head, *tail = [1, 2, 3, 4]   # head=1, tail=[2,3,4]
*front, last = [1, 2, 3, 4]   # front=[1,2,3], last=4
first, *mid, last = [1, 2, 3, 4, 5]   # first=1, mid=[2,3,4], last=5

# Вызов функции: распаковать список в аргументы
def f(a, b, c):
    return a + b + c
args = [1, 2, 3]
f(*args)   # 6 — эквивалент f(1, 2, 3)

# В цикле (распаковка кортежей/списков)
for name, age in [("Alice", 30), ("Bob", 25)]:
    print(name, age)
```

#### Shallow copy и deep copy: почему это важно

**Shallow copy** (поверхностная копия) копирует только «верхний уровень»: элементы списка остаются теми же объектами. Если элемент — вложенный список или словарь, то в копии и в оригинале будет **одна и та же** вложенная структура; изменение вложенного объекта в одной копии видно в другой.

**Deep copy** (глубокая копия) рекурсивно копирует все вложенные объекты; после **deepcopy** оригинал и копия полностью независимы.

```python
import copy
orig = [[1, 2], [3, 4]]
shallow = orig.copy()   # или orig[:], list(orig)
shallow[0][0] = 99      # меняем вложенный список
# orig[0][0] тоже 99! — shallow скопировал ссылки на [1,2] и [3,4]

orig = [[1, 2], [3, 4]]
deep = copy.deepcopy(orig)
deep[0][0] = 99         # orig[0][0] по-прежнему 1
```

**Когда использовать shallow:** когда вложенных изменяемых структур нет или вы намеренно хотите делить одни и те же вложенные объекты. **Когда использовать deep copy:** когда нужна полная независимая копия структуры (например, чтобы безопасно менять копию без влияния на оригинал).

**Почему shallow copy часто достаточно:** если элементы списка — числа, строки, кортежи или другие неизменяемые объекты, то при изменении «копии» вы меняете только сам список (добавляете/удаляете элементы, меняете порядок); элементы при этом не мутируют, так как они immutable. Shallow copy копирует ссылки на те же самые элементы — и это корректно: «скопировать список чисел» означает новый список с теми же числами. Проблема возникает только когда элемент сам по себе изменяемый (список, словарь) и вы меняете **внутренность** элемента (например, **copy[0].append(1)**) — тогда оригинал тоже «увидит» это изменение, потому что **copy[0]** и **orig[0]** указывают на один и тот же объект.

#### Миграция list с Python 2 на Python 3

В Py2 и Py3 **list** ведёт себя одинаково: **append**, **extend**, **sort**, **sorted**, срезы, распаковка. Отличия связаны с другими типами: **range** в Py3 не список — **list(range(n))** для получения списка; **map**/ **filter** в Py3 возвращают итераторы — **list(map(...))** при необходимости списка.

#### list.index(x), list.clear(), вставка по индексу

- **lst.index(x[, start[, end]])** — возвращает индекс **первого** вхождения **x** в срезе **lst[start:end]**; если элемент не найден, выбрасывает **ValueError**. Удобно, когда нужно найти позицию элемента; если отсутствие допустимо — используйте проверку **x in lst** или обёртку в **try/except ValueError**.
- **lst.clear()** (Python 3.3+) — удаляет все элементы из списка на месте; эквивалентно **del lst[:]**. Возвращает **None**.
- **lst.insert(i, x)** — вставляет **x** перед позицией **i**; **lst.insert(0, x)** вставляет в начало, **lst.insert(len(lst), x)** эквивалентно **lst.append(x)**. Остальные элементы сдвигаются. Вставка в начало списка — операция O(n), так как сдвигаются все последующие элементы; для частых вставок в начало рассмотрите **collections.deque** с **appendleft**.

```python
lst = [1, 2, 3, 2]
lst.index(2)       # 1
lst.index(2, 2)    # 3
lst.index(5)       # ValueError
lst.clear()        # lst теперь []
lst.insert(0, 10)  # [10] при пустом lst; при [1,2] станет [10, 1, 2]
```

#### Типичные ошибки при работе с list

- **Путать append и extend:** **lst.append([1,2,3])** добавляет один элемент (список); **lst.extend([1,2,3])** добавляет три элемента. Ожидали «плоский» список — получили вложенный.
- **Вызов метода в условии:** **lst.sort()** возвращает **None**; писать **if lst.sort():** бессмысленно (всегда False). Для проверки отсортированности используйте **sorted(lst) == lst** или отдельно вызовите **lst.sort()**.
- **Изменение списка во время итерации:** удаление или вставка элементов в цикле **for x in lst** может привести к пропуску элементов или неожиданному поведению. Итерируйте по копии: **for x in lst[:]:** или создайте новый список с нужными элементами.
- **Ожидать от .index() -1 при отсутствии:** в отличие от **str.find()**, у списка нет **find**; **lst.index(x)** при отсутствии **x** выбрасывает **ValueError**. Для «найти или -1» используйте **lst.index(x) if x in lst else -1** или **try/except**.

### 2.4 set и frozenset

**set** — изменяемое множество уникальных элементов; **frozenset** — неизменяемое. Элементы должны быть **hashable** (неизменяемые или с фиксированным хешем). Порядок элементов не гарантирован (в CPython 3.7+ для **set** он стабилен при неизменном наборе, но полагаться на порядок не рекомендуется).

#### Операции

| Операция                | Метод / оператор                                     | Описание                                    |
| ----------------------- | ---------------------------------------------------- | ------------------------------------------- |
| Объединение             | **`s \| other`**, **`s.union(*others)`**             | Все элементы из s и других                  |
| Пересечение             | **`s & other`**, **`s.intersection(*others)`**       | Общие элементы                              |
| Разность                | **`s - other`**, **`s.difference(*others)`**         | Элементы из s, которых нет в other          |
| Симметрическая разность | **`s ^ other`**, **`s.symmetric_difference(other)`** | Элементы, входящие ровно в одно из множеств |

In-place варианты (только для **set**): **`s |= other`**, **`s &= other`**, **`s -= other`**, **`s ^= other`** и соответствующие методы **update**, **intersection_update**, **difference_update**, **symmetric_difference_update**.

**set.pop()** — удаляет и возвращает **произвольный** элемент множества; если множество пусто, выбрасывает **KeyError**. Порядок не гарантирован, поэтому **pop()** удобен, когда нужен «любой» элемент (например, взять один элемент из множества обработанных). **frozenset** не поддерживает **pop**, **add**, **remove** — неизменяем. Создать **frozenset** из итерабельного: **frozenset([1, 2, 3])**, **frozenset(range(5))**.

```python
a, b = {1, 2, 3}, {2, 3, 4}
a | b   # {1, 2, 3, 4}
a & b   # {2, 3}
a - b   # {1}
a ^ b   # {1, 4}
```

#### Почему элементы set и frozenset должны быть hashable

**Hashable** — объект, который можно использовать как ключ словаря или элемент множества: у него есть **хеш** (целое число), не меняющийся за время жизни объекта, и он сравнивается через **равенство** (**==**). Неизменяемые встроенные типы (**int**, **float**, **str**, **tuple**, **frozenset**, **bytes**) в Python hashable по умолчанию. Изменяемые (**list**, **dict**, **set**, **bytearray**) — **не** hashable: их хеш мог бы измениться при изменении содержимого, что сломало бы словарь или множество.

Поэтому в **set** и **frozenset** нельзя добавить **list** или **dict** — только неизменяемые или объекты с реализованным \***\*hash\*\*** и \***\*eq\*\***, причём \***\*hash\*\*** не должен меняться при изменении состояния (на практике для своих классов часто делают неизменяемые «значения» или считают хеш по неизменяемым полям).

```python
{1, 2, 3}           # OK
{"a", "b"}           # OK
{(1, 2), (3, 4)}    # OK — tuple hashable
{[1, 2]}             # TypeError: unhashable type: 'list'
{{1: 2}}             # TypeError: unhashable type: 'dict'
```

#### set vs frozenset: когда что использовать

- **set** — изменяемое множество: можно добавлять (**add**, **update**), удалять (**remove**, **discard**, **pop**), очищать (**clear**). Используйте, когда набор элементов может меняться.
- **frozenset** — неизменяемое множество: после создания элементы нельзя добавлять или удалять. Используйте, когда нужен **стабильный** набор (например, ключ словаря, элемент другого set, или передача в функцию без риска изменения изнутри). **frozenset** можно использовать как ключ **dict**; **set** — нельзя (set не hashable).

```python
fs = frozenset([1, 2, 3])
d = {fs: "value"}   # OK
s = {1, 2, 3}
d = {s: "value"}    # TypeError: unhashable type: 'set'
```

#### Set comprehension (генератор множества)

Множество можно построить через **set comprehension** — синтаксис **`{выражение for элемент in итерабельное [if условие]}`** (как list comprehension, но с фигурными скобками). Результат — **set** (уникальные элементы; дубликаты автоматически отбрасываются).

```python
{x ** 2 for x in range(-3, 4)}   # {0, 1, 4, 9}
{c for c in "hello"}              # {'h', 'e', 'l', 'o'} — уникальные символы
{x for x in range(10) if x % 2 == 0}   # {0, 2, 4, 6, 8}
```

**Когда использовать:** когда нужен набор **уникальных** значений из итерабельного (дедупликация) или множество как результат преобразования с фильтром. Для упорядоченной последовательности без дубликатов можно использовать **dict.fromkeys(seq)** (сохраняет порядок вставки в Py3.7+) или список с проверкой **in**.

#### Сложность проверки «элемент в множестве» (in)

Проверка **x in s** для **set** и **frozenset** в среднем выполняется за **O(1)** — хеш-таблица позволяет быстро выяснить наличие элемента. Для **list** и **tuple** проверка **x in lst** в худшем случае **O(n)** — перебор элементов. Поэтому при частых проверках «принадлежность» выгодно преобразовать последовательность в множество: **s = set(lst)**; затем **x in s** за O(1). Учтите: множество не хранит порядок (или не гарантирует его в спецификации); при необходимости порядка используйте **dict** (ключи с Py3.7) или список с проверкой.

**Как это работает внутри: почему x in s за O(1).** Множество **set** (и **frozenset**) реализовано как **хеш-таблица**: каждый элемент должен быть **hashable**, и при добавлении вычисляется **hash(x)** (целое число). По хешу определяется **корзина** (bucket) в массиве; в корзине хранятся элементы с совпадающим хешем (при коллизиях — цепочка или открытая адресация). При проверке **x in s** интерпретатор вычисляет **hash(x)** и смотрит в соответствующую корзину: если там есть объект **y** такой, что **y == x**, возвращается **True**; иначе **False**. Обращение по индексу корзины — константное время; сравнение в корзине в среднем с малым числом элементов. Поэтому **x in s** в среднем **O(1)**. Для **list** и **tuple** проверка **x in lst** требует перебора всех элементов до первого совпадения — в худшем случае **O(n)**.

**Пошагово: что происходит при s.add(x).** (1) Интерпретатор вычисляет **hash(x)** — целое число (для hashable объектов хеш не меняется за время жизни объекта). (2) По хешу определяется **индекс корзины** в массиве хеш-таблицы (обычно **hash(x) % len(table)** или более сложная схема). (3) В корзину нужно **добавить** **x**, если его там ещё нет: перебираются элементы корзины (при коллизиях их может быть несколько); для каждого **y** проверяется **y == x**. Если найден **y** такой, что **y == x**, элемент уже есть — **add** ничего не делает. Если такого **y** нет — **x** добавляется в корзину. (4) Если таблица заполнена выше порога, выполняется **рехеширование** — массив расширяется, элементы перераспределяются по новым корзинам. Итог: **s.add(x)** в среднем **O(1)**; при коллизиях возможно сравнение **==** с несколькими элементами в корзине. Поэтому элементы должны быть **hashable** (неизменяемый хеш и \***\*eq\*\***); изменяемые объекты (list, dict, set) нельзя добавлять в множество.

**Пошагово: s.remove(x) vs s.discard(x).** (1) **s.remove(x)** — ищет **x** в множестве (через **hash(x)** и сравнение в корзине). Если **x** найден — удаляет его и возвращает **None**. Если **x** не найден — выбрасывает **KeyError**. Используйте, когда отсутствие элемента — ошибка. (2) **s.discard(x)** — то же самое, но при отсутствии **x** **ничего не делает** и не выбрасывает исключение; возвращает **None**. Используйте, когда отсутствие элемента допустимо (например, «удалить, если есть»). Итог: **remove** — строгий, **KeyError** при отсутствии; **discard** — мягкий, отсутствие игнорируется. Для проверки «есть ли элемент перед удалением» можно **if x in s: s.remove(x)** или просто **s.discard(x)**.

#### update vs union (|): на месте и новый объект

- **s.union(\*others)** и **s | other** возвращают **новое** множество; исходные **s** и **other** не меняются.
- **s.update(\*others)** и **s |= other** изменяют **s** на месте; возвращают **None** (метод **update**) или обновлённое множество (оператор **|=** в Py3 возвращает **None** для set; в некоторых версиях **|=** возвращает **self**). Аналогично для **intersection_update** (**&=**), **difference_update** (**-=**), **symmetric_difference_update** (**^=**).

**Когда что использовать:** если исходное множество нужно сохранить — **s | other** или **s.union(other)**. Если **s** можно менять и важен минимальный расход памяти (без создания нового объекта) — **s |= other** или **s.update(other)**.

```python
a, b = {1, 2}, {2, 3}
a | b    # {1, 2, 3} — новый set; a и b не изменились
a.update(b)   # a теперь {1, 2, 3}; возвращает None
```

#### Типичные ошибки при работе с set

- **Добавить изменяемый объект:** **s.add([1, 2])** даёт **TypeError: unhashable type: 'list'**. Нужен неизменяемый аналог: **s.add((1, 2))** (кортеж).
- **Полагаться на порядок элементов:** порядок в **set** в Py3.7+ в CPython стабилен при неизменном наборе, но **гарантировать** порядок язык не обязан; для упорядоченной последовательности уникальных элементов используйте список с проверкой или **dict** (ключи с Py3.7 упорядочены).
- **remove vs discard:** **s.remove(x)** при отсутствии **x** выбрасывает **KeyError**; **s.discard(x)** при отсутствии ничего не делает. Ожидали «удалить, если есть» — используйте **discard**; ожидали «элемент обязан быть» — используйте **remove** и обрабатывайте **KeyError**.
- **Путать remove и discard:** **remove(x)** выбрасывает **KeyError**, если **x** нет в множестве; **discard(x)** ничего не делает, если **x** нет. Используйте **discard**, когда отсутствие элемента не ошибка.

**Пример: когда remove, когда discard.** Если логика программы такова, что отсутствие элемента в множестве означает ошибку (например, вы удаляете «обязательный» флаг), используйте **remove** — получите **KeyError** и сможете обработать. Если удаление «если есть» (например, снять флаг, который мог и не быть установлен), используйте **discard** — код остаётся простым, без **if x in s: s.remove(x)**.

```python
allowed = {"read", "write"}
allowed.remove("read")   # OK
allowed.remove("execute")   # KeyError — такого ключа нет

flags = {"dirty"}
flags.discard("dirty")   # OK, множество станет пустым
flags.discard("dirty")   # OK, ничего не делаем, ошибки нет
```

- **Путать union и update:** **s.union(other)** не меняет **s**; **s.update(other)** меняет **s** на месте. Ожидали «объединить и сохранить в s» — используйте **update** или **|=**; ожидали «получить новое множество» — используйте **union** или **|**.

#### Миграция set/frozenset с Python 2 на Python 3

В Py2 и Py3 **set** и **frozenset** по смыслу одинаковы; синтаксис литералов **{1, 2, 3}** и операции совпадают. В Py2 не было гарантии порядка итерации; в Py3.7+ в CPython порядок стабилен (но не закреплён в спецификации для set). Специфичных изменений при миграции для set/frozenset почти нет.

### 2.5 tuple

Кортежи **неизменяемы** (immutable): после создания элементы нельзя добавлять, удалять или менять. Поддерживают индексацию, срезы, **`in`**, **`len`**, итерацию. Распаковка и **`*rest`** работают так же, как для списков. Кортеж **hashable**, если все его элементы hashable — поэтому кортеж можно использовать как **ключ словаря** и как **элемент множества** (в отличие от списка).

#### Литерал кортежа из одного элемента

В отличие от списка **`[1]`**, кортеж из одного элемента нельзя записать как **`(1)`** — скобки в этом случае означают просто группировку выражения, и тип будет **int**. Для кортежа из одного элемента **обязательна запятая**: **`(1,)`**. Без запятой **`(1)`** равно **1**; **`(1,)`** — кортеж длины 1. То же для пустого кортежа: **`()`** — пустой кортеж (тип **tuple**).

**Почему так сделано:** синтаксис выражений в Python использует круглые скобки для группировки: **(a + b) \* c**. Поэтому **(1)** интерпретируется как «выражение в скобках», то есть просто **1**. Запятая — это то, что создаёт кортеж: в Python кортеж задаётся именно запятыми, а не скобками; скобки нужны для пустого кортежа **()** и для читаемости **(1, 2)**. Выражение **1,** — это кортеж из одного элемента **(1,)**; скобки вокруг него для ясности. Поэтому **return 1,** в функции возвращает кортеж **(1,)**, а не число 1.

**Пошагово: как парсер различает (1) и (1,).** (1) При разборе **(1)** интерпретатор видит открывающую скобку **(** — это начало выражения в скобках; затем **1** — выражение (целое число); затем **)** — конец группировки. Результат — выражение **1**, тип **int**. (2) При разборе **(1,)** интерпретатор видит **(** и **1**; затем **,** — запятая означает, что это **tuple display** (кортеж): после запятой может следовать ещё элемент или закрывающая скобка. Результат — кортеж из одного элемента, тип **tuple**. Ключевое различие — наличие **запятой**: без запятой скобки трактуются как группировка; с запятой — как кортеж. Поэтому **1,** без скобок тоже кортеж: **x = 1,** присваивает **x** кортеж **(1,)**.

```python
type((1))    # int
type((1,))   # tuple
len((1,))    # 1
()           # пустой кортеж
x = 1,       # x — кортеж (1,)
```

#### Распаковка кортежа: циклы и возврат нескольких значений

Кортежи часто используются для **возврата нескольких значений** из функции: **`return a, b`** — на самом деле возвращается один объект (кортеж **`(a, b)`**); вызывающий код может распаковать: **`x, y = f()`**. В цикле **for** распаковка кортежей (и списков) позволяет сразу получать поля: **`for name, age in [( "Alice", 30 ), ( "Bob", 25 )]:`**.

```python
def get_min_max(lst):
    return min(lst), max(lst)   # возвращается кортеж (min, max)

lo, hi = get_min_max([3, 1, 4, 1, 5])   # lo=1, hi=5

for key, value in [("a", 1), ("b", 2)]:
    print(key, value)   # a 1, b 2
```

**Почему возвращать кортеж:** один объект возврата упрощает контракт функции; распаковка на стороне вызывающего даёт читаемые имена (**x, y = f()**). Альтернатива — именованный кортеж или dataclass, когда полей много и важна читаемость.

#### Кортеж как ключ dict и элемент set

Поскольку кортеж **неизменяем** и (при hashable элементах) **hashable**, его можно использовать как ключ словаря и как элемент множества. Список — изменяемый и не hashable — нельзя. Типичные сценарии: кэш по паре аргументов **`(a, b)`**, множество координат **{(0, 0), (1, 0)}**, группировка по составному ключу.

```python
cache = {}
cache[(1, 2)] = "result"
cache[(1, 2)]   # "result"

points = {(0, 0), (1, 0), (0, 1)}
(1, 0) in points   # True
# {[1, 2]}   # TypeError: unhashable type: 'list'
```

#### Сравнение кортежей (лексикографическое)

Кортежи сравниваются **лексикографически**: сначала первые элементы, при равенстве — вторые и т.д. Это удобно для сортировки списка кортежей по нескольким полям: **sorted(pairs, key=lambda p: (p[0], p[1]))** или просто **sorted(pairs)** (по умолчанию сравнение кортежей по порядку элементов). Типы элементов в кортеже при сравнении должны быть сравнимы между собой (иначе **TypeError**).

#### Именованные кортежи

Удобный способ задать структуру с именованными полями без объявления класса:

- **`collections.namedtuple(typename, field_names)`** — фабрика классов; поля доступны по имени и по индексу.
- **`typing.NamedTuple`** — подкласс с аннотациями типов: **`class Point(NamedTuple): x: int; y: int`**.

```python
from collections import namedtuple
Point = namedtuple('Point', ['x', 'y'])
p = Point(1, 2)
p.x, p.y     # 1, 2
p[0], p[1]   # 1, 2
```

#### Почему tuple: неизменяемость и использование

**Неизменяемость** кортежа означает: после создания нельзя присвоить **t[i] = x**, нельзя вызвать **append**, **extend**, **remove** и т.п. Это даёт:

1. **Возможность использовать как ключ dict и элемент set** — tuple hashable (если все его элементы hashable). Список — нет.
2. **Защиту от случайного изменения** — передали кортеж в функцию, содержимое не изменится.
3. **Экономию памяти и быстрый доступ** в реализации CPython — кортежи часто компактнее списков.

**Распаковка** кортежа (и списка) в цикле и в присваивании:

```python
for a, b in [(1, 2), (3, 4)]:
    print(a + b)   # 3, 7

x, y = (10, 20)    # x=10, y=20
a, *rest = (1, 2, 3, 4)   # a=1, rest=[2, 3, 4]
```

#### namedtuple и typing.NamedTuple: когда что использовать

- **collections.namedtuple(typename, field_names)** — фабрика классов; **field_names** может быть строка с пробелами **"x y"** или список **["x", "y"]**. Возвращает класс; экземпляры — подкласс tuple, поля доступны по имени и по индексу. Удобно для простых структур данных без методов (точка, запись, константа).
- **typing.NamedTuple** — наследование с аннотациями типов: **class Point(NamedTuple): x: int; y: int**. Совместимо с типизацией (mypy, pyright); при этом экземпляры по-прежнему tuple по поведению.

```python
from typing import NamedTuple
class Point(NamedTuple):
    x: int
    y: int
p = Point(1, 2)
p.x, p.y   # 1, 2
```

**Граничные случаи tuple:** (1) **Пустой кортеж** — **()**; литерал без элементов; **len(())** → **0**; **() in [()]** → **True**. (2) **Один элемент** — **(1,)** или **1,;** без запятой **(1)** — это просто **1** в скобках, не кортеж. (3) **Кортеж с изменяемым элементом** — **(1, [2, 3])** допустим; кортеж сам неизменяем, но элемент **[2, 3]** можно мутировать: **t[1].append(4)** — допустимо; при этом кортеж остаётся **не hashable** (содержит изменяемый объект), его нельзя использовать как ключ **dict** или элемент **set**. (4) **Распаковка с остатком** — **a, \*rest = (1, 2, 3)** присваивает **a=1**, **rest=[2, 3]**; **rest** всегда **list**, даже при одном остатке: **a, \*rest = (1, 2)** → **rest=[2]**. (5) **Несовпадение числа при распаковке** — **a, b = (1,)** → **ValueError: not enough values to unpack**; **a, b = (1, 2, 3)** → **ValueError: too many values to unpack**; использование **\*** решает: **a, \*rest = (1, 2, 3)** — допустимо.

#### Типичные ошибки при работе с tuple

- **Попытка изменить элемент:** **t[0] = 1** даёт **TypeError: 'tuple' object does not support item assignment**. Нужен новый кортеж: **t = (1,) + t[1:]** или используйте список, если нужна мутация.
- **Один элемент в литерале:** **(1)** — это просто число в скобках, не кортеж. Кортеж из одного элемента: **(1,)** — запятая обязательна.
- **Путать tuple и list при ключах dict:** **d[[1,2]] = 3** — ошибка (list не hashable). Используйте **d[(1,2)] = 3**.
- **Кортеж с list как ключ dict:** **d[(1, [2])] = 3** — **TypeError: unhashable type: 'list'**; кортеж с изменяемым элементом не hashable.

#### Миграция tuple и коллекций с Python 2 на Python 3

- **tuple** в Py2 и Py3 ведут себя одинаково (неизменяемость, распаковка, индексация). Именованные кортежи (**namedtuple**) доступны в обоих; синтаксис совпадает.
- **set** и **frozenset** в Py3 не изменились по смыслу; удалены методы **.has_key()** у dict (в Py2 был у словарей; в Py3 используйте **`key in d`**).
