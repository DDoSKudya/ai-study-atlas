### Цели `__repr__` и `__str__`

| Метод | Для кого | Где используется |
|-------|----------|------------------|
| `__repr__(self)` | Для разработчика | интерактивная консоль, дебаггер, логирование, `repr(obj)` |
| `__str__(self)` | Для пользователя | `print(obj)`, `str(obj)`, форматирование через `%s` или `{}` |

**Правило:**

- `__repr__` должен стремиться быть **однозначным**: по возможности возвращать строку, из которой можно восстановить объект (`eval(repr(obj)) == obj`, где это разумно).
- `__str__` — **читаемое, дружелюбное** представление (может быть короче и менее формальным).

Если `__str__` не определён, `str(obj)` по умолчанию вызывает `__repr__`.

### Минимальный шаблон

```python
class Point:
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"{self.__class__.__name__}(x={self.x!r}, y={self.y!r})"

    def __str__(self):
        return f"({self.x}, {self.y})"

p = Point(1, 2)
print(repr(p))  # Point(x=1, y=2)
print(p)        # (1, 2)
```

Использование `self.__class__.__name__` делает `repr` устойчивым к наследованию.

#### Шаблоны для `__repr__` в прикладном коде

Часто полезно придерживаться простых и единообразных шаблонов:

- **Данные-контейнеры / value-объекты**:

  ```python
  def __repr__(self):
      cls = self.__class__.__name__
      return f"{cls}(x={self.x!r}, y={self.y!r})"
  ```

- **Большие объекты / «тяжёлые» структуры**:  
  показывайте только ключевые поля, остальное заменяйте кратким описанием:

  ```python
  def __repr__(self):
      return f"BigModel(id={self.id!r}, fields={len(self.fields)} fields)"
  ```

- **Чувствительные данные**:  
  не выводите секреты в `__repr__` (пароли, токены, ключи):

  ```python
  def __repr__(self):
      return f"DbConfig(user={self.user!r}, password=***hidden***)"
  ```

Практический критерий: `repr(obj)` можно безопасно логировать в debug-лог **без риска утечки критичных данных**.

### `__format__` и f-строки

Функция `format(obj, spec)` и синтаксис `f"{obj:spec}"` используют **`obj.__format__(spec)`**.

Если `__format__` не определён, Python вызывает реализацию базового типа или `__str__`.

#### Пример: форматирование денежной суммы

```python
class Money:
    def __init__(self, amount: float, currency: str = "USD"):
        self.amount = amount
        self.currency = currency

    def __repr__(self):
        return f"Money({self.amount!r}, {self.currency!r})"

    def __format__(self, format_spec: str):
        # поддержим 2 стиля: "" и ".2f"
        if not format_spec:
            fmt = f"{self.amount:.2f}"
        else:
            fmt = format(self.amount, format_spec)
        return f"{fmt} {self.currency}"

price = Money(1234.5, "EUR")
print(f"Цена: {price}")          # Цена: 1234.50 EUR
print(f"Точная: {price:.4f}")    # Точная: 1234.5000 EUR
```

Дополнительно:

- `__format__` часто используют для **локализации** (разные форматы дат, чисел) — но обычно это делается в отдельном слое (через `babel`, `locale` и др.).
- Для большинства прикладных классов достаточно корректного `__str__`; `__format__` нужен, когда вы явно хотите поддерживать **несколько режимов** форматирования.

#### Строка формата: как с ней работать внутри `__format__`

Строка `format_spec` — это ровно то, что вы написали после двоеточия в `f"{obj:...}"`. Для чисел и строк Python уже реализует богатый язык форматов:

- выравнивание (`<`, `>`, `^`) и заполнение (`0`, пробел и др.);
- ширина поля (`10`, `08`, и т.п.);
- формат чисел (`f`, `e`, `%`, `d`, `x`, `b`);
- знаки (`+`, `-`, ` `) и разделители (`_`, `,`).

Если ваш объект **оборачивает** стандартный числовой тип или строку, самый простой и правильный способ — делегировать форматирование этому типу:

```python
class Percent:
    def __init__(self, value):
        self.value = float(value)  # 0.0–1.0

    def __format__(self, spec):
        # добавим, что по умолчанию хотим видеть "%"
        if not spec:
            spec = ".2%"
        return format(self.value, spec)
```

Так вы не переписываете язык форматов, а лишь задаёте **дефолт** или дополнительные соглашения.

Если наоборот, `format_spec` должен управлять **несколькими аспектами** отображения (например, и точность, и единицы измерения), удобно:

- договориться о простом внутреннем синтаксисе (например, `"km:.2f"` означает «показать в километрах с двумя знаками после запятой»);
- внутри `__format__` разобрать строку `format_spec` на части (можно через `str.partition`, `split`, простые разборы).

### `__bytes__`

Метод `__bytes__(self)` определяет поведение `bytes(obj)`. Полезен, если объект имеет **естественное байтовое представление** (протокол, зашифрованные данные, бинарный формат).

```python
class Packet:
    def __init__(self, payload: bytes):
        self.payload = payload

    def __bytes__(self):
        # пример: простой префикс длины
        length = len(self.payload).to_bytes(2, "big")
        return length + self.payload
```

При проектировании `__bytes__` полезно задать себе вопросы:

- Это **стабильный двоичный формат**, который вы сможете поддерживать в будущем?
- Должен ли он совпадать с форматом, который вы используете при сериализации/отправке по сети?
- Нужны ли вам отдельные методы `to_bytes()` / `from_bytes()`, чтобы не жёстко привязывать протокол к `__bytes__`?

#### Рекурсивные структуры и защита от бесконечного `__repr__`

Если ваш объект содержит ссылки на другие объекты, которые в свою очередь могут ссылаться **обратно**, наивный `__repr__` легко уходит в рекурсию:

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None

    def __repr__(self):
        return f"Node({self.value!r}, next={self.next!r})"

a = Node("a")
b = Node("b")
a.next = b
b.next = a   # цикл

repr(a)  # RecursionError
```

В таких случаях:

- либо ограничивайте глубину вывода (например, печатать только несколько уровней и дальше ставить `...`);
- либо выводите только **идентификаторы** связанных объектов (`id`, ключ, имя), а не целиком всю структуру.

Упрощённый безопасный подход:

```python
class SafeNode:
    def __init__(self, value):
        self.value = value
        self.next = None

    def __repr__(self):
        cls = self.__class__.__name__
        nxt = "..." if self.next is not None else "None"
        return f"{cls}(value={self.value!r}, next={nxt})"
```

Для сложных структур (графы, деревья, ORM-объекты с отношениями) часто полезно иметь:

- краткий `__repr__`, не пытающийся вывести всю структуру;
- отдельные функции/утилиты для «глубокой» печати/отладки (например, `debug_dump_tree(node)`).
