# Часть XVI. Тонкости и «подводные камни»

> **Цель:** научиться **избегать типичных ловушек Python**, понимать **как профилировать и отлаживать** код, знать основы **безопасности** и **проверки зависимостей**, чтобы:
>
> - не «стрелять себе в ногу» мутабельными аргументами по умолчанию, late binding в замыканиях и другими классическими подводными камнями;
> - осознанно подходить к производительности: знать, когда профилировать, как читать cProfile, когда использовать PyPy/Cython;
> - уверенно отлаживать код через pdb и понимать, как работают breakpoint, trace/profile хуки;
> - понимать риски SQL injection, pickle, eval/exec и уметь применять инструменты проверки (Bandit, pip-audit, ast.literal_eval);
> - знать о ReDoS и как защищаться от катастрофического backtracking в регулярных выражениях.
>
> Всё — **простым языком**, с множеством примеров, теоретическими пояснениями и устоявшейся терминологией.

---

## Общая картина: зачем знать тонкости и подводные камни

Python называют языком, где «легко писать, легко читать». Но под этой простотой скрываются **тонкости**, которые могут привести к неочевидным багам, уязвимостям и сложностям при отладке. Эта часть даёт **карту подводных камней** и инструменты, чтобы их избегать.

| Что может пойти не так | Примеры |
|------------------------|---------|
| **Неожиданное поведение** | Функция «помнит» состояние между вызовами (mutable default); замыкание в цикле возвращает одно и то же значение (late binding) |
| **Медленный или нестабильный код** | Изменение списка во время итерации — пропуск элементов; `is` вместо `==` для чисел/строк — может работать «случайно», а потом сломаться |
| **Уязвимости** | SQL injection — подстановка пользовательского ввода в запросы; pickle из ненадёжных источников — RCE; ReDoS — «подвисание» на злонамеренном вводе в регулярные выражения |

После изучения этой части вы будете знать, как всего этого избежать и как эффективно отлаживать и профилировать код.

## Краткая шпаргалка

| Понятие | API / суть |
|---------|------------|
| **Mutable default** | `def f(a=None): a = a or []` или `if a is None: a = []` — не `def f(a=[])` |
| **Late binding** | `lambda x=x: ...` — привязка по значению в цикле |
| **Проверка None** | Только `x is None`; для значений — `==` |
| **Изменение в цикле** | list: `lst[:]` или новый список; dict/set: `list(d)` / `list(s)` или comprehension |
| **cProfile** | `cProfile.run('func()')`, `pstats.Stats` |
| **timeit** | `timeit.Timer('code', setup='...').timeit(number=N)` |
| **tracemalloc** | `start()`, `get_traced_memory()`, `take_snapshot()` |
| **pdb** | `breakpoint()`, `n`/`s`/`c`/`l`/`p`/`w` |
| **PYTHONBREAKPOINT** | `=0` — отключить; `=ipdb.set_trace` — свой отладчик |
| **SQL injection** | Параметризованные запросы: `?`, `%s`, `:name`. Не склеивать строки с вводом. |
| **pickle** | Не загружать из ненадёжных источников |
| **eval/exec** | Опасно с пользовательским вводом; для литералов — `ast.literal_eval` |
| **secrets** | `token_hex()`, `token_urlsafe()` — вместо random |
| **Bandit, pip-audit** | Статический анализ, проверка CVE |
| **ReDoS** | Избегать вложенных квантификаторов; ограничить длину ввода |
| **else в цикле** | Выполняется, если цикл завершился без `break` |
| **else в try** | Выполняется, если не было исключения |

## Маршрут изучения

1. **Блок 2 (§88)** — изучить известные ловушки: mutable default, late binding, изменение в цикле (list, dict, set — у dict/set RuntimeError, у list — пропуск элементов), `is` vs `==`, интернирование, цепочки or/and, assert, рекурсия, классовые переменные, float и == (использовать `math.isclose`), мутабельные объекты как ключи dict (только hashable). Без этого легко «стрелять себе в ногу».
2. **Блок 3 (§89, §89a)** — освоить профилирование (cProfile, timeit, tracemalloc) и отладку (pdb, breakpoint). Уметь измерять, а не гадать.
3. **Блок 4 (§90)** — понять риски безопасности: SQL injection, pickle, eval/exec, и инструменты проверки (Bandit, pip-audit).
4. **Блок 5 (§90b)** — дополнительные тонкости: bound method, truthiness, else в цикле/try, scope comprehensions.

**Желательно знать заранее:** базовый синтаксис Python (Часть I), функции и области видимости (LEGB), основы ООП.

## Как изучать эту часть

**Минимальный практический минимум** — если нужно быстро избежать основных ловушек:
- Mutable default: `def f(a=None): a = a or []` — не `def f(a=[])`.
- Late binding: `lambda x=x: ...` — привязка по значению в цикле.
- Изменение коллекций при итерации: для list — `lst[:]`; для dict/set — `list(d)` или comprehension.
- Проверка None: только `x is None`; для значений — `==`.

**Базовый уровень** — после минимума: все ловушки §88, cProfile/timeit, pdb, SQL injection, pickle/eval риски, Bandit/pip-audit.

**Углублённый уровень** — для продвинутой практики: tracemalloc, py-spy/memray/scalene, settrace/setprofile, sys.audit, ReDoS, marshal/shelve.
