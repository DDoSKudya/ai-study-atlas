## §17. Распаковка и упаковка

**Цель раздела:** присваивать несколько имён из одной последовательности; передавать аргументы через `*` и `**`; использовать распаковку в `for` и в литералах; понимать ValueError при несовпадении числа элементов.

| Термин | Определение |
|--------|-------------|
| **Unpacking (распаковка)** | Присваивание нескольких имён из одной последовательности за один раз |
| **Packing (упаковка)** | Сбор «остатка» в переменную через `*` (последовательность) или `**` (словарь) |
| **Target list (целевой список)** | Левая часть присваивания или заголовок `for` — куда связываются значения |
| **`*` и `**`** | В присваивании — один остаток; в вызове — развернуть в аргументы; в def — собрать в tuple/dict |

**Суть:** `a, *rest = seq` — первый элемент в `a`, остальные в список `rest`; `f(*args, **kwargs)` — развернуть в аргументы; в одном присваивании только одна звёздочка.

---

**Распаковка (unpacking)** — присваивание нескольких имён из одной последовательности или отображения за один раз: левая часть присваивания содержит несколько целей (имён или подшаблонов), правая — одна итерабельная последовательность или отображение; интерпретатор сопоставляет элементы с целями по порядку. **Упаковка (packing)** — сбор «остатка» элементов в одну переменную с помощью **`*`** (в последовательностях) или **`**`** (в вызовах и словарях). Синтаксис **`\*`** и **`**`** единообразно используется в **присваиваниях**, в **вызовах функций** и в **литералах** списков, кортежей, множеств и словарей — это один из краеугольных камней «питонического» стиля.

**Терминология (кратко):**

- **Target list (целевой список)** — левая часть присваивания или заголовок **for**: перечень имён и/или подшаблонов, которым присваиваются значения.
- **Iterable (итерабельный объект)** — объект, по которому можно пройти (список, кортеж, диапазон, строка, ключи словаря и т.д.).
- **Unpacking в вызове** — использование **`*iterable`** или **`**dict`\*\* в вызове функции: элементы передаются как отдельные аргументы.

**Где используются `*` и `**` (сводная таблица):\*\*

| Контекст               | Синтаксис                | Назначение                                                                     |
| ---------------------- | ------------------------ | ------------------------------------------------------------------------------ |
| Присваивание           | `a, *rest = seq`         | Связать имена с элементами; `*rest` получает список «остатка»                  |
| Вызов функции          | `f(*args, **kwargs)`     | Развернуть последовательность в позиционные аргументы, словарь — в именованные |
| Литерал списка/кортежа | `[*a, *b, 1]`            | Включить элементы итерабельных объектов в новую последовательность             |
| Литерал словаря        | `{**d1, 'k': v}`         | Включить пары ключ-значение из словаря (и переопределить ключи)                |
| Параметр функции       | `def f(*args, **kwargs)` | Собрать лишние позиционные аргументы в кортеж, именованные — в словарь         |

### 17.1 Простое присваивание кортежа/списка

Несколько имён можно присвоить из одной итерабельной последовательности (кортеж, список и т.д.): количество имён должно совпадать с количеством элементов (или использоваться `*rest` — см. ниже).

```python
a, b = 1, 2
# Справа — кортеж (1, 2), слева — имена a, b
# a == 1, b == 2

x, y, z = [10, 20, 30]
# x==10, y==20, z==30

# Обмен значений без временной переменной
a, b = b, a
```

**Терминология:** справа — **итерабельный объект** (iterable), слева — **целевой список** (target list). Присваивание выполняется слева направо: сначала вычисляется правая часть (одна последовательность), затем элементы по порядку связываются с именами слева.

**Пошагово: что происходит при a, b, c = [10, 20, 30].**

1. Правая часть **`[10, 20, 30]`** вычисляется — получается список из трёх элементов.
2. Интерпретатор проверяет, что число целей слева (три: **a**, **b**, **c**) совпадает с числом элементов справа (три). Если бы не совпадало и не было звёздочки — **ValueError**.
3. Присваивание выполняется по порядку: **a = 10**, **b = 20**, **c = 30**. Важно: это не «присвоить кортеж», а три отдельных присваивания в одном операторе; справа может быть любой итерабельный объект, но он будет полностью исчерпан.
4. Если справа итератор, то после присваивания он исчерпан — повторная итерация по нему не даст элементов.

**Почему это важно на практике:** обмен значений **a, b = b, a** выполняется без временной переменной и без риска ошибиться; распаковка возврата функции **x, y = parse_pair()** делает намерение явным. Распаковка в **for** (**for key, value in d.items()**) избавляет от индексов и обращений по ключу внутри цикла.

### 17.2 Звёздочка в присваивании: a, \*rest = seq

Одна переменная в целевом списке может быть помечена **`*`** (звёздочка). Ей присваивается **список** из всех «лишних» элементов последовательности. Остальные имена получают по одному элементу в порядке слева направо.

```python
first, *rest = [1, 2, 3, 4, 5]
# first == 1, rest == [2, 3, 4, 5]

head, *middle, tail = [1, 2, 3, 4]
# head == 1, middle == [2, 3], tail == 4

*begin, last = [10, 20, 30]
# begin == [10, 20], last == 30
```

**Правила:**

- В одном целевом списке может быть не более **одной** звёздочки.
- Переменной со звёздочкой может достаться пустой список, если элементов ровно столько, сколько «обычных» имён: `a, *r = [1]` → `a==1`, `r==[]`.
- Переменная со звёздочкой всегда получает тип **list**, даже если справа кортеж или другой итератор.

```python
a, *b = (1, 2, 3)
# b — список: [2, 3]
```

### 17.3 Распаковка в вызовах функций: \*args и \*\*kwargs

В **вызове** функции одна звёздочка **`*`** распаковывает итерабельный объект в **позиционные** аргументы, две звёздочки **`**`** распаковывают словарь в **именованные\*\* аргументы (ключи — имена, значения — аргументы).

```python
def f(a, b, c):
    return a + b + c

# Распаковка последовательности в позиционные аргументы
args = [1, 2, 3]
print(f(*args))        # 6

# Распаковка словаря в именованные аргументы
kwargs = {'a': 10, 'b': 20, 'c': 30}
print(f(**kwargs))     # 60

# Комбинация
print(f(1, *[2, 3]))           # 6
print(f(1, 2, **{'c': 3}))     # 6
```

**Пошагово: что происходит при f(\*[1, 2, 3]).**

1. Интерпретатор видит вызов `f` с одним аргументом `*[1, 2, 3]`.
2. Выражение `*[1, 2, 3]` в контексте вызова означает: взять элементы итерабельного объекта и передать их как отдельные позиционные аргументы.
3. Вызов эквивалентен `f(1, 2, 3)`.
4. Параметры `a`, `b`, `c` получают значения 1, 2, 3.

Итог: **`*iterable`** в вызове «разворачивает» итерабельный объект в аргументы.

**Порядок в вызове:** позиционные аргументы (включая распакованные из `*...`) должны идти до именованных; словарь `**d` обычно ставят в конце. Комбинация `f(*args, **kwargs)` — стандартный приём для передачи аргументов «как есть» в другую функцию.

### 17.4 Распаковка в цикле for

В заголовке `for` целевой список может быть таким же, как в присваивании: несколько имён, при необходимости со звёздочкой.

```python
pairs = [(1, 'a'), (2, 'b'), (3, 'c')]
for num, letter in pairs:
    print(num, letter)
# 1 a
# 2 b
# 3 c

# Словарь: .items() даёт пары (key, value)
d = {'x': 1, 'y': 2}
for k, v in d.items():
    print(k, v)
```

Такой стиль делает код читаемым и избавляет от индексов.

**Граничный случай: звёздочка в for.** В заголовке **for** можно использовать одну переменную со звёздочкой: **for first, \*rest in [(1, 2, 3), (4, 5), (6,)]:** — на каждой итерации **first** получает первый элемент кортежа, **rest** — список остальных: **(1, [2, 3])**, **(4, [5])**, **(6, [])**. Количество элементов в каждом элементе итерации может различаться; переменная со звёздочкой «собирает» всё лишнее в список. Если бы элементов не хватило для **first** (например, пустой кортеж **()**), получили бы **ValueError**. В **for** допускается не более одной звёздочки в целевом списке — как и в присваивании.

**Пошагово: что происходит при for k, v in d.items():**

1. **d.items()** возвращает представление пар (ключ, значение) словаря **d**.
2. На каждой итерации из представления извлекается одна пара (кортеж из двух элементов).
3. Целевой список **k, v** распаковывает этот кортеж: **k** получает ключ, **v** — значение.
4. В теле цикла доступны **k** и **v** для текущей пары.

Итог: перебор словаря без обращения **d[k]** и без индексов. Аналогично **for i, x in enumerate(seq):** — **i** получает индекс, **x** — элемент.

### 17.5 Распаковка словарей при создании словаря

В литерале словаря можно «влить» другой словарь с помощью **`**dict`\*\*:

```python
d1 = {'a': 1, 'b': 2}
d2 = {'c': 3, 'd': 4}
merged = {**d1, **d2}
# {'a': 1, 'b': 2, 'c': 3, 'd': 4}

# Добавление или переопределение ключей
merged = {**d1, 'b': 20, **d2}
# 'b' перезаписан значением 20; затем ключи из d2
```

В **Python 3.9+** для слияния словарей можно также использовать операторы **`|`** и **`|=`**: `d1 | d2`, `d1 |= d2` (см. план Часть I, §2.2).

**Пошагово: что происходит при {**d1, 'b': 20, **d2}.** (1) Интерпретатор строит новый словарь; сначала обрабатываются все пары до **\*\*d2**. (2) **\*\*d1** «вливает» пары из **d1** в новый словарь (ключ за ключом). (3) Литерал **'b': 20** добавляет или перезаписывает ключ **'b'** значением **20**. (4) **\*\*d2** добавляет пары из **d2**; если ключ уже есть (например, **'b'** в **d2**), значение перезаписывается значением из **d2**. (5) Итоговый словарь возвращается. Порядок ключей в Python 3.7+ сохраняет порядок вставки: сначала ключи из **d1**, затем **'b'**, затем ключи из **d2** (при совпадении ключа побеждает последнее значение).

#### Граничные случаи распаковки словарей

- **Ключи не строки:** при **\*\*распаковке** в вызове функции или в литерале словаря ключи должны быть строками. **{1: 'a', **{2: 'b'}}** вызовет **TypeError** (ключ 2 не строка). В вызове **f(**{1: 'a'})** тоже **TypeError** — имена аргументов должны быть строками.
- **Пустой словарь:** **{**d}** при пустом **d** даёт пустой словарь; **{**d1, **d2}\*\* при двух пустых — пустой словарь.

#### Пример: передача аргументов «как есть» в обёртку

```python
def log_call(func):
    def wrapper(*args, **kwargs):
        print("Вызов", func.__name__, "с", args, kwargs)
        return func(*args, **kwargs)
    return wrapper

@log_call
def add(a, b):
    return a + b
add(1, 2)           # Вызов add с (1, 2) {}
add(1, b=2)         # Вызов add с (1,) {'b': 2}
```

### 17.6 Распаковка в литералах списков

В списке можно распаковать итерабельный объект через **`*iterable`**:

```python
a = [1, 2, 3]
b = [*a, 4, 5]
# b == [1, 2, 3, 4, 5]
```

Это удобно для объединения списков без изменения исходных: `[*list1, *list2]`.

#### Распаковка в литералах кортежа и множества

В **кортеже** и **множестве** тоже можно использовать **`*iterable`** для включения элементов итерабельного объекта без явного цикла:

```python
a = (1, 2)
b = (*a, 3, 4)       # (1, 2, 3, 4)
c = {*a, 5, 6}       # {1, 2, 5, 6} — множество

# Объединение множеств без изменения исходных
s1, s2 = {1, 2}, {2, 3}
union = {*s1, *s2}   # {1, 2, 3}
```

В кортеже **(\*a, 3)** создаётся новый кортеж: сначала распаковываются элементы **a**, затем добавляется **3**. В множестве **{*s1, *s2}** получается объединение множеств (все уникальные элементы). Обратите внимание: **{*s1, *s2}** даёт то же множество, что и **s1 | s2** (Python 3.9+), но синтаксис **\*** работает и в более старых версиях.

### 17.7 Теоретическое обоснование

- **Унификация:** один и тот же синтаксис `*`/`**` в присваивании, в вызовах и в литералах — меньше правил для запоминания.
- **Читаемость:** `for k, v in d.items():` яснее, чем перебор по индексам или по ключам с последующим взятием значения.
- **Гибкость API:** функции могут принимать `*args` и `**kwargs`, а вызывающий код передаёт списки и словари через `*` и `**`.

### 17.8 Граничные случаи

- **Несовпадение числа элементов:** `a, b = [1, 2, 3]` → **ValueError** (слишком много значений). `a, b, c = [1, 2]` → **ValueError** (недостаточно значений). Исключение — когда есть переменная со звёздочкой, которая «собирает» остаток.
- **Одна переменная со звёздочкой:** `*x = [1, 2, 3]` в Python 3 допустимо: `x == [1, 2, 3]`. Одиночная звёздочка без имени в присваивании недопустима.
- **В вызовах:** `**` распаковывает только словари с ключами-строками; имена ключей должны совпадать с именами параметров функции (или попадать в `**kwargs`).
- **Пустая последовательность:** `a, *b = []` — для **a** элемента нет → **ValueError**. `*b = []` допустимо: **b == []**.
- **Один элемент и звёздочка:** `a, *b = [1]` → **a == 1**, **b == []**; переменная со звёздочкой получает пустой список.

#### Параметры \*args и \*\*kwargs в определении функции

В **определении** функции **`*args`** собирает все «лишние» позиционные аргументы в кортеж **args**, **`**kwargs`** — все лишние именованные аргументы в словарь **kwargs**. Имена **args** и **kwargs** — соглашение; важны сами **`\*`** и **`**`**.

```python
def f(a, b, *args, **kwargs):
    print("a =", a, "b =", b, "args =", args, "kwargs =", kwargs)

f(1, 2, 3, 4, x=10, y=20)
# a = 1 b = 2 args = (3, 4) kwargs = {'x': 10, 'y': 20}
```

**Порядок параметров в def:** позиционные (обычные), затем **`*args`** или один **`*`** (за ним только keyword-only), затем keyword-only, затем **`**kwargs`**. После **`\*`\*\* все параметры можно передавать только по имени.

**Пошагово: что происходит при вызове f(1, 2, \*[3, 4], **{'x': 10}).** (1) Интерпретатор собирает все аргументы: сначала позиционные **1**, **2**, затем распаковка **\*[3, 4]** даёт позиционные **3**, **4**, затем **\*\*{'x': 10}** даёт именованный **x=10**. (2) Функция **f** получает **a=1**, **b=2**, **args=(3, 4)**, **kwargs={'x': 10}** (если сигнатура **def f(a, b, \*args, **kwargs)**). (3) Имена ключей словаря при **\*\*** должны быть строками и совпадать с именами параметров (или попадать в **kwargs**); иначе **TypeError**. (4) Порядок важен: сначала все позиционные (включая из **\*seq**), затем все именованные (включая из **\*\*dict**); один и тот же аргумент нельзя передать и позиционно, и по имени.

#### Плохо vs лучше (распаковка)

```python
# Плохо: индексация в цикле — легко ошибиться индексом
for i in range(len(pairs)):
    print(pairs[i][0], pairs[i][1])

# Лучше: распаковка в for
for a, b in pairs:
    print(a, b)

# Плохо: временная переменная для обмена
tmp = a
a = b
b = tmp

# Лучше: распаковка
a, b = b, a

# Плохо: передача списка как одного аргумента (если функция ждёт отдельные аргументы)
func(my_list)   # func получит один аргумент — список

# Лучше: распаковка в вызове
func(*my_list)
```

- **Порядок при слиянии словарей:** **`{**d1, **d2}`** — сначала применяются пары из **d1**, затем из **d2**; при совпадении ключей побеждает значение из **d2**. То же для **`{**d1, 'x': 1, **d2}`**: ключ **'x'** из литерала перезапишет **d1['x']** при наличии; затем ключи из **d2** перезапишут при совпадении.

#### Пошагово: что происходит при first, \*rest = [1, 2, 3, 4]

(1) Правая часть **`[1, 2, 3, 4]`** — итерабельный объект из четырёх элементов. (2) В целевом списке одна переменная со звёздочкой (**\*rest**). Интерпретатор отводит **первый** элемент последовательности первой «обычной» цели (**first**), а **все остальные** элементы собирает в **список** и присваивает **rest**. (3) Итог: **first = 1**, **rest = [2, 3, 4]**. Важно: переменная со звёздочкой **всегда** получает тип **list**, даже если справа был кортеж или **range** — это гарантировано языком. (4) Если бы элементов было ровно один: **first, \*rest = [1]** → **first = 1**, **rest = []** (пустой список). Если бы элементов не хватило для обязательных целей (например, **a, b, \*c = [1]**): **a = 1**, **b** не получает значения — **ValueError**, потому что для **b** элемента нет. Звёздочка может «собрать» ноль элементов, но не может компенсировать отсутствие элемента для обязательной цели.

#### Пример: разбор строки «ключ=значение» с распаковкой

```python
# Один разделитель — partition даёт ровно три части
key, _, value = "name=Alice".partition("=")   # key="name", value="Alice"

# Несколько пар в списке
pairs = ["a=1", "b=2", "c=3"]
for item in pairs:
    k, _, v = item.partition("=")
    print(k, v)
```

**Запомните (§17):** в одном присваивании только одна звёздочка; `*rest` всегда даёт список; в вызове `*` — позиционные аргументы, `**` — именованные; ключи при `**` должны быть строками; при несовпадении числа элементов без `*` — ValueError.
