## §18. match/case (сопоставление с образцом) (Python 3.10+)

**Цель раздела:** писать `match`/`case`; сопоставлять литералы, последовательности, словари и классы; использовать guard и wildcard; понимать порядок веток и роль `__match_args__`.

| Термин | Определение |
|--------|-------------|
| **Subject (подвергаемое)** | Выражение после `match`, по которому идёт сопоставление |
| **Pattern (образец)** | Шаблон в `case` — литерал, последовательность, класс, `as`, `\|` |
| **Guard** | Условие `if ...` в `case`; проверяется после успешного сопоставления образца |
| **Wildcard `_`** | Образец «любое значение»; обычно ставят последним |
| **`__match_args__`** | Кортеж имён атрибутов для позиционного образца класса |

**Суть:** `match x:` — по порядку проверяются образцы в `case`; первый совпавший выполняется; `case _:` — ловит всё; для классов с позиционным образцом нужен `__match_args__`.

---

**Сопоставление с образцом (pattern matching)** — конструкция **`match`** / **`case`**, введённая в **Python 3.10** (PEP 634 — Specification, PEP 635 — Motivation, PEP 636 — Tutorial). По значению выражения после **`match`** по очереди проверяются образцы в ветках **`case`**; первая совпавшая ветка выполняется, остальные не проверяются. Это альтернатива длинным цепочкам **`if/elif`** и особенно удобно при разборе **структурированных данных** (списки, словари, объекты с атрибутами), когда нужно проверить и форму, и извлечь части значения в переменные.

**Терминология:**

- **Subject (подвергаемое)** — значение выражения после **match**; именно оно сопоставляется с образцами.
- **Pattern (образец)** — то, что записано после **case**: литерал, последовательность образцов, отображение, тип с подобразцом, имя переменной, **\_** (wildcard) и комбинации через **|** или **as**.
- **Binding (привязка)** — при успешном сопоставлении имена в образце получают соответствующие части значения и видны в теле ветки **case**.
- **Guard (охранное условие)** — выражение **if условие** после образца; ветка выполняется только если и образец совпал, и условие истинно.

**Почему ввели match/case:** в функциональных языках (Haskell, ML, Rust) сопоставление с образцом — базовый способ разбора вариантов и структур; в Python до 3.10 писали длинные **if/elif** с **isinstance** и проверкой полей. Это засоряло код и не давало гарантии полноты вариантов. **match/case** позволяет записать варианты линейно, извлекать вложенные значения в переменные одной строкой и в связке с **assert_never** (см. §19c) — проверять исчерпывающий разбор статическим анализатором.

### 18.1 Базовый синтаксис

```python
match выражение:
    case образец_1:
        тело_1
    case образец_2:
        тело_2
    ...
```

Сначала вычисляется **выражение** после `match`. Затем по порядку проверяются образцы в **`case`**. Первый образец, с которым значение **сопоставилось**, выполняет своё тело; остальные ветки не проверяются. Если ни один образец не подошёл и нет **`case _:`**, выполнение выходит из блока **match** без входа ни в одну ветку (исключение не возбуждается); поэтому для обработки «все остальные случаи» в конце обычно ставят **`case _:`**.

**Пошагово: что происходит при выполнении match x: ... case 1: ... case 2: ... case \_: ...**

1. Вычисляется **x** — получается **subject** (например, значение **2**).
2. Проверяется **case 1:** — сопоставление с литералом **1** идёт через **==**; **2 == 1** ложно → образец не совпал.
3. Проверяется **case 2:** — **2 == 2** истинно → образец совпал.
4. Выполняется тело этой ветки; остальные **case** не проверяются.
5. После тела выполнение продолжается после всего блока **match**.

Если бы ни один образец не совпал и не было **case \_:**, после проверки всех веток выполнение также продолжилось бы после **match** (без входа в какую-либо ветку).

**Таблица типов образцов (кратко):**

| Тип образца        | Пример                              | Когда совпадает                                                       |
| ------------------ | ----------------------------------- | --------------------------------------------------------------------- |
| Литерал            | `case 1:` `case "ok":` `case None:` | Значение равно литералу (==)                                          |
| Переменная (имя)   | `case x:`                           | Всегда; в **x** попадает subject                                      |
| Wildcard           | `case _:`                           | Всегда; значение не сохраняется                                       |
| Последовательность | `case [a, b]:` `case (x, *rest):`   | Subject — последовательность, длина и элементы совпадают              |
| Отображение        | `case {"key": val}:`                | Subject — dict, есть ключ, значение по ключу совпадает с подобразцом  |
| Тип/конструктор    | `case int(n):` `case Point(x, y):`  | Subject — экземпляр типа; атрибуты/поля сопоставляются с подобразцами |
| OR                 | `case 1 \| 2 \| 3:`                 | Хотя бы один из образцов совпадает                                    |
| AS                 | `case [a, b] as pair:`              | Образец совпадает; **pair** — всё сопоставленное значение             |
| Guard              | `case x if x > 0:`                  | Образец совпал и условие после **if** истинно                         |

**Краткий справочник: синтаксис образцов**

| Что сопоставить                | Образец                      | Пример subject                      |
| ------------------------------ | ---------------------------- | ----------------------------------- |
| Конкретное число               | `case 42:`                   | `42`                                |
| Конкретная строка              | `case "ok":`                 | `"ok"`                              |
| None / True / False            | `case None:`                 | `None`                              |
| Любое значение (не сохранять)  | `case _:`                    | что угодно                          |
| Любое значение (сохранить в x) | `case x:`                    | что угодно → **x**                  |
| Список ровно из 2 элементов    | `case [a, b]:`               | `[1, 2]` → **a=1, b=2**             |
| Список: первый + остаток       | `case [first, *rest]:`       | `[1,2,3]` → **first=1, rest=[2,3]** |
| Словарь с ключом "k"           | `case {"k": v}:`             | `{"k": 10}` → **v=10**              |
| Тип (без извлечения)           | `case int():`                | любое целое                         |
| Тип (извлечь в n)              | `case int(n):`               | `42` → **n=42**                     |
| Несколько вариантов (OR)       | `case 1 \| 2 \| 3:`          | 1, 2 или 3                          |
| Образец + условие              | `case int(n) if n > 0:`      | целое и положительное               |
| Подшаблон как целое            | `case [a, [b, c] as inner]:` | **inner** = `[b, c]`                |

**Пошагово: что происходит при match [1, 2, 3]: case [first, *rest]: ...** (1) Subject — ** [1, 2, 3]** (список из трёх элементов). (2) Образец ** [first, *rest]** требует: subject — последовательность, первый элемент сопоставляется с **first**, остальные — собираются в список для **rest**. (3) Проверка: subject поддерживает **len** и итерацию; первый элемент **1** → **first = 1**; остальные ** [2, 3]** → **rest = [2, 3]** (гарантированно list). (4) Образец совпал; выполняется тело ветки, в нём доступны **first** и **rest**. Если бы subject был **[]**, образец не совпал бы (для **first** элемента нет). Если subject ** [x]**: **first = x**, **rest = []**.

### 18.2 Сопоставление литералов

Образец может быть литералом: число, строка, `True`, `False`, `None`. Сопоставление идёт по равенству (`==`).

```python
def http_status(code):
    match code:
        case 200:
            return "OK"
        case 404:
            return "Not Found"
        case 500:
            return "Server Error"
        case _:
            return "Unknown"

print(http_status(200))   # OK
print(http_status(404))   # Not Found
print(http_status(999))   # Unknown
```

**`case _:`** — **wildcard** (универсальный образец): сопоставляется с любым значением. Обычно ставят последним как «всё остальное».

**Плохо vs лучше (match vs if/elif для литералов):**

```python
# Плохо: длинная цепочка if/elif для кодов — много повторений "code =="
def status(code):
    if code == 200:
        return "OK"
    elif code == 404:
        return "Not Found"
    elif code == 500:
        return "Server Error"
    else:
        return "Unknown"

# Лучше: match/case — варианты перечислены линейно, без повторения subject
def status(code):
    match code:
        case 200:
            return "OK"
        case 404:
            return "Not Found"
        case 500:
            return "Server Error"
        case _:
            return "Unknown"
```

Для разбора по **структуре** (список, словарь, атрибуты) **match** даёт ещё больше выигрыша: извлечение частей в переменные в одном образце вместо нескольких строк с **isinstance** и индексацией.

### 18.3 Сопоставление последовательностей (последовательность с элементами)

Образец в виде последовательности (список, кортеж и т.д.) сопоставляется с последовательностью той же длины; внутри могут быть подобразцы и **`*rest`** для «хвоста».

```python
match [1, 2, 3]:
    case [a, b, c]:
        print(a, b, c)   # 1 2 3

match [1, 2, 3, 4]:
    case [first, *rest]:
        print(first, rest)   # 1 [2, 3, 4]
```

Имена в образце (**a**, **b**, **first**, **rest**) — **переменные**, которым присваиваются соответствующие части значения (binding). Они видны только в теле этого `case`.

### 18.4 Сопоставление отображений (словари)

Образец вида **`{ключ: образец, ...}`** сопоставляется со словарём: у subject должны быть **все** указанные ключи, и значение по каждому ключу сопоставляется с соответствующим подобразцом. Дополнительные ключи в словаре не мешают (образец проверяет наличие и совпадение только указанных). Ключи в образце должны быть литералами (строки, числа и т.д.), не переменными.

```python
def handle_config(config):
    match config:
        case {"host": str(host), "port": int(port)}:
            print(f"Подключение к {host}:{port}")
        case {"debug": True}:
            print("Режим отладки")
        case {}:
            print("Пустая конфигурация")
        case _:
            print("Неизвестная конфигурация")
```

**Пошагово: сопоставление {"host": str(host), "port": int(port)} с {"host": "localhost", "port": 8080}.** (1) Subject — словарь с ключами **"host"** и **"port"**. (2) Проверяется наличие ключа **"host"** и сопоставление значения **"localhost"** с образцом **str(host)** — **str** с одной переменной сопоставляет любую строку и связывает её с **host**; **host = "localhost"**. (3) Аналогично **"port"** и **int(port)** — **port = 8080**. (4) Все подобразцы совпали — сопоставление успешно; в теле доступны **host** и **port**.

### 18.5 Образец «as» — захват подшаблона

К образцу можно добавить **`as имя`**: при успешном сопоставлении в это имя попадёт всё сопоставленное значение (или его часть, в зависимости от контекста). Удобно, когда нужно и проверить структуру, и использовать целое значение.

```python
match [1, [2, 3]]:
    case [a, [b, c] as inner]:
        print(a, inner)   # 1 [2, 3]
        print(b, c)       # 2 3
```

### 18.6 Образец «|» (OR)

Несколько образцов можно объединить через **`|`**: сопоставление успешно, если подошёл хотя бы один из них.

```python
match value:
    case 0 | 1 | 2:
        print("малое число")
    case str() | bytes():
        print("строка или байты")
```

### 18.7 Guard (охранное условие)

После образца можно записать **`if условие`**: ветка выполняется только если и образец совпал, и условие истинно. Условие может использовать имена, связанные в этом образце.

```python
match x:
    case int(n) if n > 0:
        print("положительное целое:", n)
    case int(n) if n < 0:
        print("отрицательное целое:", n)
    case int():
        print("ноль")
```

**Пошагово: когда вычисляется guard.** (1) Сначала проверяется образец: subject сопоставляется с **int(n)** — если **x** не **int**, образец не совпадает и guard не вычисляется. (2) Если образец совпал, в **n** привязано значение **x** (как целое). (3) Затем вычисляется условие **if n > 0** в контексте этой ветки — **n** уже доступна. (4) Если условие истинно, выполняется тело ветки; если ложно, образец считается «не совпавшим» для этой ветки и проверяется следующий **case**. Итог: guard вычисляется **только после** успешного сопоставления образца и может использовать переменные, привязанные в образце. Порядок важен: **case int(n) if n > 0:** — сначала проверка типа и привязка **n**, затем проверка **n > 0**.

### 18.8 Сопоставление с типом и конструктором

Образец вида **`тип(подобразец)`** проверяет, что значение является экземпляром **тип**, и сопоставляет атрибуты (для типов с **`__match_args__`**) или подобразец с самим значением. Для встроенных типов (**int**, **str**, **list**, и т.д.) скобки могут быть пустыми: **`int()`**, **`str()`** — проверка только типа.

```python
match value:
    case int():
        print("целое число")
    case str(s):
        print("строка:", s)
    case list([a, b]):
        print("список из двух элементов:", a, b)
```

### 18.9 **match_args** для классов

Для пользовательских классов порядок атрибутов, используемых при **позиционном** сопоставлении (например, **`Point(x, y)`**), задаётся атрибутом класса **`__match_args__`** — кортеж имён атрибутов.

```python
class Point:
    __match_args__ = ("x", "y")
    def __init__(self, x, y):
        self.x = x
        self.y = y

def describe(p):
    match p:
        case Point(0, 0):
            return "начало координат"
        case Point(x, 0):
            return f"на оси X: {x}"
        case Point(0, y):
            return f"на оси Y: {y}"
        case Point(x, y):
            return f"точка ({x}, {y})"
```

Без **`__match_args__`** сопоставление с **Point(...)** возможно только по **именованным атрибутам**: образец **Point(x=0, y=0)** сопоставляется с объектом, у которого есть атрибуты **x** и **y** и их значения совпадают с образцом. Позиционный образец **Point(0, 0)** без ****match_args**** не поддерживается — интерпретатор не знает, какой аргумент образца соответствует какому атрибуту. С **`__match_args__ = ("x", "y")`** интерпретатор сопоставляет первый аргумент образца с **p.x**, второй с **p.y**.

**Пошагово: что происходит при match p: case Point(3, 4): при p = Point(3, 4).** (1) Subject — объект **p** типа **Point**. (2) Образец **Point(3, 4)** проверяет: subject является экземпляром **Point**; класс **Point** имеет ****match_args** = ("x", "y")**, поэтому первый аргумент образца (**3**) сопоставляется с **getattr(p, "x")** → **p.x == 3**; второй аргумент (**4**) — с **p.y == 4**. (3) Сравнение идёт по **==**: **3 == 3** и **4 == 4** — образец совпадает. (4) Выполняется тело ветки. Если бы образец был **Point(x, y)** (переменные), то при совпадении **x** получил бы **p.x**, **y** — **p.y** (binding). Порядок атрибутов в ****match_args**** определяет, какой аргумент образца с каким атрибутом сопоставляется.

**Пример: сопоставление по именованным атрибутам (класс без **match_args**):**

```python
class Config:
    def __init__(self, host: str, port: int):
        self.host = host
        self.port = port

def handle(c: Config):
    match c:
        case Config(host="localhost", port=8080):
            return "local"
        case Config(host=h, port=p):
            return f"{h}:{p}"
        case _:
            return "unknown"
```

Здесь **Config(host="localhost", port=8080)** — образец по именованным атрибутам; **Config(host=h, port=p)** извлекает **host** и **port** в переменные **h** и **p**. Для позиционного образца **Config(h, p)** классу нужен ****match_args** = ("host", "port")**.

#### Пример: разбор JSON-подобных структур

```python
def total_sum(obj):
    match obj:
        case int() | float():
            return obj
        case list() | tuple():
            return sum(total_sum(x) for x in obj)
        case dict():
            return sum(total_sum(v) for v in obj.values())
        case _:
            return 0

total_sum([1, [2, 3], {"a": 4}])   # 10
```

### 18.10 Теоретическое обоснование

- **Читаемость:** разбор вариантов (дискриминанты, структуры данных) записывается компактно и линейно.
- **Безопасность:** можно добиваться полноты вариантов (все случаи обработаны) и использовать **assert_never** (см. §19c) вместе с типизацией.
- **PEP 634** (Specification), **PEP 635** (Motivation), **PEP 636** (Tutorial).

### 18.11 Граничные случаи

- Имена в образце создают **локальные привязки** только в теле этого `case`; после выхода из `match` они не гарантированы (как и при любом присваивании в ветке).
- **`case _:`** должен быть последним, если нужен «catch-all», иначе последующие ветки никогда не выполнятся.
- Сопоставление с классом без **`__match_args__`** может поддерживать только именованные образцы (зависит от реализации).
- **Порядок case имеет значение:** первая совпавшая ветка «выигрывает»; если **case \_:** поставить первой, остальные ветки никогда не выполнятся. Специфичные образцы (конкретные литералы, типы) обычно ставят выше, общие (**\_** или переменная) — в конце.
- **Имена в образце перезаписывают существующие:** если в образце есть **case x:** и **x** уже была переменной в области видимости, после входа в ветку **x** будет указывать на subject. Не используйте в образцах имена, которые не должны перезаписываться.

#### Пошагово: сопоставление последовательности [first, *rest] с [1, 2, 3]

(1) Subject — **`[1, 2, 3]`**. (2) Образец **`[first, *rest]`** требует: subject — последовательность (list, tuple и т.д.), первый элемент сопоставляется с **first**, остальные собираются в **rest**. (3) **first** получает **1**, **rest** получает **list([2, 3])** — в Python переменная со звёздочкой в образце всегда получает **list**. (4) Сопоставление успешно; в теле ветки доступны **first == 1** и **rest == [2, 3]**.

#### Пример: разбор команд (команда + аргументы) через match

```python
def handle(command):
    match command.split():
        case ["quit"]:
            return "Выход"
        case ["load", filename]:
            return f"Загрузка {filename}"
        case ["save", filename]:
            return f"Сохранение {filename}"
        case ["run", *args]:
            return f"Запуск с аргументами: {args}"
        case _:
            return "Неизвестная команда"

handle("load file.txt")   # "Загрузка file.txt"
handle("run a b c")       # "Запуск с аргументами: ['a', 'b', 'c']"
```

#### Миграция с if/elif на match/case

Длинные цепочки **if/elif** по одному значению или по типу можно заменить на **match/case** для единообразия и удобства привязки переменных.

**Было (if/elif):**

```python
def handle(code):
    if code == 200:
        return "OK"
    elif code == 404:
        return "Not Found"
    elif code == 500:
        return "Server Error"
    else:
        return "Unknown"
```

**Стало (match/case):**

```python
def handle(code):
    match code:
        case 200:
            return "OK"
        case 404:
            return "Not Found"
        case 500:
            return "Server Error"
        case _:
            return "Unknown"
```

При разборе структур (списков, словарей, объектов) **match** даёт привязку частей к именам в образце — не нужно вручную писать **isinstance** и индексацию.

#### Плохо vs лучше (match/case)

```python
# Плохо: много вложенных if и isinstance — трудно читать
if isinstance(x, list):
    if len(x) == 2:
        a, b = x[0], x[1]
        ...
elif isinstance(x, dict):
    if "key" in x:
        val = x["key"]
        ...

# Лучше: линейный разбор через match
match x:
    case [a, b]:
        ...
    case {"key": val}:
        ...
    case _:
        ...
```

#### Ограничения образцов: что нельзя

- В образце **отображения** ключи должны быть **литералами** (строка, число и т.д.), не переменными. То есть **case {key: val}:** с именем **key** в образце — не «любой ключ», а буквальный ключ с именем **"key"**.
- Имя переменной в образце **перезаписывается** при входе в ветку — не используйте имена, которые не должны меняться.
- В одном **case** может быть только один образец с **`*`** в последовательности (как и в присваивании).

#### Граничные случаи match: subject — range, строка, итератор, None

- **Subject — range:** **range** поддерживает протокол последовательности (индексация, длина); образец **case [a, b, c]:** с subject **range(1, 4)** сопоставится: **a=1**, **b=2**, **c=3**. Образец **case [first, *rest]:** с **range(5)** даст **first=0**, **rest=[1, 2, 3, 4]**.
- **Subject — строка:** строка — последовательность символов; **match "ab":** с образцом **case [a, b]:** даст **a='a'**, **b='b'**. Образец **case "ab":** сопоставляется по равенству (**==**), то есть subject должен быть строкой **"ab"**.
- **Subject — итератор:** образцы последовательностей требуют, чтобы subject поддерживал **len()** и индексацию (или итерацию с известной длиной); произвольный итератор (например, **iter([1,2,3])**) может не поддерживать **len()** — сопоставление с **[a, b, c]** может завершиться ошибкой или неожиданным поведением. Для сопоставления с последовательностью subject обычно должен быть **list**, **tuple**, **range** или **str**, а не «голый» итератор.
- **Subject — None:** для **match value:** при **value is None** образец **case None:** совпадает (сравнение по **is** для **None**); **case \_:** тоже совпадает. Если нужно явно обработать **None** отдельно от остальных значений, ставьте **case None:** перед **case \_:**; иначе **case \_:** перехватит и **None**, и все остальные.

**Запомните (§18):** образцы проверяются по порядку, выполняется только первая совпавшая ветка; `case _:` совпадает с любым значением — ставьте последним; guard (`if условие`) проверяется после образца; для позиционного образца класса нужен `__match_args__`.
