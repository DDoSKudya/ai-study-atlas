## Частые сценарии и решения (Часть IV)

Ниже собраны **типичные задачи** и **готовые решения** в виде таблиц: в левой колонке — что нужно сделать (формулировка на понятном языке), в правой — какой синтаксис или конструкция для этого подходит (с терминологией: f-string, unpacking, match/case, walrus, Union, zip(strict), except\* и т.д.). Удобно использовать как шпаргалку при написании кода.

### Форматирование вывода (f-строки)

| Задача                                            | Решение                                       |
| ------------------------------------------------- | --------------------------------------------- |
| Вывести переменную и её имя (отладка)             | `f"{var=}"` (Python 3.8+)                     |
| Число с фиксированным числом знаков после запятой | `f"{x:.2f}"`                                  |
| Выровнять строку по ширине (влево/вправо/центр)   | `f"{s:<20}"`, `f"{s:>20}"`, `f"{s:^20}"`      |
| Целое с ведущими нулями (например, номер)         | `f"{n:05d}"`                                  |
| Дата в формате ISO (YYYY-MM-DD)                   | `f"{dt:%Y-%m-%d}"`                            |
| Разделитель тысяч в числе                         | `f"{n:,}"` или `f"{n:_}"`                     |
| Процент от доли (0.25 → 25%)                      | `f"{x:.0%}"` или `f"{x:.1%}"`                 |
| Знак числа (всегда показывать + или −)            | `f"{n:+d}"` или `f"{n: d}"` (пробел вместо +) |
| Шестнадцатеричное с префиксом 0x                  | `f"{n:#x}"` → **0x2a**                        |

### Распаковка

| Задача                                          | Решение                            |
| ----------------------------------------------- | ---------------------------------- |
| Обмен двух переменных                           | `a, b = b, a`                      |
| Взять первый элемент, остальное в список        | `first, *rest = seq`               |
| Взять последний элемент, всё остальное в список | `*begin, last = seq`               |
| Вызвать функцию с аргументами из списка/словаря | `func(*args)`, `func(**kwargs)`    |
| Объединить два списка без изменения исходных    | `[*a, *b]` или `list(a) + list(b)` |
| Слить словари с переопределением ключей         | `{**d1, 'key': val, **d2}`         |
| В цикле по парам (ключ, значение)               | `for k, v in d.items():`           |

### Сопоставление с образцом (match/case)

| Задача                                                | Решение                                                                    |
| ----------------------------------------------------- | -------------------------------------------------------------------------- |
| Обработать несколько конкретных значений              | `case 1:` `case 2:` или `case 1 \| 2:`                                     |
| «Всё остальное»                                       | `case _:` в конце                                                          |
| Разобрать список фиксированной длины                  | `case [a, b, c]:`                                                          |
| Разобрать список: первый + хвост                      | `case [first, *rest]:`                                                     |
| Проверить тип и извлечь значение                      | `case int(n):` или `case str(s):`                                          |
| Дополнительное условие на значение                    | `case int(n) if n > 0:`                                                    |
| Разобрать словарь по ключам                           | `case {"host": h, "port": p}:`                                             |
| Позиционное сопоставление с классом                   | Класс с **`__match_args__ = ("x", "y")`**, образец **`case Point(x, y):`** |
| Сохранить сопоставленное значение целиком (подшаблон) | **case [a, b] as pair:** — в **pair** попадает весь список ** [a, b]**     |
| В цикле: первый элемент и остаток в каждой итерации   | **for first, \*rest in [(1,2,3), (4,5)]:**                                 |

### Морж (:=)

| Задача                                                 | Решение                                                                                                     |
| ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------- |
| Вызвать функцию один раз и проверить результат в if    | `if data := read(): process(data)`                                                                          |
| Читать файл построчно до пустой строки/конца           | `while line := f.readline(): process(line)`                                                                 |
| Использовать результат выражения и в условии, и в теле | Скобки: `if (n := len(data)) > 0: use(n)`                                                                   |
| Отфильтровать по вычисленному значению в listcomp      | `[y for x in items if (y := f(x)) is not None]` (осторожно: **y** остаётся в области видимости после цикла) |

### Union и zip (Python 3.10+)

| Задача                                  | Решение                                       |
| --------------------------------------- | --------------------------------------------- | ---------------- | ----- |
| Аннотировать «тип A или тип B»          | `def f(x: int                                 | str):`или`x: int | None` |
| Гарантировать одинаковую длину итераций | `for a, b in zip(list1, list2, strict=True):` |

### ExceptionGroup (Python 3.11+)

| Задача                                  | Решение                                              |
| --------------------------------------- | ---------------------------------------------------- |
| Обработать только часть типов из группы | `except* ValueError: ...` — остальные пробрасываются |
| Отфильтровать подгруппу по типу         | `eg.subgroup(ValueError)`                            |

### Аннотации типов (3.10–3.12+)

| Задача                                     | Решение                                        |
| ------------------------------------------ | ---------------------------------------------- |
| «Тип A или B» без импорта (3.10+)          | `def f(x: int \| str):`                        |
| «Тип или None» (3.10+)                     | `x: int \| None`                               |
| Метод возвращает self / подкласс (3.11+)   | `def copy(self) -> Self:`                      |
| Исчерпывающий разбор в match (3.11+)       | В последней ветке **case \_: assert_never(x)** |
| Опциональные ключи в TypedDict (3.11+)     | **NotRequired[float]** для ключа               |
| Псевдоним типа без присваивания (3.12+)    | **type Point = tuple[float, float]**           |
| Переопределение метода с проверкой (3.12+) | **@override** на методе наследника             |

**Краткая шпаргалка: что выбрать в типичной ситуации**

- **Строка с подстановкой** → f-строка **f"{x}"**; отладка → **f"{x=}"**; число с двумя знаками → **f"{x:.2f}"**; дата → **f"{dt:%Y-%m-%d}"**.
- **Обмен / первый + остаток / вызов с \*args** → распаковка **a, b = b, a**, **first, \*rest = seq**, **f(\*args)**.
- **Разбор по значению или структуре** → **match/case** с литералами, **case [a, b]**, **case {"k": v}**, **case Point(x, y)** при ****match_args****.
- **Один вызов — проверить и использовать** → морж **if (x := f()): use(x)**; цикл до пустой строки → **while (line := f.readline()):**.
- **Тип A или B в аннотации (3.10+)** → **int | str**, **int | None**; одинаковые длины списков → **zip(a, b, strict=True)**.
- **Часть исключений из группы (3.11+)** → **except\* ValueError:**; исчерпывающий match → **case \_: assert_never(x)**.

## Сводная таблица: когда какую конструкцию использовать

Одна таблица для быстрого выбора: **ситуация** (что вы делаете в коде) → **рекомендуемая конструкция** (синтаксис или тип из Части IV). Термины те же, что в разделах выше (f-строка, распаковка, match/case, морж, Union, zip(strict), except\*).

| Ситуация                                           | Рекомендуемая конструкция                                        |
| -------------------------------------------------- | ---------------------------------------------------------------- |
| Подстановка переменных в строку                    | f-строка **f"{x}"**                                              |
| Отладочный вывод «имя = значение»                  | **f"{var=}"** (3.8+)                                             |
| Форматирование числа (дробная часть, выравнивание) | f-строка с format spec **f"{x:.2f}"**, **f"{n:>5}"**             |
| Форматирование даты/времени                        | f-строка с strftime **f"{dt:%Y-%m-%d}"**                         |
| Обмен двух переменных                              | Распаковка **a, b = b, a**                                       |
| «Первый элемент» и «остаток» списка                | **first, \*rest = seq**                                          |
| Вызов функции с аргументами из списка/словаря      | **f(\*args)**, **f(**kwargs)\*\*                                 |
| Слияние двух списков без изменения исходных        | **[*a, *b]**                                                     |
| Слияние словарей с переопределением ключей         | **{**d1, 'k': v, **d2}** или **d1 \| d2** (3.9+)                 |
| Разбор по значению (код, команда, литерал)         | **match/case** с литералами                                      |
| Разбор по структуре (список, словарь, объект)      | **match/case** с образцами последовательности/отображения/класса |
| «Все остальные» в match                            | **case \_:** в конце                                             |
| Один вызов — проверить и использовать результат    | Морж **if (x := f()): use(x)**                                   |
| Чтение файла до пустой строки/конца                | **while (line := f.readline()): process(line)**                  |
| Аннотация «тип A или B» (3.10+)                    | **X \| Y**, **X \| None**                                        |
| Гарантия одинаковой длины итераций (3.10+)         | **zip(a, b, strict=True)**                                       |
| Обработка части типов из группы исключений (3.11+) | **except\* ValueError:**                                         |

## Рекомендации по стилю

- **f-строки:** используйте по умолчанию для подстановки переменных и простых выражений; не помещайте в `{}` длинную логику — выносите в переменную. Для шаблонов из внешних источников — **.format()** или шаблонизатор.
- **Распаковка:** предпочитайте **for k, v in d.items()** индексации; используйте **a, b = b, a** для обмена; в вызовах **\*args, \*\*kwargs** — стандартный приём для обёрток и декораторов.
- **match/case:** используйте для разбора дискриминантов (коды, команды, типы) и структур (списки, словари, объекты с ****match_args****); ставьте **case \_:** в конце для «все остальные»; избегайте перезаписи важных имён в образцах.
- **Морж:** уместен, когда один вызов нужно и проверить, и сохранить; в условиях **if**/ **while** всегда ставьте скобки вокруг **:=**, если справа идёт сравнение или логический оператор. Не злоупотребляйте в comprehensions.
- **Версии:** при поддержке Python &lt; 3.10 не используйте match/case и **X | Y** в типах; при поддержке &lt; 3.8 — не используйте морж и **f'{x=}'**. Указывайте в документации минимальную версию Python.

## Краткое повторение терминологии

Краткий **словарь терминов** Части IV: термин (на английском и в контексте) и короткое определение на понятном языке. Используйте для быстрого напоминания, что означают f-string, format spec, unpacking, pattern, guard, walrus, ExceptionGroup, Self, type statement и др.

| Термин                          | Краткое определение                                                                        |
| ------------------------------- | ------------------------------------------------------------------------------------------ |
| **f-string**                    | Строковый литерал с префиксом `f`/`F`, выражения в `{...}` подставляются в строку          |
| **Format spec**                 | Спецификация формата после `:` в f-строке: ширина, точность, тип вывода                    |
| **Распаковка (unpacking)**      | Присваивание нескольких имён из одной последовательности; `*` собирает «остаток» в список  |
| **Упаковка в вызове**           | `*iterable` — позиционные аргументы, `**dict` — именованные аргументы                      |
| **match/case**                  | Сопоставление с образцом: выбор ветки по образцу (литерал, последовательность, тип, guard) |
| **Wildcard**                    | Образец `_` в case — совпадает с любым значением                                           |
| **Guard**                       | Условие `if ...` в case после образца; ветка выполняется только при истинном условии       |
| ****match_args****              | Атрибут класса: порядок атрибутов для позиционного сопоставления в образце                 |
| **Walrus operator (:=)**        | Присваивание в выражении; присваивает и возвращает значение                                |
| **ExceptionGroup**              | Контейнер нескольких исключений (Python 3.11+); обрабатывается через except\*              |
| **Self / Never / assert_never** | Типы и функция для аннотаций и исчерпывающего разбора вариантов                            |
| **type statement**              | Объявление псевдонима типа через оператор `type` (Python 3.12+)                            |
| **Subject / Pattern**           | В match: subject — значение после match; pattern — образец в case                          |
| **Binding**                     | Привязка: имя в образце получает часть значения при успешном сопоставлении                 |

## Вопросы по каждой теме (с скрытыми ответами)

Ответы скрыты под спойлером: нажмите «Ответ», чтобы раскрыть.

### §16. f-strings

1. **Зачем в f-строке нужны флаги `!r` и `!a` и чем они отличаются от подстановки по умолчанию?**

<details>
<summary>Ответ</summary>

По умолчанию значение подставляется через **str()**. **!r** подставляет **repr(x)** — удобно для отладки (кавычки, escape). **!a** подставляет **ascii(x)** — не-ASCII символы экранируются (например, кириллица в `\u041f`); полезно для логов и сред, где важна только ASCII.

</details>

2. **Что выведет `f"{{x}}"` и почему нельзя написать `f"\{x\}"`?**

<details>
<summary>Ответ</summary>

**f"{{x}}"** выведет буквально строку **"{x}"**: двойные фигурные скобки **{{** и **}}** экранируются в одну **{** и **}**. Внутри выражения в **{...}** обратный слэш не допускается грамматикой f-строки, поэтому **\{** недопустим — только **{{** для буквальной фигурной скобки.

</details>

3. **В чём разница между `f"{dt:%Y-%m-%d}"` и `f"{dt}"` для объекта datetime?**

<details>
<summary>Ответ</summary>

**f"{dt:%Y-%m-%d}"** передаёт в **dt.**format**** спецификацию **%Y-%m-%d** (коды strftime) и выводит, например, **2025-02-08**. **f"{dt}"** вызывает **str(dt)** и выводит стандартное строковое представление datetime (полная дата-время в формате по умолчанию).

</details>

### §17. Распаковка и упаковка

1. **Почему в одном присваивании нельзя написать `a, *b, *c = seq`? Как получить «первый», «средние» и «последний» элементы?**

<details>
<summary>Ответ</summary>

В целевом списке допускается не более одной переменной со звёздочкой — иначе интерпретатор не может однозначно распределить элементы. Чтобы получить первый, средние и последний: **first, \*middle, last = seq** (одна звёздочка у **middle**). Либо использовать срезы: **first, last = seq[0], seq[-1]; middle = seq[1:-1]**.

</details>

2. **Что произойдёт при вызове `f(**{1: "a", 2: "b"})`, если функция ждёт именованные аргументы?\*\*

<details>
<summary>Ответ</summary>

**TypeError**: при распаковке словаря в именованные аргументы (**\*\*dict**) ключи должны быть **строками**. Ключи **1** и **2** — целые числа, они не могут быть именами параметров. Нужен словарь с строковыми ключами, например **{"x": "a", "y": "b"}** для **def f(x, y)**.

</details>

3. **Чем `[*a, *b]` отличается от `a + b` для двух списков? Когда что использовать?**

<details>
<summary>Ответ</summary>

Для двух списков результат одинаковый — новый список из элементов **a** и **b**. **[*a, *b]** работает для любых итерабельных объектов (например, **[_range(3), _[10, 20]]** → **[0, 1, 2, 10, 20]**). **a + b** — только для двух списков и создаёт новый список. **[*a, *b]** удобен, когда нужно «склеить» несколько итерабельных объектов или добавить отдельные элементы: **[*a, 1, 2, *b]**.

</details>

### §18. match/case

1. **Почему образец `case _:` обычно ставят последним? Что будет, если поставить его первым?**

<details>
<summary>Ответ</summary>

**case \_:** (wildcard) совпадает с **любым** значением. Образцы проверяются по порядку; выполняется только первая совпавшая ветка. Если **case \_:** стоит первым, он всегда совпадает, и все остальные **case** никогда не выполнятся. Поэтому **case \_:** ставят в конце как «всё остальное».

</details>

2. **Зачем в классе нужен `__match_args__` и что будет, если его не задать?**

<details>
<summary>Ответ</summary>

****match_args**** — кортеж имён атрибутов, задающий **порядок** для **позиционного** образца: **case Point(x, y):** сопоставляет первый аргумент с **p.x**, второй с **p.y**. Без ****match_args**** позиционный образец **Point(0, 0)** не поддерживается — интерпретатор не знает, какой аргумент какому атрибуту соответствует. Сопоставление возможно только по **именованным** атрибутам: **Point(x=0, y=0)**.

</details>

3. **Когда вычисляется guard в `case int(n) if n > 0:` — до или после сопоставления образца?**

<details>
<summary>Ответ</summary>

Guard вычисляется **после** успешного сопоставления образца. Сначала проверяется **int(n)** — subject должен быть целым, в **n** привязывается значение. Затем вычисляется условие **if n > 0**. Если оно ложно, ветка считается не совпавшей и проверяется следующий **case**. Guard может использовать переменные, привязанные в образце (**n**).

</details>

### §19. Морж (:=)

1. **Почему пишут `if (n := len(data)) > 0`, а не `if n := len(data) > 0`?**

<details>
<summary>Ответ</summary>

Приоритет **:=** ниже, чем у сравнения. **if n := len(data) > 0** парсится как **n := (len(data) > 0)** — в **n** попадает булево значение **True**/False, а не длина. Чтобы присвоить длину и сравнить с нулём, нужны скобки: **(n := len(data)) > 0** — сначала выполняется присваивание, затем сравнение.

</details>

2. **Можно ли использовать морж в части `for` спискового включения: `[x for (x := item) in items]`?**

<details>
<summary>Ответ</summary>

Нет. В **for**-части comprehension оператор **:=** синтаксически **запрещён** — **for (x := item) in items** даёт **SyntaxError**. Морж допустим только внутри выражения (например, в условии фильтра или в выражении значения): **[y for x in items if (y := f(x)) is not None]**.

</details>

3. **Переменная, присвоенная через морж в условии `while`, видна после выхода из цикла?**

<details>
<summary>Ответ</summary>

Да. Переменная из **:=** принадлежит той же области видимости, что и цикл (функция, класс, модуль). После выхода из **while** она сохраняется и содержит значение **последней** итерации (последнее «истинное» значение или значение на момент выхода, если цикл завершился из-за «ложного» условия). На это можно опереться, если нужно знать, на каком значении цикл остановился.

</details>

### §19a. Union | None, zip(strict)

1. **Чем `isinstance(x, int | str)` отличается от `isinstance(x, (int, str))` в Python 3.10+?**

<details>
<summary>Ответ</summary>

Семантически результат одинаковый — проверка, что **x** является экземпляром **int** или **str**. **int | str** в 3.10+ создаёт объект **types.UnionType**; **(int, str)** — кортеж типов (поддерживается давно). Синтаксис **int | str** единообразен с аннотациями типов (**def f(x: int | str)**) и не требует запятой и скобок.

</details>

2. **Что вернёт `list(zip([1, 2], [10, 20], strict=True))` и при каком вызове `zip(..., strict=True)` возникнет ошибка?**

<details>
<summary>Ответ</summary>

Вернёт **[(1, 10), (2, 20)]** — пары до исчерпания кратчайшей последовательности, при равной длине то же, что без **strict**. Ошибка **ValueError** возникнет, если длины не совпадают, например **list(zip([1, 2], [10, 20, 30], strict=True))** — при попытке взять третий элемент из первой последовательности **zip** обнаружит разную длину и возбудит исключение.

</details>

3. **Эквивалентны ли аннотации `Optional[int]` и `int | None` в Python 3.10+?**

<details>
<summary>Ответ</summary>

Да, семантически эквивалентны. **Optional[int]** из **typing** — это **Union[int, None]**; **int | None** обозначает то же объединение типов. **int | None** не требует импорта и короче; в 3.10+ предпочтительнее для новых аннотаций.

</details>

### §19b. ExceptionGroup, except\*

1. **Что делает `except* ValueError:` и куда деваются исключения других типов из той же группы?**

<details>
<summary>Ответ</summary>

**except\* ValueError:** перехватывает из группы **только** исключения типа **ValueError**; они попадают в переменную после **as** (например, **eg.exceptions**). Остальные исключения из группы формируют **новую** ExceptionGroup и пробрасываются дальше — их можно обработать следующими ветками **except\*** или они уйдут наверх. Таким образом обрабатывается часть типов без «проглатывания» остальных.

</details>

2. **Для чего нужен метод `eg.subgroup(ValueError)`?**

<details>
<summary>Ответ</summary>

**subgroup(ValueError)** возвращает **новую** ExceptionGroup, в которой только исключения типа **ValueError** из **eg** (и из вложенных групп). Если таких нет, возвращается **None**. Удобно для фильтрации группы по типу без обработки через **except\*** — например, чтобы логировать только ValueError и пробросить остальное.

</details>

3. **Можно ли в одной ветке перехватывать несколько типов исключений из группы?**

<details>
<summary>Ответ</summary>

Да: **except\* (ValueError, TypeError):** — перехватываются все исключения, являющиеся **ValueError** или **TypeError** из перехваченной группы. Остальные типы образуют новую группу и пробрасываются. Синтаксис **except\* (A, B):** аналогичен **except (A, B):** для обычного **try/except**, но применяется к каждому исключению в группе по отдельности.

</details>

### §19c. Self, Never, assert_never

1. **Зачем методам, возвращающим `self`, аннотировать возвращаемый тип как `Self`?**

<details>
<summary>Ответ</summary>

**Self** обозначает «экземпляр текущего класса». В подклассе метод с **-> Self** считается возвращающим экземпляр **этого** подкласса, а не базового. Без **Self** пришлось бы писать имя класса или **TypeVar**; тип-чекер тогда не знал бы, что **SubClass().copy()** имеет тип **SubClass**, а не **Base**. Удобно для цепочек вызовов и фабричных методов (**copy()**, **with\_...()**).

</details>

2. **В каком месте кода обычно вызывают `assert_never(x)` и что произойдёт в runtime, если туда всё же попали?**

<details>
<summary>Ответ</summary>

**assert_never(x)** вызывают в ветке, которая по замыслу **недостижима** — например, в **case \_:** после исчерпывающей обработки всех вариантов enum. Тип-чекер считает, что сюда управление не попадает. Если в runtime туда всё же попали (добавили новый вариант и забыли обработать), **assert_never** возбуждает **TypeError**, что помогает отловить ошибку при тестировании.

</details>

3. **Какой тип возврата у функции, которая только вызывает `raise` или `sys.exit()`?**

<details>
<summary>Ответ</summary>

Аннотируют как **-> Never** (из **typing**). Тип **Never** (bottom type) означает «функция не возвращает управление». Статический анализатор считает код после вызова такой функции недостижимым; это полезно для **sys.exit()**, **raise**, бесконечных циклов без **return**.

</details>

### §19d. TypeVarTuple, Required, NotRequired, LiteralString

1. **Для чего нужен TypeVarTuple и чем он отличается от обычного TypeVar?**

<details>
<summary>Ответ</summary>

**TypeVar** привязан к **одному** типу. **TypeVarTuple** — к **произвольному числу** типов (variadic). Он нужен, когда тип зависит от переменного числа аргументов: например, **def make_tuple(*args: *Ts) -> tuple[*Ts]** — возвращаемый кортеж имеет те же типы, что и аргументы. Обычный TypeVar так не выразить без множества **@overload**.

</details>

2. **В TypedDict все ключи по умолчанию обязательны. Как пометить ключ как необязательный?**

<details>
<summary>Ответ</summary>

Использовать **NotRequired[...]** (Python 3.11+): **timeout: NotRequired[float]** — ключ **timeout** может отсутствовать при создании словаря. Тип-чекер не будет требовать его в литерале. **Required[key]** явно помечает ключ обязательным (по умолчанию все ключи и так обязательны; полезно в смеси с **NotRequired**).

</details>

3. **Зачем API принимать `LiteralString` вместо `str` для SQL-запросов?**

<details>
<summary>Ответ</summary>

**LiteralString** — тип для строк, составленных только из литералов (и конкатенаций таких строк). Тип-чекер не примет переменную с произвольной строкой (например, из **input()**). Это защита от SQL injection: в запрос нельзя подставить пользовательский ввод через переменную; только «зашитые» в код строки и их конкатенации.

</details>

### §19e. type statement, @override, PEP 696, PEP 709

1. **Чем `type Point = tuple[float, float]` отличается от присваивания `Point = tuple[float, float]`?**

<details>
<summary>Ответ</summary>

**type** объявляет **псевдоним типа**: имя **Point** в аннотациях обозначает тип, в runtime отдельная переменная с этим значением не создаётся в том же смысле. При присваивании **Point = tuple[float, float]** имя **Point** в runtime — обычная переменная, хранящая класс **tuple**. **type** явно сигнализирует «это тип для аннотаций» и в 3.12+ корректно обрабатывается в **isinstance** и т.п.

</details>

2. **Что проверяет декоратор `@override` и в какой момент это видно?**

<details>
<summary>Ответ</summary>

**@override** помечает метод как переопределяющий метод базового класса. **Статические анализаторы** (mypy, pyright) проверяют, что в базовом классе действительно есть метод с таким именем. Если метода нет (опечатка или переименование в базе), выдаётся предупреждение. В runtime сам декоратор в 3.12+ не выполняет проверку — только анализ кода.

</details>

3. **Что даёт PEP 709 (comprehension inlining) в Python 3.12?**

<details>
<summary>Ответ</summary>

Это **внутренняя оптимизация**: list/dict/set comprehensions компилируются так, что уменьшаются накладные расходы на создание фрейма итерации. Синтаксис и поведение не меняются — те же listcomp/dictcomp/setcomp, но выполнение быстрее. Пользовательский код менять не нужно.

</details>

### §19f. Python 3.13+

1. **Как пометить устаревшую функцию, чтобы анализаторы и при необходимости runtime предупреждали об использовании?**

<details>
<summary>Ответ</summary>

Использовать декоратор **@deprecated** из **typing** (PEP 702, Python 3.13+): **@deprecated("Используйте new_func()")** перед функцией или классом. В сообщении указывают, чем заменить. Статические анализаторы могут показывать предупреждение при обращении к помеченной сущности; в runtime при вызове может выдаваться **DeprecationWarning** (зависит от версии и настроек **warnings**).

</details>

2. **Для чего запускать интерпретатор с флагом `-P`?**

<details>
<summary>Ответ</summary>

**-P** отключает автоматическую загрузку каталога **site-packages** в **sys.path**. Окружение становится «минимальным» — скрипт не видит пакеты, установленные в пользовательский site-packages. Удобно для проверки, что скрипт не опирается на такие пакеты (например, в CI или при сборке); если при **-P** возникает **ModuleNotFoundError**, зависимость нужно явно указать или установить в venv.

</details>

3. **Что такое free-threaded Python и как его запустить?**

<details>
<summary>Ответ</summary>

**Free-threaded Python** (PEP 703) — сборка интерпретатора **без GIL** (global interpreter lock), позволяющая нескольким потокам выполнять Python-байткод одновременно. Запуск (если сборка с поддержкой): **python -X gil=0**. На 2025 год это экспериментальная возможность; не все сборки её поддерживают. Обычная сборка CPython по-прежнему с GIL.

</details>

## Вопросы для самопроверки

1. **Чем f-строка отличается от вызова `str.format()` и когда что предпочтительнее?**  
   В f-строке выражения стоят прямо в месте подстановки; при компиляции генерируется код построения строки без разбора шаблона при каждом вызове. Предпочтительнее почти всегда f-строка (3.6+); **.format()** уместен, когда шаблон приходит извне (файл, пользователь) или когда одна и та же строка подставляется с разными наборами аргументов из кода.

2. **Что выведет `f"{x=:.2f}"` при `x = 3.14159`?**  
   Строка **`x=3.14`**: слева от **=** — текст выражения (**x**), справа — значение, отформатированное как float с двумя знаками после запятой.

3. **В чём разница между `a, *b = seq` и `*a, b = seq`?**  
   В первом случае **a** получает первый элемент, **b** — список всех остальных. Во втором **a** — список всех элементов кроме последнего, **b** — последний элемент.

4. **Когда уместно использовать `zip(..., strict=True)`?**  
   Когда ожидается, что все итерабельные объекты имеют одинаковую длину, и расхождение считается ошибкой (например, два списка «имена» и «возрасты»). **strict=True** вызовет **ValueError**, если одна из последовательностей короче или длиннее — это защищает от тихих обрезок по кратчайшей.

5. **Как в match/case обработать «любое значение» и зачем нужен guard?**  
   «Любое значение» обрабатывается образцом **`case _:`** (wildcard). Guard (**`case x if x > 0:`**) нужен, когда ветка должна выполниться не просто при совпадении формы/типа, но и при выполнении дополнительного условия (например, положительное число, непустая строка).

6. **В каких контекстах для моржа `:=` обязательны скобки и почему?**  
   В условиях **if** и **while**, когда справа от **:=** идёт сравнение или логический оператор: иначе парсер свяжет **:=** с правой частью (например, **n := len(data) > 0** станет присваиванием булева значения). Скобки задают границу: **(n := len(data)) > 0**.

7. **Что такое ExceptionGroup и как обработать только часть исключений из группы?**  
   **ExceptionGroup** — контейнер из нескольких исключений (например, при параллельном выполнении в **asyncio.TaskGroup**). Обработать только часть типов можно через **except\* TypeError:** — перехватываются все **TypeError** из группы, остальные исключения формируют новую группу и пробрасываются дальше. Либо вручную через **eg.subgroup(ValueError)** и обработку подгруппы.

8. **Для чего нужны typing.Self и assert_never?**  
   **Self** — аннотация возвращаемого типа для методов, возвращающих **self** (цепочки вызовов, **copy()**); тип-чекер понимает, что возвращается экземпляр текущего класса. **assert_never(x)** — маркер «сюда управление не должно попасть»; используется в последней ветке **match** для исчерпывающей проверки: если добавили новый вариант и забыли обработать, статический анализатор или runtime сообщат об ошибке.

9. **Зачем в f-строке флаг !r и когда использовать !a?**  
   **!r** подставляет **repr(x)** вместо **str(x)** — удобно для отладки (кавычки, escape-последовательности). **!a** подставляет **ascii(x)** — не-ASCII символы экранируются (например, кириллица в **\u041f**); полезно при выводе в среду, где важна только ASCII-совместимость (логи, старые терминалы).

10. **Как вывести в f-строке буквальную фигурную скобку и почему нельзя использовать `\{`?**  
    Удвойте скобку: **{{** даёт **{**, **}}** даёт **}**. Внутри выражения в **{...}** обратный слэш не допускается грамматикой f-строки; экранирование **{** возможно только через **{{**, иначе **{** считается началом выражения и приведёт к **SyntaxError**.

- Можно ли в одной f-строке использовать и **str()**, и **repr()** для разных выражений? Как?

- Что вернёт **list(zip([1, 2, 3], strict=True))** и почему **strict** при одном аргументе допустим?

**Краткие ответы на вопросы на проверку:** (1) **f"{x}"** использует **str(x)**, **f"{x!r}"** — **repr(x)**; **!r** уместен для отладочного вывода. (2) В одном присваивании допускается только одна звёздочка; вместо двух «остатков» можно взять срезы или обработать последовательность по-другому. (3) Для subject **[1, 2]** оба образца **[a, b]** и **(a, b)** сопоставляются (и list, и tuple — последовательности); привязки **a=1**, **b=2** одинаковы. (4) Образцы проверяются по порядку до первого совпадения; имена из образца видны только в теле совпавшей ветки и не гарантированы после выхода из **match**. (5) Да: **f"{x} {y!r}"** — для **x** используется **str**, для **y** — **repr** (флаг **!r** в f-строке). (6) При первой итерации **next()** на **zip([], [1], strict=True)** будет **ValueError** (разная длина); **zip** с **strict=True** проверяет длину при итерации. (7) **int | None** и **Optional[int]** семантически эквивалентны; **int | None** не требует импорта **typing** в Python 3.10+ и короче. (8) Из-за short-circuit: в **a and (b := f())** выражение **(b := f())** вычисляется только если **a** истинно; если **a** ложно, **f()** не вызывается и **b** не присваивается. Пример: **False and (b := 1)** — **b** не будет присвоена; после такого выражения обращение к **b** даст **NameError** (если **b** не была определена ранее). (9) **list(zip([1, 2, 3], strict=True))** вернёт **[(1,), (2,), (3,)]** — кортежи из одного элемента. **strict=True** проверяет равенство длин всех переданных итераций; при одной итерации «все длины равны» тривиально выполняется, поэтому **strict** допустим и не меняет результат.

## Типичные ошибки при переходе на современный синтаксис

- **Забыли префикс f:** **"{x}"** выводит буквально **"{x}"**, а не значение переменной — нужен **f"{x}"**.
- **Морж без скобок в условии:** **if n := len(data) > 0** присваивает **n** булево значение; правильно **if (n := len(data)) > 0**.
- **Две звёздочки в одном присваивании:** **a, *b, *c = seq** — **SyntaxError**; в одном целевом списке допускается только одна переменная со звёздочкой.
- **case \_: не последний:** если поставить **case \_:** первым, остальные ветки никогда не выполнятся — ставьте **case \_:** в конце.
- **Ключи словаря при ** распаковке не строки:\*\* **{**{1: 'a'}}** даёт **TypeError\*\* — ключи должны быть строками.
- **zip без strict при разной длине:** если длины списков должны совпадать, используйте **zip(a, b, strict=True)**, иначе лишние элементы тихо отбрасываются.

## Отладка и диагностика (Часть IV)

- **f-строка не подставляет значение:** убедитесь, что используете префикс **f** или **F** и выражение в **{}**; проверьте, что имя переменной существует в области видимости в момент выполнения.
- **Распаковка: ValueError «not enough / too many values»:** проверьте длину последовательности справа и число целей слева; используйте **\*rest**, чтобы собрать «остаток», или **zip**/цикл, если длина переменная.
- **match: ни одна ветка не выполнилась:** добавьте **case \_:** в конце или проверьте порядок образцов (литералы и конкретные типы перед **case x:** и **case \_:**).
- **Морж: NameError после if/while:** переменная, присвоенная через **:=**, видна в теле ветки/цикла и в той же области видимости после; в comprehensions переменная после цикла сохраняет значение последней итерации — не полагайтесь на него снаружи.
- **zip(strict=True) ValueError:** сообщение указывает, какой аргумент длиннее/короче; проверьте источники данных (списки, файлы) на одинаковую длину или обработайте исключение.
- **ExceptionGroup: как обработать только часть:** используйте **except\* Тип:** для каждого нужного типа; оставшиеся исключения автоматически формируют новую группу и пробрасываются — не нужно вручную разбирать **.exceptions**.
- **Union в isinstance: только 3.10+:** на более старых версиях используйте **isinstance(x, (int, str))** (кортеж типов); **int | str** во втором аргументе **isinstance** поддерживается с Python 3.10.
- **ValueError при распаковке:** число элементов справа не совпадает с числом целей слева (и нет звёздочки). Проверьте **len(seq)** и количество имён; при **\*rest** «остаток» может быть пустым списком.
- **match не заходит ни в одну ветку:** проверьте порядок **case** (первый совпавший выполняется); убедитесь, что есть **case \_:** для «все остальные»; для классов проверьте ****match_args**** при позиционном образце.
- **Морж присваивает не то:** в условии **if**/ **while** при наличии сравнения справа от **:=** нужны скобки: **if (n := len(data)) > 0**, иначе **n** получит булево значение.
- **zip(strict=True) и ValueError:** одна из итераций короче или длиннее других; проверьте длины списков до вызова **zip**.
- **Буквальная фигурная скобка в f-строке:** используйте **{{** и **}}**; **\{** внутри **{...}** недопустимо и даёт **SyntaxError**.

## Резюме по Части IV

**Часть IV** охватывает современный синтаксис Python 3:

- **§16 f-strings** — основной способ форматирования строк: выражения в `{}`, format spec, debug-формат `f'{x=}'`, даты через strftime-коды.
- **§17 Распаковка** — присваивание с `*`/`**`, распаковка в вызовах и в литералах списков/словарей.
- **§18 match/case** — сопоставление с образцом: литералы, последовательности, отображения, `as`, `|`, guard, `_`, `__match_args__`.
- **§19 Walrus (:=)** — присваивание в выражении в условиях и comprehensions, с учётом скобок и области видимости.
- **§19a–f** — нововведения по версиям: Union `|`, zip(strict), ExceptionGroup/except\*, Self/Never/assert_never, TypeVarTuple/Required/NotRequired, type statement, @override, free-threaded и др.

Использование этих конструкций делает код короче, понятнее и безопаснее при соблюдении стиля и ограничений, описанных в плане и в этой части.

**Связь с другими частями плана:** Часть IV опирается на базовый синтаксис (Часть I: типы, функции, исключения), на стандартную библиотеку (Часть III: **typing**, **contextlib**) и ведёт к контекстным менеджерам (Часть V, §20) и асинхронности (Часть VI: **asyncio.TaskGroup** и **ExceptionGroup**). Нововведения §19a–f пересекаются с типизацией (Часть III, §9–§9a) и с глобальным планом по шагам (Шаг 3 — Современный синтаксис, Шаг 4 — Типизация).
