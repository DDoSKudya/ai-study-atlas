## §16. f-strings

**Цель раздела:** подставлять переменные и выражения в строку, задавать формат чисел и дат, использовать отладочный вывод; понимать, когда f-строки уместны.

| Термин | Определение |
|--------|-------------|
| **f-string** (formatted string literal) | Строковый литерал с префиксом `f`, в `{...}` — выражения; при вычислении подставляются в строку |
| **format spec** | Часть после `:` в `{expr:format_spec}`; ширина, точность, выравнивание, тип (d, f, b, x) |
| **Флаги `!s`, `!r`, `!a`** | str(), repr(), ascii() — способ преобразования значения в строку |
| **debug-формат** | `f'{x=}'` — подставляет имя выражения, `=`, repr(значения) |

**Суть:** f-строка — выражение прямо в месте подстановки; компилируется один раз; по умолчанию `str()`, для отладки `f'{x=}'`, для форматирования — `{x:format_spec}`.

---

**F-string** (formatted string literal, «форматированный строковый литерал») — строковый литерал с префиксом `f` или `F`, внутри которого в фигурных скобках `{...}` записаны выражения. При вычислении строки эти выражения подставляются в результат. F-strings появились в **Python 3.6** (PEP 498 — Literal String Interpolation) и стали **основным** способом форматирования строк в современном коде: они объединяют читаемость, производительность и единообразие в одном синтаксисе.

#### В чём суть: f-строка vs старые способы

До f-строк в Python использовались:

1. **Оператор `%` (printf-стиль):** `"Имя: %s, возраст: %d" % (name, age)` — порядок аргументов в кортеже должен совпадать с порядком плейсхолдеров; при добавлении или перестановке аргументов легко ошибиться; тип задаётся буквой (`s`, `d`, `f`).
2. **Метод `str.format()`:** `"Имя: {}, возраст: {}".format(name, age)` или `"Имя: {0}, возраст: {1}".format(name, age)` — гибче, но выражение и подстановка разнесены: нужно смотреть то на строку, то на аргументы; при длинном списке аргументов читаемость падает.
3. **Конкатенация:** `"Имя: " + name + ", возраст: " + str(age)` — много кавычек и `+`, обязательное приведение к строке вручную; неудобно для чисел с форматированием.

**F-строка** переносит **выражение прямо в место подстановки**: `f"Имя: {name}, возраст: {age}"`. Имя переменной и место, где оно появится в строке, находятся рядом; не нужно держать в голове порядок аргументов или индексы. Кроме того, внутри `{}` может стоять **любое выражение** (вызов функции, атрибут, тернарный оператор), а не только имя переменной — без отдельного вычисления заранее.

**Краткая таблица сравнения:**

| Аспект                                 | `%`                         | `.format()`                        | f-строка                                        |
| -------------------------------------- | --------------------------- | ---------------------------------- | ----------------------------------------------- |
| Где задаётся значение                  | В кортеже/словаре справа    | В аргументах вызова                | Прямо в `{}` в строке                           |
| Читаемость при нескольких подстановках | Средняя (порядок в кортеже) | Средняя (индексы/имена)            | Высокая (всё в одном месте)                     |
| Производительность                     | Хорошая                     | Хуже (вызов метода, разбор строки) | Лучшая (компиляция в байткод построения строки) |
| Выражения в подстановке                | Нет (только переменные)     | Ограниченно (в аргументах)         | Да (любое выражение в `{}`)                     |
| Версия Python                          | 2.x, 3.x                    | 2.6+, 3.x                          | 3.6+                                            |

**Почему это важно на практике:** при ревью кода или отладке f-строка позволяет сразу видеть и шаблон, и источник данных; при рефакторинге (переименование переменной, добавление поля) меньше шанс забыть обновить формат-строку в другом месте. Производительность важна в циклах или при частом логировании: f-строка компилируется один раз и при выполнении лишь вычисляет выражения и собирает строку, без разбора шаблона при каждом вызове, как у `.format()`.

### 16.1 Базовый синтаксис

Общий вид:

```text
f'текст {выражение} текст'
```

- Префикс **`f`** или **`F`** перед открывающей кавычкой (одинарной, двойной или тройной).
- Внутри строки **`{ выражение }`** — любое допустимое выражение Python; оно вычисляется в текущем контексте и результат подставляется в строку через `str()` (или через `__format__`, если указан format spec).

**Примеры:**

```python
name = "Алиса"
age = 30
msg = f"Имя: {name}, возраст: {age}"
# "Имя: Алиса, возраст: 30"

x, y = 10, 20
print(f"Сумма: {x + y}")   # Сумма: 30
```

**Пошагово: что происходит при вычислении f"Имя: {name}, возраст: {age}".**

1. Интерпретатор видит f-строку и разбивает её на фрагменты: литеральный текст и части в `{...}`.
2. Для каждой части в скобках вычисляется выражение в текущей области видимости: `name` → `"Алиса"`, `age` → `30`.
3. Каждый результат приводится к строке (здесь через `str()`).
4. Фрагменты склеиваются в одну строку: `"Имя: " + "Алиса" + ", возраст: " + "30"` → `"Имя: Алиса, возраст: 30"`.

Итог: f-строка — это выражение, которое в момент выполнения даёт обычную строку `str`.

### 16.2 Выражения внутри фигурных скобок

В `{...}` может стоять не только имя переменной, но и любое выражение: вызовы функций, атрибуты, индексация, тернарный оператор и т.д.

```python
from math import pi
print(f"Число π: {pi:.4f}")                    # Число π: 3.1416

items = [1, 2, 3]
print(f"Длина списка: {len(items)}")           # Длина списка: 3
print(f"Первый элемент: {items[0]}")           # Первый элемент: 1

def double(x):
    return x * 2
print(f"Удвоенное 5: {double(5)}")             # Удвоенное 5: 10

# Тернарный оператор
n = 7
print(f"{n} — {'чётное' if n % 2 == 0 else 'нечётное'} число")  # 7 — нечётное число
```

**Важно:** выражение внутри `{}` выполняется в том же контексте, что и окружающий код. Обратные кавычки и символ `\` внутри выражений в f-строках не допускаются (в части выражения); кавычки для строк внутри выражений — обычные `'` или `"`.

#### Преобразование значения: str, repr, ascii (флаги !s, !r, !a)

По умолчанию результат выражения в **`{выражение}`** подставляется через **str()**. Можно явно указать способ преобразования сразу после выражения (перед format spec, если он есть):

- **`{x}`** — подстановка через **str(x)** (по умолчанию).
- **`{x!s}`** — то же, **str(x)** (явно).
- **`{x!r}`** — подстановка через **repr(x)**; удобно для отладочного вывода (кавычки, escape-последовательности).
- **`{x!a}`** — подстановка через **ascii(x)**; не-ASCII символы экранируются (например, **'\u041f'** для «П»).

**Примеры:**

```python
s = "hello"
f"{s}"      # hello
f"{s!r}"    # 'hello'  (с кавычками, как в коде)
f"{s!a}"    # 'hello'  (ascii — здесь то же)

x = "Привет"
f"{x!a}"    # '\u041f\u0440\u0438\u0432\u0435\u0442'  (не-ASCII экранированы)
```

**Когда использовать !r:** когда нужно видеть «как в коде» (кавычки, тип числа, escape). В отладке **f"{var=}"** уже использует **repr** для значения справа от **=**; для отдельного выражения используйте **f"{var!r}"**.

### 16.3 Спецификация формата (format spec)

После выражения можно указать **format spec** через двоеточие: `{выражение:формат}`. Формат задаёт, как значение представляется (число знаков, выравнивание, тип и т.д.) и передаётся в метод `__format__(value, format_spec)` или встроенную логику для типов.

**Числа:**

```python
x = 3.14159
print(f"{x:.2f}")      # 3.14 — два знака после запятой
print(f"{x:>10.2f}")  # "      3.14" — ширина 10, выравнивание вправо
print(f"{x:<10.2f}")  # "3.14      " — влево
print(f"{x:^10.2f}")  # "  3.14    " — по центру

n = 42
print(f"{n:05d}")     # 00042 — целое, ширина 5, дополнение нулями
print(f"{n:b}")       # 101010 — двоичное
print(f"{n:o}")       # 52 — восьмеричное
print(f"{n:x}")       # 2a — шестнадцатеричное (нижний регистр)
print(f"{n:X}")       # 2A — шестнадцатеричное (верхний регистр)
```

**Строки:**

```python
s = "Python"
print(f"{s:>10}")     # "    Python" — ширина 10, вправо
print(f"{s:<10}")     # "Python    " — влево
print(f"{s:^10}")     # "  Python  " — по центру
print(f"{s:.3}")      # "Pyt" — первые 3 символа (усечение)
```

**Общая схема format spec:** `[[fill]align][sign][#][0][width][grouping][.precision][type]`.

- **fill** — символ заполнения (по умолчанию пробел).
- **align** — `<` (влево), `>` (вправо), `^` (центр), `=` (знак отдельно, число после).
- **width** — минимальная ширина поля.

**Таблица: format spec по типам (краткий справочник):**

| Тип значения | Пример format spec     | Результат (пример)                           |
| ------------ | ---------------------- | -------------------------------------------- |
| **int**      | `:d`                   | Обычное целое (**42**)                       |
| **int**      | `:05d`                 | Ширина 5, дополнение нулями (**00042**)      |
| **int**      | `:b`                   | Двоичное (**101010**)                        |
| **int**      | `:x`, `:X`             | Шестнадцатеричное (**2a**, **2A**)           |
| **int**      | `:o`                   | Восьмеричное (**52**)                        |
| **float**    | `:.2f`                 | Два знака после запятой (**3.14**)           |
| **float**    | `:>10.2f`              | Ширина 10, вправо, два знака (**" 3.14"**)   |
| **float**    | `:e`, `:.2e`           | Экспоненциальная запись (**3.14e+00**)       |
| **str**      | `:>10`, `:<10`, `:^10` | Выравнивание по ширине 10                    |
| **str**      | `:.3`                  | Первые 3 символа (усечение) (**"Pyt"**)      |
| **число**    | `:+`, `:-`, `: `       | Знак: всегда, только минус, пробел для плюса |
| **число**    | `:,`                   | Разделитель тысяч (**1,000,000**)            |

**Граничный случай: отрицательные числа и sign.** При **f"{n:+d}"** для отрицательного **n** выводится минус: **f"{-5:+d}"** → **"-5"**. При **f"{n: d}"** (пробел вместо плюса) для положительного числа выводится пробел вместо **+**: **f"{5: 3d}"** → **" 5"**, для отрицательного — минус: **f"{-5: 3d}"** → **" -5"**. При **f"{n:-d}"** знак выводится только для отрицательных (по умолчанию для чисел так и есть); **:** без sign ведёт себя как **:-**.

- **precision** — для float число знаков после запятой; для строк — максимальная длина.
- **type** — для чисел: `d` (int), `f` (float), `%`, `e`, `b`, `o`, `x` и т.д.

#### Полная таблица: format spec для чисел и строк

| Компонент        | Синтаксис                  | Значение                                                | Пример                           |
| ---------------- | -------------------------- | ------------------------------------------------------- | -------------------------------- |
| **fill**         | любой 1 символ перед align | Символ заполнения при width > длины значения            | `f"{3:*>5}"` → `"***3"`          |
| **align**        | `<` `>` `^` `=`            | Влево, вправо, центр, знак отдельно (для чисел)         | `f"{'x':^5}"` → `"  x  "`        |
| **sign**         | `+` `-` ` `                | Всегда знак, только минус, пробел вместо +              | `f"{1:+d}"` → `"+1"`             |
| **#**            | `#`                        | Альтернативная форма (0x для hex, 0o для oct и т.д.)    | `f"{10:#x}"` → `"0xa"`           |
| **0**            | `0`                        | Дополнение нулями (при числовом типе)                   | `f"{7:05d}"` → `"00007"`         |
| **width**        | целое число                | Минимальная ширина поля в символах                      | `f"{'hi':>5}"` → `"   hi"`       |
| **grouping**     | `,` `_`                    | Разделитель тысяч (`,` или `_`)                         | `f"{1000000:,}"` → `"1,000,000"` |
| **.precision**   | `.число`                   | Для float — знаков после запятой; для str — макс. длина | `f"{3.14159:.2f}"` → `"3.14"`    |
| **type (int)**   | `d` `b` `o` `x` `X` `c`    | Десятичное, двоичное, восьмеричное, hex, символ         | `f"{65:c}"` → `"A"`              |
| **type (float)** | `f` `e` `E` `%` `g` `G`    | Фиксированная, экспонента, процент, общий               | `f"{0.5:%}"` → `"50.000000%"`    |

**Примеры комбинаций:**

```python
# Заполнение и выравнивание
f"{42:*>8}"      # "******42"
f"{42:*<8}"      # "42******"
f"{42:*^8}"      # "***42***"

# Знак и ширина
f"{42:+5d}"      # "  +42"
f"{-42:+5d}"     # "  -42"
f"{42: 5d}"      # "   42" (пробел вместо + для положительных)

# Разделитель тысяч
f"{1_000_000:,}" # "1,000,000"
f"{1000:_}"      # "1_000" (подчёркивание как в литералах)

# Процент
f"{0.25:.0%}"    # "25%"
f"{0.255:.1%}"   # "25.5%"
```

**Порядок компонентов в format spec:** компоненты интерпретируются в фиксированном порядке: выравнивание и fill, знак, альтернативная форма (**#**), ширина, группировка (**,** или **\_**), точность (**.n**), тип (**d**, **f**, **e**, **%** и т.д.). Нельзя, например, поставить тип перед шириной — спецификация разбирается по правилам мини-языка format (см. документацию **Format Specification Mini-Language**). Пример с знаком и нулевым заполнением: **f"{n:0=+6d}"** при **n=42** даёт **"+00042"** (знак **+**, заполнение нулями до ширины 6, знак отдельно от цифр при **=**).

#### Как это работает внутри (упрощённо)

При компиляции исходного кода интерпретатор распознаёт f-строку как особый токен. Вместо одной константы строки генерируется код, который:

1. Вычисляет каждое выражение в `{...}` по очереди (слева направо).
2. Для каждого результата вызывается либо **`format(value, format_spec)`** (если после выражения указан format spec через `:`), либо **`str(value)`** (или **`repr(value)`** для формы `f'{x=}'`).
3. Полученные строки и литеральные фрагменты между скобками объединяются в одну результирующую строку (по сути конкатенация).

Таким образом, f-строка **не** хранится как «шаблон» для последующей подстановки: она превращается в последовательность операций «вычислить — преобразовать в строку — добавить к результату». Поэтому повторное использование одной и той же f-строки в цикле не приводит к повторному разбору шаблона — разбор произошёл один раз при компиляции. Метод **`.format()`** каждый раз разбирает строку шаблона и ищет в ней `{...}` — при большом числе вызовов f-строки выигрывают по скорости.

**Пошагово: что именно передаётся в **format**.** Если в f-строке указан format spec, например **`f"{x:>10.2f}"`**, то для значения **x** вызывается **x.**format**(">10.2f")**. Встроенные типы (**int**, **float**, **str**, **datetime** и т.д.) реализуют ****format**** и интерпретируют строку спецификации: **>** — выравнивание вправо, **10** — ширина, **.2** — точность (для float два знака после запятой), **f** — тип float. Пользовательские классы могут определить ****format**(self, format_spec)** и обрабатывать свою спецификацию (например, **f"{dt:%Y-%m-%d}"** для **datetime** передаёт **"%Y-%m-%d"** в **dt.**format**("%Y-%m-%d")**).

### 16.4 Debug-формат f'{x=}' (Python 3.8+)

В **Python 3.8** (PEP 553) добавлен специальный вид f-строки: **`f'{x=}'`**. При вычислении в строку подставляется не только значение выражения, но и его **текст** (имя переменной или выражение), затем `=`, затем repr значения. Это очень удобно для отладки.

```python
x = 10
y = 20
print(f"{x=}")           # x=10
print(f"{x=}, {y=}")     # x=10, y=20
print(f"{x + y=}")       # x + y=30
```

**Пошагово: что происходит при f"{x=}".** (1) Часть до `=` интерпретируется как «репрезентация» выражения (здесь имя `x`). (2) Вычисляется выражение `x` → получается значение (например, `10`). (3) В вывод попадает строка вида `"x=" + repr(10)`, т.е. `"x=10"`. Для сложного выражения, например `f"{items[0]=}"`, в левую часть попадёт текст `"items[0]"`, в правую — `repr(items[0])`. Итог: не нужно вручную писать `print("x =", x)` — одна конструкция даёт и имя, и значение.

Можно комбинировать с format spec: **`f'{x=:.2f}'`** — выведет, например, `x=3.14`.

### 16.5 Вложенные кавычки

В f-строке используются кавычки того же типа, что и у самой строки. Чтобы внутри выражения задать строковый литерал, нужно использовать другой тип кавычек или экранирование.

```python
name = "Мир"
# Внешние двойные — внутри выражения одинарные
print(f"Привет, {'гость' if name else 'мир'}!")
# Привет, гость!

# Внешние одинарные — внутри двойные
print(f'Строка: {"hello"}')
# Строка: hello
```

Тройные кавычки позволяют переносы строк и удобны для многострочных f-строк:

```python
a, b = 1, 2
text = f"""
Сумма: {a + b}
Разность: {a - b}
"""
```

### 16.6 Дата и время в f-строках

Для объектов `datetime.date`, `datetime.datetime` и `datetime.time` в format spec после `:` можно использовать те же коды форматирования, что и в методе `strftime`.

```python
from datetime import datetime
dt = datetime(2025, 2, 8, 14, 30)
print(f"{dt:%Y-%m-%d}")           # 2025-02-08
print(f"{dt:%Y-%m-%d %H:%M}")     # 2025-02-08 14:30
print(f"{dt:%d.%m.%Y}")           # 08.02.2025
```

Это возможно потому, что у этих типов реализован `__format__`, принимающий спецификацию в формате strftime.

#### Таблица: часто используемые коды strftime в f-строках

| Код    | Значение               | Пример вывода |
| ------ | ---------------------- | ------------- |
| **%Y** | Год (4 цифры)          | 2025          |
| **%m** | Месяц (01–12)          | 02            |
| **%d** | День месяца (01–31)    | 08            |
| **%H** | Час 24ч (00–23)        | 14            |
| **%M** | Минута (00–59)         | 30            |
| **%S** | Секунда (00–59)        | 00            |
| **%A** | Полное имя дня недели  | Monday        |
| **%B** | Полное имя месяца      | February      |
| **%a** | Сокращённое имя дня    | Mon           |
| **%b** | Сокращённое имя месяца | Feb           |
| **%p** | AM/PM (для 12ч)        | AM            |
| **%I** | Час 12ч (01–12)        | 02            |
| **%%** | Буквальный символ %    | %             |

**Примеры комбинаций для даты и времени:**

```python
from datetime import datetime
dt = datetime(2025, 2, 8, 14, 30, 0)
f"{dt:%Y-%m-%d}"              # "2025-02-08" — ISO-подобная дата
f"{dt:%d.%m.%Y %H:%M}"        # "08.02.2025 14:30" — европейский формат
f"{dt:%A, %d %B %Y}"          # "Saturday, 08 February 2025"
f"{dt:%H:%M}"                 # "14:30" — только время
```

#### Граничные случаи f-строк с датой/временем

- Если передать в format spec код, не поддерживаемый типом (например, **%H** для **date**, у которого нет времени), возможна ошибка или неопределённое поведение — используйте коды, подходящие для **date** / **datetime** / **time**.
- Пустая спецификация после двоеточия: **f"{dt:}"** эквивалентна **str(dt)** (стандартное строковое представление).

### 16.7 Теоретическое обоснование

- **Читаемость:** выражение и место подстановки находятся рядом; не нужно смотреть на порядок аргументов в `.format()` или `%`.
- **Производительность:** f-строка компилируется в код, который строит строку за один проход; по сравнению с конкатенацией или многократным вызовом `.format()` это часто эффективнее.
- **Единообразие:** один механизм и для простой подстановки, и для форматирования чисел/дат через format spec.
- **PEP 498** — Literal String Interpolation: мотивация — упрощение форматирования и уменьшение ошибок.

### 16.8 Граничные случаи и тонкости

- **Синтаксис `{}`:** чтобы вывести буквальную фигурную скобку, удвойте её: `f"{{ и }}"` → `"{ и }"`. Одна `{` без закрывающей `}` внутри выражения — синтаксическая ошибка.
- **Обратные слэши:** внутри **выражения** в `{...}` обратный слэш не допускается (ограничение грамматики). В литеральном тексте f-строки — как обычно (`\n`, `\t` и т.д.).

**Пошагово: как получается буквальная фигурная скобка в f-строке.**

1. В f-строке **`{{`** интерпретируется как **эскейп** для одной буквальной **`{`**: при построении результата **`{{`** заменяется на один символ **`{`**.
2. Аналогично **`}}`** заменяется на один символ **`}`**.
3. Итог: **f"{{ и }}"** при вычислении даёт строку **"{ и }"**. Одна **`{`** без пары **`}`** или наоборот трактуется как начало или конец выражения и приводит к **SyntaxError**. Чтобы вывести одну фигурную скобку в тексте, пишите **{{** или **}}** соответственно.

- **Пустое выражение:** `f"{}"` — **SyntaxError**; выражение в скобках обязательно.
- **Порядок вычисления:** выражения в одной f-строке вычисляются слева направо; побочные эффекты (например, вызов функции с side effect) выполняются ровно один раз для каждого вхождения.

**Типичные ошибки:** (1) Перепутать кавычки и получить SyntaxError внутри `{}` — использовать другой тип кавычек для вложенной строки. (2) Пытаться экранировать `{` как `\{` внутри выражения — нельзя; только двойная скобка `{{`. (3) Писать длинные или с побочными эффектами выражения в f-строке — лучше вынести в переменную и подставить её.

#### Плохо vs лучше (f-строки)

```python
# Плохо: конкатенация и ручное str() — много шума, легко забыть str()
"Цена: " + str(price) + ", количество: " + str(qty)

# Лучше: f-строка
f"Цена: {price}, количество: {qty}"

# Плохо: .format() с индексами — при добавлении аргумента легко сбить нумерацию
"{} и {} и {}".format(a, b, c)

# Лучше: f-строка — видно сразу, что куда подставляется
f"{a} и {b} и {c}"

# Плохо: отладка вручную — легко опечататься в имени переменной
print("price =", price, "qty =", qty)

# Лучше: f'{x=}' (Python 3.8+) — имя и значение автоматически
print(f"{price=}, {qty=}")
```

#### Ещё граничные случаи: пустая строка, один символ, None

- **Пустая f-строка:** **f""** — допустимо, результат **""**.
- **Одно выражение без текста:** **f"{x}"** — результат **str(x)**; если **x** — **None**, получится **"None"**.
- **None в выражении:** **f"значение: {x}"** при **x is None** даёт **"значение: None"** (через **str(None)**). Для отладки **f"{x=}"** даёт **"x=None"** (через **repr(None)**).
- **Пустая f-строка:** **f""** — допустима, результат **""**; **f"{''}"** даёт **""** (подставляется пустая строка).
- **Длина выражения:** технически в **{}** может быть многострочное выражение в скобках, но стилистически это не рекомендуется — выносите в переменную.
- **Порядок: fill и align:** при указании символа заполнения (fill) выравнивание (align) обязательно: **f"{x:0>5}"** — ширина 5, заполнение нулями справа от числа; без align fill игнорируется.

**Пошагово: что происходит при f"{x:>10.2f}" при x = 3.14159.**

1. Вычисляется выражение **x** → **3.14159**.
2. К нему применяется format spec **:>10.2f**: **>** — выравнивание вправо, **10** — минимальная ширина поля, **.2** — два знака после запятой, **f** — тип float.
3. Число форматируется как float с двумя знаками: **3.14**.
4. Ширина поля 10 символов; строка **"3.14"** короче, поэтому слева добавляются пробелы (fill по умолчанию — пробел).
5. Итог: **" 3.14"** (6 пробелов + "3.14").

#### Вложенные f-строки

Внутри выражения в **`{...}`** можно использовать другую f-строку — она вычисляется как обычное выражение и даёт строку. Такая «вложенная» f-строка полезна, когда нужно сформировать подстроку по своему шаблону и подставить её в общий шаблон:

```python
width = 10
name = "Alice"
# Внешняя f-строка, внутри выражение — ещё одна f-строка
msg = f"| {f'{name}':^{width}} |"   # |   Alice    |
```

Внутренняя **f'{name}'** вычисляется первой и даёт **"Alice"**; затем к ней применяется format spec **:^{width}** во внешней f-строке. Не злоупотребляйте вложенностью — при сложной логике выносите в переменные.

#### Терминология (f-строки)

| Термин (EN)                  | Термин (RU)                       | Пояснение                                                                                                  |
| ---------------------------- | --------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **Formatted string literal** | Форматированный строковый литерал | Строковый литерал с префиксом **f** или **F**, в котором **{...}** — подставляемые выражения               |
| **Format spec**              | Спецификация формата              | Часть после **:** в **{expr:format_spec}**; задаёт ширину, точность, выравнивание, тип (d, f, b, x и т.д.) |
| **Conversion flag**          | Флаг преобразования               | **!s**, **!r**, **!a** — способ приведения значения к строке (str, repr, ascii)                            |
| **Expression**               | Выражение                         | Всё, что в **{...}** до **:** или **!** — вычисляется в текущем контексте и подставляется в строку         |

#### Пример: отладка с f'{x=}' и format spec

```python
# Python 3.8+
price = 19.999
quantity = 3
print(f"{price=:.2f}, {quantity=}")   # price=20.00, quantity=3
```

**Пошагово: что происходит при f"{price=:.2f}" (Python 3.8+).** (1) Синтаксис **=** после выражения в f-строке — **debug format**: в вывод попадает не только значение, но и **текст выражения** (имя переменной или выражение), затем **=**, затем отформатированное значение. (2) Вычисляется выражение **price** → **19.999**. (3) К нему применяется format spec **:.2f** — два знака после запятой → **"20.00"** (округление). (4) Слева от **=** подставляется буквальный текст выражения: **"price"**. (5) Итог: **"price=20.00"**. Без format spec **f"{price=}"** дало бы **"price=19.999"** (repr по умолчанию для debug). Комбинация **=** и format spec полезна для отладочного вывода с нужным форматированием числа или строки.

#### Пример: многострочная f-строка с выравниванием

```python
rows = [
    ("Алиса", 30, "инженер"),
    ("Боб", 25, "дизайнер"),
]
for name, age, role in rows:
    print(f"| {name:>10} | {age:>5} | {role:<12} |")
# |      Алиса |    30 | инженер     |
# |        Боб |    25 | дизайнер    |
```

#### Когда не использовать f-строки

- **Шаблон приходит извне** (файл, сеть, пользователь): подстановка произвольных выражений из строки небезопасна; используйте **.format()** с фиксированным набором ключей или шаблонизатор (Jinja2 и т.п.).
- **Одна и та же строка с разными наборами аргументов:** например, шаблон логирования задан один раз, а подставляемые значения меняются — тогда **template.format(**data)\*\* удобнее, чем собирать f-строку из частей.
- **Слишком длинное или сложное выражение в `{}`:** если внутри скобок больше одной строки или тяжёлая логика — вынесите в переменную или функцию и подставьте результат: **f"{result}"**.

#### Пример: миграция с .format() на f-строку

```python
# Было (format)
"Пользователь {0} (id={1}) залогинен с {2}".format(username, user_id, ip)

# Стало (f-строка)
f"Пользователь {username} (id={user_id}) залогинен с {ip}"
# Порядок аргументов не важен; видно сразу, какая переменная куда подставляется
```

### 16.9 Практический вывод по f-strings

- Предпочитайте f-строки для подстановки переменных и выражений: `f"{x}"`, `f"{x:.2f}"`.
- Для отладки используйте `f"{var=}"` (Python 3.8+).
- Для дат/времени используйте format spec с кодами strftime: `f"{dt:%Y-%m-%d}"`.
- Не помещайте в f-строки слишком сложную логику — выносите в переменные или функции для читаемости.
- Для шаблонов из внешних источников используйте **.format()** или специализированные библиотеки, не f-строки с **eval**-подобной подстановкой.

**Запомните (§16):** f-строка = префикс `f` + `{выражение}` в строке; по умолчанию подстановка через `str()`, для отладки — `f'{x=}'`, для форматирования — `{x:format_spec}`; буквальные `{` и `}` пишут как `{{` и `}}`; сложную логику выносите из `{}` в переменные.

#### Полный пример: логирование с f-строками и датой

```python
from datetime import datetime

def log(level: str, message: str, **data):
    ts = datetime.now()
    extra = " ".join(f"{k}={v!r}" for k, v in data.items())
    line = f"{ts:%Y-%m-%d %H:%M:%S} [{level:>5}] {message}"
    if extra:
        line += f" | {extra}"
    print(line)

log("INFO", "Запуск", port=8080, debug=True)
# 2025-02-08 15:30:00 [ INFO] Запуск | port=8080 debug=True
```
