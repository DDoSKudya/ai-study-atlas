 ## 5. Компьютерное зрение (§138)

 ### 5.0. Картинка как массив чисел

 Любое изображение в компьютере — это:

 - матрица пикселей;
 - у каждого пикселя — яркость (ч/б) или набор каналов (RGB);
 - значения обычно в диапазоне [0, 255] или [0.0, 1.0].

 Пример: чтение и отображение картинки с помощью OpenCV:

 ```python
 import cv2

 img = cv2.imread("cat.jpg")  # BGR‑формат
 print(img.shape)  # (height, width, channels)

 gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
 blurred = cv2.GaussianBlur(gray, (5, 5), 0)

 cv2.imwrite("cat_gray_blur.jpg", blurred)
 ```

 Даже такие простые операции уже позволяют:

 - улучшать качество картинок;
 - готовить изображения для подачи в нейросети.

#### 5.0a. Как представить себе изображение как тензор

Полезно в голове держать три уровня:

1. **Уровень файлов.**
   - У вас есть `cat.jpg` — файл сжатого изображения;
   - внутри — байты, закодированные в формате JPEG/PNG.
2. **Уровень матрицы пикселей.**
   - Когда вы делаете `cv2.imread`, библиотека:
     - читает байты с диска;
     - декодирует их в **массив чисел**;
   - результат — тензор (обычно NumPy‑массив) формы примерно `(height, width, channels)`;
     - `height` — число строк (по вертикали);
     - `width` — число столбцов (по горизонтали);
     - `channels` — 3 для RGB (или BGR, как в OpenCV).
3. **Уровень нормализации / подготовки к модели.**
   - значения каналов могут быть в `[0, 255]` (целые числа);
   - для нейросетей часто переводят в `float32` и масштабируют в `[0.0, 1.0]` или нормируют вокруг нуля.

Мозгу помогает представить себе **чёрно‑белое изображение**:

- тогда у вас есть просто матрица чисел:
  - чем ближе значение к 0 — тем пиксель темнее;
  - чем ближе к 255 — тем светлее.

Цветное изображение — это просто **три такие матрицы** (R, G, B), сложенные в один тензор.

**Мини‑упражнение:**

1. Возьмите любую фотографию и прочитайте её через OpenCV или Pillow;
2. Выведите:
   - форму массива (`img.shape`);
   - минимум/максимум значений по каждому каналу;
3. Попробуйте:
   - обрезать часть картинки (например, верхнюю половину);
   - уменьшить её размер (`cv2.resize`);
   - сохранить и посмотреть глазами, что изменилось.

 ### 5.1. Torchvision, YOLO и др.

 - **torchvision**:
   - модели (ResNet, EfficientNet, и др.);
   - датасеты (CIFAR‑10, ImageNet, …);
   - трансформации (`transforms`) для аугментации данных.
 - **Ultralytics YOLO**:
   - удобный интерфейс для детекции объектов;
   - обученные модели, которые «из коробки» находят объекты на картинках/видео.
 - **albumentations**:
   - мощная библиотека аугментаций изображений;
   - позволяет улучшать обобщающую способность моделей.
 - **Mediapipe**:
   - быстрые модели для детекции позы, рук, лица в реальном времени.

 Ключевая идея:

 - вместо того, чтобы «писать фильтры руками», мы **обучаем модели**, которые сами учатся извлекать нужные признаки из изображений;
 - большая часть сложности скрыта за фреймворками, ваша задача — правильно подготовить данные и интерпретировать результаты.

#### 5.1a. Как устроен типичный пайплайн CV‑модели

На примере детекции объектов (YOLO) можно мыслить так:

1. **Вход.**
   - вы подаёте на вход изображение (тензор формы `(H, W, 3)` или `(3, H, W)` в зависимости от фреймворка);
2. **Предобработка.**
   - изменение размера до фиксированного (`640x640`, к примеру);
   - нормализация значений пикселей;
   - иногда — дополнительные аугментации (повороты, обрезания, изменение яркости).
3. **Модель.**
   - несколько десятков/сотен слоёв свёрточной нейросети;
   - сеть постепенно превращает «сырые пиксели» в более абстрактные признаки:
     - сначала — края/углы;
     - потом — простые формы;
     - потом — части объектов;
     - в конце — сами объекты и их местоположение.
4. **Выход.**
   - набор предсказаний вида:
     - «координаты прямоугольника» (`x, y, w, h`);
     - «класс объекта» (кошка, собака, человек…);
     - «уверенность» (score).
5. **Пост‑обработка.**
   - фильтрация по порогу уверенности;
   - устранение дублей (NMS — non‑maximum suppression).

Вам, как практикующему инженеру, важно уметь:

- правильно **подготовить данные** (разметка, форматы, аугментации);
- выбрать и настроить **готовую модель/библиотеку**;
- уметь **читать и интерпретировать результаты** (где модель ошибается, какие типы объектов путает).

Дальше, когда вы захотите углубляться, можно разбираться во внутренностях архитектур (ResNet, EfficientNet, разных версий YOLO и т.д.), но база — это понимание пайплайна «картинка → тензор → модель → боксы/классы».
