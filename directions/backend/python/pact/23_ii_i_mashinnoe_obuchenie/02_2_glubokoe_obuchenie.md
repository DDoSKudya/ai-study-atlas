 ## 2. Глубокое обучение (Deep Learning) (§135)

### 2.0. Идея нейросетей на пальцах

Нейросеть можно воспринимать как **набор последовательных трансформаций векторов**:

- вход \( x \) (например, вектор признаков или картинка → «распрямлённый» вектор);
- слой 1: \( h_1 = f(W_1 x + b_1) \);
- слой 2: \( h_2 = f(W_2 h_1 + b_2) \);
- …;
- выход \( \hat{y} \) — предсказание.

Где:

- \( W_i, b_i \) — **параметры** (веса и смещения), которые мы подбираем;
- \( f \) — нелинейная функция (ReLU, sigmoid, tanh и т.д.);
- обучаем с помощью **градиентного спуска**, автоматически считая производные (`autograd`).

#### 2.0a. Аналогия с поваром и рецептом

Представьте, что у вас есть:

- **список продуктов** — это ваш вход \( x \) (признаки);
- несколько **этапов готовки**:
  - сначала что‑то режем и маринуем;
  - потом обжариваем;
  - потом тушим с соусом;
- на каждом этапе блюдо **меняется** — это слои нейросети.

Тогда:

- **сырой продукт** → первый этап (слой) → промежуточное состояние;
- промежуточное состояние → второй этап → ...
- на выходе получаем **готовое блюдо** — предсказание.

«Рецепт» — это числа \( W_i, b_i \):

- сколько соли насыпать;
- сколько минут жарить;
- в какой момент добавить воду.

**Обучение нейросети** — это процесс, когда:

- вы много раз готовите блюдо для гостей (прогоняете данные через сеть);
- гости ставят оценку (функция потерь/ошибка);
- вы **чуть‑чуть корректируете рецепт** (обновляете веса), чтобы в следующий раз было вкуснее.

Градиентный спуск:

- «градиент» говорит, **в какую сторону крутить ручку** (увеличить/уменьшить параметр), чтобы ошибка уменьшилась;
- «спуск» — вы маленькими шагами двигаетесь по этим подсказкам, пока не найдёте «достаточно хороший» рецепт.

Если математика сейчас кажется страшной — держите в голове только картинку:

> Нейросеть = много маленьких простых преобразований (слоёв),  
> которые мы **медленно подстраиваем**, чтобы они выдавали нужный результат на примерах.

 ### 2.1. PyTorch: тензоры и autograd

 PyTorch — один из самых популярных DL‑фреймворков (особенно в исследовательском мире).

 Основные сущности:

 - `torch.Tensor` — многомерный массив (похож на `ndarray`, но с поддержкой GPU и autograd);
 - `nn.Module` — базовый класс для слоёв и моделей;
 - `optim` — оптимизаторы (SGD, Adam, …);
 - `DataLoader` — итератор по батчам данных.

**Простейший пример: линейная регрессия на PyTorch.**

 ```python
 import torch
 from torch import nn

 # 1. Подготовим данные (y ~ 2x + 1 с шумом)
 torch.manual_seed(42)

 X = torch.linspace(-1, 1, steps=100).unsqueeze(1)  # shape (100, 1)
 y = 2 * X + 1 + 0.2 * torch.randn_like(X)

 # 2. Определим модель: один линейный слой
 model = nn.Linear(in_features=1, out_features=1)

 # 3. Функция потерь и оптимизатор
 criterion = nn.MSELoss()
 optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

 # 4. Цикл обучения
 for epoch in range(1000):
     # прямой проход (forward)
     y_pred = model(X)

     # считаем loss
     loss = criterion(y_pred, y)

     # обнуляем градиенты
     optimizer.zero_grad()

     # обратный проход (backward) — autograd считает градиенты
     loss.backward()

     # шаг оптимизатора
     optimizer.step()

     if epoch % 200 == 0:
         print(f"Epoch {epoch:4d}, loss = {loss.item():.4f}")

 # 5. Смотрим итоговые параметры
 weight = model.weight.item()
 bias = model.bias.item()
 print("Learned parameters:", weight, bias)
 ```

 Важно:

 - вы **не считаете производные руками**, PyTorch делает это автоматически;
 - GPU‑ускорение включается просто `tensor.to("cuda")` / `model.to("cuda")`, если доступен CUDA‑девайс;
 - более сложные модели — это композиция слоёв (`nn.Sequential` или свой `nn.Module`).

#### 2.1a. Разбор примера PyTorch «по шагам»

1. **Создаём данные.**
   - `torch.linspace(-1, 1, steps=100)` — берём 100 чисел равномерно от -1 до 1;
   - `unsqueeze(1)` превращает вектор формы `(100,)` в матрицу `(100, 1)` — одна колонка, 100 строк;
   - `y = 2 * X + 1 + шум` — мы притворяемся, что **настоящая зависимость**: \( y = 2x + 1 \), но с небольшим случайным шумом.

2. **Определяем модель.**
   - `nn.Linear(1, 1)` — это слой, который делает ровно то же: `y_pred = w * x + b`;
   - внутри него есть параметры `weight` и `bias`, которые изначально случайные.

3. **Функция потерь (`MSELoss`).**
   - Смотрит на разницу между предсказанием `y_pred` и настоящим `y`;
   - чем сильнее промах — тем больше `loss`.

4. **Оптимизатор (`SGD`).**
   - Реализует «градиентный спуск» — знает, как по градиентам обновить веса;
   - `lr=0.1` — «размер шага» (насколько сильно мы двигаем веса в сторону уменьшения ошибки).

5. **Цикл обучения.**
   - `y_pred = model(X)` — прогоняем все входы через текущий «рецепт»;
   - `loss = criterion(y_pred, y)` — считаем, насколько мы ошиблись;
   - `optimizer.zero_grad()` — обнуляем старые градиенты (иначе они будут накапливаться);
   - `loss.backward()` — PyTorch **сам** считает градиенты по всем параметрам;
   - `optimizer.step()` — делает маленький шаг в сторону уменьшения ошибки.

6. **Печать параметров.**
   - В идеале после обучения `weight` будет близко к 2, а `bias` — к 1;
   - это значит, что сеть «выучила» скрытое правило, которым мы сгенерировали данные.

**Мини‑упражнение:**

1. Поменяйте формулу генерации данных на \( y = -3x + 0.5 \) и посмотрите, чему научится модель.
2. Попробуйте изменить `lr` на `0.001` и на `1.0` и посмотреть:
   - как меняется скорость сходимости;
   - бывают ли случаи, когда обучение «разъезжается» (loss растёт).

Не нужно сейчас понимать всю механику `backward` — важно почувствовать, что PyTorch берёт на себя **самую тяжёлую часть** (вычисление градиентов), а вам остаётся задать форму модели и цикл обучения.

 ### 2.2. PyTorch: простая классификация на MLP

 Мини‑пример бинарной классификации с помощью многослойного перцептрона:

 ```python
 import torch
 from torch import nn
 from torch.utils.data import TensorDataset, DataLoader

 # 1. Сгенерируем игрушечные данные
 torch.manual_seed(0)
 n_samples = 1000

 X = torch.randn(n_samples, 2)
 # Цель: если x0 + x1 > 0 -> 1, иначе 0
 y = (X.sum(dim=1) > 0).long()

 dataset = TensorDataset(X, y)
 dataloader = DataLoader(dataset, batch_size=32, shuffle=True)

 # 2. Определяем модель
 class SimpleMLP(nn.Module):
     def __init__(self):
         super().__init__()
         self.net = nn.Sequential(
             nn.Linear(2, 16),
             nn.ReLU(),
             nn.Linear(16, 16),
             nn.ReLU(),
             nn.Linear(16, 2),
         )

     def forward(self, x):
         return self.net(x)

 model = SimpleMLP()
 criterion = nn.CrossEntropyLoss()
 optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

 # 3. Обучение
 for epoch in range(20):
     total_loss = 0.0
     for xb, yb in dataloader:
         logits = model(xb)
         loss = criterion(logits, yb)

         optimizer.zero_grad()
         loss.backward()
         optimizer.step()

         total_loss += loss.item() * xb.size(0)

     print(f"Epoch {epoch+1}, loss = {total_loss / n_samples:.4f}")
 ```

 Идея:

 - на выходе `logits` размера `(batch_size, 2)`, соответствующие двум классам;
 - `CrossEntropyLoss` внутри применяет `softmax` + log‑loss;
 - вы можете заменить архитектуру (больше слоёв, другие активации), оставляя цикл обучения тем же.

#### 2.2a. Что здесь вообще происходит «человеческим языком»

1. **Генерация данных.**
   - `X = torch.randn(n_samples, 2)` — 1000 точек на плоскости (по сути, случайные координаты \((x_0, x_1)\));
   - `y = (X.sum(dim=1) > 0).long()` — правило:  
     если \( x_0 + x_1 > 0 \) → класс `1`, иначе `0`.  
     То есть мы делим плоскость линией `x0 + x1 = 0`.

2. **Модель `SimpleMLP`.**
   - Первый слой: `Linear(2, 16)` — сжимает/разворачивает 2 входных числа в 16;
   - `ReLU` — «обрубает» отрицательные значения (делает модель нелинейной, что важно);
   - Второй слой: `Linear(16, 16)` + `ReLU` — добавляет ещё один уровень нелинейности;
   - Последний слой: `Linear(16, 2)` — выдаёт два числа: «баллы» для класса 0 и класса 1 (`logits`).

3. **Почему именно `CrossEntropyLoss`.**
   - На вход принимает `logits` (сырые баллы) и истинные метки классов `y`;
   - внутри считает `softmax` (превращает баллы в вероятности) и логарифмический loss;
   - наказывает сильнее, когда модель очень уверенно ошибается.

4. **Цикл обучения.**
   - Берём батч (`xb, yb`) из `DataLoader` — небольшую порцию данных;
   - считаем `logits = model(xb)` — текущие предсказания;
   - считаем `loss` — насколько мы ошиблись;
   - обнуляем градиенты, делаем `backward`, `step` — как и в примере с линейной регрессией.

5. **Что в итоге «знает» модель.**
   - Она учится разделять плоскость на две области, как если бы вы провели разделяющую линию;
   - MLP в таком простом примере мог бы обойтись и одной линейной моделью,  
     но чуть более сложные архитектуры дают запас гибкости для более «кривых» границ.

**Мини‑упражнение:**

1. Измените правило генерации `y`, например:
   - класс 1, если \( x_0^2 + x_1^2 > 1 \) (точки вне круга радиуса 1);
   - класс 1, если \( x_0 > 0 \) и \( x_1 > 0 \) (один квадрант).
2. Обучите ту же модель и посмотрите, как меняется поведение loss.
3. Попробуйте уменьшить число нейронов (например, до 4) и посмотреть, справляется ли модель с задачей.

 ### 2.3. TensorFlow / Keras: альтернативный стек

 **TensorFlow** (особенно в виде высокоуровневого API Keras) — популярный фреймворк, часто встречается в прод‑системах.

 Мини‑пример на Keras:

 ```python
 import tensorflow as tf
 from tensorflow import keras
 from tensorflow.keras import layers

 # Игрушечные данные
 import numpy as np
 X = np.random.randn(1000, 2).astype("float32")
 y = (X.sum(axis=1) > 0).astype("int32")

 model = keras.Sequential(
     [
         layers.Dense(16, activation="relu", input_shape=(2,)),
         layers.Dense(16, activation="relu"),
         layers.Dense(2, activation="softmax"),
     ]
 )

 model.compile(
     optimizer="adam",
     loss="sparse_categorical_crossentropy",
     metrics=["accuracy"],
 )

 model.fit(X, y, batch_size=32, epochs=10, validation_split=0.2)
 ```

 Важное сходство:

 - те же понятия: слои, активации, функции потерь, оптимизаторы;
 - GPU‑ускорение тоже «прозрачное» при наличии подходящего окружения.

#### 2.3a. Как соотнести PyTorch и Keras в голове

Полезно думать так:

- в PyTorch вы чаще **явно пишете цикл обучения** (хотя есть и высокоуровневые обёртки);
- в Keras вы чаще говорите:  
  «вот модель, вот функция потерь, вот оптимизатор, вот данные» → `model.fit(...)` делает всё сам.

При этом базовые кирпичики одни и те же:

- `Dense` в Keras ≈ `nn.Linear` в PyTorch;
- функции активации (`relu`, `softmax`) есть в обоих стеках;
- loss‑функции (`MSE`, `crossentropy`) и оптимизаторы (`SGD`, `Adam`) тоже совпадают по смыслу.

Если вы поймёте **общую схему обучения** (forward → loss → backward → update),  
пересесть с одного фреймворка на другой будет в разы проще.

 ### 2.4. JAX, ONNX, TensorRT, OpenVINO

 - **JAX** — библиотека от Google для автоматического дифференцирования и JIT‑компиляции:
   - популярна в исследовательских проектах;
   - позволяет писать код «почти как NumPy», но с автодифом и векторизацией.
 - **ONNX** (Open Neural Network Exchange) — формат для обмена моделями между фреймворками:
   - обучили в PyTorch → экспортировали в ONNX → запустили в другом рантайме (ONNX Runtime, TensorRT, OpenVINO);
 - **TensorRT, OpenVINO** — фреймворки для **оптимизации инференса** (особенно на GPU и edge‑устройствах):
   - могут давать сильный выигрыш по скорости и памяти за счёт сжатия и оптимизации графа.

 На уровне «первого знакомства» важно просто знать:

 - **ONNX** — формат, который позволяет «не быть привязанным» к одному фреймворку;
 - **TensorRT / OpenVINO** — инструменты для **ускорения моделей в проде**, особенно на ограниченном железе.

#### 2.4a. Очень грубая интуиция

Можно представить себе:

- PyTorch / TensorFlow / JAX — как **кухни**, на которых вы готовите блюда (обучаете модели);
- ONNX — как **контейнер**, в который можно переложить готовое блюдо, чтобы его можно было разогреть в разных микроволновках;
- TensorRT / OpenVINO — как **специальные духовые шкафы**, которые разогревают еду быстрее и экономнее,  
  но требуют, чтобы блюдо было упаковано в удобный для них формат.

На этапе освоения основ вам достаточно просто знать, что такие штуки есть,  
и они пригодятся, когда вы дойдёте до реального продакшена и упираетесь в скорость/память.

#### 2.4b. Мини‑примеры: JAX и экспорт в ONNX

**JAX: «NumPy с автодифом и JIT» (очень упрощённо).**

```python
import jax
import jax.numpy as jnp

# Простая функция: y = w * x + b
def model(params, x):
    w, b = params
    return w * x + b

# Функция потерь: среднеквадратичная ошибка
def loss_fn(params, x, y_true):
    y_pred = model(params, x)
    return jnp.mean((y_pred - y_true) ** 2)

# Считаем градиент потерь по параметрам
grad_loss = jax.grad(loss_fn)

x = jnp.linspace(-1.0, 1.0, 100)
y_true = 2.0 * x + 1.0

params = (jnp.array(0.0), jnp.array(0.0))  # стартовые w, b
grads = grad_loss(params, x, y_true)

print("dL/dw, dL/db:", grads)
```

Ключевые моменты:

- `jax.numpy` выглядит почти как обычный `numpy`;
- `jax.grad` автоматически считает производные по параметрам;
- затем поверх этого можно строить более сложное обучение (оптимизаторы, JIT‑компиляция и т.п.).

**Экспорт PyTorch‑модели в ONNX.**

```python
import torch
from torch import nn

class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(4, 2)

    def forward(self, x):
        return self.fc(x)

model = SimpleNet()
dummy_input = torch.randn(1, 4)

torch.onnx.export(
    model,
    dummy_input,
    "simplenet.onnx",
    input_names=["input"],
    output_names=["output"],
    opset_version=17,
)
```

После этого файл `simplenet.onnx` можно:

- загружать в ONNX Runtime;
- оптимизировать и запускать через TensorRT / OpenVINO;
- использовать в окружениях, где PyTorch не нужен/не доступен.
