### Цель раздела

Понять, что такое хеш в контексте Python: как вызывается, что возвращает, для чего используется, и **как именно** Python вычисляет хеш для разных типов. Отличить `hash()` от криптографических хешей (`hashlib`).

### Термины

| Термин | Определение |
|--------|-------------|
| **Хеш** (hash) | Целое число фиксированного диапазона, вычисляемое по содержимому объекта. «Сжатый» числовой «отпечаток» для быстрого поиска. |
| **Хеш-функция** | Функция, преобразующая произвольные данные в хеш. В Python: `hash(obj)` → вызывает `obj.__hash__()`. |
| **Равномерное распределение** | Идеальное свойство хеш-функции: разные входы дают хеши, равномерно разбросанные по диапазону, чтобы минимизировать коллизии. |
| **sys.hash_info** | Модуль/атрибут с параметрами хеширования CPython: `width`, `modulus`, `inf`, `nan`, `algorithm` и др. |

### Правила и синтаксис

1. **Вызов:** `hash(obj)` — встроенная функция, вызывает `type(obj).__hash__(obj)`. Поиск метода идёт по классу (MRO), не по экземпляру. Если у объекта нет `__hash__` или он вернул не `int` — ошибка.
2. **Возвращаемое значение:** целое число `int`. Разрядность: 64 бита на 64-битных платформах, 32 бита на 32-битных (см. `sys.hash_info.width`). Значение может быть отрицательным.
3. **Индексация в CPython:** для dict/set используется младшие биты хеша:
   ```python
   # Упрощённо (реализация в C сложнее):
   # Хеш «сворачивается» в диапазон индексов таблицы
   index = hash(key) & ((1 << 61) - 1)  # младшие 61 бит
   # Число 61: 2^61 - 1 умещается в 64-битное слово, оставляя место для метаданных
   ```
4. **Цель:** равномерное распределение — разные ключи должны давать разные индексы (в идеале), чтобы быстрый поиск O(1).

### sys.hash_info — параметры хеширования

```python
import sys
print(sys.hash_info)
# hash_info(width=64, modulus=2305843009213693951, inf=314159, nan=0,
#           imag=1000003, algorithm='siphash24', hash_bits=64, seed_bits=128, ...)
```

- **width** — разрядность хеша в битах (64).
- **modulus** — `2^61 - 1`; используется для приведения хеша к положительному диапазону.
- **inf** — хеш для `float('inf')` (и `-inf` имеет другой фиксированный хеш).
- **nan** — хеш для `float('nan')` (особый случай: `nan != nan`, но `hash(nan)` определён).
- **algorithm** — алгоритм для str/bytes (например, `'siphash24'` с PEP 456).
- **seed_bits** — разрядность seed для рандомизации (128).

### Пошагово: что происходит при вызове hash(obj)

1. Python ищет `__hash__` на типе объекта: `type(obj).__hash__`.
2. Если метод найден — вызывается `type(obj).__hash__(obj)`.
3. Возвращаемое значение должно быть `int`; иначе `TypeError`.
4. Если `__hash__` не найден или равен `None` — `TypeError: unhashable type`.
5. Результат используется для индексации в хеш-таблице (dict, set и т.д.).

### Как Python хеширует разные типы (детально)

#### int

- **Мелкие целые числа** (часто в диапазоне -5..256): хеш обычно равен самому числу, `hash(42) == 42`.
- **Произвольные int:** используется младшие биты числа (или комбинация с модулем), чтобы хеш помещался в разрядность.
- **-1:** особый случай — `hash(-1)` возвращает `-2` в CPython, потому что `-1` зарезервирован как маркер «пустой слот» во внутренней структуре dict. В массиве indices dict хранит индексы в entries; значение `-1` (или `UINT_MAX`) означает «слот свободен». Если бы `hash(-1)` возвращал `-1`, ключ `-1` конфликтовал бы с маркером и ломал логику поиска.
- **Дополнительно:** `hash(n) == n` для `-5 <= n <= 256` (кеш мелких int).

```python
hash(42)    # 42
hash(-1)    # -2  (исключение из-за внутреннего использования -1)
hash(10**20)  # большое число, хеш — его представление по модулю 2^64
```

#### float

- Хеш вычисляется по битовому представлению числа (с учётом нормализации).
- **Особые значения:**
  - `float('nan')` — `hash(float('nan'))` возвращает фиксированное число (например, 0 в части реализаций); при этом `float('nan') != float('nan')`, но хеш у них одинаковый (это допустимо: обратная импликация `hash(a)==hash(b)` ⇒ `a==b` не требуется).
  - `float('inf')` и `float('-inf')` — имеют фиксированные хеши.
  - `-0.0` и `0.0` — равны (`-0.0 == 0.0`), поэтому `hash(-0.0) == hash(0.0)` (контракт!).

```python
hash(3.14)
hash(float('nan'))  # определён, но nan != nan
hash(0.0) == hash(-0.0)  # True, т.к. 0.0 == -0.0
```

#### str и bytes

- Используется **SipHash** (PEP 456) — криптографически стойкая хеш-функция, устойчивая к hash-flooding.
- **PYTHONHASHSEED** задаёт seed: при `random` — seed случайный при каждом запуске; при `0` — детерминированный.
- Длина строки влияет на время вычисления хеша, но не на «качество» в смысле равномерности.

#### tuple

- Хеш кортежа — комбинация хешей элементов. В CPython используется итеративная формула (реализация: `Objects/tupleobject.c`, функция `tuplehash`):
  ```python
  # Псевдокод tuple hash (концептуально):
  mult = 1000003   # sys.hash_info.imag; множитель для «размазывания» битов
  h = начальное_значение
  for x in t:
      h = (h ^ hash(x)) * mult + сдвиг  # с приведением к разрядности
  h += 97531 * len(t)   # добавка по длине для различения () и ((),) и т.д.
  return -2 if h == -1 else h   # -1 зарезервирован для «пустой слот»
  ```
  Константы взяты из `sys.hash_info` (imag, и др.); обеспечивают хорошее распределение при комбинировании хешей.
- Важно: **порядок элементов влияет**. `hash((1, 2)) != hash((2, 1))`.
- Все элементы должны быть hashable; иначе `hash(tuple)` вызовет `TypeError`.
- Пустой кортеж: `hash(())` — фиксированная константа.

#### frozenset

- Хеш вычисляется как **XOR** хешей элементов с дополнительной обработкой (в CPython — сумма `hash(x) * множитель` по всем элементам, с учётом размера множества). Порядок элементов **не влияет**: `hash(frozenset([1,2])) == hash(frozenset([2,1]))`.
- Все элементы должны быть hashable.
- Пустой frozenset: `hash(frozenset())` — фиксированная константа (0 в CPython).
- Важно: `hash(frozenset([a,b])) == hash(frozenset([b,a]))`, но `hash(frozenset([a,b]))` вообще говоря **не равен** `hash(a) ^ hash(b)` — в CPython используется более сложная формула для уменьшения коллизий.

```python
hash(frozenset([1, 2])) == hash(frozenset([2, 1]))  # True
hash(frozenset())  # 0
hash(frozenset([1]))  # зависит от hash(1)
```

#### complex

- Хеш комбинирует хеши действительной и мнимой частей. `hash(1+2j)` зависит от `hash(1)` и `hash(2)`.
- `complex` hashable; `hash(complex(nan, 0))` определён (возвращает фиксированное значение). `complex(nan, 0) != complex(nan, 0)` (nan не равен сам себе), но хеши совпадают — контракт требует только `a == b` ⇒ `hash(a) == hash(b)`; обратная импликация не нужна.

#### None, bool

- `hash(None)` — фиксированная константа.
- `bool` — подкласс `int`, поэтому `hash(True) == 1`, `hash(False) == 0`.

### Почему хеш ищется на классе, а не на экземпляре

`hash(obj)` вызывает `type(obj).__hash__(obj)`, а не `obj.__hash__()`. Причина: магические методы — часть протокола типа. Если бы поиск шёл по экземпляру, можно было бы «подменить» хеш динамически: `obj.__hash__ = lambda: 123`. Это нарушило бы инвариант: один объект в dict должен иметь стабильный хеш. Поиск по классу гарантирует, что поведение определяется типом, а не случайной подменой атрибута.

### hash() vs hashlib — критически важное различие

| Аспект | `hash(obj)` | `hashlib.md5()`, `hashlib.sha256()` и т.д. |
|--------|-------------|--------------------------------------------|
| **Назначение** | Быстрый поиск в dict/set (хеш-таблицы) | Криптография, проверка целостности, пароли |
| **Вход** | Любой hashable объект Python | Байты (bytes, bytearray) |
| **Выход** | `int` (32 или 64 бита) | Байты фиксированной длины (16, 32, 64 и т.д.) |
| **Детерминированность** | Зависит от PYTHONHASHSEED (для str/bytes) | Всегда детерминирован для одних и тех же данных |
| **Безопасность** | Не для криптографии; SipHash защищает от DoS | MD5 — устарел для паролей; SHA-256, BLAKE2 — криптографически стойкие |
| **Где используется** | `dict`, `set`, `frozenset`, `__hash__` | Подписи, хеш-файлы, проверка загрузок |

**Вывод:** никогда не используйте `hash()` для паролей или криптографических целей. Для этого — `hashlib`, `hmac`, `secrets`.

### Визуализация: поток при d["key"]

```
Пользователь:  d["key"]
       ↓
Python:        h = hash("key")           # SipHash; результат — int
       ↓
               i = h & (len(indices)-1)  # индекс в массиве индексов
       ↓
               idx = indices[i]          # индекс в entries
       ↓
               если пусто → KeyError
               иначе: entries[idx].key == "key" ?
                  да  → вернуть entries[idx].value
                  нет → perturb, следующий индекс, повторить
```

### Примеры

```python
# Хеши встроенных типов
print(hash(42))           # 42 (мелкие int)
print(hash(-1))           # -2 (особый случай)
print(hash(3.14))         # 322818021289917443
print(hash("hello"))      # зависит от PYTHONHASHSEED
print(hash((1, 2)))       # комбинация hash(1) и hash(2)
print(hash(frozenset([1,2])))  # XOR-комбинация, порядок не важен

# Хеш вызывается при добавлении в set или использовании как ключ dict
d = {}
d["key"] = 100   # внутри: hash("key") используется для поиска слота
s = {1, 2, 3}   # каждый элемент хешируется

# sys.hash_info
import sys
print(sys.hash_info.algorithm)  # 'siphash24'
```

### Граничные случаи и типичные ошибки

- **Не все объекты hashable:** `hash([1, 2])` → `TypeError: unhashable type: 'list'`.
- **Хеш не обязан быть уникальным:** коллизии возможны; важно: `a == b` ⇒ `hash(a) == hash(b)`. Обратное неверно.
- **Хеш может быть отрицательным:** CPython приводит его к нужному диапазону при индексации (через битовые операции и mask).
- **float('nan'):** `hash(nan)` определён, но `nan != nan`. Контракт требует: *если* `a == b`, *то* `hash(a) == hash(b)`. Для nan условие «если» ложно (nan никогда не равен другому объекту, включая сам себя), поэтому требование «то» не применяется. Обратная импликация (hash совпадает ⇒ объекты равны) не входит в контракт. Поэтому `hash(nan)` допустим.
- **Изменение объекта после добавления в set/dict:** для hashable типов мутабельность запрещена; если бы мутабельный объект как-то попал в set, изменение полей привело бы к «потере» объекта при поиске.
- **__hash__ возвращает не int:** `TypeError: __hash__ method should return an integer`.
- **Большие числа:** `hash(10**100)` — число сводится к разрядности (64 или 32 бита); переполнение обрабатывается через модуль или битовые операции.
- **Постоянство между сессиями:** хеши **не гарантированы** между запусками интерпретатора. Для str/bytes/datetime — зависят от PYTHONHASHSEED. Для int/float/tuple — детерминированы в рамках одной версии CPython, но не сохраняйте хеши между сессиями для сериализации, сравнения или проверки целостности; для этого используйте `hashlib`.

### Требования к __hash__ (что должен возвращать)

- **Тип:** обязательно `int`. Если `__hash__` вернёт `float`, `None` или другой тип — `TypeError`.
- **Стабильность:** один и тот же объект должен возвращать один и тот же хеш при каждом вызове (пока объект не изменится — для immutable это «никогда»).
- **Согласованность с __eq__:** если `a == b`, то `hash(a) == hash(b)`.

```python
class Bad:
    def __hash__(self):
        return "not an int"
hash(Bad())  # TypeError: __hash__ method should return an integer
```

### Запомните

> `hash(obj)` — это не «уникальный ID объекта» и не криптографический хеш. Это число для **быстрого поиска в хеш-таблицах**: стабильное для равных объектов, равномерно распределённое для разных. Используется в `dict`, `set`, `frozenset`. Для криптографии — `hashlib`.
