# Часть XI. Хеширование: как это работает

> **Цель:** глубоко понять механизм хеширования в Python — от математической идеи до реализации в `dict` и `set`. Научиться:
>
> - понимать, что такое **хеш-функция** и **контракт hashable**;

> - разбираться во внутреннем устройстве хеш-таблиц (бакеты, коллизии, perturb, resize);

> - учитывать `PYTHONHASHSEED`, SipHash и защиту от hash-flooding DoS;
> - отличать `hash()` от криптографических хешей (`hashlib`).

## Хеширование в Python: общая картина

Прежде чем погружаться в отдельные разделы, полезно понимать **зачем хеширование нужно** и **как оно связано с dict и set**. Многие типичные ошибки («unhashable type», «равные объекты разъехались в set») объясняются именно отсутствием этой общей картины.

### Зачем нужен хеш

**Хеширование** — основа быстрого поиска по ключу. Без него операции `d["key"]` и `"key" in s` требовали бы перебора всех элементов — **O(n)**. С хеш-таблицей — в среднем **O(1)**.

| Операция | list (без хеша) | dict / set (с хешем) |
|----------|-----------------|----------------------|
| Поиск элемента | O(n) | O(1) в среднем |
| При N = 1 000 000 | тысячи сравнений | один проход по хешу |

Именно поэтому **dict** и **set** — основа многих алгоритмов: уникальные элементы, подсчёт частот, кеширование.

### Ключевая идея хеш-функции

**Хеш-функция** преобразует объект произвольного размера в число фиксированного диапазона. По этому числу вычисляется индекс в массиве — «адрес» слота.

- Два **равных** объекта должны давать **одинаковый** хеш (контракт).
- Разные объекты — желательно разные хеши (равномерное распределение).

### Где в Python используется хеширование

| Конструкция | Роль хеша |
|-------------|-----------|
| `dict` | Ключи — хеш нужен для O(1)-поиска |
| `set`, `frozenset` | Элементы — проверка `x in s` за O(1) |
| `functools.lru_cache` | Аргументы функции — ключи кеша |
| `weakref.WeakKeyDictionary`, `WeakSet` | Ключи/элементы — хеш-таблица внутри |
| `Counter`, `defaultdict` | Ключи — те же требования, что у dict |
| `itertools.groupby` | Ключи группировки — **не** требуют hashable (сравнение через `==`) |
| Внутренние структуры CPython | Имена переменных, атрибуты и т.п. |

**Не используют хеш:** `functools.cached_property` хранит результат в `__dict__` экземпляра по имени атрибута — хеш объекта не нужен.

### Зачем изучать хеширование

- Корректно проектировать классы (особенно с `__eq__`).
- Избегать ошибок `TypeError: unhashable type`.
- Понимать производительность: O(1) vs O(n), когда хеш «медленный».
- Учитывать безопасность: hash-flooding DoS, PYTHONHASHSEED.

## Краткая шпаргалка (суть)

| Понятие | Суть |
|---------|------|
| **hash(obj)** | Вызывает `obj.__hash__()`; возвращает `int`; используется для индексации в хеш-таблицах |
| **Hashable** | Объект: `hash(a)` стабилен за время жизни; `a == b` ⇒ `hash(a) == hash(b)` |
| **Хеш-таблица** | Массив бакетов; индекс от хеша; dict, set реализованы через неё; O(1) в среднем |
| **Open addressing** | CPython: при коллизии ищем следующий слот (perturb-схема); не chaining |
| **Коллизия** | Два ключа дают один индекс; неизбежна (принцип Дирихле); разрешается probing |
| **Load factor** | Заполненность; при ~2/3 — resize; поддерживает O(1) |
| **__hash__ = None** | Сделать объект не hashable (при переопределённом `__eq__`) |
| **PYTHONHASHSEED** | `random` — защита от DoS (SipHash); `0` — детерминированный хеш |
| **hash() vs hashlib** | `hash()` — для dict/set; hashlib — криптография, пароли, проверка целостности |

**Сводка типов:** hashable — `int`, `float`, `str`, `bytes`, `tuple`*, `frozenset`*, `None`, `bool`, `Decimal`, `Fraction`, `datetime`, `Ellipsis`, классы, функции. Не hashable — `list`, `dict`, `set`, `bytearray`. (*Только если все элементы hashable.)

## Как изучать эту часть

**Минимальный практический минимум** — если нужно быстро использовать хеширование:
- **hashable** = объект можно положить в set/dict; `list`, `dict`, `set` — не hashable; `tuple`, `str`, `int`, `frozenset` — hashable.
- При переопределении `__eq__` — либо `__hash__` (по тем же полям), либо `__hash__ = None`.
- Уметь: `tuple(lst)`, `frozenset(s)`, `tuple(d.items())` для ключей.
- `hash()` — для dict/set; для паролей и криптографии — `hashlib`.

**Базовый уровень** — после минимума: контракт hashable, полный список типов, кастомный `__hash__`, PYTHONHASHSEED.

**Углублённый уровень** — для библиотек и CPython: indices+entries, perturb, SipHash, hash-flooding DoS, sys.hash_info.
