## Краткая шпаргалка по терминам

| Термин | Что это значит простыми словами |
|---|---|
| **CLI (Command Line Interface)** | Интерфейс управления Celery через команды в терминале. |
| **Worker** | Процесс(ы), которые берут задачи из очередей и исполняют код. |
| **Beat** | Планировщик, который по расписанию публикует задачи в broker. |
| **Inspect** | Набор read-only команд, чтобы посмотреть состояние worker-ов. |
| **Control** | Набор команд удаленного управления worker-ами (например, shutdown). |
| **Broadcast** | Механизм рассылки команды сразу нескольким worker-ам. |
| **Revoke** | Отмена задачи (до старта исполнения, либо с terminate для running-задачи). |
| **Purge** | Полная очистка очередей от ожидающих задач. |
| **QoS / prefetch** | Сколько задач worker может заранее зарезервировать. |
| **Runbook** | Пошаговая инструкция действий в штатных и аварийных сценариях. |

#### Проверь себя: термины

1. Чем `inspect` принципиально отличается от `control`?

<details><summary>Ответ</summary>

`inspect` — только чтение состояния, `control` — активное изменение состояния worker-а/пула через удаленные команды.

</details>

2. Почему broadcast-команды могут «не доходить» даже при рабочем брокере?

<details><summary>Ответ</summary>

Из-за сетевых ограничений, политик безопасности, настроек транспорта, недоступности worker-а или отключенных частей event/control-контура.

</details>
