## Углубление: границы плана vs границы vendor support (к 43.4)

| Ожидание | Реальность | Что делать |
| -------- | ---------- | ---------- |
| «Курс покроет каждый CVE до публикации» | CVE появляются непредсказуемо | Подписка на security advisory + процесс патча |
| «После курса не нужен SRE» | SRE закрывает эксплуатацию, не обучение | Runbooks из **21**, **14** |
| «Celery заменит workflow engine для всего» | Это архитектурное решение | **Части 10, 25** |

### Проверь себя

Почему таблица выше **не умаляет** ценность учебного плана?

<details><summary>Ответ</summary>

Она разводит **роли инструментов**: план строит модель и полноту навыков, а security/vendor/SRE закрывают внешние потоки изменений; без модели из плана процессы бессмысленны, без процессов план не выживает в проде.

</details>
