## Типичные ошибки процесса

| Ошибка | Почему больно | Как правильно |
| ------ | ------------- | ------------- |
| Апгрейд без чтения changelog зависимостей | Симптом выглядит «как баг кода» | Читать пакетный набор релизов |
| Диагностика только через бизнес-логику задачи | Теряются часы на SQL/ретраи | Сначала сеть/TLS/IAM per 43.2 |
| «У нас всё в Kubernetes, значит сеть идеальна» | K8s не отменяет DNS/CNI/egress | Дашборды сети + тест клиента брокера |
| Жить только на знаниях из курса без delta-log | Дрейф 2–3 года незаметен | Квартальная сверка 36–37 |
| Обещать ноль инцидентов после обучения | Ломает доверие к инженерам | Честная формулировка 43.4 + процесс |
| Делать чеклисты «для галочки» без артефактов | Через месяц решение не воспроизводится | Для каждого пункта хранить ссылку на доказательство (тикет, лог, ADR) |
| Пытаться решить всё одним владельцем | Процесс разваливается на стыках команд | Явный RACI и периодический review ролей |
| Не привязывать процесс к календарю релизов | Проверки вспоминают «после аварии» | Фиксированный cadence + событие-триггеры из 43.3 |
