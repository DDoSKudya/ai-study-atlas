## Как устроены разделы и как пользоваться оглавлением

Внутри каждого основного раздела применяется учебный маршрут:

| Блок | Зачем нужен |
|---|---|
| **Цель раздела** | Понять результат чтения. |
| **В этом разделе главное** | Быстро собрать 3–7 опорных идей. |
| **Термины** | Зафиксировать слова, без которых дальше будет путаница. |
| **Теория и правила** | Дать точные формулировки и инженерные правила. |
| **Пошагово** | Пройти процесс как “чек-лист действий”. |
| **Простыми словами** | Перевести техническую идею в человеческий язык. |
| **Картинка в голове** | Метафора/схема для запоминания. |
| **Как запомнить** | Сжать идею в короткую формулу. |
| **Примеры** | Увидеть идею в коде/командах/схемах. |
| **Практика / реальные сценарии** | Место, где это встречается в жизни. |
| **Типичные ошибки** | Узнать частые грабли заранее. |
| **Что будет если…** | Граничные случаи и последствия. |
| **Проверь себя** | 2–3 вопроса с ответами под спойлером. |
| **Запомните** | Итог. |

Оглавление помогает прыгать к нужной теме. В этой части номера соответствуют `4.1–4.6` из глобального плана.
