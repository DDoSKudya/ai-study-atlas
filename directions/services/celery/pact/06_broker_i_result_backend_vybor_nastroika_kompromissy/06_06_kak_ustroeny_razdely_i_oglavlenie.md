## Как устроены разделы и оглавление

Внутри этой части каждый основной раздел (`6.1–6.6`) следует одному маршруту:

| Блок | Зачем нужен |
|---|---|
| **Цель раздела** | Понимать результат чтения. |
| **В этом разделе главное** | Быстро ухватить опорные идеи. |
| **Термины** | Убрать двусмысленность ключевых слов. |
| **Теория и правила** | Дать точные формулировки и правила выбора/настройки. |
| **Пошагово** | Разложить процесс инженерного решения. |
| **Простыми словами** | Перевести в “картинку в голове”. |
| **Картинка в голове** | Визуализировать связь компонентов и “точки влияния”. |
| **Как запомнить** | Сжать в формулу. |
| **Примеры** | Показать настройку и код на типовых сценариях. |
| **Практика / реальные сценарии** | Что делать “после теории”. |
| **Типичные ошибки** | Что ломают чаще всего. |
| **Что будет, если…** | Последствия неправильных решений. |
| **Проверь себя** | Вопросы на понимание с ответами в `<details>`. |
| **Запомните** | Короткий вывод. |

Оглавление разбито на этапы изучения, и внутри каждого этапа ссылки ведут к нужным секциям.
