## Краткая шпаргалка по терминам

| Термин | Суть простыми словами |
|---|---|
| **Unit test** | Тест чистой логики без инфраструктуры: быстрый, детерминированный. |
| **Integration test** | Тест взаимодействия с реальными внешними компонентами (broker/backend/DB). |
| **End‑to‑end (E2E)** | Тест “как в проде”: реально запускается worker, задача проходит через broker и возвращает результат. |
| **Eager mode** | Режим Celery, где задачи выполняются “сразу” в том же процессе, без worker и broker. |
| **Контракт задачи (task contract)** | Что именно считается “входом/выходом” задачи: типы/структура аргументов, обязательные поля, семантика ошибок. |
| **Payload** | Содержимое сообщения/аргументов задачи после сериализации. |
| **Contract test** | Тест, который фиксирует формат/схему payload, чтобы изменения не ломали потребителя. |
| **Determinism** | Свойство теста “одинаковый ввод → одинаковый результат” без флейков. |
| **Flaky test** | Нестабильный тест, который то проходит, то падает из‑за гонок/таймингов/сети. |
| **Redelivery** | Повторная доставка сообщения (например, ack не дошёл или worker умер). |
| **Logical job** | Одна бизнес‑работа, которая может породить несколько попыток/исполнений задач. |
| **Test isolation** | Изоляция тестов друг от друга: отдельные очереди/префиксы/брокер, очистка состояния. |
