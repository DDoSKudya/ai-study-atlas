## Справочник по части 3

| Тема | Что обязательно помнить |
|---|---|
| `3.1` Установка и окружение | Celery нужен вместе с broker и (опционально) result backend; URL задаёт главное; extras зависят от транспорта; конфигурацию лучше держать в env vars |
| `3.2` Структура минимального проекта | `celery_app.py` с одним объектом `app` — точка сборки; `tasks.py` декорирует функции; worker должен импортировать то же через `-A` |
| `3.3` Первая задача | `delay()` публикует сообщение и возвращает task_id; аргументы должны сериализоваться; `apply_async()` даёт больше контроля публикации |
| `3.4` Запуск worker | Worker с `-A` должен регистрировать задачи; логи подтверждают publish -> consume -> execute |
| `3.5` Чтение статусов | `AsyncResult` читает backend; `PENDING` не всегда “в очереди”; `STARTED/RETRY/FAILURE/SUCCESS/REVOKED` — диагностические метки |
| `3.6` Чек-лист “не работает” | Сначала import/`-A`, потом доступность broker, потом queue/vhost/routing, потом execution и только затем backend-статусы |
| `3.7` Несколько worker-ов и очередей | `-Q` управляет обслуживаемыми очередями; разделение очередей помогает SLA и управляемости |
