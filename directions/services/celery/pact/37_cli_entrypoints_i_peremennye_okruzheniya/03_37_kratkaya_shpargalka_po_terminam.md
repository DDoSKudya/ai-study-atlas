## Краткая шпаргалка по терминам

| Термин | Простыми словами |
|---|---|
| **Entrypoint** | Точка входа в Celery app: где CLI находит приложение и registry задач. |
| **Global CLI flags** | Флаги верхнего уровня `celery` перед подкомандой (`-A`, `--workdir`, `--config`). |
| **Subcommand flags** | Флаги конкретной подкоманды (`worker`, `beat`, `inspect`, `control` и др.). |
| **Destination** | Конкретный worker (или группа), к которому направляется команда управления. |
| **Statedb** | Локальный файл состояния worker (например, revoked tasks). |
| **Pidfile** | Файл с PID процесса для операционного управления. |
| **CELERY_* env** | Переменные окружения, которыми можно задавать часть параметров запуска/конфига. |
| **Config precedence** | Приоритет источников настроек: что переопределяет что. |
| **Graceful shutdown** | Корректная остановка процесса с завершением/сохранением состояния. |
| **Control plane** | Канал операционного управления worker-ами через `inspect`/`control`. |
