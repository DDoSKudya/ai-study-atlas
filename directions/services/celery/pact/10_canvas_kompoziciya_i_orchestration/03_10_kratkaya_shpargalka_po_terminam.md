## Краткая шпаргалка по терминам

| Термин | Суть простыми словами |
| --- | --- |
| **Canvas** | Набор примитивов Celery для композиции задач в workflow-графы. |
| **Signature** | "Рецепт" будущего вызова задачи (что запускать, с какими аргументами и опциями). |
| **Partial signature** | Сигнатура, где часть аргументов уже задана заранее, а часть подставится позже. |
| **Immutable signature** | Сигнатура, которая не принимает автоматически результат предыдущего шага (`.si()`). |
| **Chain** | Последовательность задач: выход одной задачи идёт на вход следующей. |
| **Group** | Набор независимых задач для параллельного запуска (fan-out). |
| **Chord** | `group` + финальный callback после завершения всех задач в группе (fan-in). |
| **Map / starmap** | Массовый запуск одной задачи на коллекции входных данных. |
| **Chunks** | Пакетирование множества входных элементов в ограниченное число задач. |
| **Logical job** | Бизнес-операция верхнего уровня (например, "собрать отчёт"), включающая много task execution. |
| **Correlation ID** | Идентификатор для сквозной трассировки всех шагов одного logical job. |
| **Partial success** | Часть шагов workflow уже успешна, часть упала или не стартовала. |
