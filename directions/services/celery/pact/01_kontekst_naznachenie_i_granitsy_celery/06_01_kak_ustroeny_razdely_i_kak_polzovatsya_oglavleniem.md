## Как устроены разделы и как пользоваться оглавлением

Внутри каждого основного раздела используется один и тот же учебный маршрут:

| Блок                             | Зачем нужен                                                         |
| -------------------------------- | ------------------------------------------------------------------- |
| **Цель раздела**                 | Понять, какой результат должен остаться после чтения.               |
| **В этом разделе главное**       | Сразу увидеть 3-7 опорных идей.                                     |
| **Термины**                      | Зафиксировать слова, без которых дальше будет путаница.             |
| **Теория и правила**             | Получить точную формулировку и инженерные правила.                  |
| **Пошагово**                     | Пройти процесс рассуждения или выбора по шагам.                     |
| **Простыми словами**             | Перевести техническую идею в человеческий язык.                     |
| **Картинка в голове**            | Увидеть схему или метафору, чтобы знание не оставалось абстрактным. |
| **Как запомнить**                | Сжать идею в короткую формулу.                                      |
| **Примеры**                      | Увидеть, как теория выглядит в коде и архитектуре.                  |
| **Практика / реальные сценарии** | Понять, где и как это встречается в живых системах.                 |
| **Типичные ошибки**              | Узнать частые грабли заранее.                                       |
| **Что будет, если...**           | Разобрать последствия неправильного использования.                  |
| **Проверь себя**                 | Проверить понимание, а не просто запоминание.                       |
| **Запомните**                    | Зафиксировать итог раздела.                                         |

Как пользоваться этой частью:

- если Celery для тебя новый, иди **строго сверху вниз**;
- если опыт уже есть, но не хватает системности, особенно внимательно читай `1.2`, `1.3`, `1.5`, `1.7`;
- если нужна быстрая перепроверка перед внедрением, открывай **шпаргалку**, **таблицы сравнения**, **частые сценарии** и **типичные ошибки** в конце;
- если какая-то формулировка кажется очевидной, всё равно просматривай блоки **"Что будет, если..."** и **"Проверь себя"**: именно там обычно вскрываются реальные различия между "кажется понятным" и "понимаю по-настоящему".
