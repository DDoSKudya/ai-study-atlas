## Что желательно знать заранее

Желательно, чтобы ты уже прошёл:

- **часть 1** — контекст и границы Celery;
- **часть 2** — очереди, брокер, базовые семантики;
- **часть 3** — быстрый старт, первый worker;
- **часть 4** — компоненты и путь сообщения (producer → broker → worker → result backend).

Если чего-то не хватает, материал всё равно читается: термины здесь кратко повторяются там, где без них нельзя.
