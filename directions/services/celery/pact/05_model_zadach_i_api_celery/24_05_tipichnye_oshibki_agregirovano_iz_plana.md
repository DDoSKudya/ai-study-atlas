## Типичные ошибки (агрегировано из плана)

> В глобальном плане (часть 5) отдельно выделены три **частых промаха** — они сознательно повторены здесь и раскрыты по всему файлу:

1. **Слишком большой payload** вместо id — см. п. **5.3**, сценарии в конце части.  
2. **Неявная зависимость от глобального состояния процесса** (singleton’ы с «теплом» producer, кэш без инвалидации, env только на одном узле) — см. таблицу ниже и п. **5.3** (чёрный список типов).  
3. **Не различать бизнес-ошибку и временную инфраструктурную** — см. п. **5.5** (retry vs permanent), **5.6** (`autoretry_for`), диаграмму трёх слоёв.

| Ошибка | Почему больно | Как правильно |
| --- | --- | --- |
| Огромный payload | Давит брокер, растут задержки | id + внешнее хранилище |
| Зависимость от глобального состояния процесса | Невоспроизводимо на worker | явные args, конфиг из env |
| Путаница бизнес/инфра ошибок | либо вечные ретраи, либо молчаливые потери | классификация исключений |
| ORM из родительской сессии | таинственные баги | загрузка по id в worker |
| Fan-out без таймаутов | лавина, каскадные падения | лимиты, circuit breaker, chunking |
