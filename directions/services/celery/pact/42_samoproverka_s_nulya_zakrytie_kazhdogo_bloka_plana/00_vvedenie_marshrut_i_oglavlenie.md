# Часть 42. Самопроверка «с нуля»: закрытие каждого блока плана

> **Цель этой части** — превратить прочитанные главы курса в **проверяемые навыки**: после каждой учебной части у тебя есть повторяемый ритуал (термины → схема → анти-паттерны → edge case → связка с конфигом/CLI), есть **короткие микролабы** на ≤30 минут по частям 1–34 и **сквозной экзамен-проект** из восьми модулей, плюс честные «ловушки самооценки», чтобы не путать *узнавание* с *пониманием*.
>
> **Ключевая мысль:** Celery учат не глаза, а **руки и рассуждения**. Если после части ты не можешь за 10 минут нарисовать поток сообщения и назвать два параметра, которые реально меняют поведение, знание ещё не «село».

> **Как сказано в глобальном плане (`mastery_plan.md`, вступление к части 42):** для новичка после каждой **части 1–34** полезно отвечать письменно (или устной записью) **по шаблону**; без этого «мелочи» не стыкуются в цельную картину. Этот файл и есть такой шаблон в развёрнутом виде.
