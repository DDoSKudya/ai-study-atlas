## Мини-оценка зрелости (self-assessment)

Оцени каждый пункт по шкале `0-2`:
- `0` — не внедрено;
- `1` — внедрено частично;
- `2` — внедрено и стабильно работает.

| Критерий | Оценка |
|---|---:|
| Границы ответственности Celery/Kombu/Billiard понятны и применяются в диагностике |  |
| Матрица совместимости и policy апдейтов ведутся регулярно |  |
| Lock-процесс единый и воспроизводимый в локальной, CI и production среде |  |
| SBOM/сканирование/лицензионные проверки встроены в pipeline |  |
| DX-контур стандартизирован (compose профили, make/just, pre-commit) |  |
| Есть canary + rollback дисциплина для dependency-обновлений |  |

Интерпретация:
- `0-4` — знание темы есть, но внедрение слабое;
- `5-8` — рабочий уровень, есть зоны риска;
- `9-12` — зрелая практика по части 27.

#### Проверь себя: мини-оценка зрелости

1. Почему высокая самооценка без canary/rollback практики часто ошибочна?

<details><summary>Ответ</summary>

Потому что устойчивость dependency-изменений проверяется именно в контролируемом релизном цикле. Без canary/rollback оценка чаще отражает теорию, а не эксплуатационную зрелость.

</details>

2. Какой минимальный признак перехода с уровня `5-8` на `9-12`?

<details><summary>Ответ</summary>

Наличие повторяемой дисциплины: единый lock-процесс, transport-aware тесты, canary rollout, быстрый rollback и регулярный review матрицы совместимости.

</details>
