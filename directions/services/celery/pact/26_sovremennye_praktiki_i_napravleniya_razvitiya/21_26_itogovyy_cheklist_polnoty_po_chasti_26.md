## Итоговый чеклист полноты по части 26

Перед тем как считать часть усвоенной на практике, проверь:

- [ ] По `26.1` есть IaC, autoscaling guardrails, секреты и telemetry baseline.
- [ ] По `26.2` есть сквозная корреляция `HTTP -> task -> dependency` и SLO с burn-rate алертами.
- [ ] По `26.3` есть broker resilience drills и обновляемый runbook восстановления.
- [ ] По `26.4` есть корректные probes, graceful shutdown и scaling per queue.
- [ ] По `26.5` есть зафиксированные контракты границ в гибридной архитектуре.
- [ ] По `26.6` есть roadmap с измеримыми артефактами и периодическим ревью.
- [ ] Есть минимум один mini-RFC и один postmortem, оформленные по шаблону из этой части.

Если 2+ пункта не выполнены, часть лучше считать изученной теоретически, но не внедренной практически.

#### Проверь себя: итоговый чеклист

1. Почему наличие чеклиста без фактических артефактов не подтверждает внедрение?

<details><summary>Ответ</summary>

Потому что чеклист фиксирует намерение, а артефакты подтверждают выполнение и позволяют проверить качество изменений на практике.

</details>

2. Что означает статус “изучено теоретически, но не внедрено практически”?

<details><summary>Ответ</summary>

Команда понимает концепции, но не имеет работающих механизмов в проде: нет измеримости, регулярных процедур и ответственных владельцев.

</details>
