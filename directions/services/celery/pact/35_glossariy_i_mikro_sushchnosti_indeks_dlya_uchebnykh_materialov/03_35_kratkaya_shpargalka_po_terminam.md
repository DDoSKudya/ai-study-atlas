## Краткая шпаргалка по терминам

| Термин | Простыми словами |
|---|---|
| **Celery app** | Экземпляр приложения Celery, где живут конфигурация, реестр задач и связи с брокером/backend. |
| **Task** | Описание фоновой работы + правила исполнения (retry, time limits, serializer и т.д.). |
| **Request** | Текущий контекст выполнения задачи на worker-е: id, retries, headers, chain info. |
| **Signature** | "Чертеж" вызова задачи с аргументами и опциями, который можно комбинировать в workflow. |
| **AsyncResult** | Объект-доступ к состоянию и результату одной задачи по `task_id`. |
| **Queue/Exchange/Binding** | Брокерные сущности маршрутизации сообщений (куда отправить и по какому правилу). |
| **Pool** | Модель параллелизма worker-а (`prefork`, `threads`, `gevent`, `eventlet`, `solo`). |
| **Chord** | Паттерн "группа задач + финальный callback после завершения всех". |
| **Beat schedule** | Правила периодического запуска задач. |
| **accept_content** | Белый список форматов сообщений, которые worker разрешает принимать. |
