## Краткая шпаргалка по терминам

| Термин | Суть простыми словами |
|---|---|
| **Monitoring** | “Мы наблюдаем заранее выбранные показатели и алёртимся по ним”. |
| **Observability** | “По данным системы мы можем объяснить, что происходит и почему, даже если не ожидали такой поломки”. |
| **SLI / SLO / SLA** | Метрика качества / цель по метрике / договорённость (часто юридическая). |
| **Queue depth** | Сколько сообщений лежит в очереди сейчас. |
| **Queue lag** | Насколько по времени мы “отстаём” (как давно опубликовано сообщение, которое ещё не обработано). |
| **Publish rate / consume rate** | Скорость публикации / скорость потребления сообщений. |
| **Execution time** | Время выполнения кода задачи на worker-е. |
| **End-to-end latency** | Время от публикации до завершения, включая ожидание/ретраи. |
| **Correlation id** | Идентификатор, который связывает логи/трейсы одной бизнес‑операции (HTTP → tasks → downstream). |
| **Cardinality (кардинальность)** | Сколько уникальных значений у метки (label). Высокая кардинальность ломает метрики. |
| **USE / RED** | Два подхода к метрикам: Utilization/Saturation/Errors и Rate/Errors/Duration. |
| **Fan-out / fan-in** | Разветвление задач на много подзадач / сбор результатов обратно. |
