## Справочник по части 20

| Тема | Ключевая идея | Где применять |
|---|---|---|
| Fire-and-forget | Вторичные эффекты без синхронного ожидания | Уведомления, не-критичные интеграции |
| Async request/reply | Явный жизненный цикл долгого job через API | Генерация отчетов, долгие операции UI |
| Batch processing | Чанки + fan-out/fan-in + checkpoint | Ночные пересчеты, массовые регламенты |
| Event-driven handlers | Outbox/inbox + versioning + replay | Реакция на доменные события |
| Pipeline | Стадии с контрактами и отдельным SLA | ETL, модерация, enrichment |
| Bulkhead | Изоляция очередей/воркеров/квот | Защита критичных контуров |
| Anti-patterns | Раннее распознавание хрупких решений | Архитектурные ревью и рефакторинг |

#### Проверь себя

1. Чем "справочник" полезен в эксплуатации, а не только в обучении?

<details><summary>Ответ</summary>

Он ускоряет выбор решения в инцидентах и архитектурных обсуждениях: можно быстро сопоставить симптом/сценарий с подходящим паттерном и типовыми рисками.

</details>
