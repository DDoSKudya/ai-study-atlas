## Краткая шпаргалка по терминам

| Термин | Что это значит простыми словами |
|---|---|
| **PII (Personally Identifiable Information)** | Персональные данные, по которым можно прямо или косвенно определить человека. |
| **PHI (Protected Health Information)** | Медицинские данные, защищенные отраслевыми правилами. |
| **PCI DSS data** | Данные платежных карт и связанный контекст, требующий строгих ограничений. |
| **Data minimization** | Храним и передаем только минимально необходимый объем данных. |
| **Tokenization** | Вместо чувствительных данных передаем безопасный идентификатор/ссылку. |
| **Retention policy** | Политика сроков хранения и удаления данных. |
| **Legal hold** | Юридический запрет на удаление данных до завершения расследования/спора. |
| **Immutable audit log** | Журнал, который нельзя незаметно изменить задним числом. |
| **Data residency** | Требование хранить/обрабатывать данные в определенной географии. |
| **SBOM (Software Bill of Materials)** | "Состав ПО": список зависимостей и их версий для контроля рисков. |

#### Проверь себя: термины

1. Чем токенизация отличается от шифрования payload?

<details><summary>Ответ</summary>

Шифрование защищает содержимое при передаче/хранении, но данные все равно присутствуют в сообщении. Токенизация заменяет чувствительные данные ссылкой/идентификатором, уменьшая сам объем риска в очереди.

</details>

2. Почему immutable audit log обычно отделяют от result backend Celery?

<details><summary>Ответ</summary>

Result backend оптимизирован для технических статусов задач и может чиститься по retention. Аудит должен иметь отдельные гарантии неизменности, доступа и хранения.

</details>
