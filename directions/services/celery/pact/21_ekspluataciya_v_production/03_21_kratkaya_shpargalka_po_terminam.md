## Краткая шпаргалка по терминам

| Термин | Что это значит простыми словами |
|---|---|
| **Deployment model** | Способ, как запускаются worker-ы и beat: systemd, контейнеры, Kubernetes и т.д. |
| **Graceful shutdown** | Корректная остановка: не брать новые задачи и по возможности завершить текущие без порчи данных. |
| **Drain behavior** | Поведение worker-а при "осушении" очереди перед остановкой/обновлением. |
| **Rolling update** | Постепенное обновление экземпляров без полной остановки системы. |
| **Autoscaling** | Автоматическое изменение количества worker-ов по сигналам нагрузки. |
| **Lag** | Отставание очереди: сколько задач/времени нужно, чтобы догнать входящий поток. |
| **Thrashing** | Дребезг масштабирования: частые scale up/down, когда система "дергается" и тратит ресурсы впустую. |
| **PDB** | `PodDisruptionBudget`, правило в Kubernetes, ограничивающее одновременные прерывания pod-ов. |
| **Runbook** | Пошаговая инструкция для оператора: как диагностировать и чинить конкретный инцидент. |
| **Replay** | Повторная подача событий/задач после сбоя или восстановления. |
| **Backward compatibility** | Совместимость новой версии кода со старыми сообщениями в очереди. |
| **Phased rollout** | Пошаговое внедрение изменений, когда старые и новые версии некоторое время живут вместе. |
