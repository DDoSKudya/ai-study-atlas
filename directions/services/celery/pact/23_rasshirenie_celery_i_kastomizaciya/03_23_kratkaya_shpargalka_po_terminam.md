## Краткая шпаргалка по терминам

| Термин | Что это значит простыми словами |
|---|---|
| **Custom Task base class** | Свой базовый класс задач, где собраны общие правила для всех задач. |
| **Retry helper** | Унифицированный способ повторного выполнения задач (тайминги, логика ошибок, jitter). |
| **Tracing hook** | Точка, где ты подключаешь трассировку/метрики вокруг исполнения задачи. |
| **Payload schema** | Договор о структуре данных в сообщении задачи. |
| **Serializer** | Механизм преобразования Python-объектов в формат передачи (`json`, `msgpack` и т.д.). |
| **Schema evolution** | Изменение формата payload без поломки старых и новых потребителей. |
| **Signal** | Событие жизненного цикла Celery/задачи, на которое можно подписаться. |
| **Bootstep** | Шаг инициализации worker-а с зависимостями между шагами. |
| **Task annotations** | Централизованный слой "перепределения" параметров задач без редактирования их кода. |
| **Centralized policy** | Единая политика исполнения задач (лимиты, retry, таймауты) из одного места. |
| **Version lock-in** | Ситуация, когда кастомизация сильно привязывает к версии Celery/Kombu. |
| **Extension governance** | Управление расширениями: владелец, тесты, документация, SLA на изменения. |
